## 摘要
长周期软件工程智能体通常需要在有限的上下文预算下管理冗长且嘈杂的交互历史。现有的内存管理方法通常依赖静态压缩工作流，或对压缩时机和粒度施加刚性约束。此外，这些方法未能联合优化内存管理和问题解决能力，以在减少令牌使用的同时提升性能。我们提出SWE-MeM，一种面向软件工程智能体的主动按需内存管理训练框架。SWE-MeM提供灵活的内存工具，使智能体能够根据轨迹状态、任务进度和剩余上下文预算决定何时、何地以及如何压缩。我们通过合成的主动内存管理轨迹和内存感知GRPO训练智能体，后者通过内存感知轨迹分割和步骤级信用分配联合优化内存管理和问题解决。在SWE-Bench Verified上，SWE-MeM在4B和30B模型上分别达到了43.4%和60.2%的解决率，在性能和效率上均优于现有的内存管理基线。

## Abstract
Long-horizon software engineering agents often need to manage lengthy and noisy interaction histories under limited context budgets. Existing memory management methods typically rely on static compression workflows or impose rigid constraints on compression timing and granularity. Moreover, these approaches fail to jointly optimize memory management and issue resolution capabilities to improve performance while reducing token usage. We present SWE-MeM, a training framework for proactive and on-demand memory management in software engineering agents. SWE-MeM provides a flexible memory tool that lets agents decide when, what, and how to compress based on trajectory state, task progress, and remaining context budget. We train agents with synthesized proactive memory-management trajectories and Memory-aware GRPO, which jointly optimizes memory management and issue resolution through memory-aware trajectory splitting and step-level credit assignment. On SWE-Bench Verified, SWE-MeM achieves 43.4% and 60.2% resolve rate with 4B and 30B models, respectively, outperforming existing memory management baselines in both performance and efficiency.

### 论文核心问题与整体含义（研究动机与背景）

- **核心问题**：长周期（Long-Horizon）软件工程（SWE）智能体在执行任务时，会产生大量、冗长且充满噪声的交互历史（如探索、尝试、日志等）。在有限的上文预算（Context Budget）内，智能体的性能会因信息过载和注意力分散而显著下降。
- **现有方法局限**：现有的内存管理方法（如阈值触发的总结、子任务折叠）存在两大缺陷：
    1.  **灵活性不足**：采用固定的压缩规则（如仅当上下文达到阈值时压缩、固定的压缩粒度），无法适应轨迹中信息价值的异质性（如早期的关键代码不能压缩，但冗长的执行日志可以安全压缩）。
    2.  **优化目标单一**：这类方法仅优化上下文长度的压缩，而没有与提升任务解决效率联合优化。这可能导致步骤数或总 Token 消耗量增加，整体效率反而降低。
- **研究动机**：开发一种**自适应的、主动的**内存管理机制，使智能体能够根据当前任务状态和上下文预算，自主、智能地决定何时、何处以及如何压缩历史信息，从而实现任务性能与 Token 效率的协同提升。

### 论文提出的方法论：核心思想与关键技术细节

- **核心思想**：提出 **SWE-MeM** 训练框架，赋予 SWE 智能体**主动按需**（Proactive and On-demand）的内存管理能力。该框架通过**灵活的内存工具**、**合成轨迹 + 课程学习**的监督微调，以及**记忆感知 GRPO** 强化学习，实现内存与问题解决能力的联合优化。
- **关键技术细节**：
    1.  **Flexible Memory Management Tool**：
        - 设计一个内存工具 `compress(analysis, start_step_number, end_step_number, content, remaining_work)`，智能体可在任意步骤主动调用。
        - `analysis`：当前进展分析；`start/end_step_number`：指定要压缩的对话轮次范围；`content`：对该范围的总结摘要；`remaining_work`：记录剩余任务以维持连续性。
        - 该操作将替换选定范围为摘要，从而实现任意时机、任意范围、任意方式的压缩。
    2.  **Curriculum Fine-Tuning with Synthesized Memory Trajectories**：
        - **轨迹合成**：运行基础模型（如 Qwen3-Coder-30B）进行任务求解，同时使用一个**专有模型**（如 GPT-5.1）作为“上下文监控器”，判断何时需要压缩（基于预算压力或主动语义判断，如子任务完成、低信息密度、焦点分散），并合成内存工具的各个参数。
        - **数据过滤**：保留成功轨迹，并针对压缩行为（如压缩率过低、范围过短、内容不匹配）和非压缩行为（如回复质量低）进行规则清洗。
        - **课程学习**：两阶段训练。阶段一学习基本工具使用和预算触发压缩；阶段二重点学习更具挑战性的“主动”内存管理行为。
    3.  **Reinforcement Learning with Memory-aware GRPO**：
        - **记忆感知轨迹分割**：在训练时，在每次内存压缩步骤处切分子轨迹，使后续步骤的训练状态与推理时看到的状态一致，避免了标准 GRPO 因长度变异性大导致的问题。
        - **步骤级信用分配（Step-level Mask）**：为克服标准 GRPO 仅依赖最终结果进行信用分配的缺点，设计掩码机制，用于屏蔽：
            - 低质量的压缩动作（如压缩率不合适）。
            - 过晚的压缩动作（低于阈值后才执行）。
            - 溢出失败的步骤（超出长度限制未调用工具）。
            - 无效工具调用。

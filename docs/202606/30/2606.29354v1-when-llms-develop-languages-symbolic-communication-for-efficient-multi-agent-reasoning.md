## 摘要
思维链（Chain-of-Thought, CoT）提升了大型语言模型（LLMs）在复杂推理任务上的表现，但其生成的长篇自然语言推理过程往往难以与高效的机器推理对齐。我们提出通信语言符号路由（Communicative Language Symbolism Routing, CLSR），一种测试时框架，其中多个LLM智能体自主发明、演化并共享紧凑的语言符号框架（Language Symbolism Frameworks, LSFs），同时一个无潜在变量的路由器根据查询自适应地选择和组合这些语言，以优化准确率与词元开销的权衡。与优化表面指令的提示优化不同，CLSR将每个LSF视为可重用的符号协议，包含紧凑符号、使用规则和消息传递契约，并通过由正确性和词元成本驱动的进化循环不断改进。在推理时，路由器可以调用单个低成本的LSF调用、集成多个LSF，或在更难的查询上执行多轮LSF组合协议。在多个具有挑战性的基准测试中，与标准CoT相比，CLSR将面向延迟的生成词元完成量减少了3~6倍，同时保持了准确率。我们进一步推导了任意符号主义下词元开销的信息论下界，并表明在解释器可实现前提条件下，多轮LSF协议有条件地包含程序执行流水线。代码已公开（https://github.com/pzqpzq/LSF_MDia）。

## Abstract
Chain-of-Thought (CoT) improves large language models (LLMs) on difficult reasoning tasks, but it often incurs long natural-language rationales that are poorly aligned with efficient machine reasoning. We propose Communicative Language Symbolism Routing (CLSR), a test-time framework in which multiple LLM agents autonomously invent, evolve, and share compact Language Symbolism Frameworks (LSFs), while a latent-free router adaptively selects and composes these languages per query to optimize the accuracy-token trade-off. Unlike prompt optimization that refines surface instructions, CLSR treats each LSF as a reusable symbolic protocol with compact symbols, usage rules, and a message-passing contract, and improves it through an evolutionary loop driven by correctness and token cost. At inference time, the router may invoke a single low-cost LSF call, ensemble multiple LSFs, or execute a multi-round LSF composition protocol on harder queries. Across challenging benchmarks, CLSR reduces latency-oriented generated token completion by $3\sim 6\times$ compared to standard CoT while maintaining accuracy. We further derive an information-theoretic lower bound on token cost under arbitrary symbolism and show that, under an interpreter-realizability premise, multi-round LSF protocols conditionally subsume program-execution pipelines. Code is publicly available (https://github.com/pzqpzq/LSF_MDia).

### 提出的方法论

#### 核心思想
- **语言符号框架（LSF）**：每个LSF是一个可重用的符号协议，包含紧凑的词表、语法规则、使用约束及消息传递契约。它不是人类事先定义的固定形式语言，而是由LLM从示例中归纳并通过进化压力选择出来的“机器导向语言”。
- **进化循环**：多个LLM智能体（黑盒实例）通过“生成→评估→选择→变异”的迭代过程不断优化LSF池。选择压力由正确性和词元成本共同驱动，保留高杠杆（正确且紧凑）的推理轨迹。
- **路由机制**：测试时，一个无潜在变量的LLM路由器根据查询自适应选择推理模式：
    1. **单LSF直接回答**：用于简单问题。
    2. **多LSF集成**：选择多个LSF独立推理后聚合（如多数投票）。
    3. **隐式LSF组合**：针对困难问题，规划多轮通信协议，不同LSF依次执行，中间状态传递。

#### 关键技术细节
- **LSF卡**：每个LSF以文本卡形式存储，包含符号清单、轻量语法、使用规则、映射函数和实证配置文件（准确率、词元统计、失败模式等）。
- **进化过程**：将训练集划分为若干组，依次执行：
    1. 在第一批上生成初始LSF池。
    2. 在下一批上评估每个LSF，记录正确性和词元成本。
    3. 选择高杠杆轨迹（正确且词元少），用于变异生成下一代LSF。
    4. 精英保留：最佳LSF直接传递到下一代。
- **路由器规划**：路由器首先生成类别代码（如ALG、PHY）缩小候选范围，再输出格式化协议计划（模式、LSF列表、轮次结构、聚合规则、停止条件）。整个规划输出仅数十个词元。

#### 公式与理论（文字说明）
- 将CLSR建模为约束随机控制问题，目标是最大化期望正确性，同时期望词元成本不超过预算B。
- **信息论下界**：对固定查询x，若策略π达到正确率α，则期望词元成本满足E[|T|] ≥ Ireq(x,δ)/κθ(x)，其中Ireq由任务不确定性决定，κθ是每个生成词元携带的条件信息量上界。该下界表明，LSF通过提升每个词元的信息密度来提高效率。
- **程序执行子sumption定理**：在“解释器可实现”假设下（即LLM能用LSF模拟任意确定性计算），多轮LSF协议能条件性地包含程序执行流水线的能力，达到相近准确率且额外词元开销仅为O(log(1/δ))。

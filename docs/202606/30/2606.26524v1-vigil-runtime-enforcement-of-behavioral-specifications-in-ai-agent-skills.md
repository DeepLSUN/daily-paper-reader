## 摘要
代理系统越来越多地通过第三方技能运作，允许模型生成的决策影响文件、通信渠道和网络物理设备。这些技能通常包含自然语言规范，定义访问权限、披露限制、执行权限和必要前提条件。尽管此类规范描述了技能行为的预期边界，但它们本身并不提供可执行的运行时执行。执行它们会引发上下文粒度挑战：即使为特定任务上下文编写了策略，监视器仍需决定观察哪些事件、保留哪些状态、在多大执行范围内推理以及在何处干预。选择错误的粒度可能会阻止良性执行，或者遗漏仅在多个操作中出现的违规。然而，大多数现有的执行机制假设一个固定的事件模型或执行点。在这项工作中，我们提出了VIGIL，一个用于代理系统的端到端运行时执行框架。VIGIL检查代理的实际执行轨迹，对照来自技能规范、操作员定义的约束以及跨多个技能的全局策略的行为策略。为了使此类策略可执行，VIGIL引入了一种策略语言，该语言捕获代理-工具事件上的上下文特定执行要求，包括时间依赖、参数约束和值流条件。该语言配以符号评估规则，将策略转化为有限轨迹上的SMT约束，使得VIGIL能够检测依赖于事件顺序、参数关系或跨调用值流的违规，而不是依赖固定的单次调用过滤器。在涵盖办公文档、操作和工程任务的真实LLM代理运行中，VIGIL以超过95%的召回率和低于10%的误报率检测策略违规。

## Abstract
Agentic systems increasingly act through third-party skills, allowing model-generated decisions to affect files, communication channels, and cyber-physical devices. These skills often include natural-language specifications that define access permissions, disclosure limits, execution privileges, and required preconditions. Although such specifications describe the intended boundaries of skill behavior, they do not by themselves provide executable runtime enforcement. Enforcing them raises a contextual granularity challenge: even when a policy is written for a particular task context, a monitor must still decide which events to observe, what state to retain, how far across the execution to reason, and where to intervene. Choosing the wrong granularity can either block benign executions or miss violations that emerge only across multiple actions. Most existing enforcement mechanisms, however, assume a fixed event model or enforcement point.   In this work, we present VIGIL, an end-to-end runtime enforcement framework for agentic systems. VIGIL checks an agent's actual execution trace against behavioral policies from skill specifications, operator-defined constraints, and global rules spanning multiple skills. To make such policies executable, VIGIL introduces a policy language that captures context-specific enforcement requirements over agent-tool events, including temporal dependencies, argument constraints, and value-flow conditions. The language is paired with symbolic evaluation rules that translate policies into SMT constraints over finite traces, allowing VIGIL to detect violations that depend on event order, argument relationships, or cross-call value flow rather than relying on fixed single-call filters. On real LLM-agent runs spanning office-document, operational, and engineering tasks, VIGIL detects policy violations with over 95% recall and a false-positive rate below 10%.

- **目标**：提出一种运行时参考监视器，能在有限执行轨迹上强制执行行为规范，检测到时序依赖、值流条件等违规，并定位违规事件。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将自然语言规范编译为可执行策略，将执行轨迹抽象为类型化事件序列，通过SMT求解检查轨迹是否满足策略，并在不满足时利用unsatisfiable core定位违规事件。
- **关键技术细节**：
  - **轨迹抽象**：将工具调用转换为带类型的事件 `e = (action, arguments map, status, output)`，构建事实库 `JτK`（包含事件间的顺序关系、参数值等），并获取签名 Σ（动作集、参数名集、值集）。
  - **策略语言**：定义六种时序模式（absence, precedence, response, bounded response, resolve, until）以及通用 `□Θ` 形式。模式中可使用变量绑定不同事件中的相同值（例如 `k = x`）。
  - **编译**：根据签名 Σ，将自然语言规范编译为策略 `P = {φ1,...,φm}`。每条语句 φ 编译为量化自由公式 `Rφ`（对每个事件 e 的条件合取）。编译使用LLM辅助但仅用于生成候选，所有接地和验证由确定性组件完成。
  - **SMT检查**：对每条 φ，检查 `unsat(JτK ∧ Rφ)`。若不可满足，则违规；unsatisfiable core给出违规事件作为定位。
- **算法流程**（如Algorithm 1所示）：
  1. 输入执行 L 和规范 D。
  2. ABSTRACT(L) → 签名 Σ 和事实库 JτK。
  3. COMPILE(D, Σ) → 策略 P。
  4. 对每条 φ ∈ P，编码为 Rφ，调用SMT求解；若 `unsat(JτK ∧ Rφ)`，记录违规 φ 和定位事件。
  5. 若没有任何违规，返回 safe；否则返回 unsafe 并伴随见证。

## 7. 优点（方法或实验设计亮点）
- **新颖的问题定义**：将 AI 代理技能行为规范执行提升到全轨迹时序层面，弥补了单次调用过滤的盲区。
- **策略语言设计**：基于有限迹安全模式的六种预设形式，可直接表达前提、响应、禁止等常见规范；通过变量绑定支持跨调用值流约束。
- **符号编译与 SMT 求解**：将策略转化为量化自由 SMT 公式，使决策可判定、可定位（unsatisfiable core 给出精确违规事件）。
- **LLM用法受限**：LLM仅用于从自然语言规范到策略候选的翻译，所有接地和验证由确定性组件完成，防止模型偏见影响判决。

## 8. 不足与局限
- **实验覆盖**：主要评估集 SB+SI 仅152条，规模偏小；跨基准 AgentDojo/SafeAgentBench 的结果统计显著性未报告。
- **偏差风险**：SB+SI 的标注是人工筛选+手工标签，可能引入主观偏差；不包含随机采样、置信区间等。
- **编译依赖LLM**：尽管限制严格，LLM 仍可能导致过度泛化（如丢失“批量”限定导致误报）或遗漏（如错误时序模式）。对于复杂规范，LLM 编译质量缺乏系统评估。
- **可观察性边界**：VIGIL 只能观察工具调用层面的事件；如果违规发生在一个内部脚本中（如 `secure_archive.py` 内部操作），则无法检测，只能通过增强沙箱弥补。
- **性能**：虽然验证阶段快，但编译阶段（4.05 s）仍依赖外部LLM服务，延迟在实时场景中可能累积；仅讨论了缓存优化，未给出实际部署缓存命中率或最坏情况。
- **可扩展性**：策略语言仅支持有限迹安全性质；未来可能需处理无限状态、复杂数值约束或概率规范。

## 摘要
现有的多智能体辩论框架存在两个关键局限：它们依赖静态架构，其中智能体角色和协调模式在设计时固定，并且它们需要实例化多个模型副本，导致大量计算开销。我们提出了辩论者混合（MoD），这是一个统一框架，通过利用专家混合范式在单个模型内实现动态自我辩论。我们解决了将MoE应用于辩证推理的三个关键挑战：（1）双路由，将角色分配与流程解耦，动态决定何时辩论与何时综合；（2）动量切换，利用局部上下文平滑令牌级路由，减少专家切换抖动；（3）统一自我辩论，将多样的辩论人格封装为轻量级专家模块，在保持行为多样性的同时消除智能体间通信。在多模态基准上的大量实验表明，MoD优于单模型基线和传统多智能体系统，实现了更高的准确性，延迟降低3.7倍，令牌消耗减少87%。源代码可在https://github.com/YongLD/MoD获取。

## Abstract
Existing multi-agent debate frameworks suffer from two critical limitations: they rely on static architectures where agent roles and coordination patterns are fixed at design time, and they require instantiating multiple model copies, incurring substantial computational overhead. We propose Mixture of Debaters (MoD), a unified framework that enables dynamic self-debate within a single model by leveraging the Mixture-of-Experts paradigm. We address three key challenges in adapting MoE for dialectical reasoning: (1) dual-routing that decouples role allocation from process flow, dynamically determining when to debate versus when to synthesize; (2) momentum switching that smooths token-level routing with local context, reducing expert-switch jitter; and (3) unified self-debate that encapsulates diverse debating personas into lightweight expert modules, eliminating inter-agent communication while preserving behavioral diversity. Extensive experiments on multimodal benchmarks demonstrate that MoD outperforms both single-model baselines and conventional multi-agent systems, achieving superior accuracy with 3.7x lower latency and 87% reduction in token consumption.The source code can be accessed at https://github.com/YongLD/MoD.

- **整体含义**：提出“Mixture of Debaters (MoD)”框架，将辩论视为**单模型内部的专家路由问题**，利用混合专家（MoE）范式在架构层面实现可学习的辩论能力。

## 2. 方法论：核心思想与关键技术

- **核心思想**：在冻结的视觉-语言骨干模型上，用轻量级 MoD 适配器（替代标准 LoRA）实现动态专家路由，使得单模型能模拟多个具有不同辩论人格的智能体进行内部自我辩论。  
- **关键技术细节**：  
  1. **双路由机制（Dual-Routing）**：  
     - 解耦**角色分配**（谁在辩论）与**流程控制**（何时辩论、何时综合）。  
     - 使用两个独立的路由器 A（解释路由）和 B（综合路由），分别选择不同的专家子集，并通过**平方根乘积**组合门控分数。  
     - 实现动态决定是否启用辩论或直接合成，适应问题复杂度。  
  2. **动量切换（Momentum Switching）**：  
     - 针对标准 MoE 令牌级路由易导致相邻 token 频繁切换专家（破坏辩论一致性）的问题，采用**滑动窗口因果平均**：  
       \[
       x_{\text{route}}^{(t)} = \frac{\sum_{k=\max(0,t-\mathcal{W})}^{t} x_k}{\min(t+1,\mathcal{W})}
       \]  
     - 默认窗口大小 \(\mathcal{W}=16\)，兼顾稳定性与响应性，并设计专用 token 缓存兼容自回归生成。  
  3. **统一高效自我辩论（Unified Self-Debate）**：  
     - 将不同辩论人格封装为轻量级专家模块（A 侧解释专家和 B 侧合成专家），每侧 8 个专家，每次激活 Top-4。  
     - 通过解耦专家池（\(A_i \in \mathbb{R}^{1\times k}, B_i \in \mathbb{R}^{1\times d}\)）实现 \(N\times N\) 组合路径，增强表达能力。  
     - 单个前向即可模拟多角色辩论，消除智能体间通信。  

- **训练数据合成（Viewpoint-Shift Data Synthesis）**：  
  - 对每个推理实例，从基模型采样 K 次，将正确/错误答案分别放入正确池和错误池。  
  - 构造三类训练数据：  
    - \(\mathcal{T}_{\text{pos}}\)：一致正确链。  
    - \(\mathcal{T}_{\text{rev}}\)：需要纠错（识别错误并修正）。  
    - \(\mathcal{T}_{\text{rob}}\)：在误导信息下坚持正确立场。  
  - 训练时将多个历史回应和辩论提示拼接作为输入，监督模型输出最终正确答案。  

- **训练目标**：  
  - 自回归语言建模损失 \(\mathcal{L}_{\text{reg}}\) + 辅助负载均衡损失 \(\mathcal{L}_{\text{aux}}\)（鼓励两侧专家均匀使用），权重 \(\lambda=0.01\)。

## 6. 主要结论与发现

1. **MoD 有效实现单模型内动态自我辩论**，在多模态推理任务上一致优于标准 MoE-LoRA 和外部多智能体辩论，同时显著降低计算开销。  
2. **双路由机制**解耦角色分配与流程控制，使模型能自适应选择何时辩论、何时综合，是性能提升的关键。  
3. **动量切换**通过滑动窗口平滑路由决策，显著提高短程专家一致性和推理连贯性，避免 token 级噪声。  
4. **专家池解耦设计**（A/B 侧独立选择）产生 \(N\times N\) 组合路径，丰富了表达多样性。  
5. 合成的**视角转换数据**（含纠错和鲁棒性样本）对引导模型学习辩证推理至关重要。  
6. MoD 在保持 12M 可学习参数（与 MoE-LoRA 相同）的情况下获得更好精度，表明增益来自架构设计而非参数量。

## 7. 方法优点

- **创新性**：首次在架构层面（MoE）将辩论内化为单模型的专家路由问题，而非外部多智能体交互。

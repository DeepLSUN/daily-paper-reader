## 摘要
LLM智能体在数字环境中遵循一个实用的执行循环：它们对结构化状态进行推理，调用工具，检查反馈，并修正行动。将这个循环扩展到物理机器人是困难的，因为物理执行是连续的、依赖具身的、不确定的，并且受安全性约束。现有的具身人工智能系统在操作、空间理解、导航和人形控制方面取得了进展，但这些能力往往仍然是专门的模块或松散耦合的决策循环。在这项工作中，我们引入了HoloAgent-0，一个用于真实世界机器人部署的统一具身智能体框架。Embodied AgentOS将语言指令转换为可执行的技能图，调度机器人资源，监控执行，并根据运行时反馈触发澄清或重新规划。HoloAgent-0通过三个耦合层组织异构的机器人模型和控制器：用于闭环执行的Embodied AgentOS、用于物理世界锚定的3D空间记忆以及用于机器人动作的具身技能。我们在真实硬件上部署HoloAgent-0，并评估其空间记忆、长时域导航以及跨运动生成、物体搜索、跨机器人协调和移动操作的闭环执行。

## Abstract
LLM agents follow a practical execution loop in digital environments: they reason over structured states, invoke tools, inspect feedback, and revise actions. Extending this loop to physical robots is difficult because physical execution is continuous, embodiment-dependent, uncertain, and constrained by safety. Existing embodied-AI systems have advanced manipulation, spatial understanding, navigation, and humanoid control, but these capabilities often remain specialized modules or loosely coupled decision loops. In this work, we introduce HoloAgent-0, a unified embodied agent framework for real-world robot deployment. Embodied AgentOS converts language instructions into executable skill graphs, schedules robot resources, monitors execution, and triggers clarification or re-planning from runtime feedback. HoloAgent-0 organizes heterogeneous robot models and controllers through three coupled layers: Embodied AgentOS for closed-loop execution, 3D spatial memory for physical world grounding, and embodied skills for robot action. We deploy HoloAgent-0 on real hardware and evaluate its spatial memory, long-horizon navigation, and closed-loop execution across motion generation, object search, cross-robot coordination, and mobile manipulation.

## 2. 方法论

### 2.1 核心思想
- 将长时域具身任务执行视为**系统组织问题**，而非单一模型推理问题。
- 提出**Embodied AgentOS**：一个闭环运行时，负责任务规划、技能调度、执行监控与重新规划。
- 引入**类型化技能接口**（typed skill calls）和**3D空间记忆**，使异构技能可通过统一的ROS2命令/状态总线组合与监控。
- 遵循四个设计原则：闭环优先（closed-loop first）、记忆中心（memory-centric）、类型化技能接口（typed skill interface）、默认可观测（observable by default）。

### 2.2 关键技术细节

#### (a) Embodied AgentOS层
- **输入**：自然语言指令 → **输出**：可执行的技能图（skill graph）。
- **处理流程**：解析指令 → 检索任务历史与空间记忆 → 分解为技能节点，包含顺序、前提条件、恢复依赖 → 调度就绪技能调用 → 消费状态反馈 → 触发澄清或重新规划。
- 通过ROS2命令/状态总线连接记忆层、技能层与监控验证层。

#### (b) 记忆层
- **空间记忆**：
  - 度量几何：支持LiDAR后端（FAST-LIVO）或纯视觉后端（GeoFlow-SLAM++，多相机深度预测）。
  - 开放词汇3D语义映射：使用SAM2生成2D掩码，计算三个SigLIP描述子（全帧、掩码段、最小边界框），通过加权融合得到实例特征。实例关联：将新2D观测与已有3D实例通过IoU匹配，未匹配的初始化新实例。
  - **分层多模态场景图（HMSG）**：包含楼层、房间、视角、物体四层，支持粗到细的检索（如先查楼层→房间→视角→物体），减少搜索空间。
- **时间记忆**：记录目标与计划状态、执行与恢复轨迹、结果摘要，用于长时域任务恢复与经验积累。

#### (c) 技能层
- **类型化技能调用**：每个技能声明命令名、类型化参数、前提条件、预期效果。状态接口返回进度、成功/失败模式、置信度、可恢复性等。
- **主要技能家族**：
  - **HoloNavi**（导航/探索）：层次化物体导航（先基于HMSG快速匹配，再通过VLM慢速验证）；若验证失败则启动主动探索（基于信息增益的边界点评分）。
  - **HoloBrain**（操作VLA后端）：支持pick/place/fold等，输出状态包括物体未找到、抓取失败等。
  - **HoloMotion**（全身运动）：支持参考轨迹跟踪与速度控制，用于人形机器人行走、舞蹈等。
  - **跨机器人协调**：通过共享记忆与技能调用，不同机器人可协同完成同一任务。

### 2.3 算法流程（文字说明）
1. 用户输入语言指令。
2. AgentOS层检索空间记忆（HMSG）和时间记忆（任务历史）获取上下文。
3. 分解指令为技能图，调度待执行技能。
4. 技能层通过ROS2命令总线执行技能（如导航、操作），状态总线实时返回进度与失败信息。
5. 监控与验证层评估技能结果，若失败则触发重新规划或更新记忆。
6. 记忆层更新空间记录（新观测、物体关联、动态场景更新）和时间轨迹。
7. 循环直到任务完成或不可恢复。

## 6. 主要结论与发现
1. **AgentOS循环提升导航性能**：HoloAgent-Nav在仿真HM3D-ObjNav上取得SR 82.6%（比最佳基线MSGNav高8.5%），SPL 42.8%（比最佳FSR-VLN高1.8%），说明反馈驱动执行在不牺牲路径效率的情况下提高了成功率。

3. **语义映射竞争力**：HoloAgent-Memory在ScanNet上频率加权准确率最高（61.58%），在Replica上排名中上，表明在线操作下仍能提供有效的语言锚定空间记忆。
4. **闭环执行定性可行**：能够通过统一接口组合导航、操作、人形运动和跨机器人协调，展示长时域任务（如叠衣服）的分解与执行。

## 7. 优点
- **系统完整性**：首次提出从语言理解到技能执行、记忆维护、监控恢复的完整闭环框架，且已在真实异构机器人上验证。
- **设计模块化**：通过ROS2命令/状态总线实现解耦，各技能后端可独立替换，便于后续集成新模型（如更先进的VLA、视觉SLAM）。
- **记忆设计合理**：HMSG的分层检索机制（楼层→房间→视角→物体）有效减少搜索空间，结合快速匹配与慢速VLM验证，兼顾效率与精度。

- **实验双轨制**：仿真+真实机器人评估，既有可控定量指标又有定性演示，增强可复现性与说服力。

## 8. 不足与局限
- **缺乏端到端量化基准**：操作、人形运动、跨机器人协调等仅靠定性演示，无标准成功率或效率指标，可能引入选择偏差（只展示成功案例）。

- **算力资源未公开**：无法评估方法的计算效率和可复现性，特别是VLM推理和语义映射的实时性。

- **动态场景适应性有限**：记忆更新仅通过视觉观测和技能结果触发，对长期环境变化（如家具移动）的鲁棒性未充分测试。
- **人形机器人技能局限性**：目前HoloMotion仅支持参考跟踪和速度控制，尚缺乏自主平衡、摔倒恢复等复杂能力，闭环执行依赖环境安全假设。

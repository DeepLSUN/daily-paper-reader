## 摘要
长上下文大语言模型推理受到大规模键值（KV）缓存的严重瓶颈制约，然而现有的稀疏注意力方法通常受限于静态固定预算（Top-k）检索，或依赖于计算成本高且存在偏差的代理分数。为解决这些限制，我们提出RaBitQCache，一种新型稀疏注意力框架，利用随机旋转二进制量化和高吞吐量的二进制-INT4算术高效估算注意力权重。我们的代理分数作为无偏估计量，具有经证明的误差界，支持自适应Top-p检索，根据实际注意力稀疏性动态调整token预算。我们进一步实现了一个硬件感知系统，采用异步流水线和惰性更新来掩盖开销。评估表明，与最先进的基线相比，RaBitQCache在保持生成质量的同时显著加速推理并减少内存I/O。代码可访问https://github.com/Sakuraaa0/RaBitQCache.git。

## Abstract
Long-context Large Language Model inference is severely bottlenecked by the massive Key-Value (KV) cache, yet existing sparse attention methods often suffer from static fixed-budget (Top-k) retrieval or rely on proxy scores that are computationally expensive and biased. To address these limitations, we propose RaBitQCache, a novel sparse attention framework that utilizes randomized rotated binary quantization and high-throughput binary-INT4 arithmetic to efficiently estimate attention weights. Our proxy score serves as an unbiased estimator with a proven error bound, enabling adaptive Top-p retrieval that dynamically adjusts the token budget based on actual attention sparsity. We further implement a hardware-aware system with asynchronous pipelining and lazy updates to mask overhead. Evaluations demonstrate that RaBitQCache significantly accelerates inference and reduces memory I/O while preserving generation quality compared to state-of-the-art baselines. Code is available at https://github.com/Sakuraaa0/RaBitQCache.git.

## 1. 核心问题与整体含义（研究动机和背景）

长上下文大语言模型（LLM）推理面临严重的 KV 缓存瓶颈：随着序列长度增长至 128K 甚至更多，存储和访问大量键值对导致巨大的 GPU 内存消耗和计算延迟。现有稀疏注意力方法试图利用注意力稀疏性来加速推理，主要分为两类：一是静态固定预算（Top-k）检索，缺乏对输入依赖注意模式的适应性；二是动态代理分数方法，但存在代理分数不准确（如 Quest 按页选择导致无关 token 较多、DS 部分维度估计易偏离真实分布）且缺乏理论误差界，因此大多只能支持固定预算 Top-k 而无法实现自适应 Top-p 检索。本文提出 RaBitQCache，旨在同时满足高效、高精度（带理论误差界）和通用性三个核心需求，实现稀疏注意力加速。

## 2. 方法论：核心思想、关键技术细节

**核心思想**：利用 Johnson-Lindenstrauss 引理，通过随机旋转二进制量化将高维 Key 向量压缩为二进制向量，并与 INT4 量化的 Query 向量进行高效内积计算，从而获得无偏且具有理论误差界的代理分数，实现自适应 Top-p 检索。

**关键技术细节**：
- **内积分解**：将原始内积 ⟨q, k⟩ 分解为归一化后向量的内积 ⟨qc, kc⟩ 加上可预计算的常数项，使核心瓶颈落在余弦相似度估计上。
- **随机旋转与二进制量化**：对归一化 Key 向量 kc 施加随机正交矩阵 P（模型初始化时生成），然后取其符号位得到二进制码本向量 ¯c ∈ {±1/√D}，仅用 1-bit 存储（¯Cb）。同时为每个 token 存储一个标量校正因子 α = ⟨¯c, PT kc⟩ 保证无偏性。
- **Query INT4 量化**：对旋转后的 Query 向量 q′ 进行均匀标量量化得到 4-bit 整数向量 ¯qu，引入的量化误差被证明可控制在 O(1/√D) 量级。
- **高效分数估计**：计算 ⟨¯cb, ¯qu⟩（二元-INT4 点积），再通过公式重建近似内积，除以 α 得到无偏估计量。该估计量被证明是余弦相似度的无偏估计，误差界为 O(1/√D)。
- **自适应 Top-p 检索**：基于代理分数计算 softmax 概率，采用线性时间三分搜索算法（基于 FlashInfer 实现）找到覆盖概率质量 p 的最小 token 集，避免排序的 O(L log L) 开销。
- **系统调度优化**：
  - 异步预填充：在预填充阶段，主 CUDA 流执行密集注意力计算（O(L²)），同时低优先级流同步构建二进制索引（O(L)），索引延迟被完全掩盖。
  - 惰性解码更新：新增 token 暂存在全精度滑动窗口缓冲区中，达到批量容量后再一起量化，减少内核启动开销并保证局部上下文的高精度访问。

## 6. 主要结论与发现

- RaBitQCache 在 LongBench 上以约 17-21% 的 token 预算达到与 Full Attention 相当的生成质量（50.63 vs 50.58 on LLaMA-3.1-8B），注意力召回率达 90% 以上。
- 相比固定预算 Top-k 方法（Quest、DS），RaBitQCache 自适应调整预算（如 NarrativeQA 需更多 token，MultiNews 需更少 token），灵活应对不同稀疏模式。

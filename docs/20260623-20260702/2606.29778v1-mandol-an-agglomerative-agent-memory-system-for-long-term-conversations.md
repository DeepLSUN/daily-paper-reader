## 摘要
长期对话智能体需要记忆和查询跨会话、多类型且具有复杂关联的信息。现有的智能体记忆系统依赖异构的向量和图数据库，导致记忆信息碎片化，并造成跨数据库I/O延迟高。在检索方面，常见的RAG风格方法容易引入噪声、遗漏相关线索，且缺乏令牌预算控制，降低了LLM的准确性和效率。我们提出Mandol，一种凝聚式记忆系统，将碎片化的记忆表示和存储整合为统一的记忆原生架构。其核心组件包括：（1）层次化记忆模型，将记忆组织成表示原始记忆信息的基础层和将基础记忆凝聚为可追溯抽象记忆的高级抽象层，两者统一表示为结构化语义图；（2）凝聚式语义数据结构，结合SemanticMap和SemanticGraph，原生融合键值、向量和图结构，并提供统一的混合检索算子以消除跨数据库I/O；（3）定量查询机制，包括查询自适应路由、定量去噪与冲突解决以及令牌约束的上下文生成，在检索过程中无需涉及LLM。在两个广泛使用的长期对话基准LoCoMo和LongMemEval上的实验表明，Mandol在代表性智能体记忆系统中取得了最佳整体准确率。性能对比方面，在10 QPS并发负载下，Mandol还实现了5.4倍的检索加速和4.8倍的插入加速，同时在消费级硬件上仍保持低延迟。

## Abstract
Long-term conversational agents need to remember and query cross-session, multi-typed information with complex correlations. Existing agent memory systems rely on heterogeneous vector and graph databases, which fragment memory information and cause high cross-database I/O latency. For retrieval, common RAG-style methods tend to introduce noise, miss correlated clues, and lack token budget control, degrading LLM accuracy and efficiency.   We propose Mandol, an agglomerative memory system that consolidates fragmented memory representations and storage into a unified memory-native architecture. Its core components include: (1) a hierarchical memory model that organizes memory into a basic layer representing raw memory information and a high-level abstract layer that agglomerates basic memories into traceable abstract memories, both uniformly represented as structured semantic graphs; (2) an agglomerative semantic data structure combining SemanticMap and SemanticGraph, which natively fuses key-value, vector, and graph structures and provides unified hybrid retrieval operators to eliminate cross-database I/O; and (3) a quantitative query mechanism with query-adaptive routing, quantitative denoising and conflict resolution, and token-constrained context generation, all without involving LLMs during retrieval. Experiments on two widely used long-term conversation benchmarks, LoCoMo and LongMemEval, show that Mandol achieves the best overall accuracy among representative agent memory systems. For performance comparison, Mandol also obtains a 5.4x retrieval speedup and a 4.8x insertion speedup under 10 QPS concurrent load, while still maintaining low latency on consumer-grade hardware.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：长期对话智能体需要管理跨会话、多类型、动态演变的记忆（如对话历史、用户偏好、事件状态等），现有记忆系统（如Mem0、Zep、MemOS）普遍采用异构向量和图数据库组合，导致记忆信息碎片化、跨数据库I/O延迟高。在检索环节，传统RAG方法依赖相似性匹配，易引入噪声、遗漏关联线索，且缺乏令牌预算控制，造成LLM准确率和效率下降。
- **核心挑战**：①统一记忆表示（向量无法表达逻辑结构，知识图难以捕捉语义相似性）；②高效存储与混合查询（异构数据库带来高序列化开销）；③精准检索与令牌预算约束（RAG检索结果不完整、冲突、冗余）。
- **论文目标**：设计一个**凝聚式（Agglomerative）记忆系统Mandol**，将碎片化表示与存储整合为统一的内存原生架构，同时提升记忆检索质量与系统性能。

## 2. 方法论：核心思想与关键技术细节

### 核心思想
Mandol采用**层次化记忆模型 + 凝聚式语义数据结构 + 定量查询机制**，将基础记忆和抽象记忆统一表示为结构化语义图，并实现不依赖LLM的精准、高效检索。

### 关键技术细节

**（1）层次化记忆模型（Hierarchical Memory Model）**  
- **基础记忆层（Basic Memory Layer）**：将原始交互信息分解为记忆单元（Memory Unit，含文本、语义向量、元数据）、记忆空间（Memory Space，按会话/任务/主题逻辑隔离）以及显式/隐式关系（显式：时间顺序、实体引用；隐式：语义相似度）。整体构成结构化语义图。
- **高层抽象记忆层（High-Level Abstract Memory Layer）**：利用LLM自动从基础记忆中抽取并凝聚出三种抽象记忆——情节记忆（事件链）、语义记忆（实体图）、情感记忆（用户偏好变化路径）。每个抽象节点保留指向原始基础单元的可追溯链接（`source_unit_uids`），保证证据可回溯。
- **查询层**：执行查询自适应路由与定量检索。

**（2）凝聚式语义数据结构（Agglomerative Semantic Data Structure）**  
- **SemanticMap**：结合键值存储与向量索引，以记忆单元ID为键，存储原始文本、元数据及语义向量，同时支持倒排索引、稠密/稀疏向量索引。
- **SemanticGraph**：轻量级邻接表存储显式结构边，隐式关系在查询时通过SemanticMap索引实时计算。
- **统一视图**：两者共享同一套ID，所有混合检索操作（如向量搜索+图遍历+空间过滤）均在同一地址空间内完成，消除跨数据库I/O。
- **持久化**：底层通过嵌入的DuckDB实现冷存储，采用LRU/LFU策略进行页面换入换出。

**（3）定量查询机制（Quantitative Query Mechanism）**  
- **查询自适应路由（Query-Adaptive Routing）**：轻量级意图分类器根据查询特征（如时间表达式、实体提及）选择相关记忆源（基础、情节、语义、情感），并分配候选预算。在每个源内并行执行BM25关键词检索、SPLADE稀疏扩展、稠密向量搜索，通过倒排排名融合（RRF）合并候选集，再经图子图扩展和交叉编码器重排序得到语义相关性得分\( S_{ce}(c_i) \)。
- **定量去噪与冲突解决**：  
  - 源内去噪：基于**中位数绝对偏差（MAD）** 的动态阈值滤波，鲁棒性强于均值和标准差方法：
    \[
    \tau_{mad} = \text{median}(S_{ce}) - \kappa \cdot \text{MAD}(S_{ce}), \quad \text{MAD}(S_{ce}) = \text{median}(|S_{ce}(c_i) - \text{median}(S_{ce})|)
    \]
    低于阈值\(\tau_{mad}\)的候选被丢弃。
  - 跨源冲突解决：对涉及相同实体但描述矛盾的候选，计算**仲裁分数**\( S_{arb} \)综合平衡语义相关性\( S_{ce} \)、时间新鲜度\( f_{time} \)（指数衰减）和源置信度\( f_{source} \)（基础1.0，情节0.8，语义0.6），选最高分者保留。
- **令牌约束的上下文生成**：采用**最大边际相关性（MMR）** 贪心选择，联合优化相关性与多样性：
  \[
  \text{MMR}(c_i|S) = \lambda \cdot \text{Rel}(c_i) - (1-\lambda)\cdot \text{Red}(c_i, S)
  \]
  其中\(\text{Rel}\)为交叉编码器得分，\(\text{Red}\)基于实体和记忆源重叠度计算冗余度。最终在令牌预算内生成紧凑高质量上下文。

## 7. 优点

- **统一表示消除碎片化**：层次化语义图和凝聚式数据结构将键值、向量、图融合在同一内存地址空间，从根本上消除了跨数据库I/O开销。
- **低延迟高性能**：所有检索和更新操作均在进程内完成，无需外部数据库序列化，因此延迟极低。
- **定量检索不依赖LLM**：路由、去噪、冲突解决和上下文生成均基于非LLM的算法（MAD、仲裁评分、MMR），在提升准确率的同时节省推理成本。
- **令牌高效**：通过MMR多样性选择和令牌约束，在同等或更少令牌下获得更高准确率，尤其在复杂多跳查询上表现突出。
- **可追溯性**：抽象记忆节点保留到基础单元的链接，支持证据验证和消歧。

## 8. 不足与局限

- **个别指标略逊**：在LoCoMo的开放域问题和时间推理子类型上，Mandol略微低于EverMemOS（后者使用更大的4B嵌入和重排序模型）。
- **消融分析缺乏**：论文未设计系统性消融实验来量化每个组件（如MAD去噪、冲突解决、MMR等）的独立贡献，使归因不够严谨。
- **实验覆盖有限**：仅在两个中等规模基准上验证，未覆盖更长时段（如数百轮对话）或更大规模记忆规模；不同领域（如医疗、推荐）的通用性未测试。
- **依赖LLM做抽象**：虽然检索过程不依赖LLM，但抽象记忆的构造仍需调用LLM，这会引入额外成本，并且可能受LLM准确性影响。

- **无多用户并发评估**：虽然测试了QPS，但未涉及多用户隔离场景下的竞争和扩展性。

## 摘要
大型语言模型越来越多地处理长上下文场景，此时KV缓存成为主要内存瓶颈：其大小随序列长度线性增长，且必须在解码过程中全程保留，若不压缩，完全在GPU上缓存将代价高昂。现有KV缓存压缩方法难以在效率和忠实保留上下文之间取得平衡。Token驱逐会丢弃信息，而语义分组在预填充阶段固定压缩决策；两者都无法在生成过程中从压缩片段中恢复token级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，将上下文组织为熵引导的语义片段，并在GPU-CPU内存层次结构中存储它们，不丢弃信息。每个片段在GPU上保留轻量级摘要向量用于粗粒度路由，在CPU上保留低秩SVD基用于按需token级重建。训练得到的缩放机制在解码过程中选择性扩展与查询相关的片段，实现精确检索，而无需在GPU上实例化完整KV缓存。SeKV实现了自适应的token级重建，同时保持基础LLM完全冻结，并添加少于0.05%的可训练参数。在四个基准测试中，与最强的语义压缩基线相比，SeKV平均提升5.9%，同时与128K上下文下的全KV缓存相比，GPU内存减少53.3%。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将KV缓存从扁平token序列重构为分层语义记忆。上下文首先被熵引导的边界分割成语义片段，每个片段在GPU上保留轻量摘要向量用于粗粒度路由，在CPU上存储低秩SVD基用于按需token级重建。解码时，训练得到的zoom-in机制动态选择与当前查询相关的片段，异步获取其SVD基并重建token级KV对。
- **关键技术细节**：
  - **熵引导片段分割**：利用预填充阶段token的surprisal值（$H_t = -\log p(x_t|x_{<t})$）作为边界信号，当$H_t > \mu + \alpha\sigma$时标记为边界token（锚点token），保留在GPU上；两个锚点间的token构成一个语义片段。
  - **双分辨率表示**：
    - **GPU摘要**：对每个片段的token key向量，通过可学习的投影$W^{(l,h)}$降维至$d'=32$，并用surprisal加权平均生成摘要$\bar{K}^{(l,h)}_S$。同时保留原始空间中的加权平均key和value（$\bar{k}, \bar{v}$）用于粗粒度注意力。
    - **CPU低秩SVD基**：对每个片段的key和value矩阵进行截断SVD（最大秩$R=32$），通过一个轻量预测器$g_\phi$为每个奇异分量分配软门控（$m_i$），实现自适应秩；有效秩$r_S = \sum m_i$，信息密集的片段保留更多分量。
  - **训练的zoom-in机制**：
    - 计算查询与摘要的sigmoid相关性得分$\tilde{\alpha}^{(l,h)}_S$（含log|S|长度先验）。
    - 每个head-layer有独立学习阈值$\tau^{(l,h)}$，决定片段是否扩展：$z^{(l,h)}_S = \mathbb{1}[\tilde{\alpha}^{(l,h)}_S > \tau^{(l,h)}]$。
    - 扩展的片段从CPU获取SVD基，重建token级KV对；非扩展片段贡献一个粗粒度条目（$\bar{k}, \bar{v}$）。
    - 训练时使用直通估计器处理二值决策，损失函数包含蒸馏损失（对齐全KV教师）、zoom监督（基于教师注意力覆盖的BCE）、重建损失（控制SVD门控）和预算损失（惩罚扩展片段数和保留秩数）。
  - **可训练参数**：仅训练投影矩阵、阈值和秩预测器，总参数约4.3M（占LIama-3-8B的0.05%），基础模型完全冻结。

## 8. 不足与局限

- **依赖语义分割质量**：若文档结构高度碎片化（如密集表格、代码、非标准格式），熵引导的分割可能产生噪声边界，导致缓存放置次优。
- **超参数敏感**：分割阈值α、最大SVD秩R、损失权重等需要针对新模型或上下文长度进行验证，调参负担存在。
- **CPU-GPU带宽限制**：对于频繁触发大量远程span扩展的对抗性查询，CPU→GPU传输可能成为延迟瓶颈。
- **模型适应性受限**：保持基础模型冻结限制了其内部注意力行为对压缩结构的完全适应。联合训练路由与更灵活span重建可能是未来方向。
- **实验不足**：未在更大模型（如70B+）或更极端上下文（>128K）上验证；未报告统计显著性；训练数据与评估基准无关但未说明完全避免数据泄露。
- **应用限制**：需要预计算SVD和训练路由组件（数小时），不适合零成本压缩场景；实际部署时需管理CPU内存和异步传输。

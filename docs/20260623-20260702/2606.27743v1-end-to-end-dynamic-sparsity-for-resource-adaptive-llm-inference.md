## 摘要
大语言模型(LLMs)推理通常在静态资源假设下部署，即模型无论运行时环境如何都执行固定的计算图。然而，现实世界的云基础设施本质上是动态的，其特点是可用性波动（例如竞价实例抢占）和分层服务质量要求。在这种不稳定的环境中，静态模型缺乏灵活性：它们要么在资源约束下崩溃，要么在冗余操作上浪费计算。为弥补这一差距，我们提出学习分配（L2A），一种用于资源自适应推理的端到端框架。与仅取决于输入难度的先前方法不同，我们将推理建模为同时取决于输入和运行时资源预算本身的约束分配问题。我们引入了轻量级的、基于预算条件和输入感知的门控网络，并将其集成到LLM中。这些门控通过一个统一目标进行训练，该目标沿着现实世界动态表现出的三个轴联合优化任务性能、逻辑一致性和资源成本：用于内存和深度压力的层跳过、用于吞吐量争用的头部剪枝以及用于时延紧缩的推理令牌减少。这使得模型能够学习超越输入难度的预算感知策略：它根据实时资源动态自适应地配置其计算足迹，在资源允许时最大化推理深度，在预算紧张时强制严格节俭。单个L2A模型在Llama-3-8B和Qwen-3-4B上绘制出整个计算-准确率帕累托前沿：在高达34%的实用层稀疏度下，其在GSM8K上与稠密基线的差距保持在0.6%以内，在分布外任务上零样本时相同差距保持不变，而每个静态或启发式基线都需要单独调优模型，并在可比的推理时间下仍下降5-10%。

## Abstract
Large Language Models (LLMs) inference is typically deployed under a static resource assumption, where models execute a fixed computational graph regardless of the runtime environment. However, real-world cloud infrastructure is inherently dynamic, characterized by fluctuating availability (e.g., spot instance preemption) and tiered Quality-of-Service requirements. In such volatile settings, static models are inflexible: they either crash under resource constraints or waste compute on redundant operations. To bridge this gap, we propose Learning to Allocate (L2A), an end-to-end framework for resource-adaptive inference. Unlike prior methods that condition only on input difficulty, we formulate inference as a constrained allocation problem conditioned on both the input and the runtime resource budget itself. We introduce lightweight, budget-conditioned and input-aware gating networks integrated into the LLM. These gates are trained via a unified objective that jointly optimizes task performance, logical consistency, and resource costs along three axes matching how real-world dynamics manifest: layer skipping for memory and depth pressure, head pruning for throughput contention, and reasoning-token reduction for latency tightening. This lets the model learn a budget-aware policy beyond input difficulty alone: it adaptively configures its computational footprint with respect to real-time resource dynamics, maximizing reasoning depth when resources permit while enforcing strict frugality when budgets tighten. A single L2A model traces the entire compute-accuracy Pareto frontier on Llama-3-8B and Qwen-3-4B: at up to 34% realized layer sparsity, it stays within 0.6% of the dense baseline on GSM8K, with the same gap holding zero-shot on out-of-distribution tasks, while every static or heuristic baseline requires a separately tuned model and still drops by 5-10% at comparable inference time.

# 面向资源自适应的LLM推理的端到端动态稀疏性

## 1. 核心问题与整体含义
- **研究动机**：现有LLM推理通常假设静态计算图，模型无论输入难度和运行时资源状况都执行固定计算。但现实云环境中GPU内存、时延、竞价实例抢占等资源动态波动，静态模型要么因资源不足而崩溃，要么浪费计算。亟需一种能根据输入和实时资源预算动态调整计算量的方法。
- **整体意义**：提出L2A框架，将推理建模为端到端约束分配问题，使单个模型能覆盖整个计算-精度帕累托前沿，适应资源波动和服务质量差异化。

## 2. 方法论
- **核心思想**：在冻结的LLM中插入轻量级预算条件门控网络，同时使用LoRA进行参数高效微调。门控网络接收输入隐藏状态和标量预算信号b（b∈[0,1]），输出三种门控值：
  - **层跳过门**α_l：决定是否跳过整个Transformer层。
  - **头剪枝门**β_{l,h}：决定是否跳过某个注意力头。
  - **推理-答案转换门**τ_t：决定何时从<think>段切换到<answer>段，缩短推理token数。
- **关键技术细节**：
  - 预算信号b通过可学习嵌入e(b)注入门控网络；训练时从分布D中采样b，推理时通过实时系统指标（时延、内存、队列）校准得到b。
  - 训练时使用软门控（sigmoid输出），推理时使用硬门控（如0.5阈值二值化）。
  - 统一损失函数：  
    L(b) = L_ce + γ·L_kd + λ·(C(b) - B(b)) + λ_T·R_tok  
    - L_ce：交叉熵损失，保持生成能力。  
    - L_kd：KL散度蒸馏，防止稀疏执行时模型漂移。  
    - C(b)-B(b)：预算惩罚项，使实际计算量C(b)接近目标预算B(b)。  
    - R_tok：推理token长度惩罚，促使早期转换到answer段。
- **训练流程**：端到端联合优化LoRA参数和门控网络参数，每个样本随机采样一个预算b进行条件化。训练后单一模型即可在不同预算下适应。

## 6. 主要结论与发现
- 单个L2A模型在Llama-3-8B和Qwen-3-4B上能覆盖整个计算-精度帕累托前沿，在34%层稀疏度下GSM8K精度仅降0.6%（从36.5%到35.9%），且零样本泛化到分布外任务（HumanEval保持近12.2%，基线降至5.2%）。
- 静态或启发式基线（AdaSkip、FlexiDepth）在相同推理时间下精度下降5-10%。
- 跨域训练能使门控网络学到输入难度感知，而非过拟合特定领域（混合训练在困难任务上保守，在简单任务上高效稀疏）。
- 蒸馏（L_kd）和预算惩罚（λ项）至关重要：去除蒸馏导致GSM8K从35.8%暴跌至14.4%；去除预算惩罚导致模型几乎不稀疏。
- 联合层跳过+头剪枝+推理token缩短比单独层跳过更优。

## 7. 优点
- **端到端统一框架**：将动态稀疏性（层、头、推理长度）训练整合为单目标，无需分阶段优化或手工启发式。
- **预算条件化**：门控网络同时接受输入状态和外部预算信号，使一个模型可在不同资源水平下自适应运行，无需多个压缩模型。
- **单模型覆盖整个帕累托前沿**：显著降低了部署和维护成本（静态方法需要为每个预算点单独压缩微调）。
- **跨任务泛化能力强**：在分布外任务上误差仅0.6%，而基线退化严重，说明门控学到了通用难度特征而非过拟合。
- **参数高效**：使用LoRA，仅更新少量参数（秩16），冻结原始权重，保留预训练知识。
- **推理兼容性**：硬门控决策（层跳过、头剪枝）无需专用硬件，可直接减少FLOPs。

## 8. 不足与局限
- **头剪枝的加速依赖实现**：实际中注意力头剪枝的墙钟加速效果不如层跳过显著，论文中主要加速来自层跳过和推理长度控制。
- **预算校准依赖估计器**：需要在线时延/内存估计器，估计误差可能导致过高或过低的实际预算，影响效果。
- **结构化格式限制**：需要统一使用 `<think>...</think><answer>...</answer>` 格式，不适用于所有部署场景（如纯生成任务无显式推理段）。
- **缺乏大规模模型验证**：仅在8B和4B模型上实验，未在13B、70B或更大模型上验证，能否有效扩展未知。
- **未报告训练成本**：无GPU型号、数量、训练时间，难以评估实用性。
- **未对比更先进的动态方法**：如Mixture-of-Depths（MoD）、SkipGPT等，仅对比了AdaSkip和FlexiDepth。
- **统计显著性缺失**：未报告多次重复实验的标准差或置信区间，可能受随机种子影响。
- **仅支持单卡推理**：未讨论多GPU或分布式场景下的动态稀疏性对齐问题。

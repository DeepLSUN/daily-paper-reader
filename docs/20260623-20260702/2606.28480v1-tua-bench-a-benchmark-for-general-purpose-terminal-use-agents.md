## 摘要
随着大语言模型和编排框架的不断进步，在终端中运行的代理能够执行越来越广泛的通用计算机使用任务，而不仅仅是编码。然而，现有基准测试未能充分评估通用终端计算机使用代理（TUA）：通用计算机使用基准测试主要针对图形用户界面（GUI），而基于终端的基准测试则主要强调历史上原生用于 Shell 的技术和编程中心化工作流。我们引入了 TUA-Bench，一个面向终端使用代理的通用基准测试。TUA-Bench 包含跨五个任务族的 120 个真实世界任务，覆盖日常数字活动——包括文档编辑、电子邮件管理和实时网页信息检索——以及与博士级领域专家共同设计的、需要专业软件的科学与工程工作流。这种广度使 TUA-Bench 区别于此前专注于 Shell 或特定领域的基准测试。每个任务均手工设计，在具有确定性设置脚本的真实终端中运行，并通过基于执行的评分协议进行评估。我们发现，最强的前沿代理——Claude Code（搭配 Claude Opus 4.8 最大推理努力）——取得了 65.8% 的整体性能，且两条轨道间存在显著差距。通过对终端使用能力进行广泛且真实的评估，TUA-Bench 旨在加速从狭窄、任务特定的助手向能够在多样化数字环境中可靠运行的通用代理的转变。

## Abstract
As large language models and harness frameworks continue to advance, agents operating in terminals are increasingly capable of performing a broader range of general computer-use tasks beyond coding. However, existing benchmarks do not adequately evaluate general-purpose terminal computer-use agents (TUAs): general computer-use benchmarks primarily target graphical user interfaces (GUIs), whereas terminal-based benchmarks largely emphasize technical and programming-centric workflows historically native to the shell. We introduce TUA-Bench, a general-purpose benchmark for terminal-use agents. TUA-Bench includes 120 real-world tasks across five task families, covering routine digital activities-including document editing, email management, and live-web information seeking-as well as scientific and engineering workflows co-designed with PhD-level domain experts that require specialized software. This breadth distinguishes TUA-Bench from prior shell-focused or domain-specific benchmarks. Each task is manually designed, runs in a real terminal with a deterministic setup script, and is evaluated by an execution-based scoring protocol. We find that the strongest frontier agent, Claude Code with Claude Opus 4.8 max reasoning effort, achieves 65.8% overall performance, with substantial gaps across both tracks. By providing a broad and realistic evaluation of terminal-use capabilities, TUA-Bench aims to accelerate the transition from narrow, task-specific assistants to general-purpose agents capable of operating reliably across diverse digital environments.

## 6. 主要结论与发现

1. **最强配置**：Claude Code + Claude Opus 4.8（max 努力）达 **65.8%**，Codex + GPT-5.5 紧随其后（64.7%），整体性能尚有很大提升空间。
2. **模型间差距**：前三个前沿模型（GPT-5.5 60.1%、Opus 4.8 59.7%、Opus 4.7 58.0%）远高于中等模型（~44-49%），有效区分能力等级。
3. **时间预算重要性**：限制时间显著影响成功率，从 150s 的 33.0% 提升到 2400s 的 60.1%，说明很多失败源于超时而非根本能力。
4. **思考努力缩放**：成功率和 token 消耗正相关，但边际收益递减；medium-high 提供最佳性价比。
5. **代理-模型依赖**：模型性能因代理而异，例如 GPT-5.5 在 Mini-SWE-Agent 上领先 Claude Opus 4.8（+5.0%），但在 OpenHands 上落后（-2.0%），选择代理与选择模型同等重要。
6. **类别差异**：System & SW 类别普遍易解（高成功率），Office 和 Multimedia 最难（多数模型低于 45%）；无模型在所有类别占优，GPT-5.5 最一致，Opus 4.8 在 Web & Info 上领先。
7. **成本-性能帕累托前沿**：开源模型（如 MiniMax-M3、GLM-5.1）在低成本下达到 47-48% 成功率；Claude Code 最高但成本也高（$173.61/run）；Codex + GPT-5.5 在 $138/run 达到 64.7%，性价比较好。

## 8. 不足与局限

### 实验覆盖
- **专业领域有限**：仅覆盖 4 个科学工程领域，每个领域任务数较少（5-8 个），可能不足以代表该领域全貌。
- **语言限制**：所有任务指令为英文，未评估多语言或非英语场景。
- **工具版本固定**：容器内工具版本固定，随着软件更新可能需要维护，影响长期可复现性。

### 偏差风险
- **任务来源偏差**：日常任务全部源自 OSWorld（一个 GUI 基准），虽然转换为 CLI 任务，但任务本身可能偏向 GUI 环境下的常见操作（如编辑表格、设置浏览器），可能遗漏一些纯 CLI 原生任务（如系统管理、网络配置）。
- **难度筛选引入偏差**：通过模型表现筛选最难任务，可能保留了那些由于特定工具兼容性、软件缺陷等非能力因素导致难的任务。

### 应用限制
- **不覆盖所有计算机交互**：某些应用仍缺乏成熟 CLI 或 headless 支持，不能完全代表计算机使用全貌（如游戏、专业 3D 建模等）。
- **未评估多模态能力**：虽然任务涉及图像、视频等多媒体，但代理仅通过命令行操作，不评估视觉感知，可能不是最大化代理能力的界面（未来多模态 CLI 可能更优）。
- **固定设置脚本**：任务有确定性设置，真实场景中环境不断变化，基准可能高估代理在变化环境中的泛化能力。

### 其他局限
- **公开后发现模型训练数据污染**：论文承认可能需要定期刷新基准。
- **计算资源未报告**：无法复现成本效益图中的具体成本分配，用户需自行猜测 API 计费方式。

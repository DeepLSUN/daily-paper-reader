## 摘要
多模态大语言模型（MLLMs）在处理长视觉令牌序列时计算量增大。现有加速方法通常移除视觉令牌或跳过整个层中的视觉令牌更新，但这些粗粒度策略可能丢弃细粒度的证据或连同冗余算子一起抑制有用算子。本文从答案可观察角度研究视觉令牌计算，发现后期视觉令牌更新可能保持较大但对答案令牌表示影响很小。受这种答案静默冗余的启发，我们将每个Transformer层分解为注意力算子和FFN算子，并表明有用的视觉计算通常是算子主导且依赖于层。我们提出一个算子级视觉令牌跳跃框架，保留完整的视觉令牌序列，同时选择性地绕过冗余的注意力、FFN或两者。在三个MLLM架构和10个VQA基准上的实验表明，我们的方法实现了强效率-准确率权衡，在Qwen3-VL上减少33.7% TFLOPs，同时保留99.5%的原始模型性能。

## Abstract
Multimodal large language models (MLLMs) increasingly process long visual-token sequences, increasing the overall inference computation. Existing acceleration methods usually remove visual tokens or skip visual-token updates in entire layers, but these coarse strategies may discard fine-grained evidence or suppress useful operators together with redundant ones. In this paper, we study visual-token computation from an answer-observable perspective and find that late visual-token updates can remain large while having little effect on answer-token representations. Motivated by this answer-silent redundancy, we decompose each Transformer layer into attention and FFN operators and show that useful visual computation is often operator-dominant and layer-dependent. We propose an operator-level visual-token skipping framework that preserves the full visual-token sequence while selectively bypassing redundant attention, FFN, or both. Experiments across three MLLM architectures and 10 VQA benchmarks show that our method achieves strong efficiency-accuracy trade-offs, reducing \textbf{33.7\%} TFLOPs on Qwen3-VL while retaining \textbf{99.5\%} of the vanilla model performance.

- **研究动机**：多模态大语言模型（MLLMs）在推理时处理大量视觉 token（如高分辨率图像、图像分块），导致预填充阶段计算开销巨大。现有加速方法要么直接删除视觉 token（token pruning），要么跳过整个 Transformer 层的视觉更新（layer-wise skipping），但前者会永久丢失细粒度视觉证据，后者将注意力与 FFN 算子捆绑处理，无法区分哪些算子真正冗余。
- **核心问题**：如何在保留完整视觉 token 序列的前提下，细粒度地识别并跳过对答案生成影响很小的视觉计算？
- **整体含义**：论文从“答案可观察”（answer-observable）角度重新定义视觉冗余——晚期层视觉 token 的更新幅度虽大但对答案 hidden state 影响微弱（称为“answer-silent”），并提出**算子级别的视觉跳过**策略，将每个 Transformer 层拆解为注意力（Attention）和 FFN 两个独立算子，根据各算子对答案的贡献选择性跳过冗余算子，在几乎不损失精度的情况下大幅降低计算量。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **答案可观察冗余诊断**：定义三个指标：视觉更新幅度（VUM）、答案可观察影响（AOI）、答案可观察效率（AOE）。通过 Jacobian-vector product 计算视觉 token 更新对最终 prompt token hidden state 的一阶影响，发现晚期层 VUM 高但 AOI/AOE 低，即视觉更新是“答案静默”的。
- **算子风险分析**：将每个 Transformer 层分解为 attention 更新和 FFN 更新，分别衡量跳过视觉 attention（RA）、跳过视觉 FFN（RF）或同时跳过两者（RAF）对输出分布的影响（用 KL 散度衡量）。定义算子主导指标 Γ(l) = log(RA(l) / RF(l))，正值为 attention 主导，负值为 FFN 主导。
- **算子感知的跳过策略**：基于校准集（每基准 7 个样本，共 7 个基准）计算各层的 RA、RF、RAF。在给定计算预算 B 下，选择风险最小的 B 个层，为每个层分配最安全的操作：  
  - 若 RA<RF → 跳过视觉 attention，保留 FFN（FFN-ONLY）；  
  - 若 RF<RA → 跳过视觉 FFN，保留 attention（ATTENTION-ONLY）；  
  - 若 RAF 足够小 → 同时跳过两者（FREEZE-UPDATE）。  
  文本 token 的完整计算保持不变，视觉 token 序列完整保留。

### 关键技术细节
- 使用随机投影法高效估计 AOI，避免显式构造完整 Jacobian 矩阵。
- 通过小规模校准集（每个基准 7 个样本，共 49 个样本）确定层间策略，之后固定用于所有测试样本。
- 理论 FLOPs 节省量公式化：  
  ΔC_skip = Σ(l∈L_FFN)ΔC_A(l) + Σ(l∈L_Attn)ΔC_F(l) + Σ(l∈L_Freeze)(ΔC_A(l)+ΔC_F(l))  
  其中 ΔC_A = Θ(Nvd² + MVd)，ΔC_F = Θ(κNvdd_ff)。

## 6. 论文的主要结论与发现

1. **发现答案静默冗余**：晚期层视觉 token 更新幅度大但对答案影响小，冻结这些更新仅带来 <0.5% 性能损失（表 1）。
2. **揭示算子级主导性**：不同层中 attention 和 FFN 对答案的贡献不同，存在 attention-dominant、FFN-dominant 和 frozen 三种模式。
3. **算子级跳过有效**：在 Qwen3-VL-8B 上，以 66% TFLOPs 实现 99.5% 平均性能保持；在三个架构上均优于同等预算的层级别方法。
4. **与 token pruning 互补**：保留完整 token 序列并跳过冗余算子在低 token 场景优于单纯剪枝。
5. **策略鲁棒**：预算增加时性能平滑下降，无突变，易于按需调整效率/精度。

## 7. 优点

- **细粒度抽象**：首次将视觉冗余分解到算子级别，而非层级别或 token 级别，理论更精准。
- **保留完整序列**：不删除任何视觉 token，避免不可逆的细粒度信息丢失。
- **训练无关**：仅需少量无标注校准样本，无需微调或重训练。
- **广泛验证**：在 4 个架构（含 LLaVA-NeXT）、10 个基准上全面对比，消融充分。
- **可解释性**：通过答案可观察指标直观解释冗余来源，与注意力/FFN 功能解耦的研究一致（引用文献 Xi et al., 2026）。
- **效率增益稳定**：多个预算水平下性能保持率均较高，且与其它加速方法可正交联合。

## 摘要
大型语言模型和基于LLM的智能体被广泛用作个人聊天助手、企业副驾驶和自主工作流智能体。在所有这些应用中，记忆（在长上下文和多次交互中积累信息的保留、访问和推理能力）在决定任何智能体的可靠性中起着关键作用。我们提出了RECON（基于混淆叙事的长上下文推理），这是一个用于评估长上下文组合推理的基准测试。RECON涵盖三个领域（刑事、医疗和金融）的24个案例文件，每个文件包含5万到10万个token，并在六项记忆密集型任务上测试智能体：重建多跳证据链、传播级联失效、解决源冲突、反事实推理、满足时间约束和时间事实检索。最近的记忆基准测试评估智能体是否能检索散落的事实或检测事实是否发生变化，而RECON评估变化发生后会发生什么，智能体是否能追踪哪些下游结论受到影响，哪些通过独立支持幸存下来，以及替代时间线会如何展开。我们的评估揭示了当前架构的重大局限性：即使是最强的非Oracle系统也只能达到22.4%的准确率，检索和推理各自成为挑战。

## Abstract
Large language models and LLM-based agents are widely used as personal chat assistants, enterprise copilots, and autonomous workflow agents. In all these applications, memory (the ability to retain, access, and reason over information accumulated over long contexts and multiple interactions) plays a crucial role in determining the reliability of any agent. We introduce RECON (Reasoning over Extended Contexts with Obfuscated Narratives), a benchmark for evaluating compositional reasoning over long contexts. RECON spans 24 case files across three domains (criminal, medical, and financial), each ranging from 50k to 100k tokens, and tests agents on six memory intensive tasks: reconstructing multi-hop evidence chains, propagating cascading invalidations, resolving source conflicts, counterfactual reasoning, satisfying temporal constraints, and temporal fact retrieval. Recent memory benchmarks evaluate whether agents can retrieve scattered facts or detect if a fact has changed whereas RECON evaluates what happens after the change, whether agents can trace which downstream conclusions are affected, which survive through independent support, and how alternative timelines would have unfolded. Our evaluation reveals substantial limitations across current architectures: even the strongest non-Oracle system reaches only 22.4% Accuracy, with retrieval and reasoning each surfacing as challenges.

- **整体含义**：提出了一个新基准RECON，将记忆建模为**溯源图（provenance graph）**而非状态机，强调**派生历史**和**依赖传播**，比现有基准更具挑战性和现实意义。

## 6. 主要结论与发现
- **整体难度极高**：非Oracle系统最高准确率仅22.4%（gemini-2.5-pro），Oracle最高54.6%（gemini-2.5-flash）。RECON显著挑战现有架构。
- **检索与推理均为瓶颈**：即使在检索命中的情况下（获全证据），约80%的问题仍回答错误；Oracle比长上下文高39.4个百分点，说明结构化表示带来的推理增益巨大。
- **不同架构各有所长**：
  - **长上下文**在四项多源推理任务（链重建、源冲突、反事实、时间约束）领先（家族均值）。
  - **RAG**在时间事实检索领先（最佳变体0.443 vs 长上下文0.490）。
  - **记忆系统**在级联传播任务表现出色（Supermemory 0.708，接近Oracle的0.931），但其他任务弱。
- **链长度影响**：长上下文和Oracle准确率随链长度增加单调下降；非Oracle系统在所有长度上接近零，表明检索/注意力是前置瓶颈。
- **反事实和时间约束是推理天花板**：Oracle反事实准确率仅41.2%，时间约束33.3%，表明即使有完美上下文，推理仍受限制。
- **检索效率**：记忆系统使用3K–8K tokens/问，是长上下文（64K–70K）的8–20倍，但性能不输（级联任务甚至更好）。

## 7. 优点（方法/实验设计亮点）
1. **新颖的基准设计**：从状态机转向溯源图，评估信息动态变化后的因果推理，填补空白。

3. **多领域多任务覆盖**：刑事、医疗、金融；六类任务（含控制任务）覆盖推理的不同维度。
4. **严格的验证流水线**：六阶段验证（结构、接地、语义、鲁棒性、人类），人类评分κ=0.69（substantial agreement）。

## 8. 不足与局限
1. **合成数据**：叙事由LLM生成，尽管受约束，仍可能引入风格偏差，影响在真实语料上的泛化。作者提及但无法完全排除。
2. **架构泄露风险**：如果系统知道内部骨架模式（如时间戳格式、证据ID），可能通过模式匹配而非推理答题。人类基线中提供给标注者的证据包也存在类似问题。
3. **人类基线输入不对称**：给予人类标注者的是问题相关的证据子集（证据包+叙事摘录），而LLM系统面对完整文件或自检索上下文，因此人类准确率不能直接对比长上下文系统，只能作为Oracle的互补上界。
4. **LLM依赖**：虽然答案生成是确定性的，但叙事和问题润色仍依赖LLM，可能影响风格或引入细微语义偏移。作者通过多轮验证缓解。
5. **任务设计偏向文本模态**：未涉及多模态输入（如表格、图表、音频），限制了适用场景。
6. **Oracle上限仍低**：Oracle反事实准确率仅41.2%，说明推理本身（即使有完美结构）仍具挑战，基准难度可能部分来自问题设计而非记忆能力。
7. **规模有限**：24个案例，1604问题（过滤后1414），虽比许多基准大，但覆盖领域有限，未来可扩展更多领域和更长上下文。

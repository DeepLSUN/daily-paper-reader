## 摘要
监督微调（SFT）开放权重的大语言模型在专家智能体轨迹上，已成为构建不依赖专有模型的能干的代码智能体的重要方法。一个核心但尚未充分探索的问题是轨迹质量和数量如何共同塑造模型性能。我们在SWE-trajectory数据集（67,074条轨迹，其中32,161条已解决）上，针对Qwen2.5-Coder-7B-Instruct的LoRA微调，进行了轨迹数据过滤的系统性实证研究。我们提出了一个双轴质量评分框架——效率和风格——并通过16个受控实验（涵盖策略、规模和消融分析）对其进行评估。由于7B规模模型在SWE-bench上的解决率接近零，我们采用保留轨迹上的交叉熵（CE）损失作为主要指标，并通过首动作生成进行验证：CE损失和ROUGE-L完全秩相关（斯皮尔曼ρ = -1.00），有限样本证据支持但并未确凿确立这一代理指标。我们的结果揭示了规模依赖的质量-数量权衡：在小规模下，数据集翻倍（500到1,000）使CE损失降低约12.7%，而TopQ-Random差距保持在<1%（曼-惠特尼p > 0.10）；在2,000条轨迹时，这一差距扩大到3.6%（p = 0.016）。消融进一步识别出错误重试率是主要子维度，其表现与完整复合指标相当（Δ < 0.2%）。这些发现共同确立了轨迹级质量评分作为代码智能体SFT的一个可行但规模敏感的杠杆，并为端到端解决率在统计上不可行的场景提供了一个代理验证的评估协议。

## Abstract
Supervised fine-tuning (SFT) of open-weight LLMs on expert agent trajectories has emerged as a prominent approach to building capable code agents without reliance on proprietary models. A central yet underexplored question is how trajectory quality and quantity jointly shape model performance. We present a systematic empirical study of trajectory data filtering for LoRA fine-tuning of Qwen2.5-Coder-7B-Instruct on the SWE-trajectory dataset (67,074 trajectories, of which 32,161 are resolved). We propose a two-axis quality scoring framework -- Efficiency and Style -- and evaluate it through 16 controlled experiments spanning strategy, scale, and ablation analyses. Since 7B-scale models attain near-zero SWE-bench resolve rates, we adopt cross-entropy (CE) loss on held-out trajectories as the primary metric, validated via first-action generation: CE loss and ROUGE-L are perfectly rank-correlated (Spearman $ρ$ = -1.00), with limited-sample evidence supporting but not conclusively establishing this proxy. Our results reveal a scale-dependent quality-quantity trade-off: at small scales, doubling the dataset (500 to 1,000) yields ~12.7% CE-loss reduction whereas the TopQ-Random gap stays <1% (Mann-Whitney p > 0.10); at 2,000 trajectories this same gap widens to 3.6% (p = 0.016). Ablation further identifies error-retry rate as the dominant sub-dimension, performing comparably to the full composite ($Δ$ < 0.2%). Together, these findings establish trajectory-level quality scoring as a viable but scale-sensitive lever for code-agent SFT and offer a proxy-validated evaluation protocol for the regime where end-to-end resolve rate is statistically infeasible.

## 3. 实验设计

- **数据集**：SWE-trajectory（67,074条轨迹，其中32,161条已解决），来自 GitHub 仓库级任务。
- **基准模型**：Qwen2.5-Coder-7B-Instruct。
- **测试集**：三个保留集各200条轨迹：Gold（复合分前10%）、Random（均匀抽样，排除Gold）、Low-Q（后10%）。每个测试集约9.4M assistant tokens。
- **对比方法/策略**：16组实验，分为三个 Block 加一个未训练基线：
  - **Block 1**（7个实验+扩展2个）：Random、ResolvedOnly（仅保留解决轨迹）、TopQ（按复合分排名取前）、BottomQ（取后），在500/1000/2000轨迹规模下比较。
  - **Block 2**（2个实验）：Style only、Efficiency only，对比维度消融。
  - **Block 3**（4个实验）：逐个移除子维度（NoB2, NoB3, NoC2, NoC3）均在500规模下。
  - **附加实验**（2个）：2000轨迹的Random（Exp14）和TopQ（Exp15）；B2-only基线（Exp16）。
- **主要评估指标**：保留集上的**交叉熵损失**（CE loss），因为 SWE-bench 解决率对7B模型统计不可行。
- **次要验证**：首动作生成（ROUGE-L、File-Match accuracy），对4个检查点（未微调、Random-500、Random-1000、TopQ-500）进行100个实例验证。

## 6. 论文的主要结论与发现

1. **数量主导质量**（小规模）：在500→1000规模下，数据翻倍带来 **~12.7% 相对 CE 损失降低**；而质量过滤效果（TopQ vs Random）差距<1%（p>0.10，不显著）。
2. **质量效果随规模放大**（2000规模）：TopQ-Random 差距扩大到 **3.6%**（p=0.016）；TopQ 的1000→2000改善（-9.1%）优于 Random（-6.5%），暗示质量过滤的边际收益递减更慢。
3. **错误重试率（B2）是主导子维度**：B2-only 基线在500规模下与完整复合分性能相差<0.2%（Δ=+0.0010, p=0.88），且消融中移除 B2 导致最大性能下降。
4. **代理指标有效性**：CE loss 与首动作 ROUGE-L 完全秩相关（ρ=-1.00, n=4），且在定性行为上（未微调→输出结构化报告→500条后出现代理风格→1000条后接近真实）支持其合理性。
5. **工具调用格式失败**：所有16个微调配置均无法生成正确的 `<tool_call>` JSON 标签（因类不平衡，bash/edit 动作占主导），部署需用约束解码或数据增强。

## 8. 不足与局限

- **统计力不足**：测试集每臂200条轨迹，检测小效应量（如0.7%）需约2800条/臂，导致多数质量过滤效果在500-1000规模下不显著（p>0.10）。

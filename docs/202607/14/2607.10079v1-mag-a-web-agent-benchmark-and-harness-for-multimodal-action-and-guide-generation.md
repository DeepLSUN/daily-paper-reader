## 摘要
数字采用平台（DAP）是嵌入在网页系统中的覆盖层，用于引导用户完成页面内的操作，帮助用户快速上手陌生界面。然而，完成一个真实任务往往不是点击单个页面上的几个按钮那么简单，而是需要一系列在变化的页面状态下展开的操作序列。先前的研究也将自动化网页智能体操作和指南文本生成视为两个独立的问题，且大多数研究向模型输入文本化的页面表示（如DOM或可访问性树），而非人类实际操作的渲染屏幕。在本工作中，我们提出了MAG，这是首个将任务执行和指南编写统一为单一多模态操作与指南任务的基准，并在截图上提供了两种标注方案：标记集元素选择和原始像素坐标。我们进一步为这一复合任务构建了完整的测试框架，涵盖基于LLM辅助和人工验证的标注、训练、实时环境评估以及操作与指南的联合指标。利用该框架，我们评估了前沿API模型和开源多模态模型，并给出了详细分析。最后，我们设计了一种结合专家轨迹的GRPO训练方法，该方法使有监督的9B智能体的成功率几乎翻倍（从6.9%提升至13.2%），同时提升了指南质量。即使是最强的模型，其任务完成率也不到40%，这为未来研究留下了充足空间。

## Abstract
Digital Adoption Platforms (DAPs) are embedded overlays widely used on web systems to guide users through operations inside a page, helping them get started with unfamiliar interfaces quickly. Completing a real task, however, rarely means clicking a few buttons on a single page: it takes a sequence of actions that unfolds across changing page states. Prior studies have also treated automated web agent actions and guide text generation as two separate problems, and most of them feed models textual page representations such as the DOM or accessibility trees rather than the rendered screens that humans actually operate on. In this work we introduce MAG, the first benchmark that unifies task execution and guide writing into a single Multimodal Action and Guide task, with two grounding schemes over screenshots: Set-of-Mark element selection and raw pixel coordinates. We further build a complete harness for this compound task, covering annotation with LLM assistance and human verification, training, evaluation in live environments, and joint metrics for actions and guides. With this harness we evaluate frontier API models and open multimodal models, and report detailed analyses. Finally, we design a GRPO training method augmented with expert trajectories, which nearly doubles the success rate of a supervised 9B agent (from 6.9% to 13.2%) and improves guide quality at the same time. Even the strongest model completes fewer than 40% of the tasks, leaving ample room for future research.

- **研究动机**：现有数字采用平台（DAP）在网页上提供操作引导，但指南的编写和维护依赖人工，成本高昂。自动化的Web智能体研究侧重于任务完成（如WebArena、Mind2Web），指南生成研究则专注于单页面指令编写，两者相互独立，且多数方法以DOM或可访问性树等文本形式作为输入，而非人类实际操作的渲染截图。
- **核心问题**：缺乏一个统一的任务，要求智能体同时在实时多步骤Web任务中执行操作并生成每步的指南文本，同时以截图作为唯一输入。
- **整体含义**：MAG首次将动作执行与指南生成统一为单一多模态任务，采用两种接地方案（Set-of-Mark元素编号和原始像素坐标），并提供完整的数据集、评估指标和训练框架，旨在推动能够同时完成操作和解释的智能体研究，从而将指南编写从手工劳动转变为自动化任务。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 一个MAG episode定义为三元组 `(q, s0, Φ)`：自然语言意图 q、初始页面状态 s0、功能校验器 Φ。在每个步骤 t，智能体基于当前截图观测 ot 和自身历史指南 g<t，输出一个动作 at 和一个指南句子 gt，即 `(at, gt) = πθ(q, ot, g<t)`。
- 动作空间包含7个动词：click, type, select, scroll, press_enter, go_back, finish。接地方式有两种：SoM（选择元素编号）和坐标（像素位置）。指南句子必须使用用户可理解的纯文本，禁止使用标记编号或坐标。
- 评估指标：任务成功率（SR）、指南质量（Gi，基于BLEU-1/2、ROUGE-1/L的等权平均）、门控指南分数（GGS = SR * (γ + (1-γ)Gi)，γ=0.4），另设有输出格式正确率（OFCR）。

### 关键技术细节
- **数据集构建**：基于WebArena的581个成功演示轨迹（来自OpAgent），在实时站点上重放并捕获带SoM标记的截图，将每个坐标映射到候选元素；使用GPT-5.5为每步起草指南，经规则过滤和3名人类标注员审查修正，最终获得563个标注任务（396训练/167测试），共4760个SoM和5779个坐标黄金步骤。
- **训练方法**：
  - **第一阶段：SFT**：对Qwen3.5 9B VLM进行全参数微调，使用黄金步骤进行教师强制训练，学习输出格式和指南风格。SFT后OFCR从~19%提升至~97%，但任务成功率几乎没有改善（SoM甚至从8.1%降至6.9%）。
  - **第二阶段：专家增强GRPO**：在SFT权重基础上，使用GRPO进行在线强化学习，奖励为二元任务成功。关键创新在于将来自GPT-5.5的成功缓存轨迹（139个训练任务）注入到每组（group）中，与6个策略滚动（on-policy rollouts）一起构成组，以解决纯GRPO中全失败组收益方差为零的问题。优势计算为中心化但不缩放，使用非对称clip（εl=0.2, εh=0.28），无KL项。训练共10轮，前几轮优先调度有专家覆盖的任务。

## 6. 主要结论与发现

- **MAG远未解决**：最佳配置（GPT-5.5 + 坐标）仅完成37.4%的任务，最佳9B模型为13.2%。所有模型均承受指南税（GGS远低于SR）。

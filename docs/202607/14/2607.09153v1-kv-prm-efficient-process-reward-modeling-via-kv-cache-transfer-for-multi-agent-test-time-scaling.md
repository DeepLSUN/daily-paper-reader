## 摘要
过程奖励模型（PRMs）已被证明在引导测试时扩展（TTS）方法方面非常有效，显著提升了基于LLM的多智能体系统的能力。然而，现有的PRMs是基于文本的：它们从头重新编码整个轨迹文本。在长时间的多智能体展开中，评分成本随序列长度L呈二次增长，造成了严重的计算瓶颈，极大地限制了PRMs在长上下文场景中的应用。为解决这一问题，我们引入了KV-PRM，一种高效的过程奖励模型，通过直接读取LLM生成阶段自然产生的KV缓存，消除了繁重的文本重新编码。通过针对预先存在的KV缓存处理单个“验证令牌”，KV-PRM将评分成本从O(L^2)降低到O(L)。我们正式证明KV缓存包含比文本严格更多的信息容量，并且对于下游奖励建模更高效。在MATH、GSM8K和AIME基准测试上的实验表明，在Beam Search、MCTS和Weighted Voting等多种TTS方法下，KV-PRM匹配或严格优于文本PRMs，与基于文本的PRMs相比，评分FLOPs减少了高达5000倍，延迟减少了37倍，每个序列的内存占用减少了34倍。

## Abstract
Process Reward Models (PRMs) have been proven to be highly effective in guiding test-time scaling (TTS) methods, which significantly boost the capabilities of LLM-based multi-agent systems. However, existing PRMs are text-based: they re-encode the entire trajectory text from scratch. In long multi-agent rollouts, the scoring cost, growing quadratically with respect to sequence length L, creates a severe computational bottleneck, severely limiting PRMs' application in long-context scenarios. To resolve this, we introduce KV-PRM, a highly efficient process reward model that eliminates the heavy text re-encoding by directly reading the KV cache produced naturally during the LLM's generation phase. By processing a single "verify token" against the pre-existing KV cache, KV-PRM reduces the scoring cost from O(L^2) to O(L). We formally prove that the KV cache contains strictly greater information capacity than text, and is more efficient for downstream reward modeling. Empirically, across the MATH, GSM8K, and AIME benchmarks, KV-PRM matches or strictly outperforms text-PRMs under various TTS methods such as Beam Search, MCTS, and Weighted Voting, with up to a 5,000x reduction in scoring FLOPs, a 37x reduction in latency, and a 34x reduction in per-sequence memory footprint compared to text-based PRMs.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：过程奖励模型（Process Reward Model, PRM）在 LLM 多智能体测试时扩展（Test-Time Scaling, TTS）中扮演关键角色，通过评估中间推理步骤的正确性引导搜索算法（如 Beam Search、MCTS）分配计算资源，从而提升复杂推理任务的性能。
- **核心问题**：现有 PRM 都是**文本基**的——每次评分必须从头重新编码整个轨迹文本，自注意力机制导致计算成本随序列长度 L 呈 **O(L²) 二次增长**。在多智能体长轨迹（通常数千 token）和大量候选轨迹的搜索场景下，PRM 的评分成本几乎与生成成本相当，形成严重计算瓶颈，限制了 PRM 在长上下文推理中的应用。
- **整体含义**：本文旨在通过复用 LLM 生成阶段自然产生的 KV 缓存，将每轮评分的复杂度从 O(L²) 降至 O(L)，从而在保证评分质量的同时实现数量级的计算和延迟节省，推动 PRM 在真实系统中实用化。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **洞察**：LLM 在自回归生成过程中会自然形成 Key-Value (KV) 缓存，编码了轨迹的丰富连续表示（隐藏状态、注意力模式等），信息容量**严格大于**离散文本 token（理论证明）。现有文本 PRM 丢弃了这个宝贵表示，进行冗余的从头编码。
- **方案**：提出 **KV-PRM**，通过**KV 缓存传输**直接读取生成阶段已有的 KV 缓存，仅向序列末尾追加一个“验证令牌”（verify token），用轻量 LoRA 适配器处理该令牌对 KV 缓存的注意力，输出评分。评分复杂度从 O(d·L²) 降至 O(d·L)。

### 关键技术细节

#### 理论支撑
- **命题 1（KV 缓存表示优势）**：在“线性表示假设”下，KV 隐藏层每位置的表示容量是文本 token 的 Ω(d/ log|V|) 倍（d 为隐藏维度，|V| 为词汇表大小）。
- **定理 1（验证误差分解）**：KV 缓存的贝叶斯最优误差**不高于**文本，且近似间隙随读出深度 k 单调非增。
- **定理 2（读出深度递减收益）**：边际信息增益 ΔIₖ 随 k 指数衰减（约 e^{-α·n_h N_L·(k-1)}）；k=1 即捕获大部分可提取奖励信息；每额外 FLOP 的信息增益也随 k 指数衰减。这为单 token 验证设计提供了理论最优性。

#### 架构设计
- **输入**：生成阶段积累的 KV 缓存 (K, V)（长度 L，层数 N_L，隐藏维 d）。
- **处理**：激活 LoRA 适配器（r=256, α=32），基础模型参数冻结。输入一个验证令牌（如 “?”），通过 LoRA 适配器的前向传播（单层注意力 + FFN）产出一个 logit 向量。
- **评分**：两个判断 token “+” 和 “-”，取 logits 经 softmax 得到正收益概率 P(+)，即为当前中间步骤的得分 s_j。
- **成本**：每步评分仅需 O(d·L) FLOPs（单 query 对 L 个键值的注意力 + 单 FFN），相比文本 PRM 的 O(d·L²)，获得了 **L 倍加速**（典型 L≈5000 => 约 5000× FLOPs 减少）。

#### 训练流程（Algorithm 1）
1. 使用 MCTS 对训练问题生成轨迹，并以 exact-match 为依据产生 Q 值（范围 [-1,1]）作为训练标签。
2. 对于每个样本 (x, y)：
   - 基础模型（适配器关闭）处理 x 得到 KV 缓存 (K, V)（无梯度）。
   - 模型（适配器打开）处理验证令牌 v，基于 (K, V) 得到得分 P(+)。
   - 计算 MSE 损失：L_KV = MSE( P(+), (y+1)/2 )。
3. 仅更新 LoRA 参数 Δθ，梯度只经过单令牌前向。

## 6. 主要结论与发现

1. **计算效率大幅提升**：KV-PRM 将每步评分复杂度从 O(dL²) 降至 O(dL)，在实际中实现 **~5000× FLOPs 减少**、**15-37× 延迟降低**（8B 模型、L=4096 下 4.6ms vs 172.0ms）、**34.2× 每序列内存占用降低**。
2. **评分质量持平或更优**：在绝大多数配置下，KV-PRM 准确率**匹配或超过**文本 PRM（平均高约 1-2 个百分点，AIME 高 3-6 个百分点），验证了 KV 缓存的信息优越性。
3. **理论预测得到验证**：定理 1 和 2 的预测被实验证实——KV 缓存信息量 > 文本，且单 token 读出的近似间隙已很小，边际增益指数衰减。
4. **模型大小不是替代方案**：通过缩小文本 PRM 模型大小（8B→0.6B）无法弥补效率差距，KV-PRM 在更低成本下取得更高准确率。
5. **KV 缓存的差异化能力**：KV-PRM 评分对 KV 缓存可微，支持梯度优化，初步实验（KV Steering）表明可在无搜索情况下提升准确率（+1-3 pp），开启测试时潜在优化新方向。

## 7. 优点

- **理论深度**：首次形式化证明了 KV 缓存较文本的表示优势，并给出读出深度与信息增益的定界，为方法设计提供了坚实理论基础。
- **工程实用性**：直接复用生成阶段自然产生的 KV 缓存，无需额外存储或预处理，作为“即插即用”模块可无缝替代现有文本 PRM 在所有搜索算法中。
- **计算效率突破**：O(L²)→O(L) 的复杂度降阶在实际中带来三个数量级的 FLOPs 节省和两个数量级的延迟/内存节省，使长场景 TTS 变得可行。
- **实验全面且公平**：多数据集、多模型、多种搜索算法和拓扑，且与文本 PRM 保持相同架构/训练数据，消融丰富，结果可信。
- **前瞻性探讨**：提出了 KV Steering 作为潜在新方向，展现了 KV 缓存可微性带来的额外能力（梯度优化），而文本 PRM 无法做到。

## 8. 不足与局限

1. **架构绑定限制**：KV-PRM 要求【生成器与验证器共用同一基础模型架构】，因为 KV 缓存依赖于特定模型的参数和表示空间。不同家族的模型无法直接复用，限制了异构系统中的应用。
2. **理论假设强**：证明依赖“线性表示假设”（Assumption 1）——隐藏状态是独立语义基的稀疏线性组合，这在现实模型中可能近似但非精确成立；定理 2 还假设低秩奖励结构和谱衰减，具有理想化色彩。
3. **领域泛化有限**：实验仅覆盖数学推理（MATH/GSM8K/AIME），未验证代码生成、常识推理、对话等任务，亦未涉及其他模型族（LLaMA、DeepSeek 等）。在非数学领域的有效性尚不清楚。
4. **KV Steering 尚属概念验证**：仅在小规模实验中展示，未提供收敛性保证、效率定量分析或与搜索方法的联合优化。其实际收益和稳定性有待深入研究。
5. **训练数据依赖**：训练标签由 MCTS 生成，MCTS 本身有计算成本；若生成质量偏差可能导致偏好过拟合。与人工标注相比，可能引入噪声。
6. **未与基线效率充分对比**：虽然与 Text-PRM 严格对比，但未与更轻量的基于 logits 的“隐式”PRM（如直接使用生成模型自身 logit 变化）或基于蒸馏的小模型进行效率对比。
7. **单 token 读出的最优性较理论**：尽管实验显示 k=1 已近最优，但未测试不同验证 token 的选择策略（如学习最优 token），忽略了 token 表示可能带来的额外信息。

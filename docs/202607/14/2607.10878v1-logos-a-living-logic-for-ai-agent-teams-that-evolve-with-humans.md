## 摘要
AI智能体正从答案引擎演变为持久化的团队，它们使用工具、分配工作、从经验中学习，并修改影响其未来行为的工件。部署的核心问题不再仅仅是智能体能够做什么，而是谁控制它们被允许成为什么。我们引入了logos，这是一个可插拔的自我进化与治理层，旨在增强而非取代现有的多智能体框架。logos将异构多模态输入（包括文档、图像、音频、表格、数据库、API以及人类指令）编译成版本化的智能体包，其中包含智能体、工具、知识、测试、权限和策略。在运行过程中，它将智能体活动转换为可移植、可审计的事件追踪，并在各框架和后端间执行失败关闭验证。每一个学习到的提示、记忆、技能、工具、角色或工作流，在获得保留的执行证据、人工控制的策略以及明确的授权批准之前，始终被视为不可信的候选版本。这种架构实现了“可验证的人-智能体循环工程”：智能体可以行动、询问、学习并提出改进建议，而人类可以引导目标、权限、审批和不可逆操作，同时不中断持续运行。logos为可问责的自动化提供了活逻辑。智能体可以以机器速度进化，但只有证据和人类权威才能闭合这个循环。

## Abstract
AI agents are evolving from answer engines into persistent teams that use tools, delegate work, learn from experience, and modify the artifacts that shape their future behavior. The defining question for deployment is no longer merely what agents can do, but who controls what they are allowed to become. We introduce logos, a pluggable layer for self-evolution and governance that strengthens existing multiagent frameworks rather than replacing them. logos compiles heterogeneous multimodal inputs, including documents, images, audio, tables, databases, APIs, and human instructions into versioned agent packs containing agents, tools, knowledge, tests, permissions, and policies. During operation, it transforms agent activity into portable, auditable event traces and applies fail-closed verification across frameworks and backends. Every learned prompt, memory, skill, tool, role, or workflow remains an untrusted release candidate until held-out execution evidence, human-controlled policy, and explicit authorization permit its promotion. This architecture enables "verifiable human-agent loop engineering": agents can act, ask, learn, and propose improvements, while humans can steer objectives, permissions, approvals, and irreversible actions without interrupting continuous operation. logos provides a living logic for accountable automation. Agents may evolve at machine speed, but only evidence and human authority can close the loop.

### 论文的主要结论与发现

1. **编译器**：LOGOS 编译的包在生成压力测试中达到了较高的 C3/C4/C5 通过率（0.882/0.720/0.757），超过单次 LLM 和框架模板，接近手工代理。
2. **自我进化门控**：配对执行门控消除了有害采纳，在固定候选重放中净效用为正；无门控和代理门控则采纳大量有害候选。
3. **记忆**：有门控的值感知记忆在合成语料上有效；在 LongMemEval 和 LoCoMo 上，被动记忆无益甚至有害，但有显式训练信号时，门控记忆可带来提升并阻止退化。
4. **路由**：验证门控级联在代码任务（HumanEval、MBPP）上显著优于便宜模型，并接近强模型；集体路由器可在保留准确率的同时降低成本 47%。
5. **人类安全委派**：LOGOS 在 400,000 模拟任务中未出现未授权外部效应，同时将人工干预率从 100% 降至 39%。
6. **合约遵守**：Q9-Q22 的故障注入测试显示，LOGOS 在文档漂移、API 漂移、目标侧重验、根策略优先、证据接地、工具安全、租户隔离、记忆过时、回滚审计等方面均优于负控制策略。

### 优点

1. **架构创新**：清晰分离证据与权威，提出可插拔的治理层，不替代现有框架，而是增强它们。
2. **全面的合约设计**：定义了 Agent Pack、标准化追踪、失败关闭验证、门控推广、根策略、人工授权事件等，形成统一的发布治理合约。
3. **实验严谨性**：将实验证据分类（外部基准、机制研究、故障注入、一致性检查），区分不同证据的边界；包含负控制；配对比较消除系统差异。
4. **覆盖关键安全挑战**：直接处理误进化（misevolution）、信用稀释、代理评估、提示注入、记忆中毒等。
5. **可审计性**：提供标准化追踪、门控证据记录、审计日志、提议安全视图等。

### 不足与局限

1. **实验覆盖有限**：
    - 所有实验使用闭源托管 API，未开源模型或本地部署，可重复性不足。
    - 生成测试基于模板，非真实业务部署，无法估计真实故障频率。
    - 未包含大规模长期部署研究（如数周自主演化）。
    - 一些基准（如 GSM8K）上编译带来的开销导致收益不明显，仅部分场景（代码、MATH-500）有增益。
2. **依赖验证器质量**：整个门控依赖验证器的质量。若验证器存在假接受或覆盖不足，门控可能失效。
3. **开销**：编译、运行时追踪、门控重跑引入额外计算成本，文中未全面量化。
4. **门控统计限制**：配对门控仅为经验规则，缺乏严格统计错误控制（除非启用任意时间有效门控，其又可能过于保守）。LORD 账本仅在特定假设下有正式FDR控制。
5. **记忆与转移**：跨部署转移需重新验证，限制了即时迁移效率。被动记忆在无训练信号时效果为负。
6. **人工干预成本**：虽然减少了手工干预，但仍需 39% 任务有人工参与，可能不适用于高自动化场景。

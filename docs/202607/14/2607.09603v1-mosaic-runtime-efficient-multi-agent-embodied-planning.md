## 摘要
基于LLM的多智能体具身规划由于执行延迟过高而仍然不实用。我们发现失败动作是主要瓶颈，源于两个核心挑战：部分可观测性下的不准确状态追踪以及产生冗余或冲突动作的低效协调。我们提出Mosaic，一个运行时高效的多智能体规划框架，同时应对这两个挑战。Mosaic通过以智能体为中心的语义记忆维护准确且轻量的状态追踪，该记忆以相对坐标存储对象，实现几何变换和协调。它通过整数线性规划确保高效协调，在每个规划步骤分配动作，强制执行物理可行性和智能体间协调约束。在AI2-THOR和搜索救援基准测试中，Mosaic实现了27-32%更快的执行速度、30-33%更少的LLM调用、25-31%更少的步骤和4-10%更高的成功率。这些结果表明，高效记忆和约束引导的协调对于可扩展、低延迟的多智能体规划至关重要。

## Abstract
LLM-based multi-agent embodied planning remains impractical due to prohibitively high execution latency. We identify failed actions as the dominant bottleneck, stemming from two core challenges: inaccurate state tracking under partial observability and inefficient coordination that produces redundant or conflicting actions. We introduce Mosaic, a runtime-efficient multi-agent planning framework that addresses both challenges. Mosaic maintains accurate yet lightweight state tracking through agent-centric semantic memory that stores objects in relative coordinates, enabling geometric transformations and coordination. It ensures efficient coordination through Integer Linear Programming that allocates actions at every planning step, enforcing physical feasibility and inter-agent coordination constraints. Across AI2-THOR and search-and-rescue benchmarks, Mosaic achieves 27-32% faster execution, 30-33% fewer LLM calls, 25-31% fewer steps, and 4-10% points higher success rates. These results demonstrate that efficient memory and constraint-guided coordination are critical for scalable, low-latency multi-agent planning.

### 论文核心问题与整体含义（研究动机和背景）
- **背景**：多智能体具身规划（如协同搜索救援、家庭整理）利用多个智能体并行处理任务，可提高效率和鲁棒性。然而，基于大语言模型（LLM）的多智能体规划系统在部署时面临极高的执行延迟，导致实用性不足。
- **核心问题**：延迟的主要来源是**失败动作**（failed actions）。失败动作产生于两大挑战：**部分可观测性下的不准确状态追踪**（智能体无法准确记忆已探索物体的位置，导致重复导航或冲突）；**低效的多智能体协调**（任务分配粗放，产生冗余、冲突或违反前提条件的动作）。当前系统往往被动处理失败，而非主动预防，导致大量重规划和空闲等待。

### 方法论：核心思想、关键技术细节
- **整体架构**：Mosaic 是一个兼容通用LLM规划框架的模块化插件，基于“分解-提议-验证”的集中式规划架构（包含LLM-Planner、LLM-Actor、LLM-Verifier）。
- **核心组件1：智能体中心语义记忆（Agent-centric Semantic Memory, ASM）**
  - **思路**：轻量级空间追踪，每个智能体维护与任务相关物体和目的地的**相对坐标**（曼哈顿距离 + 方位角），而非全局地图或稠密3D表示。
  - **实现细节**：相对距离按语义层级离散化（如“就在前方”“远左”），但底层保留连续数值以避免累积误差。支持**智能体间坐标转换**：当智能体A观察到物体时，可通过几何变换更新其他智能体对该物体的相对位置，避免重复探索。
  - **作用**：提升动作提议质量，降低因空间推理错误导致的失败。
- **核心组件2：基于约束优化的动作选择（Action Selection via ILP）**
  - **思想**：LLM为每个智能体生成多个候选动作，然后由整数线性规划（ILP）选择全局一致的联合动作。LLM负责语义理解和候选生成，ILP强制执行可行性约束并优化代价。
  - **约束**：
    - 资格约束：禁止违反动作前提（如空手才能拾取）。
    - 每智能体单动作：每个智能体必须选一个动作（允许no-op）。
    - 多智能体联合任务：需要多个智能体共同完成的子任务必须足额分配。
    - 资源独占：同一物体/容器不能被多个智能体同时操作。
    - 碰撞/干扰：空间上不兼容的动作不能同时选中。
  - **代价函数**：综合考虑空间与时间惩罚（循环、重复失败、振荡、回溯、停滞）以及负载均衡（鼓励未被充分利用的智能体执行动作）。
  - **流程**：每个规划步，LLM-Actor根据ASM提议候选 → ILP选择联合动作 → 执行 → 更新ASM和历史 → 验证子任务完成。

### 主要结论与发现
1. **失败动作是延迟主因**：基线方法中16-51%的步骤失败，导致重规划和空闲。
2. **Mosaic显著降低失败率**：在AI2-THOR上失败率降低7-12百分点，在SAR上降低3-7百分点。
3. **效率大幅提升**：相比LLaMAR，运行时减少27-32%，LLM调用减少30-33%，智能体步骤减少25-31%。
4. **成功率提高4-10百分点**：在固定规划步预算下，Mosaic任务成功率更高。
5. **轻量且可扩展**：ILP开销仅1-2%总运行时；5智能体时仍保持1.25-1.30倍加速。
6. **跨模型鲁棒**：在GPT-4o、Claude、Gemini上均一致提升。

### 优点
- **模块化设计**：ASM和ILP可作为即插即用组件增强现有LLM规划器。
- **主动预防失败**：ILP在每一步强制执行可行性约束，避免执行时冲突。
- **高效状态追踪**：相对坐标+语义离散化避免稠密建图开销，支持智能体间信息传递。
- **综合代价优化**：空间/时间惩罚+负载均衡，兼顾效率与公平。
- **可扩展性**：候选动作剪枝（K=3）保持ILP复杂度可控；支持分层分组进一步扩展。
- **实用性**：在嘈杂传感器条件下仍保持较好性能（70-90%无噪声水平），适用于真实世界。

### 不足与局限
- **依赖定位精度**：ASM假设合理位姿估计；严重传感器漂移可能降低性能（论文通过噪声实验部分验证，但未深度探讨控制层整合）。

- **候选动作质量依赖LLM**：若LLM未生成关键动作，ILP无法恢复；虽然ASM和反馈迭代部分缓解，仍是根本限制。
- **实验统计稳健性**：每个条件仅3个episode，可能掩盖方差；未提供置信区间或显著性检验（论文附录有标准差/方差，但未与基线做统计对比）。

## 摘要
智能体检索增强生成（Agentic RAG）通过允许语言模型迭代地进行推理、生成搜索查询、检索证据和预测答案，扩展了静态RAG。然而，模型何时检索、使用词汇匹配还是语义相似度，以及如何控制上下文粒度以防止无关标记干扰智能体推理，仍然具有挑战性。在本文中，我们介绍了GRASP，这是一个强化学习（RL）框架，用于训练智能体在多步推理过程中自适应地协调互补的检索工具。GRASP为智能体提供语义搜索、关键词搜索和段落阅读动作，使其能够检索句子级证据，并仅在需要时扩展更多上下文。我们使用一个同时考虑答案准确性、基于证据的阅读、互补搜索和轮次效率的奖励来训练策略。在多跳推理基准上的实验表明，与单步检索、基于提示的智能体RAG和基于强化学习的检索基线相比，GRASP提高了检索召回率和下游问答性能。定性和消融分析表明，学习到的策略发展出可解释的略读和扫描行为：它使用语义搜索进行广泛探索，使用段落阅读进行局部验证，使用关键词搜索获取实体特定证据。这些结果表明，学习协调检索信号和上下文粒度对于智能体的正确推理至关重要。

## Abstract
Agentic retrieval-augmented generation (RAG) extends static RAG by allowing language models to iteratively reason, generate search queries, retrieve evidence, and predict answers. However, it remains challenging for models to decide when to retrieve, whether to use lexical matching or semantic similarity, and how to control context granularity to prevent irrelevant tokens from interfering with agent reasoning. In this paper, we introduce GRASP, a reinforcement learning (RL) framework for training agents to adaptively coordinate complementary retrieval tools during multi-step reasoning. GRASP provides the agent with semantic search, keyword search, and paragraph-reading actions, enabling it to retrieve sentence-level evidence and expand further context only when needed. We train the policy with a reward that jointly accounts for answer accuracy, grounded reading, complementary search, and turn efficiency. Experiments on multi-hop reasoning benchmarks show that GRASP improves both retrieval recall and downstream question answering performance compared with single-step retrieval, prompting-based agentic RAG, and RL-based retrieval baselines. Qualitative and ablation analyses show that the learned policy develops interpretable skimming and scanning behavior: it uses semantic search for broad exploration, paragraph reading for local verification, and keyword search for entity-specific evidence. These results suggest that learning to coordinate retrieval signals and context granularity is critical for agent's correct reasoning.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究背景**：检索增强生成（RAG）是解决大型语言模型（LLM）知识局限性的主流方法。传统RAG是静态的、单步的流水线，但难以应对需要多步推理的复杂问题。智能体RAG（Agentic RAG）应运而生，允许语言模型像智能体一样，在多步推理中迭代地生成查询、检索信息并综合答案。
- **核心问题**：现有智能体RAG面临三个关键挑战：
    1.  **如何互补利用检索信号**：是使用基于关键词的词汇匹配（如BM25），还是基于语义的相似度匹配（如稠密检索）？不同推理步骤可能需要不同的信号，但现有方法通常只使用单一检索器。
    2.  **如何控制上下文粒度**：检索到的段落往往包含大量无关信息，这些“噪音”会分散智能体注意力，导致其忽略关键证据或产生幻觉。模型需要能够获取句子级的精炼证据，并在必要时才扩展上下文。
    3.  **如何训练自适应策略**：需要一种策略，能决定在何时、以何种方式、检索何种粒度的信息，并协调多个工具在多步推理中的使用。
- **整体含义**：本文旨在解决上述挑战，通过学习一个能够自适应协调多种检索工具和上下文集度的智能体策略，提升智能体RAG在多跳推理任务上的准确性和可靠性。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将智能体RAG的检索过程形式化为一个有限时域的马尔可夫决策过程（MDP），并使用强化学习（RL）来训练策略模型 `πθ`。这个策略智能体可以执行一系列预定义的检索动作，并通过精心设计的奖励函数来学习最优的检索行为。

- **关键技术细节**：
    - **动作空间设计（关键方法）**：定义了四种动作，赋予智能体协调不同工具和粒度证据的能力：
        - **语义搜索 (τs)**：基于稠密向量表示，捕捉语义相似性，用于概念级探索。
        - **关键词搜索 (τk)**：基于词汇匹配（如BM25），用于精确查找特定实体。
        - **阅读段落 (τr)**：给定一个检索到的句子ID，返回其所在的完整段落。此动作允许智能体在获得句子级线索后，按需获取更广泛的上下文进行验证。
        - **终止 (τa)**：结束检索并给出最终答案。
        - **粒度控制**：前两种搜索工具默认返回句子级证据，这是实现精确检索的关键，避免了粗粒度段落带来的噪音干扰。

    - **奖励函数设计（驱动行为）**：设计了联合奖励，以引导智能体学习期望的检索行为。
        - **答案准确性奖励 (RA)**：基于预测答案与真实答案的Token级F1分数，确保最终答案的正确性。
        - **接地阅读奖励 (RR)**：评估被“阅读段落”动作读取的段落是否属于包含正确答案证据的黄金段落，鼓励阅读相关证据，惩罚读取干扰信息。
        - **互补搜索奖励 (RS)**：奖励语义搜索和关键词搜索都至少检索到一条黄金证据，鼓励两者互补使用，而非依赖单一工具。
        - **轮次效率奖励 (RE)**：在答案正确的前提下，鼓励用更少的轮次完成任务，提高效率。
        - **总奖励 (R)**： `R = RA + αRR + βRS + γRE`，其中`RA`权重最大，确保准确率是首要目标，辅助奖励共同塑造高效、精确、互补的检索行为。

    - **学习算法**：使用**组相对策略优化（GRPO）**。对于同一个问题，采样一组轨迹（rollouts），通过比较组内各轨迹的奖励来更新策略，无需依赖绝对奖励值或额外的价值函数模型。

    - **训练细节**：
        - 策略模型：Qwen2.5-3B-Instruct。
        - 训练数据：训练时使用HotpotQA的distractor split，其中包含问题、答案、句子级支持事实标注和2个黄金段落与8个干扰段落。
        - 损失屏蔽：在计算策略损失时，屏蔽检索工具返回的观察文本（tool observation）的token，只对模型自身生成的推理、工具调用和答案进行优化，防止模型学习复述检索结果。

### 6. 论文的主要结论与发现

1.  **GRASP显著提升性能**：与单步检索、基于提示的智能体RAG（IRCoT）和基于强化学习的基线（Search-R1）相比，GRASP在三个多跳推理基准上的**检索召回率**和**下游问答性能**上均取得了最好结果。
2.  **策略学习行为**：通过定性分析，GRASP学习到的策略展现出可解释且高效的检索行为，类似于人类的“略读-扫描”模式：
    - **语义搜索 (τs)** 用于**广泛探索**相关概念或主题。
    - **段落阅读 (τr)** 用于**局部验证**，在获得句子级线索后精读上下文以确认细节。
    - **关键词搜索 (τk)** 用于**实体特定搜索**，获取确切的名称或术语证据。
3.  **消融分析支持设计**：去除任何一种搜索工具或固定段落粒度，性能都会下降。其中，移除语义搜索的负面影响最大，证实了该方法中多种工具和粒度控制的必要性。

1.  **对黄金标注的依赖**：奖励函数`RR`和`RS`依赖于黄金支持事实的标注，这限制了方法在缺乏此类细粒度标注的真实场景中的应用。这是一个重要的实际限制。
2.  **过早终止问题**：尽管训练后工具调用趋于稳定，但策略仍可能在没有收集到足够证据时就提交答案（“过早承诺”）。论文将其归为局限性并提出了潜在改进方向。
3.  **模型规模**：实验仅使用了一个3B参数模型，其学习到的行为模式（如工具使用频率和策略稳定性）可能无法直接推广到更大或更小的模型。

## 摘要
训练工具使用代理从自身经验中改进仍然具有挑战性，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供了弱监督。我们提出了DeepSearch-Evolve，一个基于DeepSearch-World构建的网页代理自我蒸馏框架，DeepSearch-World是一个确定且可验证的环境，具有可复现的搜索和页面阅读工具。DeepSearch-World包含42万个由实体级随机游走构建的多跳问答任务，并支持对自我进化有用的关键代理认知行为，包括进度验证、基于事实的反思和失败恢复。DeepSearch-Evolve迭代执行轨迹生成、过滤、数据混合和微调，以训练更强的代理。无需从更强大的模型进行蒸馏，DeepSearch-World-9B在与开源代理相比时达到了竞争性能，在BrowseComp上为31.2%，在GAIA上为61.5%，在HotpotQA上为93.4%，表明可验证环境能够实现长程网页代理的可扩展自我进化。我们将公开环境、42万训练池、验证集、模型和代码，以促进未来在自我改进深度搜索代理方面的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

## 1. 核心问题与整体含义（研究动机和背景）

- 训练能够自主改进的工具使用智能体（如网页搜索代理）是一个核心挑战。
- **现有方法的局限**：
  - **监督微调（SFT）**：依赖固定教师蒸馏轨迹，性能易受限于教师能力和轨迹多样性，几轮后饱和。
  - **稀疏奖励强化学习（RL）**：仅提供最终奖励，缺乏对中间步骤（查询构建、工具选择、证据提取等）的细粒度指导。
  - **在策略自蒸馏（OPSD）**：匹配软教师分布，但在长程工具使用场景中，教师分布可能因环境不确定性而噪音过大。
- **本文动机**：需要一个**确定性、可验证的环境**，提供过程级监督信号，使代理能从自身经验稳定地自我进化。据此提出 DeepSearch-World 环境及 DeepSearch-Evolve 自蒸馏框架。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 构建一个离线、可确定、可验证的维基百科工具环境，提供实体级进度验证和基于环境的反射信号。
- 利用 scaffold 教师代理生成高质量轨迹，通过拒绝采样和质量过滤筛选成功轨迹，再转换为标准 ReAct 格式，最后用 SFT 训练学生代理。迭代多轮实现自我进化。

### 关键技术细节
1. **DeepSearch-World 环境**：
   - 基于 Wikipedia 超链接图，从实体级随机游走构造 42 万条多跳 QA 任务。
   - 离线语料库约 1000 万条目（标题、摘要、全文），使用 Pyserini 的 BM25 索引实现**确定性搜索**（`web_search_wiki`）和**确定性页面访问**（`visit_wiki` 通过 SQLite 偏移索引）。
   - 环境维护每个问题的目标实体集 \( T_i \)，在每步工具调用后自动验证是否命中未解决实体，并触发**分级反射信号**（首次失败泛泛提示，重复失败给出更强引导如实体名称）。
   - 这种实体级验证无需 LLM 判断，提供客观的进度追踪。

2. **Scaffold 教师代理**：
   - 三步流程：**计划（Plan）** → **行动（Act）** → **结束（End）**。
   - 计划阶段：初始化状态 \( s_0 \)，包含 `completed_list`、`todo_list`、`experience`、`information` 四字段。
   - 行动阶段：循环调用工具，接收观测和反射，更新状态 \( s_{t+1} = U(s_t, a_t, o_t, r_t) \)。
   - 结束阶段：基于已确认的工作记忆生成简洁答案，减少幻觉。
   - 教师轨迹包含成功工具使用和可恢复失败，记录了规划的、记忆、反思和纠错行为。

3. **自进化训练循环**：
   - 每轮 \( R \)：当前模型 \( \pi_{\theta_R} \) 作为教师在环境中生成轨迹。
   - 筛选出答案正确的轨迹，再经质量过滤（冗余、弱目标对齐、不一致推理）保留高质量轨迹。
   - **Scaffold→ReAct 转换**：移除特定阶段提示，将进度状态和反射重写为推理片段（`<think>` 块），保持动作和观测不变，训练标准 ReAct 策略。
   - 对转换后的轨迹进行**SFT**，损失函数为：
     \[
     L_{\text{SFT}}(\theta) = \mathbb{E}_{\tilde{\tau}\sim\tilde{D}^{(R)}} \sum_{t=1}^T \text{KL}(\delta_{y_t} \| \pi_\theta(\cdot|x, y_{<t}))
     \]
     等同于负对数似然。这种方式比全在策略 OPSD 更稳定，因为使用验证过的离线前缀而非有噪声的软标签。
   - 数据混合采用重要性采样，对历史轮轨迹施加衰减因子 \( \gamma = 0.5 \)。

## 3. 实验设计

### 数据集 / 场景
- **评估基准**：BrowseComp（英文网页浏览）、BrowseComp-ZH（中文）、HLE（专家级学术推理）、GAIA（文本版）、xbench（中文深度搜索）、HotpotQA（多跳检索）以及 SearchQA（综合检索）。
- **训练数据**：DeepSearch-World 自带的 42 万条多跳 QA（训练池），以及一个 377 条的手工验证集 DeepSearch-Val 用于行为分析。

### 对比方法
- **专有模型**：OpenAI Deep Research, OpenAI o3。
- **开源基线**（7B–9B 规模）：R1-Searcher、Search-R1、ZeroSearch、ASearcher、DeepResearcher、PokeeResearch、WebSailor、OffSeeker-DPO、WebExplorer、Marco-DR、MiroThinker、DeepDive 等。同时对比基础模型 Qwen3.5-9B-Instruct。

### 评价设置
- 11 轮自进化，每轮生成 10K 实例，至少 4K 正确轨迹后触发训练。
- 使用 Llama Factory 训练，学习率 5e-6，余弦退火，1 轮/轮，最大序列长度 32768。
- 训练后进一步在 1600 个真实工具实例上使用 GRPO 微调（Google SerpAPI + Jina）以缩小离线-真实差距。

## 6. 主要结论与发现

- **DeepSearch-World-9B 在 7 个基准上显著超过其基础模型 Qwen3.5-9B-Instruct**，例如 BrowseComp 提升 23.8 个点，GAIA 提升 37.6 个点，HotpotQA 提升 48.1 个点。
- **在开源代理中达到竞争水平**，无需任何更强大模型的蒸馏（与依赖教师轨迹或合成数据的 Marco-DR、MiroThinker 等具有竞争力）。
- **可验证环境 + 自蒸馏有效**：过程级实体验证能稳定监督，Scaffold→ReAct 转换成功移植了规划、记忆、错误恢复等高级技能。
- **数据池多样性是关键**：420K 池比 100K 池提供更高验证平台、更低格式错误率和更稳定的工具成功及实体命中率。

## 7. 优点

- **环境设计的优势**：
  - 确定性、可复制；支持实体级细粒度验证，无需昂贵 LLM 评判。
  - 离线工具模式与真实工具对齐（搜索→摘要+URL，访问→全文），便于迁移到线上。
- **方法简洁高效**：
  - 仅使用 SFT 而非复杂 RL，降低了训练不稳定性。
  - 自蒸馏不需要更强大的教师模型，全自主进化。
  - 异步生成-训练调度提高了资源利用率。
- **开放贡献**：即将开源环境、42 万数据集、验证集、模型和代码，促进社区研究。
- **丰富的消融与分析**：系统验证了各组件（过滤、转换策略、数据规模）的必要性，并提供工具行为可视化。

## 8. 不足与局限

- **领域覆盖有限**：环境仅基于 Wikipedia，知识覆盖和领域多样性受限。扩展到更广泛知识源（如新闻、学术论文）可能提升泛化性。
- **训练策略仍有改进空间**：目前使用 SFT，而 RL 或 OPSD 可能带来进一步提升，但如何将规划、纠错等高阶能力注入 RL 训练尚待探索。作者已指出这一点。
- **离线-真实工具差距**：虽然用 GRPO 在真实工具上微调，但核心训练仍在离线环境；真实的动态网页、反爬虫机制等未模拟。
- **跨语言泛化不足**：训练数据仅英文，在中文基准（BrowseComp-ZH）上提升相对较小，表明跨语言迁移能力有限。

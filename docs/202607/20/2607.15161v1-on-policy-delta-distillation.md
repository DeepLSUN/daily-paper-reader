## 摘要
在策略蒸馏是一种强化学习中的替代后训练方法，通过从教师模型提供token级监督来缓解奖励模型施加的约束。尽管在策略蒸馏已在各种设置中得到研究和应用，但其基本设计仍未被充分探索。在本文中，我们引入一种新的蒸馏奖励，称为delta信号，而不是直接模仿教师模型的输出分布。Delta信号定义为教师模型与其在推理能力指令调优之前的基模型之间的差异。因此，它捕捉了推理调优引起的变化，并为传递推理能力提供了更直接的信号。通过大量实证证据，我们表明delta信号显著改进了在策略蒸馏，并将这种新蒸馏方法称为在策略Delta蒸馏（OPD$^2$）。在数学、科学和代码推理基准上的实验表明，OPD$^2$始终优于传统的在策略蒸馏，使推理大语言模型仅需短时间后训练即可实现强性能。代码将在https://github.com/naver-ai/opd2提供。

## Abstract
On-policy distillation is an alternative post-training method in reinforcement learning that alleviates the constraints imposed by reward models by providing token-level supervision from a teacher model. Although on-policy distillation has been studied and applied across various settings, its fundamental design remains underexplored. In this paper, we introduce a new distillation reward, termed the delta signal, instead of directly imitating the teacher's output distribution. The delta signal is defined as the difference between the teacher model and its base model prior to instruction tuning for reasoning capability. It therefore captures the changes induced by reasoning tuning and provides a more direct signal for transferring reasoning capabilities. Using extensive empirical evidence, we show that the delta signal substantially improves on-policy distillation and refer to the new distillation method as On-Policy Delta Distillation (OPD$^2$). Experiments across mathematics, science, and code-reasoning benchmarks demonstrate that OPD$^2$ consistently outperforms conventional on-policy distillation, enabling reasoning LLMs to achieve strong performance with only a short post-training period. Code will be available at https://github.com/naver-ai/opd2

# On-Policy Delta Distillation (OPD²) 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：大型语言模型后训练中，强化学习（RL）常用奖励模型，但存在奖励设计困难、信号噪声等问题。On-Policy Distillation（OPD）作为替代方案，通过教师模型提供 token 级监督，减轻了奖励模型约束，已在 Qwen3 等模型中得到应用。
- **核心问题**：现有 OPD 直接模仿教师模型的输出分布（log π* - log π_θ），但教师模型经过推理调优后仍保留了基模型的自然偏好和风格倾向，蒸馏信号中混杂了非推理知识，未能显式聚焦于推理能力微调带来的变化。
- **整体含义**：本文提出 **delta 信号**（教师模型与其基模型的 log 概率差），作为更直接的推理知识传递信号，从而提升 OPD 的效果，使小模型在短后训练周期内获得更强的推理能力。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **delta 信号定义**：R_t^Δ = log π*(y_t|x,y_<t) - log π*_base(y_t|x,y_<t)，其中 π* 是推理调优后的教师，π*_base 是同一教师调优前的基模型。该信号捕捉了推理调优带来的变化，提供推理知识的学习轨迹。
- **相比传统 OPD**：传统 OPD 使用 R_t^OPD = log π* - log π_θ，而 OPD² 以 delta 信号为主奖励，并通过 **centering（中心化）** 和 **joint conditioning（联合条件）** 解决 delta 信号的收敛点问题和不稳定性。

### 关键技术细节
1. **Centering（中心化）**：对奖励减去 on-policy 分布下的期望值，得到优势函数 A_t^Δ = R_t^Δ - E[log π* - log π*_base]，消除基线偏差，使信号更稳定。
2. **Joint conditioning（联合条件）**：当 delta 信号优势 A_t^Δ 与原始 OPD 优势 A_t^OPD 符号不一致时（A_t^Δ * A_t^OPD ≤ 0），将优势置为 0，防止学生偏离教师信号。最终优势函数为：
   - AD2_t = A_t^Δ  if A_t^Δ * A_t^OPD > 0, else 0。
3. **训练梯度**：∇_θ J_OPD²(θ) = E[ Σ AD2_t ∇_θ log π_θ(y_t) ]，与 RL 形式一致，只更新采样到的 token。

### 算法流程（文字说明）
- 对每个问题 x，学生模型 π_θ 采样一个回答 y。
- 同时计算教师模型 π* 和教师基模型 π*_base 在 y 每个 token 上的 log 概率。
- 依次计算 R_t^Δ、A_t^Δ、A_t^OPD，然后根据条件得到 AD2_t。
- 使用 AD2_t 作为 token 级奖励，通过策略梯度更新学生模型。

## 6. 主要结论与发现

1. **delta 信号显著优于传统 OPD**：在所有模型规模和领域中，OPD² 均取得最高平均分，尤其在数学领域提升幅度大（如 Qwen3-4B 非思考数学平均 70.3 对比 ExOPD 的 66.4）。
2. **在思考模式（强基线下）仍有效**：标准 OPD 和 ExOPD 常导致性能下降，而 OPD² 持续改进，尤其在 AIME25 和 HMMT25 等竞赛基准上。
3. **跨模型家族泛化**：Gemma4-E4B 上 OPD² 同样优于对比方法，且保持原始能力（代码领域虽未超越但退化最少）。
4. **训练稳定性强**：OPD² 在训练曲线上保持较高水平，不像 OPD/ExOPD 后期易退化。
5. **delta 信号是主要贡献源**：消融实验显示去掉 delta 信号后性能下降最大，条件和中心化提供次要增益。
6. **可解释性**：词云、token 级信号可视化、统计分析表明 delta 信号更聚焦推理连接词，抑制不确定表达和无关叙述。

## 7. 优点

- **方法论创新**：首次利用教师与基模型差异作为蒸馏信号，显式传递推理微调知识，而非简单模仿最终输出。
- **全面实验验证**：覆盖多领域、多规模、多模式、多模型家族，对比先进方法 ExOPD，证明普适性。
- **训练稳定高效**：在短训练周期（100 step）内达到或超越更长时间的蒸馏效果，且最终步骤即可稳定高性能。
- **可解释性分析**：从词云、token 级信号、统计词频三个维度揭示 delta 信号特性，增强方法可信度。
- **代码开源**：提供完整实现，促进复现和后续研究。

## 8. 不足与局限

- **计算开销增加**：相比标准 OPD，需额外前向传播教师基模型，训练时间增加 24%~28%（Qwen3），虽可接受但并非零成本。
- **依赖基模型可用性**：方法需要对应教师模型的基模型（调优前 checkpoint），若基模型未公开或无法获取则无法应用。
- **代码领域表现有限**：在 Gemma4 代码基准上，OPD² 仍未超越原始模型，仅减轻退化，可能代码推理知识蒸馏更复杂。
- **规模验证范围**：学生模型仅在 1.7B~8B 验证，更大规模（如 70B）或更小（如 500M）未测试，泛化性有待扩展。
- **训练数据来源**：仅使用三个公开推理数据集（各 100k 采样），未探索不同数据比例或更大数据量影响。
- **超参数敏感性**：λ（ExOPD）和条件阈值（0）可能未全面调优，公平性有保证但最佳设置未知。
- **未与 RL 方法直接对比**：论文标题提及 RL 替代，但实验中仅对比了 OPD 变体，未与 GRPO、PPO 等 RL 方法比较性能与成本。

## 摘要
混合专家模型（MoE）是一类流行的大语言模型（LLM），具有高效率和准确性。然而，在KV缓存密集型的服务场景中，MoE模型常面临模型权重的GPU内存需求与不断增长的KV缓存之间的紧张关系。我们提出PagedWeight，一种新颖的MoE LLM服务管理方法，它在运行时动态量化MoE模型的权重，并平衡专家权重精度与KV缓存大小。PagedWeight揭示并有效导航模型任务准确性、内存消耗与吞吐量/延迟之间的复杂权衡。在多个内存敏感的MoE服务场景中，PagedWeight在质量-内存权衡上优于几种现有的量化基线。PagedWeight实现了与FP16等效的准确性，同时最高节省72.0%的GPU内存并提升1.94倍吞吐量，在相似内存预算下，量化方法的质量最高提升39.3%，而吞吐量损失最多为4.1%。

## Abstract
Mixture-of-Experts (MoE) is a popular class of large language models (LLMs), offering high efficiency and accuracy. However, in KV-cache-intensive serving scenarios, MoEs often exhibit a tension between the GPU memory requirements of the model weights and the growing KV cache. We propose PagedWeight, a novel management method for MoE LLM serving that dynamically quantizes MoE model's weights at runtime and balances expert-weight precision with the KV cache sizes. PagedWeight exposes and effectively navigates the complex tradeoff between the model's task accuracy, memory consumption, and throughput/latency. Across several memory-sensitive MoE serving scenarios, PagedWeight improves the quality-memory tradeoff over several existing quantization baselines. PagedWeight achieves FP16-equivalent accuracy with up to 72.0% GPU memory savings and 1.94$\times$ throughput improvement, and improves quality over quantization methods by up to 39.3% at a similar memory budget with at most 4.1% throughput loss.

- **目标**：提出一种运行时动态调整MoE专家权重量化精度的方法，在任务精度、内存消耗和吞吐量/延迟之间实现智能平衡，为KV缓存腾出内存。

## 2. 方法论
- **核心思想**：将MoE专家线性块的任意精度（Any-Precision）位平面和查找表视为“权重页”（weight pages），运行时根据KV缓存压力动态降低或恢复位宽，实现质量感知的权重页移动。
- **关键技术细节**：
  1. **权重页表示**：每个专家线性块（按层、专家、类型索引）维护一个页面表项，记录支持位宽集、当前提交位宽 `q_i` 和期望位宽 `d_i`；权重以APL的过采样位平面格式存储，位平面+查找表为权重页。
  2. **质量感知运行时规划器**：综合三个因素估计位宽降低的损伤：
     - *离线全局敏感度*：基于Hessian的量化损伤分数 `g_i^{b→b′}`。
     - *在线路由统计*：根据专家被路由频率（路由质量）分组，热点专家使用更高的损伤乘数和位宽下限。
     - *提示残差*：利用线性回归头从输入特征（路由加权范数）预测当前序列的特异性修正 `η_i^{b→b′}`。
  3. **规划目标**：将KV缓存压力转化为字节目标 `D`，对每个合法动作按预测损伤/释放字节排序，贪心选择直到满足目标。
  4. **异步页移动**：计划生成与推理流水线并行，内存压力触发时异步卸载/重新加载权重页，将精度提交延迟到安全边界（图4）。
  5. **融合混合精度MoE内核**：读取任意精度位平面和查找表，在每个专家线性块遵循规划器选择的位宽进行前向计算。
- **公式与算法**：损伤计算 `bd_i^{b→b′} = max(ε, μ_{β_i} · g_i^{b→b′} · η_i^{b→b′})`；规划器算法（Algorithm 1）包含多阶段位宽地面、评分、贪心选择和计划刷新。

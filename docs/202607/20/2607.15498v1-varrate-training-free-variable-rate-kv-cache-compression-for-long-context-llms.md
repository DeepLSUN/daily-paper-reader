## 摘要
键值（KV）缓存是长上下文大语言模型（LLM）推理中的主要内存瓶颈。两种领先的无训练方法族在结构上均存在局限性：token-选择方法（SnapKV、Ada-KV）从观察窗口中评估重要性并驱逐低分token，但驱逐是不可逆的——因此当重要性信号在查询无关复用下降级时，准确率会下降11-15个百分点；均匀低秩编码保留每个token但分配相同秩，从而浪费预算。我们观察到这两种失败有一个共同对策：应分配秩而非驱逐。我们提出VarRate，一种无训练的KV编解码器，通过查询显著性为每个token分配可变的低秩预算，使每个token保持非零秩。类似的自适应秩编解码器只有通过训练才能实现这种分配；VarRate无需训练。由于不丢弃任何token，在查询感知选择崩溃的情况下，它仅下降3.5-5.5个百分点。在LongBench（16个任务）上匹配20%预算时，VarRate在Llama-3.1-8B和Qwen2.5-7B上均保持在未压缩模型的0.8个百分点以内。在两个模型上平均，它是最强的匹配内存压缩器。在两个模型上均显著优于其均匀秩消融版本。与针对查询无关复用专门设计的方法KVzip相比，它在四个设置中的三个上与准确率相当，总体在一个百分点以内，预填充开销约为八分之一。

## Abstract
The key-value (KV) cache is the main memory bottleneck in long-context large language model (LLM) inference. Two leading training-free families are both structurally limited: token-selection methods (SnapKV, Ada-KV) score importance from an observation window and evict low-scoring tokens, but eviction is irreversible -- so when the importance signal degrades under query-agnostic reuse, accuracy collapses by 11-15 points; uniform low-rank coding keeps every token but spends equal rank everywhere, wasting budget. We observe that both failures share one cure: rank should be allocated, not evicted. We present VarRate, a training-free KV codec that assigns each token a variable low-rank budget by its query salience, keeping every token at a nonzero rank. Comparable adaptive-rank codecs reach this allocation only through training; VarRate requires none. Because no token is dropped, it degrades by only 3.5-5.5 points where query-aware selection collapses. At a matched 20% budget on LongBench (16 tasks), VarRate stays within 0.8 points of the uncompressed model on both Llama-3.1-8B and Qwen2.5-7B. Averaged over the two, it is the strongest matched-memory compressor. It significantly beats its uniform-rank ablation on both models. Against KVzip, a method purpose-built for query-agnostic reuse, it is accuracy-equivalent in three of four settings and within a point overall, at about one-eighth the prefill overhead.

- **核心问题**：长上下文大语言模型（LLM）推理中，KV缓存是主要内存瓶颈。现有无训练压缩方法存在两类缺陷：
  - **Token-选择方法（如SnapKV、Ada-KV）**：从观察窗口评估重要性并驱逐低分token。当重要性信号在查询无关复用（如预缓存文档后多轮对话）下过时时，驱逐不可逆，导致准确率下降11-15个百分点。
  - **均匀低秩编码（如Palu）**：保留所有token但分配相同秩，浪费预算在不重要token上，而重要token精度不足。
- **整体含义**：需要一种无需训练、不驱逐token、根据token重要度分配可变秩的方法，以同时避免上述两种缺陷，在查询感知和查询无关复用场景下均保持高精度。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 分配秩（rank）而非驱逐token：为每个token分配可变低秩预算，基于查询显著性（query salience）进行水填（water-filling），保留所有token非零秩，无需训练。

### 关键技术细节
- **预处理**：
  1. 将每个token的key反向旋转（RoPE⁻¹），并与value拼接成联合向量 $z_t = [\tilde{k}_t; v_t] \in \mathbb{R}^D$，$D=2Gd_h$（G为KV头数）。
  2. 保留步长为Δ的锚点（anchor）和最近w个token为精确存储；其他token为编码token。
- **残差编码**：
  - 对每个非锚点token，用其c个最近锚点的均值 $\bar{z}_t$ 计算残差 $e_t = z_t - \bar{z}_t$。
- **共享低秩基**：离线对少量校准文本的残差进行SVD，获取前R个右奇异向量构成基矩阵 $V \in \mathbb{R}^{R \times D}$（每层独立，无训练）。
- **显著性评分**：使用SnapKV的注意力窗口（长度为$w_{obs}$）计算每个token的平均注意力权重 $s_t$，经过移动平均和min-max归一化得 $\hat{s}_t \in [0,1]$。
- **水填分配**：在总预算B（固定内存）下，为每个编码token分配秩 $r_t$：
  $$r_t = \text{clip}\left(r_{\min} + \lambda \hat{s}_t, \; r_{\min}, R\right)$$
  其中 $\lambda$ 通过水填算法求解，使 $\sum_{t \in C} r_t = B$。
- **重建**：对每个编码token，投影到基上得系数 $c_t = V e_t$，保留前 $r_t$ 个系数重建 $\hat{z}_t = \bar{z}_t + V^\top (m_t \odot c_t)$，其中 $m_t$ 为截断掩码。然后拆分、重新旋转key至post-RoPE。
- **特性**：$r_{\min}>0$ 确保无token被丢弃；总存储与均匀秩编码器匹配，实现公平比较；可与量化正交组合。

## 6. 论文的主要结论与发现

- VarRate在20%预算下，在LongBench上距未压缩模型平均0.8分以内（Llama：45.69 vs 46.01；Qwen：48.00 vs 48.79），在两个模型上均为最强的匹配内存压缩器。

- 方法可量化组合：3-bit系数量化后缓存占用降至9%，精度几乎不变。

- **简洁高效**：无需训练、微调或架构更改；离线基只需少量校准文本；算法简单（水填、投影、截断）。

- **实验严谨性**：统一管道比较所有方法，内存匹配精确，统计检验全面，消融实验系统，基线复现验证，发现可跨模型泛化。

## 8. 不足与局限

- **计算开销**：预填充阶段+44%（未优化），高于选择方法（~2%）。可通过bf16融合和GPU缓存基优化至+25%，但仍非零。

- **低预算失效**：在8%预算下，VarRate在某些任务上（尤其Qwen）性能崩溃至与均匀秩相似，不如选择方法（SnapKV）。仅当预算≥12%时恢复。
- **对校准基的依赖**：虽然校准鲁棒（不同基差异≤1.8分），但基仍需要少量校准数据；若领域严重偏离校准分布，可能需重新计算。

- **无延迟突破**：在短生成任务中，预填充开销显著（+41.8% end-to-end）；在长生成任务中可忽略（+2.6%）。但未报告完整的端到端吞吐量比较。

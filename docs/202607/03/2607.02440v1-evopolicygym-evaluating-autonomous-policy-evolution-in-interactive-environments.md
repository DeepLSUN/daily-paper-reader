## 摘要
自主智能体越来越期望通过反馈来改进可执行策略，然而现有评估往往将此过程简化为最终分数，或将其与开放式软件工程进展相混淆。我们引入自主策略演化（Autonomous Policy Evolution），这是一种受控的评估设置，其中有一个桥接模型（harness-model）智能体在固定的交互预算下反复编辑可执行策略系统。我们在EvoPolicyGym中实例化了这一设置，这是一个由紧凑的交互式强化学习环境构建的基准，用于评估智能体如何迭代改进探索到的策略。在EvoPolicyGym套件上，GPT-5.5在所有16个环境中获得了最强的综合排名得分和前两名表现。除了排行榜结果外，EvoPolicyGym还提供轨迹级别的诊断，区分智能体如何分配预算、将反馈转化为参数调整。这些分析表明，强大的自主策略演化不仅取决于孤立任务的成功，还取决于在有限反馈下发现适合任务的机制并细化策略。

## Abstract
Autonomous agents are increasingly expected to improve executable policies through feedback, yet existing evaluations often collapse this process into a final score or confound it with open-ended software-engineering progress. We introduce Autonomous Policy Evolution, a controlled evaluation setting in which a harness-model agent repeatedly edits an executable policy system under a fixed interaction budget. We instantiate this setting in EvoPolicyGym, a benchmark built from compact interactive RL environments that evaluates how agents iteratively improve explored policies. On the EvoPolicyGym suite, GPT-5.5 achieves the strongest aggregate rank score and top-two performance on all 16 environments. Beyond leaderboard results, EvoPolicyGym also provides trajectory-level diagnostics that distinguish how agents allocate budget, convert feedback into parametric tuning. These analyses show that strong autonomous policy evolution depends not only on isolated task wins, but on discovering task-appropriate mechanisms and refining policies under bounded feedback.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有对自主智能体的评估往往将策略改进过程简化为最终分数，或与开放式软件工程进度混淆（如SWE-bench）。这忽略了迭代过程中重要的轨迹级行为，如盲目重试、过拟合、验证缺失等。

- **整体含义**：提出“自主策略演化”（Autonomous Policy Evolution）作为独立可基准的问题，强调智能体在固定交互预算下，通过反复编辑可执行策略系统，利用有限的轨迹反馈进行持续改进。

## 2. 提出的方法论

### 核心思想
- 将自主策略演化建模为**受控的优化循环**：一个编码智能体（coding agent）持有一个可执行策略工作区，提交候选策略到服务器，服务器在沙箱中执行并返回训练集上的汇总和轨迹反馈，智能体据此修改策略。整个过程受限于固定预算（训练episode数）。
- 最终评估基于**隐藏的验证集**选择最佳检查点，并在**隐藏的测试集**上报告回报，防止反馈泄漏。

### 关键技术细节
- **环境与策略定义**：环境为标准gymnasium接口（reset/step），策略为Python可执行工件，包含`Policy`类（`reset`和`act`方法），可以包含辅助模块、常量、记忆、诊断代码等。
- **交互协议**：
  - 智能体观察工作区状态、剩余预算、历史反馈，决定编辑和提交。
  - 每次提交消耗若干episode预算，服务器返回训练反馈（摘要、轨迹、视频等）。
  - 智能体不能看到验证/测试结果。
- **可见性边界**：训练episode相关数据可见，验证和测试数据完全隐藏。
- **评估指标**：每个环境采用**验证选择的测试回报**（native reward scale），跨环境聚合使用**排名得分**（rank score，将所有模型+随机策略在一个环境内排名，映射到0-1）。

### 辅助诊断
- 结构合成（Structural Synthesis） vs. 参数调优（Parametric Tuning）：
  - 合成：引入新的AST拓扑（感知、记忆、规划等结构）。
  - 调优：在相同AST拓扑下调整常量、阈值等参数。
  - 通过代码差异分类编辑类型，并计算各类编辑带来验证集改进的比例。

## 6. 主要结论与发现

- **GPT-5.5**取得最高综合排名得分（0.891），在所有16个环境中位列前两名，9次第一。
- **Claude Opus 4.7**排名第二（0.750），在MiniGrid家族表现最强（0.938），但在BipedalWalker等任务上显著落后。
- **MiniMax-M3**和**DeepSeek-V4-Pro**各有一次环境获胜，但综合排名较低（0.531和0.359），尤其在MiniGrid上几乎全失败。
- **性能分水岭**出现在“合成主导”任务（需要构建感知、规划等结构）上：强模型合成编辑成功率高达41%-48%，弱模型仅3%-10%。在“调优主导”任务上（低维控制），差距缩小。
- **策略结构复杂度**：强模型在合成任务上产生更大、更复杂的策略代码（更多函数、分支、状态），但复杂代码不保证性能（需任务适配）。
- **轨迹级分析**揭示：高效智能体能在早期通过结构合成获得较大改进，并进行有限的参数调优；弱智能体常陷入无效结构替换或重复测试，无法突破特定抽象层次。

## 7. 优点

1. **受控且分离的设置**：将策略改进过程从开放式软件工程中剥离，使评估聚焦于迭代优化能力。
2. **轨迹级诊断**：不只看最终得分，还提供编辑类型分类、反馈利用案例、后验得分演化等细粒度分析，有助于理解行为差异。
3. **环境多样性**：覆盖像素感知、符号规划、连续控制、机器人操作等多种挑战，适合评估通用性。
4. **清晰可见性边界**：训练反馈可见，验证/测试隐藏，避免作弊，保证泛化评估的公平性。
5. **可扩展性**：架构设计支持任何gymnasium兼容环境，为未来增加任务提供基础。

## 8. 不足与局限

1. **模型和实验范围有限**：仅评估4个LLM驱动的编码智能体，每个环境单次运行，缺乏多次重复以统计方差。预算固定128 episodes，未探索不同预算下的表现曲线。
2. **Harness差异未完全控制**：GPT-5.5使用Codex harness，其他使用Claude Code harness，工具能力差异可能影响结果，论文未提供等价的跨harness比较。
3. **诊断的保守性**：结构合成/参数调优分类基于AST拓扑，无法捕捉语义等价但拓扑不同的实现，也无法识别混合行为（一个拓扑内既有改进又有退化）。
4. **缺乏与标准RL方法的基准比较**：论文解释理由合理（接口/预算不同），但读者难以判断这些智能体的绝对水平与传统RL相比如何。
5. **未讨论计算成本**：未报告GPU资源、训练耗时等，使可复现性和效率比较困难。
6. **应用局限性**：所有任务基于模拟器，真实世界应用（如机器人实体交互）的直接迁移性未验证。同时，策略以代码形式表达，限制了如神经网络参数直接调优的方式（仅通过代码修改参数）。

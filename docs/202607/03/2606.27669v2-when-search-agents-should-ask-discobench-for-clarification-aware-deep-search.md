## 摘要
由大型语言模型驱动的搜索代理越来越多地被用于解决复杂的信息寻求任务，需要多步检索和推理来实现用户目标。然而，现有基准通常假设用户查询是完整且明确的，忽略了现实世界中的搜索请求常常模糊、不明确甚至事实上错误的情况。在深度搜索场景中，这种歧义可能沿着多步推理链传播，导致代理走向错误的搜索轨迹。为了解决这一差距，我们引入了 DiscoBench，一个用于澄清感知深度搜索的基准，旨在评估搜索代理是否能够主动识别歧义、提出有效的澄清问题，并通过用户交互恢复正确的推理路径。DiscoBench 包含 211 个样本和 463 个歧义实例，涵盖 11 个真实世界领域，涉及四种歧义类型。我们进一步设计了一个用户模拟器用于多轮交互，并从任务实用性、歧义检测、交互策略和成本效率四个角度评估模型性能。在代表性大型语言模型上的实验表明，歧义检测和有效澄清是不同的能力，反复搜索而不请求澄清往往比直接猜测更差，突显了当前搜索代理在检索能力与交互式解决问题之间的关键差距。

## Abstract
Search agents powered by large language models (LLMs) are increasingly used to solve complex information-seeking tasks, requiring multi-step retrieval and reasoning to fulfill user goals. However, existing benchmarks often assume that user queries are complete and explicit, overlooking the fact that real-world search requests are frequently vague, underspecified, or even factually incorrect. In deep search scenarios, such ambiguity can propagate along multi-step reasoning chains and lead agents toward incorrect search trajectories. To address this gap, we introduce DiscoBench, a benchmark for clarification-aware deep search, designed to evaluate whether search agents can proactively identify ambiguity, ask effective clarification questions, and recover correct reasoning paths through user interaction. DiscoBench contains 211 samples and 463 ambiguity instances across 11 real-world domains, covering four ambiguity types. We further design a user simulator for multi-turn interaction and evaluate model performance from four perspectives: task utility, ambiguity detection, interaction strategy, and cost efficiency. Experiments on representative LLMs show that ambiguity detection and effective clarification are distinct capabilities, and that repeatedly searching instead of asking for clarification often performs worse than direct guessing, highlighting a critical gap between retrieval ability and interactive problem-solving in current search agents.

- **问题**：现有搜索代理基准（如 GAIA、BrowseComp）通常假定用户查询是完整、明确的，但真实世界的搜索请求往往模糊、不完整甚至事实错误。在需要多步推理的深度搜索场景中，初始查询中的微小歧义会沿推理链级联传播，导致代理走向错误轨迹。
- **背景**：用户常因记忆模糊或认知负担给出碎片化查询；而现有基准要么忽略歧义（检索基准），要么仅处理静态歧义（如 AmbigQA），要么缺乏开放域深度搜索（交互基准如 IN3 限于沙盒环境）。缺乏一个能评估代理在多步搜索中主动识别歧义、提出澄清问题并恢复正确路径的基准。
- **整体含义**：代理不仅需要强检索和推理能力，还需要“歧义意识”和主动交互策略，而当前模型在此方面存在显著短板。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建 DiscoBench，将歧义建模为多步推理链中动态出现的现象，而非单个查询的静态属性。通过结构化检查点（Checkpoint）评估代理行为，并设计用户模拟器提供渐进线索。
- **关键技术细节**：
  - **数据构建**：两阶段半自动流程：
    1. **种子数据准备**：从 11 个领域手工收集主题，利用 LLM 初构 + 人工验证生成多跳种子问题，每个问题分解为有序检查点序列。
    2. **歧义数据构建**：在检查点上注入四种歧义类型：
       - **实体歧义**（Entity）：多个实体满足同一描述。
       - **版本歧义**（Version）：同一实体的不同时间/版本状态。
       - **标准歧义**（Criteria）：不同评价标准或排名准则。
       - **事实不准确**（Factual Inaccuracy）：描述与客观事实冲突。
       - 为每个歧义点生成可区分的线索（用户模拟器提供）。
  - **任务形式化**：每个问题分为 n 个检查点，类型为 Unambi（无歧义）或 Ambi（有歧义）。代理在每个检查点可执行 SEARCH、ASK（向用户请求线索）、ANSWER 三种动作。用户模拟器在接收到 ASK 时释放预定义线索。
  - **评估框架**：从四类指标评估：
    - 任务效用：端到端准确率（End-to-End Accuracy）、检查点通过率（Checkpoint Pass Rate）。
    - 歧义检测：检测准确率（Detection Accuracy）、检测 F1。
    - 交互策略：澄清问题准确性（CE-A）、澄清-进展率（CE-B）。
    - 成本效率：平均提问次数、工具使用次数、Token 消耗。
  - **用户模拟器**：使用 Gemini-3-Flash-Medium 作为模拟用户，支持多轮渐进式线索披露。

## 6. 论文的主要结论与发现

- **歧义检测与有效澄清是不同能力**：如 Qwen3.6-Max 检测 F1 极低（16%），但提问质量很高（CE-A 94.7%）；MiniMax-M2.7 提问较多但澄清效果差。两者不必然对齐。
- **反复搜索不如直接猜测**：在行为画像中，SearchHeavyGuess（多次搜索但不提问）平均通过率仅 51.9%，低于 DirectGuess 的 56.5%，更远低于 SearchThenAsk 的 93.4%。说明重复检索不能替代主动澄清。
- **引导提示（Guided）能改善检测但效果有限**：Neutral 和 Guided 下平均端到端准确率从 28.6% 升至 33.7%，检测 F1 从 45.3% 升至 64.9%，但少数模型（如 Claude）在 Guided 下准确率反而微降。当前模型仍缺乏稳健的澄清能力。
- **更多工具使用不必然带来更好性能**：高检索频率的模型（如 Claude）准确率低于检索频率适中的模型。成功的关键在于能否在正确检查点识别歧义并有效利用线索。
- **事实不准确类型较易检测**（因与检索证据冲突），而实体和标准歧义更难（需主动提问区分），暴露了模型在需要主动澄清而非事实检查的场景中的短板。
- **外部搜索工具和歧义都是难度来源**：移除搜索工具后准确率大幅下降（如 Doubao 从 43.1%→2.4%）；移除歧义后准确率显著提升（+26.8%至+40.2%）。

## 7. 优点

- **任务设计新颖**：首次将歧义建模为沿多步推理链动态传播的现象，而非静态查询属性。结构化检查点设计使评估粒度更细。
- **覆盖多种真实歧义类型**：实体、版本、标准、事实不准确四种，贴近现实需求。
- **多维评估框架**：不仅看最终准确率，还评估歧义检测、交互质量、成本效率，提供了更全面的能力画像。
- **用户模拟器设计**：渐进式线索披露，模拟真实交互，避免了二进制反馈的局限性。
- **消融实验充分**：验证了搜索工具和歧义的必要性，揭示了不同行为模式的影响。
- **数据质量控制严格**：两阶段构建 + 五阶段质量控制（结构验证、LLM探测、人工审查等），确保样本可靠。

## 8. 不足与局限

- **歧义类型覆盖有限**：仅聚焦四种客观问答相关的歧义，未涉及主观偏好歧义（如“推荐什么电影？”）等更复杂的交互场景。论文自身承认这一点。
- **用户模拟器非真实人类**：使用 LLM 驱动的用户模拟器，可能无法完全反映真实用户的多样化、噪声和不可预测性。交互的自然性和真实性有待加强。
- **语言与地域偏倚**：基准主要构建在中文网络环境，可能不直接适用于其他语言或文化背景下的歧义模式。虽然实验涉及多语言模型，但数据来源偏中文。

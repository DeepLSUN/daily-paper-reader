## 摘要
理解大型复杂代码库，尤其是那些具有混淆结构和不完整文档的代码库，仍然是一个重大挑战。现有的代码摘要解决方案通常依赖单一语言模型或像Claude Code这样的编码助手，并将源代码视为平面文本，未能充分利用代码库中丰富的相互依赖关系和层次结构信息。为解决这些不足，我们提出了Agent4cs——一个自底向上对大型代码库进行摘要的多智能体框架，其中摘要智能体专注于生成稳健的摘要；关键词提取智能体主动从子文件夹中识别关键信息；质量保证智能体迭代优化输出的可读性、连贯性和完整性。在7个前沿模型上的评估表明，与两种带有代码片段的结构化提示基线相比，Agent4cs在所有文件夹层级上的语义一致性平均提高了8%。此外，在真实世界数据集上的广泛评估显示，与相同基线相比，归一化关键词覆盖率最高提升了38%。

## Abstract
Understanding large, complex codebases, especially those with obfuscated structures and incomplete documentation, remains a significant challenge. Existing code summarization solutions often rely on a single language model or coding assistant like Claude Code, and treat source code as flat text, underutilizing the rich interdependencies and hierarchical information within a repository. To address these shortcomings, we propose Agent4cs - a multi-agent framework that summarizes large codebases in a bottom-up fashion, where a summarization agent focuses on producing robust summaries; a keyword-extraction agent proactively identifies critical information from subfolders; and a quality-assurance agent iteratively refines the outputs for readability, coherence, and completeness. Evaluated on 7 frontier models, Agent4cs improves semantic consistency across all folder levels by average 8% compared to two structured prompting baselines with code segments. Furthermore, extensive evaluation on real-world datasets demonstrates up to 38% gains in normalized keyword coverage rate over the same baselines.

### 1. 核心问题与研究动机
- **背景与挑战**：大型软件项目代码库规模庞大、结构复杂，文档往往不完整、过时或不一致，导致开发人员理解障碍、维护困难、架构偏离风险增加。
- **现有方法不足**：传统代码摘要技术多聚焦于函数级别，将代码视为平面文本，忽略了代码库中丰富的跨文件依赖和层次结构信息；即使引入LLM，单模型或Claude Code等交互式工具也难以系统化地生成持久、全面的仓库级文档，尤其当代码库超过300K tokens时更为突出。
- **研究目标**：提出一种能够有效利用代码库层次结构、实现从函数到整个仓库逐层摘要的多智能体框架，以提升摘要的语义一致性、关键信息覆盖率和可读性。

### 2. 方法论：Agent4cs 核心思想与关键技术
- **总体框架**：Agent4cs 采用**自底向上**的分层摘要策略，由三个协作智能体（Agent）组成：
  - **摘要智能体（Summarization Agent）**：负责根据当前层级的输入（函数代码或下层摘要）生成摘要。
  - **关键词提取智能体（Keyword Extraction Agent）**：在生成父文件夹摘要时，主动从子文件夹摘要中提取关键信息（关键词），以浓缩下层内容、增强层级连接，同时控制输入长度。
  - **质量保证智能体（Quality Assurance Agent）**：对生成的摘要草稿进行评审，提供反馈用于迭代改进，确保可读性、连贯性和完整性。
- **函数级摘要**：支持两种策略——直接提示（Direct Prompting）和基于抽象语法树（AST）的增强提示。该阶段也包含QA智能体反馈循环。
- **层级摘要流程**（以祖-父-子三层为例）：
  1. 先完成最深层的函数级摘要。
  2. 从第三层（祖层）开始，将子文件夹摘要经关键词提取后，与当前父文件夹摘要拼接，输入摘要智能体生成草稿。
  3. QA智能体审查草稿并提供改进建议，摘要智能体据此迭代优化，直至生成最终版本。
  4. 重复上述步骤逐层向上，直至最高仓库层。
- **技术特点**：关键词提取替代直接拼接所有子摘要，有效应对上下文窗口限制；QA反馈循环提升摘要质量；整体流程轻量、无需微调，可即插即用。

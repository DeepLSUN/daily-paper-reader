## 摘要
全模态大语言模型（OmniLLMs）因能够联合处理音频和视频而受到广泛关注，但它们在音视频输入下会生成大量令牌序列，导致推理成本显著增加。现有的音视频令牌压缩方法通常依赖单模态引导，忽略了音视频输入中查询相关证据的时间局部性，并隐含假设两种模态具有时间对齐的信息密度分布。我们提出OmniFocus，一种针对OmniLLMs的无需训练的查询引导令牌压缩方法，该方法对视频和音频进行独立的显著性估计，实现模态对称的压缩设计，在保持音视频对齐的同时保留模态特定的显著证据，从而缓解单模态引导压缩可能带来的模态偏差问题。在Qwen2.5-Omni模型系列上的四个音视频基准测试实验表明，OmniFocus在低令牌保留率下仍能保持强劲的压缩性能，并在25%令牌保留率下在多个主要基准分数上优于现有基线。在DailyOmni上使用Qwen2.5-Omni-7B且令牌保留率为25%时，OmniFocus保持了59.40的准确率，同时相较于全令牌基线实现了高达1.38倍的预填充加速，展示了良好的实际准确率-效率权衡。

## Abstract
Omni modal large language models (OmniLLMs) have attracted wide attention for their ability to jointly process audio and video, but they generate large token sequences under audio-visual inputs, leading to substantial inference cost. Existing audio-visual token compression methods often rely on unimodal guidance, overlooking the temporal locality of query-relevant evidence in audio-visual inputs and implicitly assuming that the two modalities share a temporally aligned information density distribution. We propose \textbf{OmniFocus}, a training-free query-guided token compression method for OmniLLMs that performs independent importance estimation for video and audio, enabling a modality-symmetric compression design that preserves modality-specific salient evidence while maintaining audio-visual alignment, thereby mitigating the modality bias issue that can arise from unimodal-guided compression. Experiments on the Qwen2.5-Omni model family across four audio-visual benchmarks show that OmniFocus maintains strong compressed performance at low token retention ratios and outperforms existing baselines on several major benchmark scores at 25\% token retention. On DailyOmni with Qwen2.5-Omni-7B at 25\% token retention, OmniFocus maintains 59.40 accuracy while delivering up to 1.38$\times$ prefill speedup relative to the full-token baseline, highlighting a favorable practical accuracy-efficiency trade-off.

- **问题背景**：全模态大语言模型（OmniLLMs，如 Qwen2.5-Omni）能联合处理音频和视频输入，但音视频输入会生成极长的 token 序列，导致推理成本过高，限制了其在长视频和资源受限场景下的应用。

- **核心问题**：如何在不训练的前提下，对音视频 token 进行高效压缩，同时保留跨模态对齐和单模态关键证据，避免模态偏差。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **查询引导 + 模态对称**：利用文本查询对每个时间块（chunk）的视频和音频 token 分别进行重要性打分，而非仅依赖单一模态；在压缩时对两种模态采用对称的处理流程，独立分配预算与选择 token。
- **双分数选择**：同时考虑“模态间关联”（与另一模态的对齐）和“模态内峰值”（自身模态内的独特性），确保保留跨模态对应关系和单模态特异性证据。

### 关键技术步骤（文字说明）
1. **查询感知模态重要性评分**
   - 获取 LLM 输入嵌入层中查询 token 的嵌入，进行 ℓ2 归一化后取均值得到查询表示 $\bar{q}$。
   - 对每个时间块 $i$ 的视频和音频 token，分别计算每个 token 与 $\bar{q}$ 的余弦相似度，取块内最大值作为该块该模态的查询相关性得分 $s_i^v$ 和 $s_i^a$。
   - 将得分通过 Softmax 反转为压缩得分 $c_i^m = 1 - \text{Softmax}(s_i^m)$，高相关性 → 低压缩得分 → 分配更多预算。

2. **块级局部丢弃率计算**
   - 设定全局保留比例 $r_{\text{global}}^m$，将压缩得分线性映射到 $[r_{\min}=0.35, r_{\max}=0.75]$ 范围内得到初步丢弃率 $\tilde{r}_i^m$。
   - 通过二分搜索校准偏移量 $\delta$，使实际保留的 token 数满足全局预算：$r_i^m = \text{clamp}(\tilde{r}_i^m + \delta, r_{\min}, r_{\max})$。
   - 结果：不相关的块被丢弃更多 token，相关块保留更多。

3. **双分数 token 选择**
   - **模态间关联分数**（Inter-modal Association）：视频 token 与当前块音频平均表示的余弦相似度（反之亦然），保留跨模态对齐的 token。
   - **模态内峰值分数**（Intra-modal Peak）：1 减去 token 与自身模态平均表示的余弦相似度，保留模态内独特的 token。
   - 对每个块的每种模态，按保留数 $k_i^m$ 的 1:1 比例分别选取最高关联分数的 token 和最高峰值分数的 token，组合后输入 LLM。

### 公式与算法流程（文字描述）
- 公式 (2)-(4) 描述查询表示和相关性得分；公式 (5)-(6) 描述丢弃率映射与校准；公式 (7)-(9) 描述双分数计算。
- 整体流程：输入文本查询和交错音视频序列 → 分块 → 计算块级查询相关性 → 局部丢弃率分配 → 块内双分数 token 选择 → 将压缩后的序列送入 LLM 骨干。

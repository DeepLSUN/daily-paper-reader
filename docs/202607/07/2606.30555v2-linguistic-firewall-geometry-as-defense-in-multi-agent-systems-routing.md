## 摘要
大型语言模型（LLM）的快速集成推动了多智能体系统（MAS）的发展，其中专门的智能体协作执行复杂工作流。在这些环境中，有效的编排需要强大的路由机制，以高效地将任务分配给最合适的智能体。然而，现有路由器从根本上依赖未经验证的代理（从文本自描述到静态替代表示）来衡量智能体的能力。这种对非经验数据的依赖在智能体的投影概况与其实际操作能力之间造成了关键差距，引入了严重的安全漏洞。恶意智能体可以轻易地歪曲其能力或隐藏后门，从而规避标准外部分析和静态表示学习技术。在这项工作中，我们介绍了ANTAP（自动非文本智能体选择器），一种评估驱动的路由架构，它摒弃间接代理，转而采用主动能力测试。通过动态查询智能体以经验性地确定其真实能力，ANTAP将性能提炼为共享语义空间内的固定行为算子。在推理时，通过纯粹的非文本代数投影进行路由，建立了一个“语言防火墙”，使得基于元数据的攻击无法表达。在我们的实验中，ANTAP针对基于描述的注入攻击实现了接近零的攻击成功率（ASR），而基于描述的路由基线则达到67.3%及以上。针对自适应嵌入攻击，ANTAP的ASR显著低于基于嵌入的基线，降低了20%，同时其设计上对描述操纵保持鲁棒性。

## Abstract
The rapid integration of Large Language Models (LLMs) has driven the evolution of Multi-Agent Systems (MAS), where specialized agents collaborate to execute complex workflows. Effective orchestration in these environments requires robust routing mechanisms to efficiently allocate tasks to the most suitable agent. However, existing routers fundamentally rely on unverified proxies, ranging from textual self-descriptions to static surrogate representations, to gauge an agent's competence. This reliance on non-empirical data creates a critical gap between an agent's projected profile and its actual operational capabilities, introducing severe security vulnerabilities. Malicious agents can easily misrepresent their proficiencies or harbor covert backdoors that evade both standard external analysis and static representation-learning techniques. In this work, we introduce ANTAP (Automatic Non-Textual Agent Picker), an evaluation-driven routing architecture that discards indirect proxies in favor of active capability testing. By dynamically querying agents to ascertain their true competencies empirically, ANTAP distills performance into fixed behavioral operators within a shared semantic space. At inference time, routing is performed via a purely non-textual algebraic projection, establishing a "linguistic firewall" that renders metadata-based attacks inexpressible. In our experiments, ANTAP achieves near-zero ASR against description-based injection attacks, compared to 67.3\% and above for the description-based router baseline. Against adaptive embedding attacks, ANTAP achieves substantially lower ASR than the embedding-based baseline, with a 20\% reduction, while remaining resilient to description manipulation by design.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有多智能体系统（MAS）的路由机制严重依赖智能体的 **文本自描述**（如能力描述、系统提示摘要等）作为路由选择的代理。这种设计存在根本性安全漏洞：恶意智能体可以通过嵌入对抗性指令（Prompt Injection）篡改描述，从而劫持路由决策，使路由器将用户查询分配给恶意智能体。
- **背景**：MAS 中通常存在一个特权路由器，它根据智能体提供的自然语言元数据（文本）来选择处理查询的智能体。然而，这种对未经验证文本的依赖导致了 **路由器劫持**（Router Hijacking）攻击，且文本描述与实际能力之间可能存在巨大偏差，导致路由质量下降。
- **整体含义**：作者认为，要根本解决此类安全威胁，必须改变路由的信息来源——从依赖智能体“声称”的能力，转向基于其 **经验验证的行为**，从而建立一道“语言防火墙”，使基于元数据的攻击在数学上不可表达。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出 **ANTAP（Automatic Non-Textual Agent Picker）**，一种 **评估驱动、非文本** 的路由架构。它在离线阶段通过可信基准测试主动评估各智能体的真实能力，将其行为提炼为 **固定线性算子**（behavioral operator），在线路由时仅对查询嵌入进行几何投影，完全摒弃对智能体描述文本的实时处理。
- **关键技术细节**：
  - **离线阶段（注册）**：给定一组可信基准查询 B = {q₁, …, qₙ}，对每个智能体 aᵢ 执行：
    - 获取嵌入矩阵 Q ∈ Rⁿˣᵈ（使用固定句子编码器，如 all-mpnet-base-v2）。
    - 构建性能矩阵 Y ∈ Rᵏˣⁿ，元素 yᵢⱼ = +1（回答正确且无非法工具调用）或 -1（失败或非法调用）。该二元编码最大化了正负样本的几何分离度，并隐含了安全约束（非法调用被标记为失败）。
  - **学习线性算子**：通过 Tikhonov 正则化最小二乘问题求解：
    - W* = Y(QᵀQ + λI)⁻¹Qᵀ（正则化伪逆），其中 λ 默认为 1.0。该算子仅依赖基准查询，在线下缓存。
    - 添加新智能体只需 O(N) 时间：评估新智能体获 y_new，然后一次矩阵乘法 w_new = y_new · Q†ᵧ。
  - **在线阶段（推理）**：对新查询 q_new，计算嵌入 z = E(q_new)，然后通过一次矩阵-向量乘法 s = W*·z 得到所有智能体的预测效用分数，最终路由到分数最高的智能体 a* = argmax(s)。
  - **安全特性**：路由决策仅涉及固定数值算子和嵌入式向量，不处理任何文本描述，因此注入指令无法影响路由函数，形成 “语言防火墙”。

## 3. 实验设计：数据集、基准与对比方法

- **数据集**：
  - **MMLU**（通用能力评估，选择计算机安全子集）
  - **Big-Bench Hard (BBH)**（复杂推理任务）
  - **AgentHarm**（衡量对有害请求的拒绝/顺从）
- **基准（Baseline）**：
  - **AutoGen**（经典文本路由器，使用 LLM 基于描述选择智能体）
  - **EmbedLLM**（基于嵌入的路由器，使用投影矩阵进行路由）
- **对比方法**：在同一 AutoGen 框架下搭建包含 4 个良性专家和 1 个恶意智能体的 MAS。恶意智能体按攻击向量不同进行不同配置：
  - **描述注入攻击**：恶意智能体使用模拟模型（始终失败），通过描述中的 prompt injection 试图劫持路由器。
  - **潜伏智能体攻击（Sleeper Agent）**：恶意智能体使用 LLaMA2-7B-Chat 及五种数据投毒（DPA）LoRA 适配器（BadNet、Sleeper、VPI、MTBA、CTBA），触发词激活后执行非法工具调用。
  - **自适应嵌入攻击**：白盒攻击，通过贪心令牌替换优化用户查询，使查询嵌入靠近目标智能体。

## 5. 实验数量与充分性

- **实验数量丰富**：
  - **RQ1**：对比三种攻击向量（描述注入、五种 DPAs、有害请求），在 MMLU、BBH、AgentHarm 上报告准确率和 ASR，使用 bootstrap 95% 置信区间（10,000 次重采样）。

- **潜在不足**：实验主要在 MMLU 和 AgentHarm 等有限领域进行，缺乏在更真实、更复杂的多智能体工作流上的验证；对自适应攻击的评估仅在 1-16 token 范围，更长触发词的表现未探索。

## 6. 主要结论与发现

- **安全性**：ANTAP 对 **描述注入攻击** 实现 **近零 ASR**（MMLU 上 0.2%，BBH 上 0.4%），对比 AutoGen（≥67.3%）有质的飞跃；对 **数据投毒潜伏攻击**，ASR 均低于 2.4%（AutoGen 约 73%）；对 **自适应嵌入攻击**，ASR 保持在 55-63% 之间，远低于 EmbedLLM（3 个令牌后超过 86%）。
- **可靠性**：ANTAP 的路由准确率（ACC）在多数场景下优于或相当于文本路由器（例如 MMLU 65.1% vs AutoGen 12.3%），且 **延迟更低**（27ms vs 428ms）。
- **结构优势**：收益源于路由机制本身（不依赖文本），而非模型多样性或提示工程。智能体数量增加时 ASR 下降（从 3.35% 降至 0.46%），而基准上升。
- **泛化性**：在不同场景、模型同质/异质性、提示长度变化下，ANTAP 均保持稳定优势。

## 7. 优点：方法或实验设计上的亮点

- **结构防御**：通过 **完全消除推理时文本输入**，使基于元数据的攻击在数学上不可实现，而非依赖启发式过滤。
- **经验性路由**：以真实行为数据（性能 + 安全约束）为基础，而非智能体声称的能力，校准了激励（鼓励实际能力，抑制优化描述）。
- **效率与可扩展**：在线路由为 O(Kd) 矩阵乘法，与描述长度无关，适用于大规模智能体池；新智能体注册只需 O(N) 时间。
- **隐式安全训练**：通过将非法工具调用标记为失败（-1），自动将安全知识编码进行为算子，无需额外分类器。
- **透明可解释**：决策边界可通过几何投影可视化，便于理解和审计。
- **实验设计全面**：覆盖三种独立攻击向量、五种潜伏攻击变体、六类消融实验，且报告了统计显著性（置信区间）。

## 8. 不足与局限

- **依赖可信管道**：假设离线校准环境和嵌入模块完全可信；若基准数据集本身被毒化或嵌入模型被攻破，安全假设将失效（论文承认超出范围）。
- **线性近似限制**：将智能体行为建模为线性算子，可能无法捕捉高度复杂或非线性的能力模式，虽然对路由（粗粒度比对）足够，但对需精细区分能力的场景可能不足。
- **二元评分限制**：目前使用 +1/-1 的严格极性评分，忽略不同程度的成功/失败连续性，未来可探索连续或多维评分。
- **实验覆盖**：主要基于 MMLU、BBH、AgentHarm 等标准基准，缺乏在更动态、多步骤真实工作流（如代码生成、对话管理）中的验证。
- **自适应攻击评估范围有限**：仅探索了 1-16 token 的触发长度，更长触发词可能改变结论；且未评估对时序依赖攻击的鲁棒性。
- **横向对比缺失**：未与最新的安全路由方法（如 AMRO-S、MasRouter）直接比较（但理由是其仍需处理文本，保留攻击面）。

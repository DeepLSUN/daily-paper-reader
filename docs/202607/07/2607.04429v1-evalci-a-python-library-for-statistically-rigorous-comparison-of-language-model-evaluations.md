## 摘要
语言模型评估中的主流做法是报告每个模型的单一准确率数字，并声明较高的结果更好，而不测试差距是否可能只是采样噪声。在包含几千个样本的基准测试中，以及在温度采样的条件下（模型在不同次运行之间的差异可能大于模型间报告的差距），这种做法通常夸大了头条结论的可信度。解决这一问题的统计工具——置信区间、配对显著性检验、功效分析、聚类标准误、多重比较校正——已经成熟，但还没有一个标准的、可通过pip安装的工具包将其封装成评估实际采用的形式：一个逐项结果表格。我们提出evalci，一个纯Python库（仅依赖numpy/scipy/pandas），它可以通过一次函数调用将逐项结果表格转化为可发表的结论——例如，“模型A击败模型B，$\Delta=3.1$分，95%置信区间[1.2, 5.0]，配对置换检验$p=0.002$，$n=1{,}319$”——并提供了针对lm-evaluation-harness和HELM输出的适配器。每个程序都经过了独立参考（statsmodels或穷举精确枚举）的验证，而不仅仅是自我验证。作为案例研究，我们重新分析了九个语言模型在MMLU准确率上的公开比较，发现在校正排名隐含的36次两两比较后，8个相邻排行榜排名差距中的3个不具有统计显著性。evalci可在https://pypi.org/project/evalci/获取（源代码：https://github.com/Shreyaskc/evalci，DOI：https://doi.org/10.5281/zenodo.21201815）

## Abstract
The dominant practice in language model evaluation is to report a single accuracy number per model and declare the higher one better, without testing whether the gap could plausibly be sampling noise. On benchmarks of a few thousand items, and under temperature sampling where a model can differ from itself run to run by more than the reported gap between models, this practice routinely overstates confidence in headline claims. The statistical machinery to fix this -- confidence intervals, paired significance tests, power analysis, clustered standard errors, multiple-comparison correction -- is well established, but no standard, pip-installable tool packages it in the shape an evaluation actually takes: a per-item results table. We present evalci, a pure-Python library (numpy/scipy/pandas only) that turns a per-item results table into a publication-ready claim -- e.g., "Model A beats Model B, $Δ=3.1$ pts, 95% CI [1.2, 5.0], paired permutation $p=0.002$, $n=1{,}319$" -- in one function call, with adapters for lm-evaluation-harness and HELM output. Every routine is validated against an independent reference (statsmodels, or brute-force exact enumeration) rather than only against itself. As a case study, we re-analyze a public comparison of nine language models' MMLU accuracy and find that 3 of the 8 adjacent leaderboard-rank gaps are not statistically significant after correcting for the 36 pairwise comparisons the ranking implies. evalci is available at https://pypi.org/project/evalci/ (source: https://github.com/Shreyaskc/evalci, DOI: https://doi.org/10.5281/zenodo.21201815)

- **evalci** 正是为填补这一空白而设计：它把统计方法包装成接收“逐项结果表”并输出“可发表声明”的简单函数，从而降低语言模型评估中统计检验的使用门槛，提升实证研究的严谨性。

## 2. 方法论
### 核心思想
- 将语言模型评估视为一项**实验**：每个测试项上的得分是随机变量，准确率是对总体参数的噪声估计。因此，必须报告不确定性（置信区间、p值、效应量），并进行显著性检验。
- 基于**一个共享的契约**：输入为一个包含 `item_id`, `model`, `score` 以及可选的 `subset`, `sample_idx` 列的 pandas DataFrame（逐项结果表）。所有函数均基于此格式工作，并可与主流评估框架（lm-evaluation-harness、HELM）的输出直接对接。

### 关键技术细节与函数
- **置信区间** (`evalci.ci`)：
  - 对于二值得分（正确/错误）：Wilson 区间、Clopper-Pearson 精确区间。
  - 对于连续得分或非二项情形：bootstrap（百分位、BCa）。
- **配对/非配对比较** (`evalci.compare`)：
  - 三种显著性检验：配对符号翻转置换检验（小样本穷举）、零移位bootstrap、McNemar检验（精确或渐近）。
  - 同时返回配对差异的bootstrap置信区间。
  - 当项目不对齐时提供非配对版本（标签置换、独立样本bootstrap）。
- **多重比较校正** (`evalci.multi_compare`)：
  - 对所有两两模型比较进行 Holm 逐步下降或 Benjamini-Hochberg FDR 控制，输出校正后的p值和显著性标记。
- **聚类标准误** (`evalci.cluster_ci`)：
  - 以聚类为单位进行bootstrap重采样（如当同一问题被多次解码或共享同一段落时，观测不独立），给出正确的标准误。
- **功效分析** (`evalci.power`)：
  - 给定效应量、样本量或功效，通过正态近似或蒙特卡洛模拟计算统计功效，可引入相关系数 ρ 模拟跨模型共享难度。

### 验证方法
- 每个统计例程都**对照独立参考实现（statsmodels）或穷举精确枚举**进行验证，而非仅自我测试。测试套件包含100个测试，统计核心代码覆盖率达92–99%。

## 6. 主要结论与发现

  - Nemotron-4 340B vs. Llama 3 70B（Δ=1.0 pts）
  - GPT-4o vs. Claude 3.5 Sonnet（Δ=0.8 pts）
- 这表明即使排行榜上相邻位置，部分宣称的差距可能仅仅是采样噪声，**多重比较校正会进一步削弱显著性**。

## 7. 优点
- **易用性高**：pip 安装、无 GPU 依赖、一行函数调用即可生成可发布格式（Markdown/LaTeX）的统计声明。
- **适配主流框架**：提供适配器直接读取 lm-evaluation-harness 和 HELM 的输出，无需数据格式转换。
- **验证严格**：所有统计例程均对照 statsmodels 或穷举枚举验证，而非仅自我测试，确保正确性。
- **功能全面**：涵盖了置信区间、配对/非配对检验、多重比较校正、聚类标准误、功效分析等评估所需的核心统计工具。
- **开源社区友好**：MIT 许可证，代码托管在 GitHub，Zenodo 归档，持续集成覆盖 Python 3.9–3.12。

## 8. 不足与局限
- **案例研究的数据限制**：由于 MMLU 的逐项正确性未公开，只能重建 i.i.d. 二值向量，无法恢复真实项目间相关性结构（如共享难度），也无法利用更强大的配对检验。结论只给出了保守的上界。
- **上游数据不确定性**：原始表格未说明所有模型是否在同一评估框架下运行，evalci 无法校正评估协议不一致的问题。
- **蒙特卡洛分辨率有限**：置换检验和 bootstrap p 值的精度受重采样次数限制，极小 p 值需更多重采样。
- **假设限制**：Wilson 和 Clopper-Pearson 区间假设独立二项得分，不适用于连续得分或相关数据；clustering 和 power 分析中的相关系数 ρ 是简化参数，非拟合双变量模型。
- **实验范围有限**：仅作了一个案例，未进行多数据集、多模型、多场景的系统性消融实验来充分展示库在不同条件下的行为。

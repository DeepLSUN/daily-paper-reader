## 摘要
虽然自主智能体的技能优化已受到关注，但现有方法依赖复杂流程。这留下一个根本问题未解决：什么是技能优化的最小可行流程，其中每个组件都有理论或经验必要性证明？我们通过零阶优化将技能优化形式化，将经典对应物（中心差分、信赖域）映射到最新文献。注意到与经典零阶中的盲数值扰动不同，技能轨迹可作为可解释的调试反馈。基于Claude Code哲学和PAC学习，我们建立了三个收敛和泛化原则：基于文件系统的轨迹探索、共识属性挖掘和独立验证门控。消除冗余后，我们提出了SkillOpt-Lite。它加速收敛并超越完整SkillOpt：在LiveMath上提升GPT-5.5 +8.8分，GPT-5.4-nano +25.4分，使nano模型超越由SkillOpt优化的标准GPT-5.4。最后，我们将我们的框架集成到生产编码智能体中，如VSCode Copilot，使开发者能通过一行Vibe演化智能体技能。因为我们的框架将所有智能体组件简单地视为标准可编辑代码，这个最小流程自然泛化到完整框架优化（HarnessOpt）。在SpreadsheetBench上，HarnessOpt使GPT-5.4-nano达到0.7758准确率，优于运行标准流程的更大GPT-5.5（0.7620）。代码见：https://github.com/EvolvingLMMs-Lab/SkillOpt-Lite。

## Abstract
While skill optimization for autonomous agents has gained traction, existing methods rely on complex pipelines. This leaves a fundamental question unaddressed: What constitutes a minimal viable pipeline for skill optimization, where every component is justified by theory or empirical necessity? We formalize skill optimization via Zeroth-Order (ZO) optimization, mapping classical counterparts (central difference, trust regions) to recent literature. Noting that unlike blind numerical perturbations in classical ZO, skill trajectories serve as interpretable debugging feedback. Grounded in Claude Code philosophy and PAC learning, we establish three principles for convergence and generalization: file-system-based trajectory exploration, consensus attribute mining, and independent validation gating. Eliminating redundancies, we propose SkillOpt-Lite. It accelerates convergence and outperforms full SkillOpt: improving LiveMath by +8.8 points on GPT-5.5 and +25.4 points on GPT-5.4-nano, allowing the nano model to surpass standard GPT-5.4 optimized by SkillOpt. Finally, we integrate our framework into production coding agents like VSCode Copilot, enabling developers to evolve agent skills via one line of vibe. Because our framework treats all agent components simply as standard editable code, this minimal pipeline naturally generalizes to full harness optimization (HarnessOpt). On SpreadsheetBench, HarnessOpt enables GPT-5.4-nano to achieve 0.7758 accuracy, outperforming the larger GPT-5.5 running standard pipelines (0.7620). Code is available at https://github.com/EvolvingLMMs-Lab/SkillOpt-Lite.

### 核心思想
- 将智能体技能优化形式化为**零阶优化（Zeroth-Order Optimization, ZO）**：技能文本s的梯度∇sf不可解析计算，环境反馈（奖励、轨迹日志）作为标量函数f(s)的评估值。经典ZO操作（如中心差分、坐标下降、信赖域）与现有技能优化启发式建立结构对应（见表1）。
- **关键概念分歧**：经典ZO依赖盲数值扰动，而智能体技能优化具有**程序编译本质**——每次LLM rollout产生语义丰富的可解释执行轨迹，相当于程序运行日志。技能优化应视为基于语言的程序编译，轨迹作为可调试反馈。
- 基于**PAC学习理论**推导出两个关键原则：
  1. **共识属性挖掘（Consensus Mining）**：算法必须充当β_exp-稳定算子，压缩单次轨迹异常，提取跨任务不变结构，防止过拟合。对应减少稳定系数β_exp。
  2. **独立验证门控（Independent Validation Gating）**：使用严格独立于训练集的验证集进行模型选择，可完全将β_exp排除在泛化误差上界之外，避免评估偏差。

  3. **技能优化的苦教训（Bitter Lesson）**：随着基础模型规模增大，为抑制更新速度而设计的复杂拓扑结构适得其反；授予智能体原始shell工具直接检查日志文件，效果一致优于高度工程化的基线。

### 关键技术细节与流程
- **SkillOpt-Lite流水线结构**（图3(b)）：
  1. **轨迹分阶段写入磁盘**：每次批量 rollout 后，将每个轨迹独立存储为文本文件，而非聚合到结构化小批量中。
  2. **轨迹探索**：优化器模型使用文件系统工具（ls、cat等）在固定token预算下主动导航目录，聚类失败类型，选择高杠杆文件进行详细检查。
  3. **共识挖掘与最小编辑**：直接从文件中读取轨迹，发现跨任务共享失败模式；遵循最小修改原则，生成紧凑的代码差异或补丁，满足信赖域约束（有界文本更新）。
  4. **验证门控**：候选技能库立即在独立验证集上评估。若指标提升，则接受为活跃技能St+1；若超过历史最佳，则以best_skill.md覆盖磁盘文件。被拒绝的更新归档到历史日志，无需缓冲池。

- **HarnessOpt扩展**（图3(c)）：基于“一切皆文件”理念，将优化范围从单一技能文件扩展到整个框架代码库（脚本、工具调用、控制流）。设计：
  - **第0轮引导与人在回路门**：先执行完整rollout收集日志，由诊断子智能体进行共识挖掘，识别体系结构缺陷（如工具缺失、API解析错配），生成结构化配置提案，经用户批准后才修改核心代码。

## 6. 主要结论与发现

1. **SkillOpt-Lite全面超越完整SkillOpt**：在所有6个基准上，SkillOpt-Lite的平均性能均优于SkillOpt。在逻辑密集型任务（LiveMath、Spreadsheet）上提升尤为显著，如LiveMath GPT-5.5从64.8提升至73.6（+8.8分），GPT-5.4-nano从30.3提升至55.7（+25.4分），且nano模型跨越了标准GPT-5.4的SkillOpt性能（表2）。

3. **最小可行流水线有效**：消除复杂的反射池化、慢更新阻尼和拒绝缓冲不仅没有导致过早收敛或不稳定，反而通过直接的文件系统探索实现了更精确的局部编辑，避免了一致性模糊。

5. **模型行为瓶颈差异**：HarnessOpt揭示不同模型层级的瓶颈不同，例如GPT-5.4-mini和GPT-5.4主要受限于文件观测不足和最终答案验证弱，而GPT-5.5和GPT-5.4-nano常陷入重复循环，HarnessOpt自动添加退避策略解决。

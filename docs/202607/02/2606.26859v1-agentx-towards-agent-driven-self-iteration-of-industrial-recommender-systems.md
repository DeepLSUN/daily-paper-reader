## 摘要
推荐算法的迭代正从依赖人工工程师的手工流程转向工业化研究循环，但这一转变仍受到结构性执行瓶颈的阻碍：从想法到上线的周期仍依赖人类工程师生成假设、修改生产代码、启动A/B实验以及归因线上结果。因此，创新规模随人力线性增长，而非随着证据、算力和积累的实验知识复合增长。我们提出AgentX，一个生产级多智能体系统，从根本上重构了这一生产函数。AgentX作为一个自我进化的开发引擎运行：它以任何手动工作流都无法维持的规模和速度自主生成、实现、评估推荐实验并从中学习。该系统在闭环中编排四个紧密耦合的阶段。头脑风暴智能体综合历史实验、系统架构、数据分析和外部研究的证据，生成排序后的可执行提案。开发智能体通过基于仓库的生成和多维可靠性验证将每个提案转化为生产就绪代码。评估智能体通过带有防护栏否决的A/B判断进行安全在线部署，并将成功和失败转化为结构化知识资产。一个驾驭进化层（SGPO）将执行轨迹提炼为语义梯度更新，持续增强智能体自身——使系统不仅自动化，而且自我改进。

## Abstract
Recommendation algorithm iteration is moving from an artisanal, engineer-bound process toward an industrialized research loop, but this transition remains blocked by a structural execution bottleneck: the idea-to-launch cycle still depends on human engineers to generate hypotheses, modify production code, launch A/B experiments, and attribute online results. Innovation therefore scales linearly with headcount rather than compounding with evidence, compute, and accumulated experimental knowledge. We present AgentX, a production-deployed multi-agent system that fundamentally restructures this production function. AgentX operates as a self-evolving development engine: it autonomously generates, implements, evaluates, and learns from recommendation experiments at a scale and pace that no manual workflow can sustain.   The system orchestrates four tightly coupled stages in a closed loop. A Brainstorm Agent synthesizes evidence from historical experiments, system architecture, data analysis, and external research into ranked, executable proposals. A Developing Agent translates each proposal into production-ready code through repository-grounded generation and multi-dimensional reliability verification. An Evaluation Agent conducts safe online rollout with guardrail-vetoed A/B judgment, converting both successes and failures into structured knowledge assets. A Harness Evolution layer (SGPO) then distills execution trajectories into semantic-gradient updates that continuously sharpen the agents themselves -- making the system not merely automated, but self-improving.

## 2. 方法论

### 2.1 核心思想

- AgentX是一个生产部署的多智能体系统，围绕四个紧密耦合的阶段形成闭环：**头脑风暴智能体**（Brainstorm Agent）生成假设→**开发智能体**（Developing Agent）将假设转为生产代码→**评估智能体**（Evaluation Agent）安全上线并进行A/B判断→**驾驭进化层**（Harness Evolution）通过执行轨迹优化智能体本身。闭环的反馈信号是真实的在线A/B结果，而非离线指标。
- 设计原则：将模糊的业务意图逐步精炼为可执行的实验提案，确保代码与仓库一致，通过安全部署和防护栏机制验证，正负轨迹均反馈给系统实现自进化。

### 2.2 关键技术细节

- **头脑风暴智能体**：接收模糊的用户意图，通过有界探索和证据加权生成少量可执行提案。证据来源包括实验知识库（Experiment KB，记录历史上线评审和教训）、系统知识库（System KB，记录架构、特征、管道等结构化知识）、数据分析（Data Analysis，提供经验性检查）和模型研究（Model Research，将论文成果转化为可执行提案）。候选提案按成熟度分为“就绪可实施”、“需先探测”、“远景积压”三类。评分公式为综合目标对齐、业务有效性、实现可行性、交接完整度、证据加权得分和风险惩罚。
- **开发智能体**：分两条轨道——在线策略轨道（特征/策略级提案）和离线模型探索轨道（模型架构提案）。在线轨道通过基于仓库的代码生成和验证导向的实现循环确保可靠性，包含八个维度的质量评分（如属性幻觉、DSL检查、编译构建通过次数等）。离线模型轨道强调**可证伪的归因**：每个实验必须先声明声称的因果机制和预期可观测信号，训练后通过确定性代码提取指标并进行因果链验证（清晰/不清晰），避免无归因的增益进入知识库。
- **评估智能体**：负责任务安全上线（映射到正确业务域、世界、流量桶）、灰度和金丝雀发布；利用CUPED等方差缩减方法分析A/B结果，采用业务范围化、复合经济交换指标的分级防护栏机制（硬阻塞、升级、仅监控），给出KEEP/EXTEND/DISCARD的结构化判决。失败实验结果被资产化为可复用的负向记忆。
- **驾驭进化层（SGPO）**：基于语义梯度的提示词优化。SGPO-I应用于头脑风暴阶段，从执行轨迹抽取训练数据，通过评估器生成自然语言诊断（语义梯度），由改进器生成候选提示修改，然后通过配对重放（同时运行旧版和候选版在相同任务上）仅当得分提升>阈值时接受。SGPO-II应用于开发智能体，使用历史合并请求作为代码重放证据。

7. **模块化和可组合性**：头脑风暴、开发、评估、进化各阶段职责分离，可独立演进，支持人类专家智能体与AgentX协同进化。

## 8. 不足与局限

1. **算力消耗未公开**：论文未报告AgentX运行所需的具体GPU算力、存储成本和训练时间，难以量化资源效率与业务收益的平衡。
2. **长期稳定性未充分验证**：3周部署窗口相对较短；随着系统持续运行，可能出现策略同质化、探索与利用平衡问题、防护栏过严导致的长期退化等风险。
3. **泛化性受限**：实验集中在快手推荐场景，系统对其他平台（如电商、广告、社交媒体）的适用性未验证；离线模块探索仅在少量公开数据集上展开。
4. **人工干预仍有依赖**：虽然系统自动化程度高，但想法审查、A/B实验创建、灰度扩量等关键步骤仍需要人工参与，并非完全自主。
5. **安全性与可审计性挑战**：智能体自动修改生产代码和配置，若防护栏未覆盖或误判，可能引发线上事故；目前缺乏对智能体行为的全面审计机制和鲁棒性保障。
6. **进化机制局限性**：SGPO当前只更新单个子智能体规范，且依赖LLM-as-Judge生成诊断，可能存在判断偏差；配对重放虽然能过滤明显退化，但无法完全避免未来性能下降。
7. **实验充分性不足**：离线模块组合实验中，仅验证了7种单模块和少量组合，未对所有组合进行网格搜索；工业数据集转移验证仅7组，统计显著性未明确报告。
8. **负向结果的可信度**：智能体自我诊断的失败原因（如参数冲突）可能只是表面，更深层的因果原因未分析，可能隐藏系统性偏差。

## 摘要
基于大型语言模型（LLM）构建的多智能体系统（MAS）通过角色专业化和结构化交互为解决复杂任务提供了有前景的框架。然而，其性能常受到协调不足的限制，更根本的是缺乏跨智能体的细粒度信用分配。现有方法通常依赖粗粒度的反馈，难以识别哪个智能体或交互步骤导致错误。我们提出基于梯度的连接（GBC），一种用于多智能体系统细粒度归因和优化的方法。GBC将MAS建模为计算图，并引入基于梯度的连接权重，以在token级别量化每个智能体输出对下游智能体的影响。通过构建归因图并将特定任务的损失信号反向传播，我们的方法能够精确识别错误来源并进行目标性提示优化。我们进一步开发了AgentChord，一种利用基于前缀的梯度计算的高效实现。在MultiWOZ和τ-bench上的实验表明，GBC提升了多智能体性能，并优于强单智能体和多智能体基线，且更高的归因质量与更大的优化效果相关。代码可在https://github.com/yxc-cyber/AgentChord获取。

## Abstract
Multi-agent systems (MAS) built on large language models (LLMs) provide a promising framework for solving complex tasks through role specialization and structured interaction. However, their performance is often limited by miscoordination and, more fundamentally, the lack of fine-grained credit assignment across agents. Existing approaches typically rely on coarse-grained feedback, making it difficult to identify which agents or interaction steps are responsible for errors. We propose Gradient-Based Connections (GBC), an approach for fine-grained attribution and optimization of multi-agent systems. GBC models a MAS as a computational graph and introduces gradient-based connection weights to quantify the influence of each agent's output on downstream agents at the token level. By constructing an attribution graph and propagating task-specific loss signals backward, our method enables precise identification of error sources and targeted prompt optimization. We further develop AgentChord, an efficient implementation that leverages prefix-based gradient computation. Experiments on MultiWOZ and τ-bench show that GBC improves multi-agent performance and outperforms strong single-agent and multi-agent baselines, and higher attribution quality is associated with greater optimization effectiveness. Code is available at: https://github.com/yxc-cyber/AgentChord.

- **核心问题**：当前基于大语言模型的多智能体系统（MAS）虽然通过角色分工和结构化交互在复杂任务中展现出潜力，但其性能常常受到**协调不足**和**缺乏细粒度信用分配**的严重制约。现有的系统优化方法大多依赖粗粒度的反馈信号（如最终任务是否成功），难以定位错误究竟源于哪个智能体或哪一步交互，从而限制了有效的手动调试或自动优化。简而言之，如何精准地识别并修正多智能体系统中的错误源是一个核心挑战。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：作者提出**基于梯度的连接（Gradient-Based Connections, GBC）**，将多智能体系统的运行建模为一个**计算图**。通过模拟神经网络的梯度反向传播思想，在 **Token 级别**量化每个智能体输出对最终结果的影响，从而实现对错误源的精准归因和定向优化。

- **关键技术细节**：
    - **Agent 图**：将 MAS 抽象为有向无环图（DAG），每个节点是一个具有特定提示和模型的智能体，边表示信息流。
    - **梯度连接计算**：对于每个智能体的输出，计算其与所有前驱智能体输出的**连接权重**。权重计算方法有四种变体：
        - `均值/最大 L1 范数`：计算输出 Token 对前驱输出嵌入的梯度 L1 范数，衡量敏感度。
        - `均值/最大 梯度-输入乘积`：结合梯度和输入嵌入，捕捉一阶贡献（类似积分梯度思想）。
    - **归因图构建**：根据连接权重，保留权重最高的前 `m`（默认 `m=1`）个前驱连接，构建一个更精简的**归因图**。
    - **损失函数**：定义一个任务特定的**语言损失**，将最终输出与真实值进行比较，以自然语言形式编码正确性和质量信号。
    - **反向传播与优化器**：将损失信号在归因图中反向传播，提取出导致最终错误的**归因轨迹**。然后，使用一个大语言模型作为优化器，基于这些轨迹和优化历史来迭代地更新智能体的提示词。
- **算法流程**：
    1.  **前向传播**：Agent 图按拓扑顺序处理输入，产生最终输出。
    2.  **归因**：计算梯度连接权重，构建归因图。
    3.  **反向传播**：将损失通过归因图反向传播，提取归因轨迹。
    4.  **更新**：语言模型优化器根据归因轨迹更新智能体的提示词。
- **效率优化**：提出 **AgentChord** 框架，通过**前缀梯度计算**策略，将提示词作为固定前缀，仅对输入部分计算梯度，从而显著降低内存开销。

### 6. 论文的主要结论与发现

1.  **性能提升显著**：GBC 能够**显著提升**多智能体系统的性能，在多个指标上均超越未优化的多智能体基线和强单智能体基线。例如，在MultiWOZ上，Qwen-3-32B的JGA从28.9提升至54.4；在τ-bench上，Overall Reward从13.0提升至24.3，超过了单智能体的22.6。
2.  **归因质量驱动优化效果**：论文通过分析发现，**更高的归因质量与更大的优化效果正相关**。其中，使用 L1 范数（均值/最大）的连接权重方法在归因准确性和最终性能上均表现最佳，这有力地验证了 GBC 核心设计——精准归因是有效优化的关键。
3.  **误差模式洞察**：误差分析揭示了不同任务的系统性弱点。MultiWOZ 的错误主要源于**跨领域协调**和**对话状态追踪**（信息遗漏、过度预测），而 τ-bench 的错误主要源于**检索/实体识别失败**。这为未来改进提供了方向。

### 7. 优点

1.  **细粒度归因的创新性**：将梯度计算引入多智能体系统，实现了 **Token 级别的归因**，超越了传统的粗粒度反馈，这是一个重要且有新意的贡献。
2.  **端到端优化的框架**：GBC 提供了一个完整的从归因到优化的闭环框架。它不仅诊断问题，还利用诊断结果指导提示词的更新，实现了端到端的自动化优化。
3.  **良好的可解释性**：归因图和归因轨迹提供了清晰的“错误来源路径”，使系统优化过程具备一定的**可解释性**，有助于理解错误是如何在智能体间传递的。
4.  **高效的工程实现**：**前缀梯度计算**策略显著降低了内存消耗，使得在现有硬件上对大型LLM进行梯度计算和优化成为可能，是其实用性的重要保障。

### 8. 不足与局限

1.  **计算成本高**：尽管有优化，但梯度计算仍需要多次前向和反向传播，相比纯黑箱优化方法，**GPU 资源和时间成本依然较高**，限制了在更大规模系统上的应用。
2.  **依赖语言损失函数的质量**：GBC 的性能高度依赖于设计的语言损失函数。若损失函数设计不当，无法准确反映错误本质（如噪声太大），则会产生误导归因，影响优化效果。这是一个潜在的**系统性风险**。
3.  **一阶近似局限性**：GBC 基于一阶梯度来近似影响，可能无法完全捕捉智能体间复杂的**高阶非线性交互**，尤其是在长距离或长序列的任务中。
4.  **领域泛化性有待验证**：实验主要局限于任务导向对话和特定工具使用场景。其泛化能力到其他领域，如**开放式推理、代码生成、视频游戏**等，尚未得到验证。
5.  **缺乏与其他先进方法的直接对比**：未能与 `TextGrad`、`GPTSwarm` 等同类优化框架在同等实验设置下进行公平、直接地比较，这使得其先进性未能得到最强力度的证明。

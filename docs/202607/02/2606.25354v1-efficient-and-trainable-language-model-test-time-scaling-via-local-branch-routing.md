## 摘要
测试时扩展提升了语言模型的推理能力，但现有方法常面临艰难的权衡：长思维链采样仍是单线程的，而句子级或解决方案级搜索可能计算成本高昂且难以端到端训练。我们提出局部分支路由（LBR），一种词元级测试时扩展框架，它扩展一个较小的局部前瞻树，将所有采样分支前向通过语言模型，并使用轻量级路由器选择要提交的深度为1的子树。通过在候选局部未来的隐藏状态上路由，LBR允许每个词元决策利用根节点下一词元分布之外的证据，同时避免完整的解决方案级搜索。由此产生的剪枝-移动-增长解码过程保留了离散分支身份，并定义了可处理的树轨迹似然：新增长的节点在首次采样时被计数，路由器决策被赋予显式概率。这实现了带有可验证奖励的端到端强化学习，在相同的似然比原则下联合优化基础模型和路由器，与离散词元RLVR一致。在合成层次规划任务上，LBR表明候选后隐藏状态提供了有用的路由证据。在数学推理基准上，LBR在Pass@1和Pass@32上均优于离散思维链、标准离散词元RLVR以及兼容RL的软词元分支基线。这些结果表明，轻量级局部分支为语言模型测试时扩展提供了一种高效、可训练且离散的形式。

## Abstract
Test-time scaling improves language-model reasoning, but existing approaches often face a difficult trade-off: long chain-of-thought sampling remains single-threaded, while sentence- or solution-level search can be computationally expensive and hard to train end-to-end. We introduce Local Branch Routing (LBR), a token-level test-time scaling framework that expands a small local lookahead tree, forwards all sampled branches through the language model, and uses a lightweight router to select the depth-1 subtree to commit. By routing over the hidden states of candidate local futures, LBR allows each token decision to use evidence beyond the root next-token distribution while avoiding full solution-level search. The resulting prune-shift-grow decoding process preserves discrete branch identities and defines a tractable tree-trajectory likelihood: newly grown nodes are counted when first sampled, and router decisions are assigned explicit probabilities. This enables end-to-end reinforcement learning with verifiable rewards, jointly optimizing the base model and router under the same likelihood-ratio principle as discrete-token RLVR. On synthetic hierarchical-planning tasks, LBR shows that post-candidate hidden states provide useful routing evidence. On mathematical reasoning benchmarks, LBR improves both Pass@1 and Pass@32 over discrete chain-of-thought, vanilla discrete-token RLVR, and RL-compatible soft-token branching baselines. These results suggest that lightweight local branching offers an efficient, trainable, and discrete form of language-model test-time scaling.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

**核心思想**：在每个解码步骤，LBR 扩展一个小的局部前瞻树（宽度 K、深度 L），将所有采样分支前向通过语言模型，然后使用轻量级路由器选择其中一个深度为1的子树提交。所选 token 被追加到输出，未选中的子树被剪枝，选中的子树前移并重新生长，从而形成一个滚动局部前瞻树解码过程。

**关键技术细节**：
- **四阶段解码流程**：
  1. **生长局部树**：从当前前缀出发，每个活跃节点从过滤后的模型分布中采样 K 个子节点并前向传播，存储所有节点的隐藏状态。
  2. **路由选择**：路由器观察前向树，输出深度-1 子树的分布，并采样一个子树提交其根 token。
  3. **剪枝**：丢弃未选中的子树。
  4. **前移与再生**：选中子树的根 token 已提交，剩余后代成为下一解码步骤的部分前瞻树，然后生长一层新节点以恢复深度 L。
- **路由器设计**：采用集合注意力路由器（Set-attention Router）：
  - 首先将每个深度-1候选子树编码为一个向量（当 L=1 时基于单 token 隐状态；当 L>1 时汇总多个 rollout 路径）。
  - 然后通过跨子树集合注意力得到上下文感知的表示。
  - 用共享的评分头获得分数，softmax 得到路由分布。
- **轨迹似然**：树轨迹的似然被分解为两部分：
  - 新生长节点的采样对数概率（仅首次采样时计数）；
  - 路由器决策的对数概率。
  - 公式为：  
    `log p_θ,φ(F|q) = Σ_t Σ_{v in G_t} log π̃_θ(tok(v)|ctx(v)) + Σ_t log ρ_φ(k*_t|x_<t, A_t; θ)`
- **训练目标**：在可验证奖励的强化学习（RLVR）框架下，使用 GRPO 优化，损失函数为：  
  `L_LBR(θ,φ; F) = -A(F) log p_θ,φ(F|q)`  
  其中 A 是优势函数，联合优化基础模型 θ 和路由器 φ。

## 6. 论文的主要结论与发现

- **主要结论**：LBR 在数学推理基准上显著优于离散 CoT、标准离散 token RLVR 以及软 token 分支基线。Pass@1 和 Pass@32 均有提升，表明单样本准确率和多次采样探索能力都得到增强。
- **发现细节**：
  - **后候选隐状态提供有用信息**：在合成任务中，后正确候选的隐状态对最终目标的预测能力远强于预分支状态，而 LBR 的路由器能利用这种信息。
  - **交叉子树注意力提升路由质量**：消融表明，跨候选的集合注意力比独立评分更有效。
  - **训练动态**：训练过程中路由器熵降低（决策更确定），基础模型在路由位置的熵也降低，表明模型与路由器协同适应，使局部候选更易区分。
  - **深度增加带来收益**：L=2 在 7B 模型上进一步提升 Pass@1/32，说明更深的局部前瞻提供更多路由证据。
  - **与软 token 方法对比**：LBR 的离散分支避免了软 token 混合导致的概念模糊，在合成任务上通过探针证实。

## 7. 优点：方法或实验设计上的亮点

- **方法优点**：
  - **高效轻量**：仅扩展小规模局部树（K=3, L=1/2），前向成本低，路由器参数少，适合实际应用。
  - **端到端可训练**：定义了可微的树轨迹似然，能与 RLVR 无缝结合，无需外部奖励模型或手工搜索策略。
  - **保留离散分支身份**：与软 token 方法不同，LBR 维持离散 token 路径，避免了概念模糊，且后续计算基于确定性离散 token，与标准 LM 兼容。
  - **滚动树结构**：通过剪枝-前移-再生实现“边走边看”式的局部前瞻，每次仅生长一层，高效利用缓存。

  - 合成任务设计巧妙：通过 radix 翻译将概念级规划转化为 token 级层次分支，便于控制变量分析分支机制。

  - 路由器消融和深度消融充分，验证了组件贡献。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **主要局限：局部性**。LBR 使用浅层分支信息（深度最多 2），其预测能力限于局部前瞻窗口内暴露的证据，无法实现完整的长期规划或全局搜索。
- **效率-效果权衡**：增大前瞻深度 L 可提供更强路由信号，但前向成本线性增加（需前向更多节点），经济性需评估。论文未深入探讨 L>2 时的成本效益。
- **超参数敏感性**：宽度 K、深度 L、采样温度、路由温度等可能影响性能，论文未进行全面分析。
- **实验覆盖局限**：
  - 仅测试了数学推理和合成规划任务，未涉及代码生成、常识推理、多步问答等其他测试时扩展场景。
  - 仅使用 Qwen 系列蒸馏模型（1.5B/7B），未在更大模型（如 70B）或其他架构（如 Llama）上验证。

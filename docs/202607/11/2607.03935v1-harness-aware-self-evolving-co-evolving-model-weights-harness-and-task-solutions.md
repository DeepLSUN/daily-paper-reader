## 摘要
自进化框架通常优化任务解决方案，而将周围的环境视为固定不变。我们提出感知环境自进化（HASE），一种智能体强化学习框架，其中单个模型可以在多轮动作空间中生成任务解决方案或编辑选定的环境组件。HASE使得单个Qwen3-8B模型能够匹敌使用Claude Code作为环境提议器的GPT-OSS-120B模型的文本分类性能。在Alpha因子挖掘中，HASE超过了报告的GPT-OSS-120B基线。HASE还能修复不完美的评估组件，并在圆填充算法发现中收敛到最先进的性能。这些结果表明，HASE通过统一的智能体过程同时改进了环境和解决方案。

## Abstract
Self-evolving frameworks usually optimize task solutions while treating the surrounding harness as fixed. We introduce Harness-Aware Self-Evolving (HASE), an agentic reinforcement-learning framework in which a single model can generate task solutions or edit selected harness components in a multi-turn action space. HASE enables a single Qwen3-8B model to match the text-classification performance of a GPT-OSS-120B model that uses Claude Code as the harness proposer. In alpha factor mining, HASE outperforms the reported GPT-OSS-120B baseline. HASE also repairs imperfect evaluation components and converges to state-of-the-art performance in circle-packing algorithm discovery. These results show that HASE improves the harness and the solution through one unified agentic process.

### 1. 论文的核心问题与整体含义（研究动机和背景）
现有自进化（self-evolving）系统（如FunSearch、AlphaEvolve、OpenEvolve等）通常只优化**对象级解决方案**（代码、启发式策略），而将周围的环境（Harness）视为固定不变。这里的Harness包括提示词、记忆、检索策略、工具接口、评估器等组件。这种固定假设存在两个主要局限：
- 人工编写的评估器可能不完整或与外部环境不一致，模型在分数驱动优化下可能利用这些漏洞（reward hacking）而非解决真正任务。
- 即使评估器正确，引导组件（如提示词、记忆）可能提供差搜索先验，限制了模型改进其操作的计算环境的能力。

论文提出**Harness-Aware Self-Evolving (HASE)**，让同一模型拥有一个**统一的动作空间**，既能生成解决方案，又能编辑选定的Harness组件，从而实现**模型权重、Harness、任务解三者的共同进化**。

### 2. 论文提出的方法论：核心思想、关键技术细节
**核心思想**：将Harness组件分为两类，分别采用不同的进化策略：
- **引导组件**（Guidance components）：如提示词、记忆格式化、检索脚本。可以自由编辑以改善搜索质量。
- **评估组件**（Evaluation components）：如验证器、评分器。编辑受限于真实世界反馈，仅当本地评估器分数（代理分数）与真实评估器分数（神谕分数）不一致时才对评估器进行修复，防止无约束的奖励黑客行为。

**关键技术细节**：
1. **统一动作空间**：模型在多轮交互中可以提交解决方案或使用沙盒命令（ls, cat, grep, replace等）检查/修改Harness。模型没有硬编码的计划器来决定何时编辑Harness，而是由当前上下文决定有用动作。
2. **强化学习训练**：使用**组相对策略优化（GRPO）**。每阶段包含多个GRPO epoch，模型生成多轮轨迹，最终提交解决方案。奖励只基于解决方案质量，对Harness编辑动作本身不直接奖励，但若编辑后方案分数提升，则全局轨迹获得高奖励。
3. **阶段级审查**：
   - **引导组件**：将每个轨迹中导致分数提升的解析候选编辑放入阶段候选池，阶段结束时按验证分数筛选并提交最佳编辑到持久Harness。
   - **评估组件**：阶段结束时查询真实评估器，若代理分数与神谕分数不一致，则将该情况添加到**不匹配集M**。评估器补丁必须修复M中的不匹配并保持接口完整性才会被接受。
4. **算法流程**（Algorithm 1）：
   - 初始化策略πθ、持久Harness h、可选不匹配集M。
   - 每阶段内进行多个GRPO epoch，生成轨迹，用本地评估器评分，GRPO更新策略。
   - 阶段边界处进行评估器修复（若允许）和引导组件审查，提交最佳Harness更新。

### 7. 优点：方法或实验设计上的亮点
- **统一框架**：单一模型同时处理任务求解和Harness编辑，无需外部强大提议器（如Claude Code），节省成本。
- **安全性设计**：评估器编辑受真实世界反馈锚定，只有代理与神谕不一致时才允许修改，防止奖励黑客。
- **阶段级审查机制**：引导组件通过PoC分数短名单+验证提交，评估组件通过不匹配集+回归测试，避免单次幸运轨迹直接改变持久Harness。
- **广泛的实验覆盖**：涵盖引导组件和评估组件两种类型，且包含消融实验（冻结vs进化）、未来信息泄露检测、多种子报告，证明了方法的鲁棒性。
- **实际应用价值**：Alpha挖掘中达到IR 1.70，远超现有方法，在金融领域具有明确实用性。

### 8. 不足与局限
- **非全面缩放研究**：仅使用8B模型，未测试更大模型（如70B、120B）在HASE下的行为。作者承认每个领域仍有更难的设置未探索。
- **保守的工程设计**：
  - 可编辑Harness组件是白名单的，可能排除有用但更有侵入性的修改。
  - 评估器编辑需要真实世界反馈，这在某些场景（如实时系统）可能不可用。
  - 阶段边界审查可能拒绝一些暂时不兼容但未来有用的修改。
- **泛化性风险**：圆填充和Heilbronn中对评估器的修复依赖预先定义的不匹配集，可能无法覆盖所有漏洞。

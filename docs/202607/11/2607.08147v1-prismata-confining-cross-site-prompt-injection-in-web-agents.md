## 摘要
自主Web代理有望自动化日常浏览任务，但继承了Web最古老的攻击面之一。跨站脚本已经证明，即使是在良性页面上，混合可信与不可信内容也是危险的。代理通过将自然语言解释为指令重新引发了这一风险，使得第三方和用户生成的内容能够通过提示注入劫持代理。核心挑战在于，推导任务特定的安全策略需要对与攻击者内容纠缠在一起的页面结构进行推理。我们提出Prismata，一种为Web代理实施上下文最小权限的防御机制，既约束代理能看到的，也约束代理能做的。Prismata的动态信任推导为页面内容生成权限标签，具有结构限制保证，受经典完整性模型启发，限制任何标注错误，使得标签只能降低权限，且错误标注有界。Prismata的机械限制通过删改内容和限制代理能力来强制执行这些标签。重要的是，这些机制不需要开发者注释，因此Prismata支持长尾网站。在最近发布的Web代理攻击（包括自适应变种）中，Prismata显著降低了攻击成功率，同时保持了良性任务的效用。

## Abstract
Autonomous web agents promise to automate everyday browsing tasks, but inherit one of the web's oldest attack surfaces. Cross-Site Scripting proved that mixing trusted and untrusted content is dangerous, even on benign pages. Agents resurface this risk by interpreting natural language as instructions, allowing third-party and user-generated content to hijack the agent via prompt injection. The core challenge is that deriving a task-specific security policy requires reasoning over page structure that is entangled with the attacker's content.   We present Prismata, a defense enforcing contextual least privilege for web agents, constraining both what the agent sees and what it can do. Prismata's dynamic trust derivation produces permission labels for page content, with structural confinement guarantees, inspired by classical integrity models, that bound any labeling errors so that labels can only decrease in privilege and mislabelings are bounded. Prismata's mechanical confinement enforces these labels by redacting content and restricting agent capabilities. Importantly, these mechanisms require no developer annotations, so Prismata supports the long tail of websites. Across recent published web agent attacks, including adaptive variants, Prismata substantially reduces attack success while preserving benign task utility.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：自主Web代理在浏览网页时会将自然语言解释为指令，因此继承了一个古老的攻击面——即使是良性网站，也会混合可信内容（开发者内容）与不可信内容（用户评论、外部广告等）。攻击者可以通过在这些不可信区域注入恶意提示（称为**跨站提示注入，XSP**），劫持代理执行有害操作（如泄露用户信用卡信息、修改账户设置等）。
- **研究动机**：已有的XSS防御（输入净化、沙箱等）无法直接迁移到XSP场景，因为注入载荷是自然语言而非可执行代码，LLM无法区分指令与数据，且外部内容无法在LLM上下文中被沙箱化。现有防御分为模型级防御（依赖LLM自身抗性）和系统级防御（应用经典安全原则），但前者易被自适应攻击绕过，后者（如CaMeL）要求预规划所有动作，难以处理数据依赖的Web任务，且无法提供细粒度的页面级策略。
- **整体含义**：需要一种无需开发者注释、能够动态推导页面上下文中最小权限的防御机制，既能约束代理的观察范围，又能约束其可执行的动作，从而阻断XSP攻击。

## 2. 论文提出的方法论

### 2.1 核心思想：上下文最小权限（Contextual Least Privilege）

- 对于每个交互式元素，追踪其从DOM根节点到该元素的**关键路径（critical path）**。该路径几乎完全由开发者编写的结构元素组成，极少包含不可信内容。
- 基于关键路径和用户任务，通过LLM判断元素是否在任务范围内，并赋予能力标签（可交互/只读/禁止）。
- 结合经典的**Biba完整性模型**（1977），对关键路径进行递归解析：在展示子内容之前，先通过祖先元素的结构信号（如标题、aria-label、class名）推断是否存在不可信内容，从而在不暴露攻击载荷的情况下做出标签决策。

### 2.2 关键技术细节

1. **动态信任推导（Dynamic Trust Derivation）**
   - **动作门（Action Gate）**：为每个交互元素独立评估其关键路径，仅参考任务和路径上的结构上下文，不允许看到页面上其他内容。若攻击者注入不在关键路径上，则标签完全不受影响（Case 1）。
   - **Biba解析（Biba Parsing）**：沿关键路径递归，每层只看到当前节点及其祖先，屏蔽子节点和兄弟节点（no-read-down），一旦某节点被标记为不可信，则其所有后代继承相同标签且不再深入查询（no-write-up）。若路径上存在结构信号（如“Reviews”标题），则在暴露不可信内容前就将其降级（只读）或裁剪（Case 2）。
   - 综合两者取最小值作为元素最终能力。

2. **机械限制（Mechanical Confinement）**
   - 在浏览器与代理之间插入防御层，对捕获的DOM快照进行标注，生成元素-ID到能力标签的映射。
   - 对观察进行过滤：移除不在任务策略内的节点，将只读元素的目标ID删除或禁用。
   - 对代理的动作进行拦截：在动作到达Playwright之前，检查目标标识符是否具有足够能力，若不足则拒绝执行。
   - 整个执行是确定性的代码，独立于LLM推理。

3. **三种注入位置情形**（表1）
   - Case 1：注入不在关键路径上 → 动作门给出无注入标签。
   - Case 2：注入在关键路径上，但有结构信号 → Biba解析在暴露前给出无注入标签。
   - Case 3：注入在关键路径上且无结构信号 → 这是残余风险，经验上非常罕见（约0.1%样本路径，在有最佳实践的网站上降至0.017%）。

### 2.3 安全性论证（非正式证明）

- 攻击者必须同时绕过动作门和Biba解析才能提升元素权限；机械限制不可绕过（闭环动作空间，无任意代码执行）。
- 错误标签只会影响包含注入的关键路径上的元素，其他元素不受影响。

## 7. 优点

- **无需开发者标注**：自动从DOM结构推断权限，支持长尾网站。
- **形式化安全保证**：基于Biba完整性模型提供了标签边界保证，即使模型出错，错误也仅局限在包含注入的关键路径内。
- **分层防御**：动作门和Biba解析双重机制，覆盖不同注入位置，机械限制确定性执行，不依赖LLM鲁棒性。
- **实用性**：在实际环境中测量了延迟和成本，并展示缓存的有效性，表明可部署。
- **可扩展性**：提出模块化评估框架，便于未来添加新攻击和防御。
- 在自适应攻击和WASP等公开基准上验证了有效性，证据充分。

## 8. 不足与局限

- **残余风险Case 3**：极少数无可信信号的关键路径上的注入仍可能影响标签（约0.1%，最佳实践下0.017%）。作者承认该情况无法完全保证无注入。
- **仅保护文本模态**：不支持图像、音频等多模态输入（如梯度扰动图像、非文本越狱）。作者指出未来可扩展但未实现。
- **任务范围假设**：Prismata仅保护任务权限之外的操作；如果任务本身需要阅读用户评论（如选择性购买），则虚假评论依然可能影响行为（属于平台审核问题）。
- **模型依赖性**：标签精度受限于底层LLM能力，虽然验证了多个模型，但仍有误标风险。
- **标注验证规模有限**：人类标注仅覆盖20个观察，可能存在疲劳和误差，且未对极端边缘案例进行全面测试。
- **攻击场景局限**：实验仅使用弹出式攻击模板，未覆盖所有可能的XSP变体（如通过图像注入、跨页面链式注入等）。
- **行动空间假设**：假设代理使用BrowserGym的有限动作接口（无任意JS执行）。若未来代理支持更开放的动作，则需额外加固。
- **与真实Web的差距**：实验基于WebArena模拟环境，DOM结构可能与真实Web存在细微差异，但作者指出WebArena基于真实开源栈，保真度较高。

## 摘要
自主Web代理有望自动化日常浏览任务，但继承了Web最古老的攻击面之一。跨站脚本已证明，混合可信与不可信内容是危险的，即使在良性页面上也是如此。代理通过将自然语言解释为指令重新浮现这一风险，允许第三方和用户生成的内容通过提示注入劫持代理。核心挑战在于，推导任务特定的安全策略需要对页面结构进行推理，而该结构与攻击者的内容纠缠在一起。
我们提出Prismata，一种为Web代理强制执行上下文最小权限的防御机制，同时限制代理能看到的内容和能执行的操作。Prismata的动态信任推导为页面内容生成权限标签，并具有结构约束保证，灵感来自经典完整性模型，该模型限制了任何标签错误，使得标签只能降低权限，且错误标记被限制。Prismata的机械约束通过编辑内容和限制代理能力来强制执行这些标签。重要的是，这些机制不需要开发者注释，因此Prismata支持长尾网站。在最近发布的Web代理攻击（包括自适应变体）中，Prismata显著降低了攻击成功率，同时保持了良性任务的实用性。

## Abstract
Autonomous web agents promise to automate everyday browsing tasks, but inherit one of the web's oldest attack surfaces. Cross-Site Scripting proved that mixing trusted and untrusted content is dangerous, even on benign pages. Agents resurface this risk by interpreting natural language as instructions, allowing third-party and user-generated content to hijack the agent via prompt injection. The core challenge is that deriving a task-specific security policy requires reasoning over page structure that is entangled with the attacker's content.   We present Prismata, a defense enforcing contextual least privilege for web agents, constraining both what the agent sees and what it can do. Prismata's dynamic trust derivation produces permission labels for page content, with structural confinement guarantees, inspired by classical integrity models, that bound any labeling errors so that labels can only decrease in privilege and mislabelings are bounded. Prismata's mechanical confinement enforces these labels by redacting content and restricting agent capabilities. Importantly, these mechanisms require no developer annotations, so Prismata supports the long tail of websites. Across recent published web agent attacks, including adaptive variants, Prismata substantially reduces attack success while preserving benign task utility.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：自主Web代理将自然语言解释为指令，导致第三方和用户生成内容可通过**跨站提示注入（Cross-Site Prompting, XSP）** 劫持代理行为。例如，攻击者在产品评论中嵌入恶意指令，使代理执行泄露信用卡信息等危险操作。
- **根本挑战**：制定任务特定安全策略需要对页面结构进行推理，但该推理过程本身与攻击者控制的不可信内容纠缠在一起，称为**Web纠缠问题（Web Entanglement Problem）**。现有防御（如CaMeL）依赖结构化工具API的语义，而Web代理的动作（如`click(id)`）无语义含义，必须结合页面上下文才能判断安全性。
- **背景**：XSP是跨站脚本（XSS）在LLM时代的对等物。XSS的防御（输入消毒、沙箱）无法迁移，因为自然语言文本中指令与数据不可区分。现有模型级防御易被自适应攻击绕过，系统级防御要么需预规划动作，要么需开发者手动注释，难以扩展到长尾网站。
- **整体含义**：本文提出Prismata，一种**上下文最小权限**防御，无需开发者注释，适用于任何现有网站，可显著降低XSP攻击成功率并保持良性任务效用。

## 2. 论文提出的方法论

### 核心思想
- **动态信任推导**：从DOM结构中提取每个可交互元素的**关键路径（critical path）**，利用结构线索（如标题、ARIA属性、class名）推断内容来源，生成权限标签（开发者/用户/托管方/外部）。灵感来自Biba完整性模型，确保标签只能降低权限（no-read-down, no-write-up），且误标记被限制在局部。
- **机械约束（Mechanical Confinement）**：基于标签规则，对不可信内容进行**删除或降级为只读**，并限制代理只能执行任务范围内的动作。约束在捕获的页面快照上执行，不修改实时DOM，引入的页面漂移与标准观察-动作循环一致。

### 关键技术细节
1. **动作门（Action Gate）**：为每个可交互元素独立调用LLM，仅输入其关键路径和用户任务，判断该元素是否在任务范围内。攻击者内容若不在关键路径上，则无法影响决策（情况1）。
2. **Biba解析（Biba Parsing）**：递归处理关键路径，逐层解析当前节点的来源属性。遇到结构线索（如`<h2>Reviews</h2>`）时，将后续子树标记为不可信并降级权限，不再继续查询子节点（情况2）。对于任务必需但不可信的内容（如购买需读评论），保留只读权限。
3. **策略模型**：仅根据任务（无DOM内容）判断需要保留哪些来源类的内容，实现全局剪枝。
4. **组合策略**：每个元素的最终权限 = min(动作门赋值, Biba解析赋值)。攻击者必须同时绕过两者才能提升权限（情况3，经验上仅占0.1%的路径）。

### 安全论证（非正式证明）
- 情况1：注入不在关键路径 → 动作门标签无注入影响。
- 情况2：注入在关键路径但结构线索先出现 → Biba解析在暴露注入前已完成标签锁定。
- 情况3：注入在关键路径且无结构线索 → 残留风险，但仅影响该元素路径上的内容，其余元素仍受约束。
- 动作空间封闭（仅限标识符的点击/填充等），不支持任意JS，机械约束为确定性代码，无法绕过。

## 3. 实验设计

- **数据集/场景**：
  - **WebArena**：基于GitLab、购物、Reddit论坛、CMS等生产级开源栈搭建的可靠环境，提供真实DOM结构。
  - **Common Crawl**（Tranco top 10K域，每域100页）和**Mind2Web**（任务轨迹），共5,664个DOM用于经验验证。
  - **攻击模板**：三种弹出式攻击——**Shortcut**（提供“一键完成”捷径）、**Fake Completion**（假装任务已完）、**Ignore Instruction**（覆盖原始指令）。另加**WASP**基准测试（邮件更改任务）和**自适应WA20**攻击。
- **基准**：基线为无Prismata的Web agent（gpt-5.4-mini）。对比模型包括gpt-5.4、gpt-5.4-nano、Gemini 3 Flash、GPT-OSS 120B、Qwen3 8B、MiniMax M2.5等。
- **评估指标**：
  - **攻击成功率（ASR）**：代理与恶意内容交互。
  - **任务成功率（TSR）**：WebArena标准（程序化检查+LLM评判+字符串匹配）。
  - **防御成功率（DSR）**：Prismata是否移除或降级攻击内容。
  - **允许元素标签精度/召回/F1**：比较模型标签与人类专家标签的允许元素集合。
- **验证研究**：20个观察样本（每域4个），由Web安全专家手动标注，与模型标签对比。

## 6. 论文的主要结论与发现

1. **攻击缓解效果显著**：有Prismata时，平均ASR从85.5%降至0.7%，平均TSR从4.5%提升至23.0%（+18.5pp，相对提升约5.1倍）。在Completion和Ignore攻击下ASR降至0%，Shortcut降至2.1%。
2. **良性任务效用损失小**：无攻击时TSR从29.9%降至26.6%（仅下降3.3pp）。
3. **WASP/自适应攻击完全阻断**：Prismata消除了WA20自适应攻击成功，并阻止了WASP邮件更改攻击。
4. **标签精度高**：所有测试模型精度>93%，F1>87%。gpt-5.4-nano精度最高（98.69%），但召回最低（82.69%）；Gemini 3 Flash F1最高（95.14%）。
5. **经验域支撑**：在Common Crawl和Mind2Web语料中，99.8%+的不可信路径满足Prismata的安全条件（情况1或2），仅有约0.1%属于情况3（残留风险）。
6. **缓存效率**：DOM结构跨页面重用，缓存覆盖率随同一站点DOM数量增加而快速提升，降低了推理成本。

## 7. 优点

- **无需开发者注释**：完全基于页面现有结构（HTML属性、祖先路径、class名等），支持长尾网站，可部署于现有环境。
- **形式化安全保证**：基于Biba完整性模型，通过动作门和Biba解析的组合，提供可证明的约束（在封闭动作空间下）。
- **系统级防御**：独立于模型，不依赖LLM的对抗鲁棒性，机械约束为确定性代码，不可绕过。
- **细粒度上下文最小权限**：不仅限制可见内容，还限制交互能力，区分只读/读写/删除。
- **通用性**：可防御多种XSP变体（Shortcut、Fake Completion、Ignore Instruction）及WASP这类真实场景。
- **实用考虑**：设计了缓存层降低部署成本，并提供了模块化评估框架（prismata evaluation framework），便于未来扩展攻击和防御。

## 8. 不足与局限

- **非文本攻击未覆盖**：只处理文本模态，不覆盖图像、音频等非文本注入（如梯度扰动、视觉越狱）。作者承认多模态代理仍易受此类攻击。
- **依赖于模型精度**：标签错误会导致权限放行（降低安全性）或过度限制（降低效用）。虽然论文验证了高精度，但最坏情况下单个关键元素误标可能允许攻击通过。
- **残留风险：Case 3**：约0.1%的路径上注入位于关键路径且无结构线索，此时Prismata无法保证标签不受影响。作者指出在遵循Web最佳实践的站点上该风险降至0.017%，但仍非零。
- **页面漂移**：Prismata操作的是捕获的快照而非实时DOM，快照与实时的差距既影响代理也影响Prismata，这是所有基于序列化文本的Web代理的固有局限。
- **高风险任务不适用**：如果任务需要完全信任所有来源内容（如操作评论功能），最小权限会限制必要交互。作者建议未来使用隔离代理处理此类情况。
- **实验规模有限**：WebArena环境仅包含少量网站和任务，真实世界的长尾网站结构可能更复杂。验证研究仅20个观察样本，统计代表性有限。运行次数少（3次）可能低估方差。
- **计算成本**：虽然缓存有效，但Prismata仍需大量LLM调用（动作门+ Biba解析）。在成本敏感场景下可能不实用。

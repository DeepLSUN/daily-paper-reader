## 摘要
基于大语言模型（LLM）的网页搜索智能体正在将信息检索从简单的事实性问答转变为复杂的深度与广度搜索及研究导向型任务。单个ReAct风格的智能体受限于单一长轨迹和有限的上下文，难以同时处理深度和覆盖范围。现有的多智能体系统通过并行执行和聚合提高了搜索覆盖范围，但在递归深度、协作适应性和基于证据的扩展方面仍存在明显不足。我们提出WebSwarm，一种渐进式递归委派框架，在推理过程中联合构建任务分解、递归扩展和智能体协作。WebSwarm动态实例化智能搜索节点，每个节点将局部目标与指定节点如何组织搜索和协作的搜索模式相结合。每个节点可以自行解决其目标，或进一步委派子节点；解决后，它向上返回证据和结果，使父节点能够进一步扩展、修订或聚合搜索过程。为指导这一过程，WebSwarm首先探测任务相关信息在网页上的组织方式，以奠定后续节点扩展的基础，并在同质兄弟节点之间重用过程级经验。在BrowseComp-Plus、WideSearch、DeepWideSearch和GISA上的实验表明，WebSwarm在深度、广度以及交织的深度与广度任务上始终优于单智能体和多智能体基线。进一步的消融分析、任务难度、网页工具效率及模型泛化性分析解释了WebSwarm的有效性，并为多智能体搜索系统提供了见解。

## Abstract
Large language model (LLM)-based web search agents are transforming information seeking from simple factoid question answering into complex, deep-and-wide search and research-oriented tasks. A single ReAct-style agent is constrained by one long trajectory and limited context, making it difficult to handle depth and coverage simultaneously. Existing multi-agent systems improve search coverage through parallel execution and aggregation, but still exhibit clear limitations in recursive depth, collaboration adaptability, and evidence-grounded expansion. We propose WebSwarm, a progressive recursive delegation framework that jointly constructs task decomposition, recursive expansion, and agent collaboration during inference. WebSwarm dynamically instantiates agentic search nodes, each coupling a local objective with a search mode that specifies how the node should organize search and collaboration. Each node can either solve its objective itself or further delegate child nodes; after solving, it returns evidence and results upward, enabling parent nodes to further expand, revise, or aggregate the search process. To guide this process, WebSwarm first probes how task-relevant information is organized on the web to ground subsequent node expansion, and reuses process-level experience across homogeneous sibling nodes. Experiments on BrowseComp-Plus, WideSearch, DeepWideSearch, and GISA show that WebSwarm consistently outperforms single-agent and multi-agent baselines on deep, wide, and interleaved deep-and-wide tasks. Further analyses of ablation, task difficulty, web tool efficiency, and model generalization explain WebSwarm's effectiveness and provide insights for multi-agent search systems.

## 1. 核心问题与整体含义（研究动机与背景）

- **背景**：基于大语言模型的网络搜索智能体正从简单事实问答转向复杂的深度与广度搜索（如研究级搜索、报告调查）。单个 ReAct 风格智能体被单一长轨迹和有限上下文所限制，难以同时兼顾深度（多跳约束推理）和广度（覆盖大量实体/来源）。
- **现有多智能体系统**的不足：
  1. **递归深度浅**：大部分仅限于根级任务分解，深层子任务仍需在子智能体中展开长轨迹，接近单智能体。
  2. **协作适应性弱**：使用固定全局协作模式（如多样本聚合、串行传递、并行分治），无法随着搜索过程灵活切换不同子任务所需的求解协议（如快速事实查找 vs. 开放集枚举）。
  3. **证据基础不足**：任务分解仅基于查询表面语义，未对齐网页信息实际组织方式（集中型 vs. 分散型），导致过度分解或错误维度展开。
- **本文目标**：提出 WebSwarm，一种渐进式递归委派框架，使任务分解、递归扩展和智能体协同随证据积累而动态构建。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将复杂搜索组织为动态构建的递归委派树 `T = (V, E)`：
  - 根节点接收原始任务，委派子节点。
  - 每个搜索节点 `v` 拥有 **局部目标 `q_v`** 和 **搜索模式 `m_v`**（决定了求解协议和协作结构）。
  - 节点可自行求解（调用网页工具）或进一步委派子节点；子节点返回结果后，父节点决定继续扩展、修订或聚合。
  - 过程交替进行自上而下的委派和自下而上的证据反馈，实现渐进式结构展开。

- **四种搜索模式** `M = {atom, deep, wide, entity_collect}`：
  - `atom`：原子事实查找，单智能体直接调用工具。
  - `deep`：未知目标识别或多约束推理，串行执行“搜索器-验证器”对（搜索器提出候选，验证器独立校验）。
  - `wide`：并行分治，适用于在多个对象/维度上收集类似信息（如填充多实体属性），子节点可为任意模式。
  - `entity_collect`：开放集枚举，同时派发多个 `atom` 节点从不同来源召回候选，合并去重并验证低置信度条目。

- **两个指导信号**：
  1. **网页结构探测（Web-Probing Agent）**：在需要广泛扩展的节点前，进行轻量级搜索并阅读网页，返回信息组织提示（`centralized`, `centralized_with_gaps`, `distributed`），指导后续委派的维度与粒度。
  2. **内部经验重用（Within-Instance Experience Transfer）**：对同一父节点下的同质子节点，先执行少量探路节点，从轨迹中提取可复用的查询模式、可靠来源、无效路径等过程经验，注入后续兄弟节点以提升求解质量和一致性。

- **算法流程**（伪代码概述）：
  - 根节点创建 → 调用 `SolveNode`（包含循环直到目标解决）。
  - 每一步：可能调用 Web-Probing 获取提示 → 决定是直接求解还是委派子节点。
  - 若委派，先执行探路子集并提取经验，再用经验指导其余子节点。
  - 子节点结果返回后更新证据，循环直到聚合后返回上级。

## 6. 主要结论与发现

- **性能领先**：WebSwarm 在四个基准上一致超越所有单智能体和多智能体基线。在 BrowseComp-Plus 上准确率提升 17.50（相比 ReAct），在 WideSearch 上 Row-F1 提升 10.91，Item-F1 提升 9.76，在 DeepWideSearch 和 GISA 也取得最佳或竞争结果。
- **递归委派与模式匹配至关重要**：消融实验显示，移除递归委派或强制单模式均导致性能下降，说明灵活适配子任务需求的重要性。
- **Web-Probing 提升效率，经验重用提升质量**：移除 Web-Probing 后工具调用数大幅增加（如 WideSearch 从 137 → 240），而移除经验重用则直接降低 Item-F1。
- **随难度增加优势更明显**：在困难子集上（Hard），WebSwarm 将 BrowseComp-Plus 准确率从 0% 提升至 35.7%，WideSearch Item-F1 从 24.5 提升至 55.8%，突破其他方法天花板。
- **模型无关性**：在较弱（Qwen3-32B）和较强（Qwen3.5-35B）骨干上均持续改进，表明方法具有普遍适用性。

## 7. 优点

- **方法论创新**：首次将递归委派与多样性搜索模式（atom/deep/wide/entity_collect）耦合，允许搜索结构随证据积累动态展开，而非静态预先规划。
- **指导信号设计精巧**：Web-Probing 轻量探测网页信息拓扑，避免盲目展开；内部经验重用跨同质子节点共享过程知识，减少重复试错。
- **实验全面且公平**：覆盖四个有代表性的基准、多种基线（包括近期的 SOTA）、多骨干模型、多难度层次，并有详细消融和案例研究，验证了各组件贡献。
- **实际意义**：为复杂网络搜索任务提供了可扩展的推理时架构，直接提升了 agent 在挑战性查询上的能力，尤其对长尾、高难度查询效果显著。

## 8. 不足与局限

- **未涉及多智能体训练优化**：WebSwarm 仅聚焦推理时编排，未进行数据构建或策略学习，节点委派和模式选择完全依赖基础 LLM 的理解与推理能力，可能无法充分发掘模型潜力。
- **资源消耗较高**：相比单智能体 ReAct，WebSwarm 需要更多 LLM 调用和网页工具请求（尤其在困难任务上），导致更高的推理成本和延迟。
- **模态覆盖有限**：当前仅基于文本网页工具，未处理图像、视频、音频等多模态网络信息。

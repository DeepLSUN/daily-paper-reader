## 摘要
当前的智能体工作流通常涉及将用户请求分解为具有正确解析参数的工具调用序列，其结果通过语言模型上下文窗口中的推理轨迹进行处理。改进此类推理的主流方法是测试时扩展，即训练模型在长思维链上搜索；但由此产生的能力与模型权重纠缠在一起，无法逐步验证，并且推理成本高昂。我们提出Forethought，一种神经符号推理系统，它将推理视为一个明确、可验证的程序，构建于通过领域特定语言组合的符号和神经原始函数库之上。其结果是推理程序，即模型工作的具体表示，因此可以在部署前进行检查和修改。作为工具调用执行内核实例化，并在五个基准上进行评估，Forethought将基础模型准确率相对提升约30%，并优于基础提示、强化学习脚手架和提示进化方法，使小型模型能够匹配或超越前沿模型的能力。在直接比较中，一个非推理模型通过Forethought增强后，与专用推理模型竞争，同时需要的训练后投资减少了约三个数量级，并且保持模型无关且可审计。

## Abstract
Current agentic workflows usually involve decomposing user requests into sequences of tool calls with correctly resolved parameters, the results of which are processed through reasoning traces in the language model's context window. The prevailing route to improve such reasoning is test-time scaling, which trains models to search over long chains of thought; but the resulting capability is entangled in model weights, is not verifiable step-by-step, and is costly at inference. We present Forethought, a neurosymbolic reasoning system that instead treats reasoning as an explicit, verifiable program, that builds from a library of symbolic and neural primitives which are composed through a domain-specific language. The result are reasoning programs, which are concrete representations of the model's work, and as such can be inspected and modified before deployment. Instantiated as a tool-calling execution kernel and evaluated across five benchmarks, Forethought improves base-model accuracy by about 30% relative and outperforms vanilla prompting, reinforcement learning scaffolds, and prompt-evolution methods, enabling small models to match or exceed frontier models capabilities. In a direct comparison, a non-reasoning model augmented with Forethought competes with a dedicated reasoning model while requiring roughly three orders of magnitude less post-training investment, and remains model-agnostic and auditable.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前大语言模型（LLM）在智能体工作流中需要多步推理和工具调用，但主流改进方法——测试时扩展（如 DeepSeek-R1）存在三大缺陷：
  - 推理能力与模型权重纠缠，无法迁移。
  - 推理过程是随机的、不可逐步骤验证的。
  - 推理成本高昂，往往产生数倍于任务本身的 token 消耗。
- **整体含义**：需要一种将推理显式化、可验证、模型无关且成本高效的替代方案。Forethought 提出将推理视为可编程的神经符号程序，通过组合带输出契约的原语，实现设计时验证，从而在提升准确率的同时满足审计需求。

## 2. 论文提出的方法论

### 核心思想
- 将推理分解为一系列狭窄的推理原语（Primitive），每个原语有明确的输出契约（Output Contract）。原语分为两类：
  - **符号原语**：确定性操作（如类型检查、格式验证），输出可完全验证。
  - **SLM 原语**：基于微调的小语言模型（如 Gemma 12B），负责实体提取、依赖判定等自然语言操作，其输出通过契约在运行时检查。
- 通过 Python 嵌入式领域特定语言（DSL）将原语组合成结构化的推理程序。DSL 提供序列、并行、条件、迭代、回退等组合算子，支持类型检查、成本估计和正确性推理。
- 推理程序是可分析的数据结构（AST），可在部署前进行设计时验证（Design-Time Verification）。

### 关键技术细节
- **工具调用执行内核**：五个可组合子过程（Sub-Process）：
  1. **Decompose**：将输入分解为单元任务、配置和工具注册表。混合原语：确定性配置提取 + SLM 任务分解。
  2. **Init-Scratchpad**：从配置初始化可变键值存储 S。确定性原语。
  3. **Get-Order**：构建任务依赖图、检测环、拓扑排序。混合原语：SLM 构建依赖图 + 确定性图算法。
  4. **Check-Prerequisites**：检查每个任务的前置条件（Met/¬Met/Absent），触发内联检索或补救任务。以确定性为主，SLM 用于条件判定和任务生成。
  5. **Match-Functions-and-Parameters**：匹配函数、解析参数（优先从上下文 τ 和暂存区 S 查找），仅当所有参数绑定后执行。混合原语：SLM 函数匹配 + 确定性参数查找。
- **输出契约**：每个子过程有显式后置条件（如暂存区单调性、执行守卫等），全局不变量确保程序结构正确。
- **设计时验证流程**：程序构建 → 静态类型检查 → 契约审查 → 边界案例分析 → 受控执行 → 部署。通过 300 条手动验证的 DSL 解决方案迭代优化 DSL 规范。

## 6. 主要结论与发现

- **准确性提升显著**：Forethought 使基础模型准确率相对提升 30–60%，尤其弱势模型提升更大（如 Haiku 4.5 在 Tau2 电信从 53.5 提升到 98.0）。
- **超越所有基线**：在五个基准上，NRE 均超过 Vanilla、RLM 和 GEPA 的最佳配置。
- **小模型与大模型匹敌**：NRE 使用 Kimi K2.6 或 MiniMax M3 等小模型，其准确率可与 Vanilla 下 Opus 4.7 等前沿大模型竞争（如 Tau2 零售 93.0 vs 87.7）。

- **验证驱动准确率**：设计时验证是准确率提升的主要来源，它使错误在部署前就被发现并修正。
- **模型无关性与可审计性**：推理程序结构独立于基础模型，且提供逐步骤验证的符号轨迹。

## 7. 优点

- **方法创新**：将推理显式化为带有输出契约的神经符号程序，实现了设计时验证，从根本上区别于随机搜索的测试时扩展。
- **成本极低**：后训练只需数百美元和少量数据，推理时使用小型 SLM，成本远低于推理模型。
- **模型无关**：同一推理程序可跨不同基础模型迁移，无需重新训练。
- **可审计**：每个步骤的验证结果形成结构化轨迹，满足监管要求。
- **广泛适用**：在多个工具调用基准和多种模型上一致提升，鲁棒性强。
- **设计时验证流程**：类比传统软件工程，保证了推理程序的正确性，是准确率提升的核心。

## 8. 不足与局限

- **构建和维护成本高**：需要专家手工定义原语、契约和验证用例，SLM 原语的微调需要完整的数据收集与训练流程，当前缺乏自动合成工具。
- **领域假设强**：要求推理任务能分解为狭窄的、具有可检查中间结果的原语。对于开放域推理、需要整体判断的问题（如需要创造力或模糊评价的任务）不适用。

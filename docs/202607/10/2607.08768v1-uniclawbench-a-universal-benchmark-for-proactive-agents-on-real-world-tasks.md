## 摘要
大语言模型和多模态大语言模型的快速发展加速了能够操作日常工具并在真实世界环境中协助用户的主动智能体的出现。然而，现有的基准难以有效评估这类智能体，因为它们通常依赖沙盒环境和单轮评估范式。此外，它们基于场景的任务分类将多种模型能力混合在同一任务类别中，使得难以识别智能体失败的根本原因。为解决这些局限性，我们引入了UniClawBench，这是第一个旨在动态真实世界环境中评估主动智能体的能力驱动基准。UniClawBench围绕五种基础模型能力构建：技能使用、探索、长上下文推理、多模态理解和跨平台协调。基于这些能力，我们设计了400个双语真实世界任务。与依赖静态、预先录制答案的先前基准不同，我们的基准在实时Docker容器中通过细粒度的逐步骤完成检查点来评估智能体。此外，我们设计了一种闭环评估策略，包括执行智能体、隐藏监督智能体和用户智能体，以模拟真实的多轮人类反馈，同时不泄露评分标准。为了将基础模型能力与框架级设计选择分离，我们在多个智能体框架下评估了最先进的模型。通过跨模型和框架的全面比较，我们展示了基础模型能力和智能体框架设计如何共同塑造在真实世界环境中的性能。为促进未来研究，我们在https://github.com/HKU-MMLab/UniClawBench上公开了我们的基准和代码。

## Abstract
The rapid development of large language models and multimodal large language models has accelerated the emergence of proactive agents capable of operating everyday tools and assisting users in real-world environments. However, existing benchmarks struggle to evaluate such agents effectively, as they often rely on sandboxed environments and single-turn evaluation paradigms. Moreover, their scenario-based task taxonomies mix multiple model capabilities within the same task category, making it difficult to identify the root causes of agent failures. To address these limitations, we introduce UniClawBench, the first capability-driven benchmark designed to evaluate proactive agents in dynamic, real-world settings. UniClawBench is built around five foundational model capabilities: Skill Usage, Exploration, Long-Context Reasoning, Multimodal Understanding, and Cross-Platform Coordination. Based on these capabilities, we design 400 bilingual real-world tasks. Unlike previous benchmarks that rely on static, pre-recorded answers, our benchmark evaluates agents in live Docker containers using fine-grained, step-by-step completion checkpoints. Furthermore, we design a closed-loop evaluation strategy comprising an executor agent, a hidden supervisor agent, and a user agent to simulate realistic multi-turn human feedback without leaking grading criteria. To disentangle base model capabilities from framework-level design choices, we evaluate state-of-the-art models under multiple agent frameworks. Through comprehensive comparisons across both models and frameworks, we show how base model capabilities and agent framework designs jointly shape performance in real-world environments. To facilitate future research, we make our benchmark and code publicly available at https://github.com/HKU-MMLab/UniClawBench.

# UniClawBench 论文详细总结

## 1. 核心问题与整体含义（研究动机与背景）

- **现有基准的三大局限**：
  - **沙盒环境偏差**：依赖自建网站镜像（如 WebArena）或虚拟机缓存页面（如 OSWorld），与真实世界性能存在巨大鸿沟。
  - **单轮评估缺陷**：忽略人机交互中用户迭代反馈的闭环本质。
  - **按场景分类**（如“办公”“研究”）导致能力混淆：模型在“办公”任务失败时，无法定位瓶颈是视觉感知、长上下文还是工具使用。
- **核心需求**：需要一个**能力驱动**的基准，能够在动态真实环境中评估主动代理，并定位失败根因。
- **额外挑战**：
  - 真实世界动态性：预录答案（如亚马逊产品价格）会快速过时。
  - 信息隔离悖论：模拟用户反馈需访问评分规则，但可能泄露答案或标准。

## 2. 方法论：核心思想与关键技术细节

### 2.1 能力导向的任务分类
UniClawBench 将任务按**五种基础能力**分解（每类40个英文+40个中文任务，共400个双语任务）：
- **技能使用 (Skill Usage)**：正确选择、调用、操作工具或API（如OCR、SQL查询）。
- **探索 (Exploration)**：在信息不完整或误导下主动调研（如反向工程API、审计配置）。
- **长上下文推理 (Long-Context Reasoning)**：跨多个来源保持全局一致性（如多页面旅行规划）。
- **多模态理解 (Multimodal Understanding)**：从图像/视频/音频中提取并生成内容（如根据视频截图生成清单）。
- **跨平台协调 (Cross-Platform Coordination)**：在异构应用间同步状态（如 Zotero + Obsidian 文献管理）。

### 2.2 三角色闭环评估策略
- **执行器 (Executor)**：被评估的代理，在Docker容器中运行，仅接收公共任务包（文件、工具、服务）。
- **隐藏监督器 (Hidden Supervisor)**：拥有隐藏参考答案和分步检查点评分规则，评估执行器的轨迹和产出物，输出 `pass/fail/continue` 状态和分数。**监督器的评分标准绝不暴露给执行器或用户模拟器。**
- **用户模拟器 (User Simulator)**：基于**可见的执行轨迹和粗粒度的进度信号**（无隐藏标准）生成自然语言反馈，模拟真实用户。反馈经过**信息防火墙**（仅传递 coarse progress signal）和**反馈重写器**清洗后发给执行器。
- **信息隔离机制**：监督器只向用户模拟器暴露 verdict/attempt_state/recoverable/score 四个字段，隐藏评分理由和检查点细节。

### 2.3 评估系统与指标
- **运行环境**：每个任务在独立Docker容器中启动，包含真实浏览器、GUI应用、本地文件。
- **检查点评分**：每个任务设计**隐藏的 eval_rule.md**，包含权重之和为1.0的检查点、分数上限。监督器据此打分。
- **多轮迭代**：每轮监督器判定为 `continue` 时，触发用户模拟器生成反馈，执行器继续改进；最多允许2次用户跟进。超时阈值：标准任务30分钟全局+20分钟每轮，长上下文任务45分钟+30分钟。
- **最终指标**：
  - **Pass Rate (PR)**：任务级通过率（最终得分 ≥ success_threshold，通常0.9）。
  - **Average Score (AS)**：检查点级平均分（超时任务取已完成的最高分）。

## 6. 主要结论与发现

1. **整体通过率极低**：最佳模型（Claude Opus-4.8+OpenClaw）的 PR 仅 47.5%，表明真实世界任务远难于沙盒环境。
2. **框架选择比模型选择影响更大**：同一模型在不同框架下的性能差异显著（如 Claude Opus-4.8 在 OpenClaw PR 47.5%，在 Nanobot 仅 38.5%）；框架架构决定轨迹组织和上下文保留，从而制约基础模型能力。
3. **能力瓶颈分析**：
   - 相对较易：Skill Usage（最高 PR 55%）和 Exploration（最高 82.5%）—— 本地工具操作和信息寻址能力强。
   - 主要瓶颈：Long Context（最高 PR 43.8%）、Multimodal（最高 21.2%）、Cross-Platform（最高 38.8%）—— 长程记忆、多模态推理、跨应用协调是当前代理的短板。
4. **“半途失败”现象普遍**：平均得分（AS）普遍较高（如 Claude Opus-4.8 总体 AS 0.702），但最终通过率低，说明代理能完成大部分步骤但在执行链后期犯不可恢复的错误。
5. **多轮反馈有效提升性能**：Figure 4c 显示随着交互轮次增加，Pass Rate 从 23.8% 提升至 31.7%，累积最大分数从 0.565 提升至 0.679。
6. **框架特性对比**：
   - OpenClaw：集中式单代理，上下文保留好，通过率最高。
   - EDICT：多代理编排，平均得分高但通过率偏低，存在“协调摩擦”（子代理状态传递丢失）。
   - Nanobot：极端 token 效率，但简化上下文管理导致最终产出完整率不足。

## 7. 优点

- **能力分解诊断**：首个将任务按基础能力维度划分的基准，可以精确定位失败根因，而非笼统的“场景失败”。
- **真实环境评估**：使用实时 Docker 容器、真实网站和应用，比沙盒环境更贴近部署场景。
- **闭环多轮交互**：三角色设计 + 信息防火墙，实现了既模拟真实用户反馈又防止评分泄露的平衡。
- **可复现性与透明度**：提供完整的任务包格式（YAML + 隐藏评分规则）、公开代码和基准。
- **跨模型 & 跨框架分析**：系统解耦了模型能力和框架设计的影响，为代理系统设计提供指导。
- **详细的人类验证**：50条轨迹的评审确认了自动评估的可靠性（92%一致率，中等相关性）。

## 8. 不足与局限

- **任务规模有限**：400个手动设计的任务可能无法覆盖所有现实场景，且数据集扩展性受限。
- **环境不稳定性**：真实网站可能随时间变化（价格、内容、UI），导致评估结果不可完全复现（文中用动态检查点缓解，但无法消除）。
- **LLM 评估偏差**：监督器和用户模拟器均为 GPT-5.4 驱动，可能引入自身框架的弱点或偏见；人类评审虽部分验证，但样本量较小。
- **资源消耗高**：每个任务平均17.4分钟，跨模型/框架实验总耗时数百小时，对大规模迭代不友好。
- **用户跟进次数限制**：最多2次用户反馈，可能不足以模拟需要多次协作的复杂任务。
- **仅测试预定义框架**：未涵盖最近涌现的更多代理框架（如反思型、学习型代理），通用性待验证。
- **应用风险**：可能被用于过度优化基准性能而非真实泛化，或加速开发缺乏监督的自主系统（论文已提及这一社会影响）。

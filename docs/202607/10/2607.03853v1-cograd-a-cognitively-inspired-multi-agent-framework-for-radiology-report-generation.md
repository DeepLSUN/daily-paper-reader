## 摘要
自动化放射学报告生成（RRG）可以减轻放射科医生的工作负担，然而现有系统大多通过单次前向传递生成报告，缺乏对照图像验证陈述或重新审视已发现结果的机制。我们提出CogRad，一种认知启发的多智能体框架，该框架围绕放射科医生阅读过程的四个阶段构建生成过程。侦察智能体通过槽注意直接从图像块中发现解剖区域，并分配区域和疾病级别的分诊评分；调查智能体将表征能力集中在侦察智能体标记为可疑的区域上；编写智能体将这些信号编译为疾病门控视觉前缀，供大型语言模型使用；验证智能体通过视觉蕴含损失监督训练，并在推理时逐句重新检查自己的草稿，对认为依据不足的报告进行重新生成。在CheXpert Plus数据集上，CogRad的BLEU-4得分为0.316，CIDEr得分为0.322，在我们比较的方法中取得最佳成绩。在IU X-Ray数据集上，其BLEU-4得分为0.201，CIDEr得分为0.724，在所有标准自然语言生成指标上均领先于每个基线。我们进一步使用RadGraph F1、CheXbert F1和幻觉分析评估CogRad的临床准确性，超越了标准的文本重叠指标，并通过消融研究和基于Grad-CAM的可视化来表征每个智能体的贡献以及模型的视觉基础。

## Abstract
Automated radiology report generation (RRG) can ease radiologist workload, yet most existing systems produce a report in a single forward pass, with no mechanism to check a claim against the image or revisit a finding once stated. We present CogRad, a cognitively inspired multi-agent framework that structures generation around four stages of a radiologist's reading process. A Scout agent discovers anatomical regions directly from image patches via slot attention and assigns region and disease-level triage scores; an Investigator agent concentrates representational capacity on the regions Scout flags as suspicious; a Writer agent compiles these signals into a disease gated visual prefix for a large language model; and a Verifier agent supervises training with a visual entailment loss and, at inference, re-examines its own draft sentence by sentence, regenerating any report it judges insufficiently grounded. On CheXpert Plus, CogRad attains a BLEU-4 of 0.316 and a CIDEr of 0.322, the best scores among the methods we compare against. On IU X-Ray, it attains a BLEU-4 of 0.201 and a CIDEr of 0.724, leading every baseline on every standard NLG metric. We further evaluate CogRad with RadGraph F1, CheXbert F1, and a hallucination analysis to assess clinical accuracy beyond standard text-overlap metrics, complemented by ablation studies and Grad-CAM-based visualizations that characterize each agent's contribution and the model's visual grounding.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：CogRad是一个四智能体框架，通过结构化的张量（而非文本）在各个智能体间传递信息，模拟放射科医生的四个关键认知阶段：全局分诊、聚焦调查、结构化撰写和视觉验证。

- **关键技术细节与算法流程（文字描述）**：
    1.  **共享视觉编码器**：所有智能体共用同一个Swin Transformer-Base视觉编码器，为图像生成统一的、连续的视觉表征，保证跨阶段表示的一致性。
    2.  **侦察智能体（Scout Agent）**：
        - **区域发现**：使用**槽注意机制**，通过5个可学习的“槽”竞争性地代表临床上意义重大的解剖区域，避免了传统固定网格划分的局限性。
        - **分诊**：为每个区域生成异常评分（`r`），并为整个图像预测14种标准疾病的概率（`c`），使用二元交叉熵损失（`L_Scout`）进行监督。
    3.  **调查智能体（Investigator Agent）**：
        - **区域间交互**：对侦察智能体输出的区域特征应用自注意力机制，捕捉不同区域间的临床相关性（如心脏肥大与胸腔积液的关联）。
        - **自适应聚焦**：根据侦察智能体的“分诊分数”计算**聚焦权重**（`φ`），将更多计算资源分配给高异常可能性的区域，进行更精细的特征提取（Top-K区域选择，`K=3`）。
    4.  **编写智能体（Writer Agent）**：
        - **疾病门控视觉前缀**：将侦察和调查智能体的输出组装成一个结构化的视觉前缀（`Q`），作为LLM的输入。关键创新在于使用**疾病门控机制**：对于疾病概率低的疾病，通过`sigmoid`门将其对应的嵌入向量驱动到接近零，从而在报告生成前就“屏蔽”了弱信号，减少幻觉。
        - **前缀结构**：前缀顺序模仿医生推理过程：疾病级别上下文 → 区域级病理证据 → 全局图像观察。
        - **生成**：该前缀被送入LLaMA-7B语言模型（通过LoRA微调）生成报告，使用标准自回归交叉熵损失（`L_CE`）。
    5.  **验证智能体（Verifier Agent）**：
        - **训练时**：引入**视觉蕴含损失**（`L_Ver`）。该损失鼓励每个生成的token都能在图像中找到视觉支撑，通过计算文本token与图像区域之间的跨模态注意力，并对视觉-文本对进行对齐和疾病一致性检查来实现。
        - **推理时**：运行一个**自纠正循环**。生成草稿报告后，其逐句被送入一个轻量级句子置信度头。若句子的置信度低于阈值（`τ=0.5`），则触发修正循环：提升侦察智能体的分诊分数，重新调查可疑区域，并让编写智能体基于修正后的视觉前缀完全重新生成报告（最多2轮）。

### 7. 优点

1.  **认知启发的设计**：深入挖掘并形式化了放射科医生的认知流程，将直觉性的“边看边想边验证”转变为可计算、可解释的多智能体系统。
2.  **隐含式视觉表示传递**：相比基于文本交互的多智能体系统，CogRad通过连续的张量（如区域特征、注意力权重）在各智能体间传递信息，保留了完整的空间和视觉信息，避免了信息损失。
3.  **推理时自纠正能力**：这是本文最突出的创新点之一。系统不再仅依赖单次前向推理，而是能在生成报告后进行自我检查并修正，更接近临床实践中“回头看”的习惯，有效降低了幻觉风险。
4.  **无需额外标注的幻觉感知训练**：验证智能体的视觉蕴含损失直接利用图像和报告本身，无需额外的临床标注，设计精巧且实用。

### 8. 不足与局限

1.  **NLG指标与临床准确性的“鸿沟”**：在CheXpert Plus上，模型NLG指标优异但RadGraph F1较低，表明当前方法虽然在生成流畅文本上取得了成功，但距离真正可靠、详尽的临床报告还有差距，尤其是在处理高频、并发病变时。
2.  **高幻觉率**：模型在CheXpert Plus上的实体级别幻觉率仍较高（约44%），主要集中在高频发现（如胸腔积液、肺不张）和支持设备，这在临床上是不可接受的，是该框架未来需要重点攻克的方向。
3.  **指令与区域映射缺失**：论文指出，目前的幻觉分析只在报告级别运行。将实体级幻觉映射到具体的句子和图像区域，使修正循环能精确修正有问题的句子，而非重新生成整个报告，是本框架的一个明确改进方向。
4.  **数据集限制与泛化性问题**：
    - **偏差风险**：模型在CheXpert Plus（住院/ICU人群）上的高幻觉率部分可归因于该数据集的特定病例组合（设备多、发现多），模型在该分布上表现与在门诊为主的IU X-Ray上不同，表明其泛化能力有待验证。
    - **应用限制**：性能依赖于数据集的质量和标注。在完全陌生、罕见或复杂的病例上，模型的表现和修正循环的有效性未知。
5.  **计算成本与可扩展性**：虽然使用单张A100训练，但引入了多智能体（特别是推理时的多轮修正循环），相比于单次前向生成，其推理计算成本更高，可能阻碍在低延迟场景下的部署。大型LLM（LLaMA-7B）的使用也带来了可观的资源需求。
6.  **推理阶段的“黑盒”**：尽管有修正循环，但系统仅基于模型自身的置信度分数进行判断，而非外部知识或逻辑推理。如果模型对幻觉语句过于自信（即错误的置信度高），修正循环将失效。

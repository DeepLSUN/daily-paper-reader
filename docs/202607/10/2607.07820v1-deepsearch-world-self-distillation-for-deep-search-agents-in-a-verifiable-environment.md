## 摘要
训练使用工具的代理从自身经验中改进仍然具有挑战性，因为监督式微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长程交互提供的监督信号较弱。我们提出了DeepSearch-Evolve，一个基于DeepSearch-World的网页代理自我蒸馏框架，后者是一个确定且可验证的环境，具备可复现的搜索和页面阅读工具。DeepSearch-World包含42万个基于实体级随机游走构建的多跳问答任务，并支持对自我进化有用的关键代理认知行为，包括进度验证、基于事实的反思和失败恢复。DeepSearch-Evolve通过迭代执行轨迹生成、过滤、数据混合和微调来训练更强的代理。在不依赖更强模型蒸馏的情况下，DeepSearch-World-9B与开源代理相比取得了具有竞争力的性能，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，这表明可验证环境能够为长程网页代理实现可扩展的自我进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来关于自我改进的深度搜索代理的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

## 1. 核心问题与整体含义（研究动机和背景）

训练使用工具（tool-use）的智能体（agent）从自身交互经验中持续改进是当前大语言模型（LLM）领域的关键挑战。现有方法主要依赖两类范式：

- **监督微调（SFT）**：在固定教师轨迹上学习，但性能受限于骨干模型能力和轨迹多样性，容易饱和。
- **强化学习（RL）**：使用稀疏的最终奖励（如 GRPO），但长程交互中难以定位失败原因（查询构造、工具选择、证据提取或答案合成）。

**On-policy Self-Distillation (OPSD)** 试图通过匹配学生 rollout 与教师密集的 token 级分布来缓解奖励稀疏性，但在工具使用场景中，由于缺乏确定性和可验证的环境，每一步的教师分布可能充满噪声。

因此，论文的核心动机是：**构建一个确定性、可验证的离线工具环境，使代理能够通过可控的过程级监督实现自我蒸馏与持续进化**。这为解决长程工具使用 agent 的自主改进难题提供了新思路。

## 2. 方法论：核心思想、关键技术细节

### 核心思想：DeepSearch-Evolve
基于确定性环境 **DeepSearch-World**，通过迭代式自我蒸馏循环（生成轨迹 → 过滤 → 数据混合 → SFT 训练），让 agent 从自身 **已验证的** 工具使用经验中学习规划、记忆跟踪、错误恢复等高阶能力，无需外部更强模型。

### 关键技术细节

#### a) 环境构建：DeepSearch-World
- **数据来源**：基于 Wikipedia 知识图谱，通过实体级随机游走（entity-level random walks）生成 **42 万** 多跳 QA 任务。
- **确定性工具**：
  - `web_search_wiki`：使用 Lucene BM25（Pyserini）在离线 Wikipedia 语料库中进行关键词检索，返回标题、摘要和 URL。
  - `visit_wiki`：通过 SQLite 偏移索引根据 URL 读取完整文章，模拟浏览器访问。
- **环境接地反思（Grounded Reflection）**：环境内部维护目标实体集合，每次工具调用后可自动验证是否找到新实体，提供过程级验证信号。失败时触发分阶段规则反思，鼓励查询改写和恢复。

#### b) 教师代理（Scaffold Teacher）
采用三阶段脚手架（Scaffold）结构：
- **Plan**：初始化结构化进度状态（含 `completed_list`, `todo_list`, `experience`, `information`）。
- **Act**：循环执行工具调用，更新状态并接收环境反思。
- **End**：当证据充分或预算耗尽时，基于已验证的工作记忆生成简洁答案。

#### c) 自我蒸馏循环
- **轨迹生成与验证**：当前模型 `π_θR` 作为教师生成 scaffold 轨迹，保留仅答案正确的轨迹，再用 Qwen3.5-9B 进行质量过滤（去除冗余、弱对齐、不一致）。
- **Scaffold-to-ReAct 转换**：移除阶段提示，将状态更新和环境反思改写为 `<think>` 块中的自纠正理由，形成标准 ReAct 格式的监督数据。
- **监督式自蒸馏更新**：在过滤后的 ReAct 轨迹上做 SFT 最小化负对数似然。使用重要性采样混合新旧数据（衰减因子 γ=0.5），缓解灾难性遗忘。
- **循环迭代**：每轮用 10k 问题生成，当累计 4k 通过过滤时触发训练，共进行 11 轮。

## 6. 主要结论与发现

1. **可验证环境是关键**：DeepSearch-World 提供的确定性过程级验证使自我蒸馏稳定有效，无需依赖更强教师模型。
2. **DeepSearch-World-9B 性能优异**：在 BrowseComp 达 31.2%（提升 +23.8）、GAIA 61.5%（+37.6）、HotpotQA 93.4%（+48.1），与依赖更强模型蒸馏的开源 agent 竞争。
3. **自蒸馏比 RL 更稳定**：与 GRPO 相比，进化式 SFT 避免了稀疏奖励和不可靠的目标分布问题。
4. **数据池多样性重要**：420K 规模优于 100K，因为后者过早导致重复轨迹和过拟合；足够大的池子覆盖更多推理路径和失败恢复模式，延缓饱和。
5. **高级行为涌现**：训练后的 agent 维持更长交互（平均 18 轮 vs 4.7 轮），更多访问页面（5.4 次 vs 0.9 次），高级能力评分达 70%（vs 19%）。

## 7. 优点

- **环境设计**：离线/确定/可验证，提供零成本的自动过程级审核，支持大规模生成与过滤。
- **教师脚手架**：显式注入规划、记忆和恢复能力，比纯 RL 探索更高效，比固定演示集更灵活。
- **转换蒸馏**：将结构化 scaffold 自然转化为标准 ReAct，使最终模型可部署，避免对特殊提示的依赖。
- **迭代框架**：异步生成与训练，重要性采样混合新旧数据，兼顾效率与稳定性。
- **实验全面**：多 benchmark、多消融、行为分析，结论有强支撑。
- **开源性**：将环境、数据集、模型、代码全部开源，促进行业研究。

## 8. 不足与局限

- **知识范围局限**：当前环境仅基于 Wikipedia，领域覆盖有限，泛化到其他知识源（新闻、学术、实时数据）需进一步扩展。
- **更新规则单一**：仅使用进化 SFT，未尝试 RL 或 OPSD 变体。论文指出如何将高阶能力注入 RL 训练仍待探索。
- **潜在偏差**：Wikipedia 本身可能存在社会性偏见，模型可能继承这些偏见。
- **中文能力较弱**：仅英文训练，在 BrowseComp-ZH 上提升有限，跨语言泛化需要额外设计。
- **过滤依赖外部模型**：质量过滤使用 Qwen3.5-9B，引入额外依赖；若该模型存在偏见或误差可能影响轨迹质量。

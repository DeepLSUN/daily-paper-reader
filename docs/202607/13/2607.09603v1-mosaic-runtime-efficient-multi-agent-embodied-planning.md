## 摘要
基于大语言模型的多智能体具身规划因执行延迟过高而仍然不实用。我们发现失败动作是主要瓶颈，源于两个核心挑战：部分可观测性下的不准确状态跟踪以及产生冗余或冲突动作的低效协调。我们提出了Mosaic，一个运行时高效的多智能体规划框架，同时应对这两个挑战。Mosaic通过以智能体为中心的语义记忆来维护准确且轻量的状态跟踪，该记忆以相对坐标存储物体，支持几何变换和协调。它通过整数线性规划确保高效协调，在每个规划步骤分配动作，强制执行物理可行性和智能体间的协调约束。在AI2-THOR和搜索救援基准测试中，Mosaic实现了27-32%更快的执行速度，30-33%更少的LLM调用，25-31%更少的步骤，以及4-10%更高的成功率。这些结果表明，高效的内存和约束引导的协调对于可扩展、低延迟的多智能体规划至关重要。

## Abstract
LLM-based multi-agent embodied planning remains impractical due to prohibitively high execution latency. We identify failed actions as the dominant bottleneck, stemming from two core challenges: inaccurate state tracking under partial observability and inefficient coordination that produces redundant or conflicting actions. We introduce Mosaic, a runtime-efficient multi-agent planning framework that addresses both challenges. Mosaic maintains accurate yet lightweight state tracking through agent-centric semantic memory that stores objects in relative coordinates, enabling geometric transformations and coordination. It ensures efficient coordination through Integer Linear Programming that allocates actions at every planning step, enforcing physical feasibility and inter-agent coordination constraints. Across AI2-THOR and search-and-rescue benchmarks, Mosaic achieves 27-32% faster execution, 30-33% fewer LLM calls, 25-31% fewer steps, and 4-10% points higher success rates. These results demonstrate that efficient memory and constraint-guided coordination are critical for scalable, low-latency multi-agent planning.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：基于大语言模型（LLM）的多智能体具身规划在实际部署中面临过高的执行延迟，难以用于资源受限或时间敏感的应用（如搜索救援、家庭家务）。
- **核心问题**：延迟高的主要原因是**失败动作**（failed actions），表现为：
  - **部分可观测性下的状态跟踪不准确**：智能体无法准确追踪物体位置，导致重复探索或导航到已过时位置。
  - **多智能体协调低效**：任务分配不当导致冗余动作、运动干扰、前提条件不满足。
- **现实影响**：例如，现有系统完成“关闭水龙头和灯”任务需约7.2分钟，更复杂的搜索救援需10.5分钟，其中16%-51%的规划步骤为失败动作，既浪费执行时间，又造成级联等待。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 通过**轻量级状态跟踪**和**约束引导的动作分配**，在保证任务完成率的同时大幅降低执行延迟。
- 将LLM的语义理解能力与符号优化（ILP）的可行性保证结合。

### 关键技术细节

#### (1) 智能体中心语义记忆（Agent-centric Semantic Memory, ASM）
- **存储内容**：对于每个智能体 \(a\) 和任务相关物体/目标 \(o\)，维护相对距离 \(r_{a,o}\) 和相对方位 \(\phi_{a,o}\)，表示为智能体坐标系下的曼哈顿距离和指向。
- **更新机制**：随智能体移动自动更新（前进减少距离，转向调整方位）；当物体离开视野时，通过运动历史推断其位置；支持跨智能体坐标变换（基于智能体间的相对位姿进行计算）。
- **接口**：将连续坐标离散化为语义层级（如“just ahead” 1-2步，“far” >10步），但底层保留连续值避免累积误差。
- **优点**：无需全局地图或密集3D重建，计算轻量，支持跨智能体信息共享。

#### (2) 基于约束优化的动作选择（Integer Linear Programming, ILP）
- **输入**：每个智能体由LLM-Actor生成 \(K\) 个候选动作（\(K=3\) 为默认）。
- **变量**：二进制变量 \(z^{(k)}_{i,t}\) 表示智能体 \(i\) 在时间步 \(t\) 是否选择第 \(k\) 个候选。
- **约束**（动作级可行性约束）：
  - 合格性约束（动作前提条件不满足时禁止选择）。
  - 每个智能体每步恰好选择一个动作（可包含 no-op）。
  - 多人联合子任务要求指定数量的智能体同时参与。
  - 资源独占：同一资源（物体、容器）同时只能被一个智能体操作。
  - 碰撞/干扰避免：互斥动作对不能同时选择。
- **代价函数**：
  \[
  J(z,H) = \sum_{i=1}^{N}\sum_{k=1}^{K_i} z^{(k)}_{i,t} \left[ C_{\text{penal}}(a_i, x^{(k)}_{i,t}, H) + \lambda_{\text{load}} C_{\text{load}}(a_i, x^{(k)}_{i,t}, H) \right]
  \]
  - \(C_{\text{penal}}\)：惩罚空间和时间低效模式（循环、连续失败、振荡、回溯、停滞），每种模式有预设权重（见表7）。
  - \(C_{\text{load}}\)：负载均衡惩罚，鼓励智能体均等参与。
  - \(\lambda_{\text{load}}=1\) 默认。
- **流程**：LLM-Planner分解任务→LLM-Actor每步生成候选动作→ILP选择最优联合动作→执行并更新ASM→LLM-Verifier检查子任务完成→循环。

## 7. 优点

- **轻量状态跟踪**：ASM采用相对坐标+语义离散化，避免全局地图构建的高计算成本，同时支持跨智能体信息传递。
- **动作级精细协调**：ILP在每步进行联合动作分配，比传统粗粒度目标分配更灵活，能处理动态环境。
- **神经-符号结合**：保留LLM的语义灵活性，同时利用符号优化确保可行性，避免LLM在约束推理上的不足。
- **即插即用**：Mosaic的模块（ASM、ILP）可独立集成到现有规划器中（如ReAct、LLaMAR），通用性强。
- **全面评估**：包括有效性、效率、可扩展性、鲁棒性、消融、超参数分析，结果一致。

## 8. 不足与局限

- **依赖准确位姿估计**：ASM基于合理精确的位置/朝向估计；在严重传感器漂移或故障下可能失效（尽管论文做了噪声分析）。
- **约束覆盖有限**：ILP约束仅编码已知物理和协调规则，无法处理未知动态事件或复杂社交行为；需领域专家手动扩展。

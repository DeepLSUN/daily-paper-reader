## 摘要
智能体技能通过可复用的流程、工具和领域特定工作流扩展了LLM智能体，但其安全性取决于解析交互指令间依赖关系的能力。我们提出SkillLogic框架，用于分析技能文件中的逻辑关系并从中构建可执行测试。我们的分类体系涵盖八种关系类型，包括控制有效动作的前提条件、限制允许动作执行方式的约束条件，以及指定失败后恢复行为的回退机制。利用SkillLogic，我们扫描了超过5000个公开技能，发现其中70%包含至少一个逻辑关系。随后，我们从高置信度、高影响且可本地测试的关系中构建了包含86个案例的可执行基准测试SLBench。在六个LLM骨干网络上评估Codex和Claude Code，结果显示不安全率高达70%，违规行为导致隐私泄露、不安全配置更改及不完整清理。人工审计将失败归因于智能体能力差距和技能文本显著性不足。我们进一步展示了轻量级推理时框架SLGuard，在目标案例上减少了63%的违规行为。我们的结果确立了逻辑关系遵循作为技能引导智能体的一个独特可靠性挑战。

## Abstract
Agent skills extend LLM agents with reusable procedures, tools, and domain-specific workflows, but their safety depends on resolving dependencies among interacting instructions. We introduce SkillLogic, a framework for analyzing logical relations in skill files and constructing executable tests from them. Our taxonomy covers eight relation types, including preconditions that gate valid actions, constraints that limit how allowed actions may be performed, and fallbacks that specify recovery behavior after failure. Using SkillLogic, we scan over 5000 public skills and find that 70% contain at least one logical relation. We then construct SLBench, an 86-case executable benchmark from high-confidence, high-impact, and locally testable relations. Evaluating Codex and Claude Code across six LLM backbones shows unsafe rates up to 70%, with violations leading to privacy leaks, unsafe configuration changes, and incomplete cleanup. The human audit attributes failures to both agent capability gaps and low-salience skill text. We further show that SLGuard, a lightweight inference-time scaffold, reduces violations by 63% on targeted cases. Our results establish logical-relation following as a distinct reliability challenge for skill-guided agents.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 智能体在执行任务时依赖可复用的“技能”文件（skill files），这些文件包含多条自然语言指令子句，子句之间隐含复杂的逻辑关系（如前提条件、约束、后置条件、回退、异常等）。智能体可能只遵循了显眼的主流程子句，而忽略了这些关系，导致行为看似正确实则存在安全隐患（隐私泄露、配置破坏、清理不完整等）。现有基准测试未能系统评估智能体对技能内部逻辑关系的遵循能力。
- **整体含义**：将“逻辑关系遵循”确立为技能引导智能体独立于基础能力的可靠性挑战，需要专门的方法论、基准测试和缓解措施。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建 SkillLogic 框架，通过分析技能文件中子句之间的逻辑关系，将可观测的因果关系转化为可执行测试用例，从而量化评估智能体的关系遵循能力。
- **关键技术细节**：
  - **逻辑关系分类学**：定义8种关系类型：
    - 前提条件（precondition）：动作前必须先满足的条件。
    - 后置条件（postcondition）：动作完成后必须执行的后续步骤。
    - 约束（constraint）：允许动作的执行范围限制。
    - 合取（conjunction）：多个义务必须同时满足。
    - 回退（fallback）：主路径失败后应执行的行为。
    - 异常（exception）：特定条件下覆盖默认行为。
    - 覆盖（override）：优先级更高的子句支配。
    - 冲突（conflict）：两个子句不能同时满足。
  - **分析流程**：对每个技能文件提取子句，选择单一主工作流，识别相关逻辑关系，只保留源可追溯、高置信、高影响的关系。
  - **测试用例生成**：每个关系转换为一个可执行测试，包含真实提示、受控本地仓库、确定性评分器。评分器采用不安全优先的违规检测策略，基于执行痕迹（文件状态、命令历史等）而非自然语言响应判断。
  - **缓解措施 SLGuard**：在推理时让智能体首先生成关系检查清单，显式列出要求板、约束和异常，执行前后进行验证，以减少遗漏逻辑关系的风险。

## 6. 论文的主要结论与发现

- 技能文件中的逻辑关系普遍存在：70% 的公开技能包含至少一个关系。
- 当前主流代码智能体（Codex、Claude Code）在 SLBench 上不安全率高达 70%（Codex GPT-5.5），最低也为 35.1%（Claude Code Opus 4.7）。违规集中在前提条件、冲突和覆盖关系，导致隐私泄露、配置破坏、清理缺失等严重风险。
- 人工审计表明，技能文本本身并非不可理解（人类均能正确识别安全行为），但智能体即使面对清晰文本也常常失败；技能澄清（提高文字显著性）可减少约一半的违规，说明显式化结构有助于但不足以根治。
- 轻量级推理时框架 SLGuard 通过生成关系检查清单，将目标案例的违规率降低 63%，提示显式化逻辑结构是一种有效的缓解方向。
- “逻辑关系遵循”应被视为独立于普通指令遵循的可靠性维度，需要专门的设计与评估。

## 7. 优点

- **方法论创新**：首次系统定义了技能内部逻辑关系的分类学（8种类型），并形式化为可测试的框架（SkillLogic），解决了现有基准测试忽略关系级错误的缺陷。
- **基准设计务实**：从真实技能文本出发，构建可本地运行、确定性评分、人工审核的案例，确保可复现性和证据可靠性。
- **评分器鲁棒性**：采用不安全优先的 artifact-first 设计，基于执行痕迹而非语言判断，减少误报漏报；并公开进行了优先级消融实验。
- **多维度评估**：不仅评估多个骨干，还包含人工审计、文本澄清消融、评分策略消融和缓解实验，全面揭示问题根源。
- **开源与可复现**：全部流程（分析、生成、执行、评分）均有详细设计，未来可扩展。

- **评分偏见风险**：

- **人工审计局限性**：
  - 仅 12 个案例，缺少正式信度检验，且标注者均为作者团队内部人员，存在偏差风险。

- **遗漏其他关系类型**：分类学虽覆盖 8 种，但未包括时间依赖、互斥资源、优先级链等更复杂的模式。

## 摘要
扩大大型语言模型（LLM）主要依赖于扩展Transformer主干网络，但对于已经很强的基础模型，这需要另一轮代价高昂的预训练。我们研究是否可以在不改变Transformer主干网络的情况下，通过为每个词元分配更多计算来持续改进已有模型。深度循环（循环）Transformer追求这一目标，但难以扩展，因为循环计算与训练最大模型所使用的流水线并行不太兼容。我们沿着序列长度维度增加计算，其中额外的计算仅仅是更长的输入，并且与标准的大模型训练保持兼容。我们提出了隐藏解码（Hidden Decoding），一种在持续预训练（CPT）中应用的序列长度扩展方法。它将每个词元扩展为n个流，使用独立的嵌入表，并保留中间流的键值缓存作为上下文，这样每个词元在内部执行更多计算，而无需增加或拓宽Transformer层。为了使其在大规模下可行，我们引入了流分解注意力（Stream-Factorized Attention），其中大多数层仅在每个流内进行注意力计算，只有少数层在不同流之间混合，从而将注意力成本从n的二次方降低到近似线性。实验支持两个扩展结果。在前沿规模下，我们训练了n=4的WeLM-HD4-80B和WeLM-HD4-617B模型，并改进了它们匹配的非HD基线，使隐藏解码成为首个在100B+ MoE规模下得到验证的序列长度扩展方法。在不同的扩展因子下，性能增益随着n的增加而增长，表明序列长度扩展是前沿大规模LLM的一种实用的固定主干扩展路径。

## Abstract
Scaling Large Language Models (LLMs) has been driven mainly by enlarging the Transformer backbone, but for an already-strong model this requires another round of costly pretraining. We study whether an existing backbone can keep improving by allocating more computation to each token while leaving the Transformer backbone fixed. Depth-recurrent (looped) Transformers pursue this goal but are hard to scale, because looped computation does not fit naturally with the pipeline parallelism used to train the largest models. We add computation along the sequence-length dimension, where the extra computation is simply a longer input and stays compatible with standard large-model training. We propose Hidden Decoding, a sequence-length scaling method applied during continued pretraining (CPT). It expands each token into n streams with independent embedding tables and keeps the intermediate streams' key-value cache as context, so each token performs more internal computation without adding or widening Transformer layers. To keep this affordable at scale, we introduce Stream-Factorized Attention, in which most layers attend only within each stream and only a few layers mix across streams, reducing the attention cost from quadratic to roughly linear in n. Experiments support two scaling results. At frontier scale, we train WeLM-HD4-80B and WeLM-HD4-617B at n=4 and improve their matched non-HD baselines, making Hidden Decoding the first demonstrated sequence-length scaling method at the 100B+ MoE scale. Across expansion factors, the gains grow as n increases, showing that sequence-length expansion is a practical fixed-backbone scaling path for frontier-scale LLMs.

# 论文核心总结：Hidden Decoding at Scale

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：当前大语言模型（LLM）性能提升主要依赖扩大Transformer主干（增加层数或宽度），但已有强模型若要进一步扩展，需要昂贵的大规模重预训练。能否在不改变主干的前提下，通过增加每个token的内部计算量来继续提升模型能力？
- **现有尝试**：深度循环（looped）Transformer通过重复使用同一主干逐token进行多次前向传播来增加计算，但难以扩展——因为循环执行模式与训练超大模型所必需的流水线并行不兼容，导致GPU空闲。
- **本文思路**：沿序列长度维度扩展计算，即将每个token展开为多个并行“流”，使得额外计算体现为更长的输入序列，从而兼容标准的训练优化（如流水线并行、张量并行）。该方法称为**Hidden Decoding**，应用于持续预训练（Continued Pretraining, CPT）阶段。

## 2. 方法论
### 核心思想
- 将每个输入token复制为**n个流**，每个流拥有独立的嵌入表（embedding table），形成长度为nL的扩展序列。
- 通过单次前向传播处理整个扩展序列，因果注意力自然覆盖所有流。
- 只对每个token的**最后一个流**施加下一个token预测的交叉熵损失（监督），前n-1个流不接收直接损失，作为潜在计算状态（latent computation states）。
- 保留所有中间流的KV缓存，使得前面流的计算成果可作为后续token的上下文。

### 关键技术：流分解注意力（Stream-Factorized Attention）
- 为避免对扩展序列进行全注意力带来的O(n²L²)成本，设计两种注意力层：
  - **流内层（intra-stream）**：各token仅在其所属流内进行因果注意力，成本O(nL²)。
  - **跨流层（cross-stream）**：允许跨流混合，沿用基础模型原有的注意力模式（全注意力或滑动窗口注意力）。
- 大多数层为流内层，仅少数层（如80B模型6层全注意力、23层滑动窗口、20层流内）进行跨流混合。这样注意力总成本接近线性（O(nL²)），而非二次。

### 渐进扩展（Progressive Expansion）
- 在CPT过程中逐步增大n：1→2→4→8。
- 初始化新流嵌入表时使用**循环复制**：将已有的n个嵌入表复制一份作为新n个流，使得初始行为与之前一致，训练平稳。

### 其他优化（特定于WeLM架构）
- 利用**KV镜像（KV-mirror）**设计：后半部分层的KV从前半部分层计算得到，因此中间流在镜像层可以跳过计算，只处理最后一个流。这进一步降低训练成本约20%。

## 6. 主要结论与发现
1. **Hidden Decoding提升性能**：在80B和617B MoE上，n=4的HD模型在全部9个共享基准上均超过非HD基线，尤其在数学和科学任务上增益显著（如80B上SciCode +4.2, PHYBench +4.0, FrontierMath +3.2）。
2. **扩展因子作为缩放旋钮**：在密集8B模型上，n从2增至8，平均准确率从78.2升至81.6；在80B MoE上，n=8时MMLU达到87.5（基数为85.1），Pile-test BPB降至0.378（基线0.386）。
3. **训练成本接近线性**：n=4时，实际训练时间增长为4.4×–5.1×，远低于密集注意力的16×，使大模型训练可行。
4. **流分解注意力高效**：仅需少量全注意力层（如1层）即可提升性能，大部分层可保持流内注意力。
5. **最终流监督最优**：优于对所有流施加损失或求和输出。
6. **中间流形成不同状态并被最终流利用**：探针显示隐藏状态分离，最终流对其他流有较高注意力，且中间流LM头探针显示较宽的概率分布。

## 7. 优点
- **工程实用性**：将额外计算转换为序列长度扩展，完美兼容流水线并行，解决了循环模型难以大规模训练的痛点。首次在100B+ MoE规模验证。
- **效率设计**：流分解注意力使注意力成本接近线性，配合WeLM特有的KV镜像进一步降低。渐进扩展策略有助于稳定训练。

- **视角创新**：从深度维度转向序列维度进行固定主干的计算扩展，开辟了新路径。

## 摘要
我们提出了MyAG，一个基于图的可组合LLM智能体系统设计与分析框架。我们的框架将智能体系统的构建分离为三个图抽象：组件图（用于智能体、环境和模块）、工作流图（用于执行控制）以及搜索图（用于运行时执行）。这种分离使得用户能够灵活地复用相同的组件并采用不同的策略。我们进一步通过递归系统节点支持层次化组合，并提供监控和可视化工具来检查智能体执行。在代表性智能体应用上的实验表明，我们的框架支持灵活的智能体系统设计，并有助于分析性能与效率的权衡。我们的框架已经公开并提供完全开源。

## Abstract
We present MyAG, a graph-based framework for designing and analyzing composable LLM agent systems. Our framework separates agent system construction into three graph abstractions: a component graph for agents, environments, and modules; a workflow graph for execution control; and a search graph for runtime execution. This separation allows users to flexibly reuse the same components with different strategies. We further support hierarchical composition through recursive system nodes and provide monitoring and visualization tools for inspecting agent execution. Experiments on representative agent applications show that our framework supports flexible agent system design and helps analyze performance-efficiency tradeoffs. Our framework is publicly available and fully open-source.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：随着大语言模型（LLM）的发展，构建 LLM 智能体系统时涉及组件组合、工作流执行和运行时搜索策略等多个设计维度。现有框架（如 CAMEL、MetaGPT、AutoGen、LangGraph 等）要么聚焦于特定方面，要么代码库庞大、抽象复杂，导致组件、工作流和搜索之间的边界模糊，难以灵活复用组件、切换策略，也缺乏全面的效率分析。
- **整体含义**：MyAG 旨在提供一种轻量级、图驱动的框架，通过分离组件图、工作流图和搜索图，实现关注点分离；支持层次化组合；并内置效率分析工具，帮助用户在不同设计选择下进行性能-效率权衡评估。

## 2. 论文提出的方法论

- **核心思想**：使用三种图抽象分别描述智能体系统的不同方面：
  - **组件图（Component Graph）**：定义系统中可用的组件（智能体、环境、工具、子系统等）及其交互关系。
  - **工作流图（Workflow Graph）**：指定组件如何按流程执行，包括控制流（循环、分支、停止条件），节点关联组件图中的智能体。
  - **搜索图（Search Graph）**：记录运行时执行过程（状态、分支、轨迹），支持通过状态恢复探索不同路径，从而实现不同搜索策略（贪心、Beam Search、Best-First 等）。
- **关键技术细节**：
  - **层次化组合**：通过系统节点（System Node）包裹另一个完整智能体系统，每个子系统拥有自己的组件图、工作流图和搜索图。提供 stateless 和 stateful 两种模式管理子系统状态。
  - **效率分析**：定义两种成本：
    - LLM 推理成本 \( C_L = \sum_{t=0}^{T-1} (\alpha \cdot L_t^{in} + \beta \cdot L_t^{out}) \)，其中 \( L_t^{in}, L_t^{out} \) 为输入/输出 token 数，\( \alpha, \beta \) 为可配置权重（如按价格或延迟设置）。
    - 环境交互成本 \( C_E = \sum_{t=0}^{T-1} w(s_t, a_t) \)，其中 \( w \) 为针对动作 \( a_t \) 在状态 \( s_t \) 下的可配置成本函数（实验中用经验延迟）。
  - **算法流程**：执行时，工作流图驱动从 start 节点开始，每次到达一个工作流节点即调用对应组件图中的智能体，根据输出变量决定下一节点。搜索图记录状态，支持任意搜索策略（如贪心、Best-First、Rollback、Best-of-N）。

## 6. 论文的主要结论与发现

- Greedy 策略最节省计算资源（单轨迹），但性能不如 Rollback。
- Rollback 策略在充足预算下获得最佳整体性能，但额外带来一点开销；Best-first 和 Best-of-N 能通过更多计算提升性能，但引入较大开销，且其效果依赖验证模块（如评分器）的质量。
- 不同 LLM 大小对性能-效率权衡有显著影响：MoE 模型（如 35B-A3B）在激活参数成本模型下表现更优，但大模型仍需更多内存。
- 动作级分析显示不同动作（如 scroll vs wait）的成本差异很大，表明细粒度成本记录对优化有价值。
- 总体表明 MyAG 框架能够灵活设计系统并有效分析性能-效率权衡。

## 7. 优点

- **轻量级与清晰抽象**：核心代码仅约 4K 行，三种图抽象分离关注点，复用组件灵活。
- **层次化组合**：通过系统节点支持嵌套子系统，每个子系统独立定义图，适用于复杂任务分解。
- **效率分析工具**：不仅记录总成本，还分解为 LLM 成本和环境成本，支持用户自定义成本模型，便于优化。
- **监控与可视化**：提供 dashboard 展示执行轨迹、图结构、变量、成本分布等，利于调试和理解。
- **完全开源**，易于研究社区复用和扩展。

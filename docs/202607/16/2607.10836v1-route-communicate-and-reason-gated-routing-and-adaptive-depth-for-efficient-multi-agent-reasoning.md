## 摘要
多智能体集成会在不回答三个基本问题的情况下倍增活跃参数和推理成本：应咨询哪些智能体、查询应在智能体层级中深入多少层、以及智能体间的通信何时值得其成本。我们提出GRADE（面向高效推理的门控路由与自适应深度），这是一个层级式多智能体系统，其中四个轻量级学习门共同控制智能体选择、层级深度、智能体间通信以及分支剪枝。训练采用CoGRPO（协作组相对策略优化），这是一种新颖的无评论家训练方法，将GRPO适应于多智能体层级，并为参与策略展开的每个门和智能体分配共享优势信号。智能体模型来自可热插拔的专家注册表；每个智能体的校准映射允许在推理时无需重新训练即可替换专家。在约170亿平均活跃参数下，GRADE在GSM8K、MMLUPro和GPQA上超越所有基线，在MMLUPro上以一半的活跃计算量超越最强基线4.8个百分点。在模型深度主导的AIME-2025上，GRADE与现有框架保持竞争力。消融实验将层级和掩码交叉注意力列为准确性的最大贡献因素，并表明每个智能体的校准对于安全热插拔是必要的。

## Abstract
Multi-agent ensembling multiplies active parameters and inference cost without answering three basic questions: which agents to consult, how deeply a query should traverse a hierarchy of agents, and when inter-agent communication is worth its cost. We present GRADE (Gated Routing and Adaptive Depth for Efficient Reasoning), a hierarchical multi-agent system in which four lightweight learned gates jointly govern agent selection, hierarchy depth, inter-agent communication, and branch pruning. Training uses CoGRPO (Collaborative Group-Relative Policy Optimization), a novel critic-free recipe that adapts GRPO to multi-agent hierarchies and assigns a shared advantage signal to every gate and agent that participated in a rollout. Agent models are drawn from a hot-swappable Expert Registry; per-agent calibration maps allow experts to be replaced at inference time without retraining. At $\sim$17B average active parameters, GRADE outperforms all baselines on GSM8K, MMLUPro, and GPQA, surpassing the strongest baseline by 4.8 points on MMLUPro at half the active compute. On AIME-2025, where model depth dominates, GRADE remains competitive to existing frameworks. Ablations isolate the hierarchy and masked cross-attention as the largest contributors to accuracy, and show that per-agent calibration is necessary for safe hot-swapping.

## 1. 核心问题与整体含义

- **研究动机**：现有多智能体集成系统在推理时面临三个根本问题：面对一个查询应该咨询哪些智能体（路由选择）、查询应该在智能体层级中深入多少层（自适应深度）、以及智能体间的通信何时值得其计算成本（通信控制）。现有系统要么固定拓扑、要么全激活、要么无选择地广播，缺乏对这些问题的联合优化。
- **整体含义**：本文提出GRADE（Gated Routing and Adaptive Depth for Efficient Reasoning），一个层次化的多智能体系统，通过四个轻量级学习门控（分配门、深度门、交叉阅读门、剪枝门）端到端地控制代理选择、层级深度、通信和分支剪枝，并引入无评论家的协作策略优化算法CoGRPO。该系统在约170亿平均活跃参数下，在GSM8K、MMLUPro、GPQA上超越所有基线，尤其在MMLUPro上以一半计算量领先最强基线4.8分，展示了“克制式计算”优于“叠加容量”的范式。

## 2. 方法论

- **核心思想**：GRADE构建一个三级层次架构（Orchestrator M0 → Managers → Sub-agents），通过四个可学习门控协同决策。所有门控、适配器、组合头和编码器最后两层可训练，而子智能体的底层语言模型保持冻结。训练使用CoGRPO，将GRPO扩展到多智能体联合策略，无需评论家，对每个参与展开的门和智能体分配相同的组相对优势。
- **关键技术细节**：
  - **分配门（Assignment Gate）**：基于查询表示与智能体描述嵌入的亲和力加上所属管理者的分组信号，为每个智能体计算分数，然后用Bernoulli采样决定激活集，保证至少一个智能体被激活。加入负载均衡正则化防止坍缩。
  - **深度门（Depth Gate）**：MLP输出深度层级ℓ*（0/1/2），分别对应只走Orchestrator、加入管理者、激活完整子智能体池。训练时采样探索，加入深度惩罚。
  - **交叉阅读门（Cross-Read Gate）**：对每对智能体的消息嵌入，使用Gumbel-sigmoid输出二进制通信决策，决定是否可在随后的掩码交叉注意力中互相关注。加入稀疏性惩罚。
  - **剪枝门（Prune Gate）**：为每个活跃分支输出标量值va，低于阈值0.35的分支在组合前被丢弃，至少保留一个分支。
  - **答案组合**：幸存智能体的校准消息通过掩码多头交叉注意力融合，再经组合头解码得到最终答案。使用一个持久性记忆缓冲（批级指数移动平均）提供稳定的上下文偏置。
  - **训练算法CoGRPO**：对每个查询采样G=8次联合展开，每个展开得到完整门控动作序列。奖励包含四项：正确性终端奖励（权重2）、对数边际奖励、每个幸存智能体的对数答案概率奖励、计算成本惩罚。在组内对奖励进行标准化得到组相对优势。目标函数包含裁剪重要性比率和KL散度，加上门控正则化和监督辅助损失。
  - **热插拔专家**：每个子智能体配备一个可学习校准映射κa（仿射归一化），在替换专家后仅需64个锚点查询重新估计该映射的参数，其他参数不动。

## 6. 主要结论与发现

- GRADE在GSM8K（94.8%）、MMLUPro（78.2%）、GPQA（57.4%）上超越所有基线，总体平均63.9%，优于最强基线Puppeteer（62.1%），而活跃参数仅为Puppeteer的约一半（17B vs 28B）。在AIME-2025上，Puppeteer凭借更大模型深度获得1.9分优势（27.2 vs 25.3）。
- 组件消融表明：**层次结构是最大贡献者**（移除后总体下降7.2分），其次为交叉阅读（-4.2分）和交叉注意力（-3.1分）。动态路由（Bernoulli采样）优于任何固定k路由，且k=3时性能最高，k=5时下降，印证“克制式计算”观点。
- CoGRPO优于所有对比的RL目标：无评论家+组标准化的组合效果最好，MAPoRL和MAPPO受限于评论家拟合困难。
- **校准映射对于安全热插拔至关重要**：无校准时立即下降18分，且20个查询后仍未恢复；而校准后各类替换在0-9个查询内恢复。
- 门控学习到了难度感知分配：GSM8K平均使用2.1个智能体、深度1.8；AIME-2025使用3.8个智能体、深度2.0。交叉阅读频率稳定在ρ≈0.5，过少或过多通信均损害性能。

## 7. 优点

- **自适应计算分配**：深度门允许简单查询快速终止于Orchestrator，节省计算；剪枝门去除低价值分支，避免浪费。
- **细粒度通信控制**：交叉阅读门学习每对智能体是否需要通信，而非全广播或无通信，在GPQA等多领域问题上特别有效。
- **热插拔机制**：专家注册表和每智能体校准映射使得推理时可替换专家，无需重新训练门控，校准只需64个查询，实用性强。
- **无评论家训练**：CoGRPO避免了价值函数拟合的困难，直接组内标准化得到优势，简化训练并提升稳定性。
- **系统性消融**：所有组件贡献被单独量化，结论可信；分析了不同替换场景下的恢复曲线，提供了工程指导。
- **公平比较**：考虑了活跃参数、VRAM、延迟多方维度，且对比了多种同类方法（MAGRPO、MAPoRL等），代码开源促进可复现性。

## 8. 不足与局限

- **AIME-2025表现不足**：GRADE在需要长链推理的奥林匹克数学上落后于Puppeteer，表明层次化路由无法完全补偿单智能体深度推理能力。作者也指出加入数学专用后端可能是解决方案，但未验证。
- **训练收敛问题**：AIME-2025的token使用趋势在200步后仍上升，说明门控尚未稳定，可能训练步数不足或超参数未对该任务充分调优。
- **小规模验证**：所有实验在单GPU、约17B平均活跃参数下进行，更大规模（如使用更大专家模型、更多智能体）的行为未知。层次架构的扩展性未讨论。
- **校准映射局限性**：校准仅基于64个锚点查询，对于与原始专家分布差异极大的替换（如从GPT-4o-mini切换到本地小模型），恢复需要9个查询，在极端分布偏移下可能仍需更多样本。
- **领域覆盖有限**：仅在推理类基准评估，未在代码生成、创意写作、工具使用等任务上测试，这些场景可能对通信和路由有不同要求。
- **对比公平性讨论**：尽管控制了活跃参数，但各基线的推理策略差异大（如Puppeteer使用4个8B模型顺序讨论），GRADE用更小平均参数胜出，但AIME上反而不如，说明存在任务相关trade-off，需要更细粒度分析。

## 摘要
多智能体系统（MAS）通过结构化工作流协调多个基于LLM的智能体，在获得推理能力的同时，由于多步执行和重复模型调用带来了高推理延迟。现有的编排方法主要优化任务性能和推理成本，而延迟问题在很大程度上未得到解决。在MAS中，端到端延迟由关键执行路径决定，因此仅降低总成本并不能可靠地减少延迟。此外，在保持准确性的同时优化延迟仍然具有挑战性：朴素的延迟优化可能会错误分配操作级信用并降低任务准确性。为解决这一问题，我们提出了延迟感知多智能体系统（LAMaS），一种用于基于学习的多智能体系统的延迟感知编排框架。LAMaS在两层面上应对这一挑战：在训练时，它通过具有关键路径感知信用分配的约束优化来学习延迟感知的执行图；在推理时，由于训练时确定的图无法利用运行时证据，它通过一个轻量级控制器补充图构建，该控制器在执行过程中自适应地消除冗余的未来智能体交互。在四个基准测试上的实验表明，LAMaS在评估的基于学习的MAS基线中实现了最低延迟，将端到端延迟降低了50%以上，同时保持了具有竞争力或更高的准确性。LAMaS还具有模块化特性，可以以最小更改迁移到其他MAS，持续带来延迟降低。

## Abstract
Multi-agent systems (MAS) coordinate multiple LLM-powered agents through structured workflows, gaining reasoning power but incurring high inference latency from multi-step execution and repeated model invocations. Existing orchestration methods primarily optimize task performance and inference cost, leaving latency largely unaddressed. In MAS, end-to-end latency is governed by the critical execution path, so reducing total cost alone does not reliably reduce latency. Moreover, optimizing latency while preserving accuracy remains non-trivial: naive latency optimization can misassign operator-level credit and degrade task accuracy. To address this gap, we propose Latency-Aware Multi-agent System (LAMaS), a latency-aware orchestration framework for learning-based multi-agent systems. LAMaS addresses this challenge at two levels: at training time, it learns latency-aware execution graphs through constrained optimization with critical-path-aware credit assignment; at inference time, since a graph committed at training time cannot exploit runtime evidence, it complements graph construction with a lightweight controller that adaptively eliminates redundant future agent interactions as execution unfolds. Experiments on four benchmarks show that LAMaS achieves the best latency among evaluated learning-based MAS baselines, reducing end-to-end latency by over 50\% while maintaining competitive or better accuracy. LAMaS is also modular and transfers to other MAS with minimal changes, consistently yielding latency reductions.

## 1. 论文的核心问题与整体含义（研究动机和背景）

多智能体系统（MAS）通过结构化工作流协调多个基于 LLM 的智能体协作，获得更强的推理能力，但多步执行和重复模型调用导致高昂的推理延迟。现有自动化编排方法（如 MaAS、GDesigner、AgentDropout）主要优化任务性能和推理成本（通常是 token 数量或 API 费用），而延迟作为图结构的系统级目标被忽视。**关键洞察**：端到端延迟由执行图中的关键路径（critical path）决定，而非所有操作的总成本；延迟与成本在结构上不同，降低总成本并不一定能减少延迟。此外，简单地在奖励中加入延迟项会带来操作级信用分配错误，且没有准确率下限时延迟优化会严重损害任务精度。

## 2. 论文提出的方法论

### 核心思想
提出 **LAMaS（Latency-Aware Multi-agent System）**，在训练时通过**关键路径感知的信用分配**和**约束优化**学习延迟感知的执行图；在推理时通过**轻量级执行控制器**动态消除冗余的未来智能体交互，从而在保持精度的前提下显著降低端到端延迟。

### 关键技术细节

- **问题形式化**：给定查询 \(x\)，编排器构建有向无环图 \(G=(V,E)\)。关键路径长度 \(T = \max_{p\in\text{Paths}(G)}\sum_{o\in p} t(o)\) 决定端到端延迟。目标是学习一个策略 \(\pi_\theta(G|x)\)，同时最小化期望延迟和成本，并满足准确率下限约束。

- **编排器架构**：基于 agentic supernet（类似 MaAS），策略输出操作的概率，通过累积概率阈值 \(\tau\) 动态选择每层的操作子集。

- **训练时约束优化**：
  - 原始优化：\(\min_\theta \mathbb{E}[\lambda_t T + \lambda_c C]\)，s.t. \(\mathbb{E}[S] \ge S_0\)。
  - 通过拉格朗日松弛转化为奖励：\(R = \lambda_S \frac{S}{S_0} - \lambda_t T - \lambda_c C\)，其中 \(\lambda_S\) 是自适应更新的对偶变量（式6），当精度低于 \(S_0\) 时增大以优先正确性。

- **关键路径感知信用分配**：
  - 定义每个操作 \(o\) 的路径关键性权重 \(w(o) = \ell(o)/T\)，其中 \(\ell(o)\) 是经过 \(o\) 的最长路径延迟。
  - 操作级奖励：\(R(o) = \lambda_S \frac{S}{S_0} - \lambda_t w(o) T - \lambda_c C\)。关键路径上的操作获得完整延迟惩罚，近关键路径操作得到平滑衰减，非关键操作几乎无延迟信号。

- **学习目标**：策略梯度损失（式9），奖励用 EMA 归一化以降低方差。

- **推理时执行控制**：
  - 训练收敛后，冻结编排器并收集执行轨迹，训练一个轻量 MLP 控制器来预测**控制优势** \(\Delta_k = R_k - R_N\)（当前部分执行奖励 vs 全执行奖励）。
  - 控制器特征包括：已完成算子比例、最大共识比例、查询嵌入的压缩表示。
  - 决策：当 \(\hat{\Delta}_k > 0\) 时以概率 \(\sigma(\hat{\Delta}_k/\sigma_\Delta)\) 保留当前部分执行并终止后续算子（即在线消除冗余交互）。

## 6. 论文的主要结论与发现

- LAMaS 在所有四个基准上实现**最佳延迟**，相比最强学习型 MAS 基线（MaAS）降低 55.6%–75.6%，同时保持**竞争或更优精度**（多数情况下精度略高）。
- 关键路径 token 数与墙钟延迟强相关（Pearson r=0.901），验证了延迟优化的有效性。
- 推理时控制器贡献额外 16–21% 延迟降低，且接近 Oracle 性能（差距约 2s，0.25–0.46 pp）。
- 完整 LAMaS 流程可迁移至其他 MAS 架构（GDesigner、AgentDropout），延迟降低 27–39%，精度基本不变。
- 关键路径信用分配优于均匀惩罚：去掉后延迟和成本均上升，精度下降。
- 延迟权重 \(\lambda_t\)、激活阈值 \(\tau\)、网络层数 \(L\)、精度下限 \(S_0\) 提供平滑可控的灵敏度调节。

## 7. 优点

- **问题立意新颖**：首次将延迟作为显式结构目标纳入 MAS 编排优化，并揭示关键路径的核心作用。
- **方法设计合理**：
  - 约束优化 + 拉格朗日松弛避免了手动平衡精度与延迟。
  - 关键路径信用分配精准定位瓶颈，避免噪声梯度。
  - 推理时控制器利用运行时证据在线修剪，互补训练时图优化。
- **模块化与可迁移性**：管道不限于特定编排架构，可轻松集成到现有方法中。
- **实验全面**：覆盖多任务、多基线、多消融、多参数分析，并验证跨骨干和跨服务泛化，证据充分。

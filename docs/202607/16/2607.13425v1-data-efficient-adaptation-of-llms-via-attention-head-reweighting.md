## 摘要
在安全等领域中，标注样本稀缺时，从有限数据中有效学习至关重要。大语言模型（LLM）已展现出一定的数据高效学习能力，尤其是通过参数高效的适应方法，但在面对困难任务的少量样本时仍然存在困难。为应对这一挑战，我们提出注意力头重加权（AHR），这是一种数据高效的方法，通过为每个注意力头仅学习一个标量来使LLM适应新的文本分类任务。这利用了个体注意力头的功能专门化，大幅减少了需要学习的参数数量。在多种开源文本分类数据集上的实验表明，尽管AHR仅修改模型约0.0001%的参数（可训练参数比LoRA少200-1000倍），但在有限样本学习时其性能可超越LoRA等标准基线。此外，我们学习到的权重易于解释，可通过分析来更好地理解LLM中负责上下文学习能力的机制和注意力头。

## Abstract
Learning effectively from limited data is critical in domains like security where labeled examples are scarce. Large language models (LLMs) have demonstrated some capabilities for data-efficient learning, especially through parameter-efficient adaptation methods, but continue to struggle when faced with few samples for difficult tasks. To meet this challenge, we propose Attention Head Reweighting (AHR), a data-efficient method that adapts LLMs to new text-classification tasks by learning only a single scalar per attention head. This drastically reduces the number of parameters that need to be learned by making use of the functional specialization of individual attention heads. Experiments on diverse open-source text classification datasets show that AHR can outperform standard baselines like LoRA when learning from limited samples, despite having 200-1000x fewer trainable parameters, as our AHR only modifies ~0.0001% of the model's parameters. In addition, our learned weights are easy to interpret and can be analyzed to better understand the mechanisms and attention heads responsible for in-context learning abilities in LLMs.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：在 AI 安全等高危领域，攻击和威胁不断演变，标注样本稀缺，需要一种极数据高效的适应方法，既能利用 LLM 的上下文学习能力，又能避免过拟合。
- **整体含义**：通过极少量参数（每个注意力头仅一个标量）对 LLM 进行调整，既能有效适应新任务，又能保留模型原有能力并提供可解释性。

## 2. 方法论

- **核心思想**：利用注意力头的功能专门化，为每个注意力头学习一个可训练的标量系数 \(\beta_h\)，该系数缩放该头对残差流的贡献，而不修改头内部的权重矩阵。这样只调整极少量参数，避免过拟合。
- **关键技术细节**：
  - 修改多头注意力的更新规则：
    \[
    z^{l+1}_{\text{Att}} = z^l_r + \sum_{h \in H^{l+1}} (1 + \beta_h) \, h(z^l_r)
    \]
    其中 \(\beta_h\) 初始化为 0，训练时仅优化这些标量。
  - 训练时使用 **In-Context Finetuning (IC-FT)**：构建包含 k 个示例的上下文输入，最大化所有答案 token 的交叉熵损失，并加入 L1 或 L2 正则化：
    \[
    \min_{\beta} \sum_{(x,y) \in D_{\text{train}}} -\log P(y|x;\theta,\beta) + \lambda \left(\sum_h |\beta_h|^p\right)^{1/p}
    \]
    在测试时，\(\beta\) 可合并到 \(W_O\) 矩阵中，无额外推理开销。
- **算法流程**：
  1. 为每个注意力头创建可学习标量 \(\beta_h\)（初始为 0）。
  2. 对每个训练输入，构造包含 k 个示例的上下文序列。
  3. 前向传播时按修改后的注意力输出计算 logits。
  4. 计算答案 token 的交叉熵损失 + 正则化项。
  5. 反向传播仅更新 \(\beta_h\)，保持其他参数冻结。
  6. 可选将 \(\beta_h\) 合并到 \(W_O\) 以实现零额外推理成本。

## 6. 主要结论与发现

- AHR 在极少量训练数据（|D_train| ≤ 30）下一致优于 LoRA、AdaLoRA、IA³ 等基线，平均绝对准确率提升 2-4%（相对错误率降低约 10%），而可训练参数少 200-1000 倍。
- 在安全任务（Web 钓鱼检测、越狱检测）上表现尤为突出，例如仅用 10 个样本时比最佳基线高 6-7%。
- AHR 能有效避免过拟合，在测试集上对模型其他任务能力影响较小，甚至略有提升。
- 通过学习到的 \(\beta\) 权重可以识别出两类注意力头：
  - **任务专用头**：仅对特定任务（如钓鱼检测）有重要作用。
  - **通用 ICL 头**（如 L15H14、L15H3）：对所有文本分类任务都有正面贡献，可能与归纳头（induction heads）相关。
- 使用 In-Context Finetuning 可以进一步提升约 10% 准确率。

## 7. 优点

- **极参数高效**：仅修改模型约 0.0001% 的参数（512-1152 个标量），远少于 LoRA 等。
- **抗过拟合**：冻结头部内部权重，仅调整贡献度，在数据极少时表现稳定。
- **可解释性**：学习到的 \(\beta\) 权重直接反映每个注意力头对任务的重要性，可帮助分析 LLM 的上下文学习机制。
- **低资源需求**：训练速度快，存储开销小（合并后无额外推理成本），易于部署。
- **与 In-Context Finetuning 兼容**：充分利用上下文学习的信息，进一步提升少样本性能。
- **实验设计全面**：覆盖多种模型、任务、样本规模、消融和可解释性分析，结论可信。

## 8. 不足与局限

- **任务范围有限**：仅验证了文本分类任务。对于需要精细生成控制的任务（如摘要、翻译）可能不适用，特别是当模型原本没有专注于此的注意力头时。

- **分析深度有限**：对注意力头功能的解释仅限于本文使用的文本分类数据集，其在其他上下文中可能还有更复杂的作用。

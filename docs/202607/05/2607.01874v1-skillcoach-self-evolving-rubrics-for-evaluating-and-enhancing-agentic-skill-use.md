## 摘要
技能正成为LLM智能体可重用的操作层，编码了标准操作流程、领域规则、工具工作流、脚本和验证程序。在现实的技能库中，重叠的技能使得可靠的技能使用变得困难。最终验证器的成功对于评估和训练来说都过于粗糙，因为智能体可能通过试错方式选择干扰技能、跳过必要步骤、错误组合工作流或遗漏最终检查。我们提出SkillCoach，一种自我演进的评分标准框架，用于评估和增强智能体技能使用。SkillCoach从真实运行轨迹中推导出基于技能的过程评分标准，并沿四个维度评估轨迹：技能选择、技能遵循、技能组合和基于技能的反思。它保留外部验证器作为独立的结果信号，使得过程质量能够与偶然的任务成功区分开来。演进的评分标准进一步作为过程监督，用于选择高质量的训练轨迹。实验表明，演进的评分标准显著提升了评估质量，揭示了被最终准确性掩盖的失败，并为增强智能体技能使用提供了比仅基于结果过滤更强的监督信号。

## Abstract
Skills are becoming a reusable operational layer for LLM agents, encoding SOPs, domain rules, tool workflows, scripts, and validation routines. In realistic skill repositories, overlapping skills make reliable skill-use difficult. Final verifier success is too coarse for both evaluation and training, since an agent may pass through trial and error while selecting distractor skills, skipping required steps, composing workflows incorrectly or omitting final checks. We introduce SkillCoach, a self-evolving rubric framework for evaluating and enhancing agentic skill-use. SkillCoach derives skill-grounded process rubrics from real rollouts and evaluates trajectories along four dimensions: skill selection, skill following, skill composition, and skill-grounded reflection. It keeps the external verifier as a separate outcome signal, allowing process quality to be distinguished from accidental task success. The evolved rubrics further serve as process supervision for selecting high-quality training trajectories. Experiments show that evolved rubrics substantially improve evaluation quality, expose failures hidden by final accuracy, and provide stronger supervision signals than outcome-only filtering for enhancing agentic skill-use.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：
  - **SkillCoach**是一个自演化（self-evolving）的评分标准（rubric）框架。
  - **核心概念**：将智能体的技能使用能力定义为沿四个维度的**轨迹级元能力**：
    1.  **技能选择 (Skill Selection)**：是否选择了正确的金技能（Gold skills），避开了干扰技能（Distractor skills）。
    2.  **技能遵循 (Skill Following)**：是否严格按照金技能中的关键步骤操作。
    3.  **技能组合 (Skill Composition)**：是否正确地组合多个技能，并确保正确的执行顺序和中间产物传递。
    4.  **技能反思 (Skill-grounded Reflection)**：在提交最终结果前，是否对自己的输出进行了明确的检查或验证。
  - **关键设计**：将**外部验证器（External Verifier）** 作为独立的结果信号，与过程评估分开。这使得框架能区分出：*轨迹A*（正确结果+错误过程，如使用干扰技能后trial-and-error蒙对）和*轨迹B*（正确结果+完美遵循技能）。

- **关键技术细节**：
  1.  **任务过滤**：通过一个“技能依赖性”筛选协议，确保被研究任务在没有金技能时很难完成，有金技能后成功率大幅提升。
  2.  **干扰技能库**：模拟真实工业场景，将任务对应的金技能与无关联的和语义相似的干扰技能混合，增加评估难度。
  3.  **自演化评分框架**：
     - **初始构建**：根据任务指令、金技能、Oracle解决方案和代表性轨迹，生成初始评分标准。
     - **轨迹评判**：智能体在真实环境中运行，产生轨迹。轨迹评判器（Judge）应用当前评分标准，从四个维度进行评分。
     - **演化与验证门控**：
       - **仲裁（Arbitration）**：分析评判结果，提出局部补丁（Patch）来修正评分标准的问题（如增加证据要求、修正负面案例等）。
       - **验证门控（Validation Gate）**：对提出的补丁进行严格检验，只有通过“硬性检查”（不造成破坏性变化）和“软性目标”（提升证据覆盖度、一致性等）的补丁才会被接受。
       - **迭代**：不断重复上述过程，自动优化评分标准。
  4.  **训练**：利用最终的、最优的演化评分标准（R_best）作为过滤器，从所有通过验证器的轨迹中，筛选出同时满足四个维度高分标准的轨迹，用于监督式微调（SFT）训练。

### 6. 论文的主要结论与发现

1.  **技能使用是可测量和可诊断的**：通过从“技能选择、遵循、组合和反思”四个过程维度进行评估，可以显著揭示被最终准确性所掩盖的失败模式。例如，在干扰技能下，智能体可能因为“技能选择”能力下降而导致任务失败，而不仅仅是执行错误。
2.  **自演化评分标准更优质**：个人工验证显示，经过自演化的评分标准（R_best）相比初始版本（R0），在**关键点覆盖度、可用性、过滤一致性**上均有显著提升，且**无幻觉**。
3.  **基于过程的监督信号更强**：在SF训练中，使用评分标准过滤后的轨迹（Rubric-filtered SFT）训练效果**显著优于**仅依赖结果过滤的SFT（Outcome-only SFT）。具体来说，R_best过滤后的SFT帮助4B模型从8%提升至24%，而仅结果过滤反而导致性能下降。
4.  **技能选择是关键失败点**：在大规模、高相似度的技能库中，智能体的**技能选择**能力会率先退化并最终崩溃。不同模型有不同的“退化边界”和“崩溃边界”，这与模型能力相关。
5.  **过程维度的不同重要性**：在训练中，**“关键步骤遵循”** 是最有价值的监督信号，移除后性能下降最大；其次是技能组合顺序；反思维度的影响相对较小（但仍有正面作用）。

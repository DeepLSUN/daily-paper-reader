## 摘要
我们提出了Dynamo，一个无需训练的框架，能够在不进行任何权重更新的情况下适配冻结的VLM。在小型标注训练子集上，代理检查自身正确与错误的尝试，并演化出两种互补能力：用于认知瓶颈的可复用推理技能，以及用于感知瓶颈的可执行视觉工具。每个生成的工具都与一个指定何时调用该工具的技能配对，两种能力类型都累积在一个持久库中。在四个视觉推理基准和五个VLM主干网络上，Dynamo在所有20个模型-基准设置上提升了直接推理性能（平均准确率提升+5.6）。当预先给定工具集时，该框架学习何时调用每个工具，并且每步的工具选择在所有测试的主干网络上均有改进。针对任务特定强化学习（VTool-R1, DeepEyes），Dynamo以极少的计算量缩小了65-99%的强化学习差距，并在可用时与强化学习可加性地结合。

## Abstract
Improving vision-language models (VLMs) on visual reasoning typically requires retraining or hand-designed prompts and tools. We present Dynamo, a training-free framework that adapts a frozen VLM without any weight updates. On a small labeled training subset, the agent inspects its own correct and incorrect attempts and evolves two complementary capabilities: reusable reasoning skills for cognitive bottlenecks, and executable visual tools for perceptual ones. Each generated tool is paired with a skill that specifies when to invoke it, and both capability types accumulate in a persistent library. Across four visual reasoning benchmarks and five VLM backbones, Dynamo improves direct inference on all 20 model--benchmark settings (avg. +5.6 acc). When the tool set is given in advance, the framework learns when to call each tool, and per-step tool choice improves on every tested backbone. Against task-specific RL (VTool-R1, DeepEyes), Dynamo closes 65--99% of the RL gap at a fraction of the compute, and combines additively with RL when available.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：Dynamo是一个无需训练的框架，它在**小型标注训练子集**上运行一个“进化循环”。该循环通过分析自身的成功和失败案例，自动演化出两种互补能力：
  1.  **可复用的推理技能（Skills）**：用于解决认知瓶颈（如：复杂的多步推理）。
  2.  **可执行的视觉工具（Tools）**：用于解决感知瓶颈（如：视觉模糊、需要局部放大）。
  这些能力被存储在一个持久化的库中，并在推理时动态检索和调用。
- **关键技术细节**：

  - **问题形式化**：定义了一个能力集合 `C = (S, T)`，其中 `S`是技能库，`T`是工具库。智能体的推理过程为 `f_C(x, v) = π_θ(· | x, v, Retrieve(x, v; C))`，即先检索相关能力，再生成答案。目标是在不更新模型参数 `θ` 的前提下，学习最优的能力集 `C` 以最大化在验证集上的准确率。

  - **进化循环**（核心算法）：每轮迭代包含三个阶段：
    - **诊断**：智能体在训练子集上运行，收集正确和错误的推理轨迹。然后，**分析决策器**（AnalyzerDecider）检查这些案例（包括图像本身），进行**根因分析**，判断瓶颈类型是“认知”还是“感知”，并决定采取的动作：`skill`、`tool` 或 `both`。
    - **探索**：根据诊断结果，**生成器**（Generator）提出 `M` 个候选的`技能+工具`组合，每个组合用不同策略解决诊断出的瓶颈。
    - **验证与提升**：在训练子集上评估每个候选组合的性能（`f_{C∪c}`）。只有能**单调提升**当前性能的候选才会被**提升**（promote）并加入到能力库 `C` 中。

  - **技能（Skills）与工具（Tools）** 的设计：
    - **技能**：一个结构化的Markdown文档，包含“何时使用”、“策略”、“常见陷阱”和“工作示例”四个部分。
    - **工具**：一个`<=150行`的Python函数，用于处理图像（如裁剪、放大、增强对比度等）。
    - **掌握机制（Mastery）**：当一个工具被生成时，它会与一个配套的技能配对。这个技能是该工具的“掌握SOP”，其“何时使用”的谓词指定了工具的适用场景，“常见陷阱”则指出了有害场景。这使得工具的使用变得**选择性**（selective）而非不加区分，这是与以往方法的关键区别。
    - **外部工具集（Mode B）**：如果预先提供了高质量工具集，Dynamo可以跳过工具生成，转而生成每个工具的“掌握技能”，学习何时调用它们。

  - **在线分布漂移适应**：采用滚动窗口机制，持续监控新查询的分布。当检测到主导任务族变化时，自动在新的子数据流上触发进化循环，更新技能和工具库，使能力与时俱进。

### 8. 不足与局限

- **对骨干模型内省能力的依赖**：诊断器`AnalyzerDecider`和生成器`Generator`都基于同一个冻结的VLM。对于自我内省能力较弱的模型，可能导致瓶颈分类错误，进而产生不匹配的能力，影响进化效率和质量。
- **对正确信号的依赖**：进化循环和在线适应都需要**每个样本的正确性信号**作为“监督”。在完全无监督的生产环境中（无人类反馈、无自动QA、无LLM验证器），该方法无法直接应用。这是一个重要的应用限制。
- **基准范围有限**：实验仅限于图像领域的视觉推理基准。论文提及3D空间推理、医学影像等其他模态不在当前研究范围内，方法的泛化性有待验证。
- **未探索的组件**：论文研究的“多候选探索”（`M>1`）机制对性能的影响，被作者留作未来工作，目前其贡献尚未被量化评估。
- **可重复性细节**：虽然提供了算法伪代码和超参数，但核心的`AnalyzerDecider`和`Generator`的具体提示内容（Prompts）和具体实现的细节代码，由于篇幅限制未在正文中完全展示，可能对完全复现带来挑战。作者仅提及代码将在匿名仓库中公开，但并未在论文中。

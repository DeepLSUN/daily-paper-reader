## 摘要
理解和推理长上下文已成为在现实应用中部署大语言模型（LLMs）的关键需求。尽管最近的LLMs支持越来越长的上下文窗口，但它们常常无法利用输入中已存在的相关证据，揭示了上下文访问与有效上下文利用之间的差距。在这项工作中，我们提出了递归证据重放作为大语言模型的长上下文推理工具（RECONTEXT），这是一种无需训练的推理方法，用于改进长上下文推理。RECONTEXT利用模型内部的相关性信号构建查询条件化的证据池，并在最终生成前重放该证据池，同时保留完整的原始上下文。这种递归选择过程将证据组织与答案生成分离，无需训练、外部记忆或上下文剪枝。我们还基于联想记忆提供了理论分析，将上下文表征为记忆存储，问题表征为检索线索，注意力表征为线索-痕迹关联，重放表征为痕迹再激活。在八个128K上下文长度的长上下文数据集上的实验表明，RECONTEXT在Qwen3-4B、Qwen3-8B和Llama3-8B上持续改善证据利用，在所有三个基座上取得了最佳平均排名。代码可在https://github.com/Yanjun-Zhao/ReContext获取。

## Abstract
Understanding and reasoning over long contexts has become a key requirement for deploying large language models (LLMs) in realistic applications. Although recent LLMs support increasingly long context windows, they often fail to use relevant evidence that is already present in the input, revealing a gap between context access and effective context utilization. In this work, we propose Recursive Evidence Replay as LLM Harness for Long-Context Reasoning (RECONTEXT), a training-free inference method for improving long-context reasoning. RECONTEXT uses model-internal relevance signals to construct a query-conditioned evidence pool and replays it before final generation while preserving the full original context. This recursive selection process separates evidence organization from answer generation without training, external memory, or context pruning. We also provide a theoretical analysis based on associative memory, which characterizes the context as a memory store, the question as a retrieval cue, attention as cue-trace association, and replay as trace reactivation. Experiments on eight long-context datasets with 128K context length show that RECONTEXT consistently improves evidence utilization across Qwen3-4B, Qwen3-8B, and Llama3-8B, achieving the best average rank on all three backbones. Code is available at https://github.com/Yanjun-Zhao/ReContext.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出**RECONTEXT**（Recursive Evidence Replay），一种无需训练、无需外部记忆、无需上下文剪枝的推理方法。它利用模型内部的注意力信号（查询-上下文相关性）作为候选证据提案，将选中的token转换为文本语句（证据片段），然后在最终生成前将这些证据片段“重放”到原上下文中，同时保留完整原始上下文。证据选择是递归的：每一轮基于当前累积的证据池重新计算相关性得分，从而逐步发现与问题更相关的证据。

- **关键技术细节**：
  - **证据选择**：在每一轮，使用最后 `w`（默认8）个提示词（又称cue tokens）的注意力权重，在选定层头集合上平均，获得每个上下文token的 relevance score `r_i`。然后从原始上下文token位置中选择 top-K 候选位置 `P`。
  - **证据物化**：将选中的token位置映射到它们所在的句子或局部跨度，形成证据池 `E`。新证据通过句子拷贝得到，保持原始文本。
  - **递归证据筛选**：初始证据池为空 `E^(0)=∅`。在第 j 轮，当前提示为 `[C; ϕ(E^(j-1)); q]`，即原始上下文、之前证据池和查询。根据该提示计算新的相关性得分，选择新证据 `ΔE^(j)`，然后更新证据池 `E^(j)=E^(j-1) ∪ ΔE^(j)`。重复固定小的轮数 R（默认2）。
  - **最终生成**：使用 `[C; ϕ(E^(R)); q]` 作为提示生成答案。完整上下文始终保留，证据只用于强调而非排除。

- **公式与算法流程**（文字说明）：
  - 读一遍提示，计算所有tokencue的注意力聚合得分。
  - 选择 top-K 高分token（来自原始上下文）。
  - 将这些token扩展为整个句子作为证据。
  - 将证据插入到上下文和问题之间（格式如 `[C; evidence_list; q]`）。
  - 在新的提示上重复上述步骤R次，每次只添加新证据（去重）。
  - 最后一次生成最终答案。

- **理论分析**：基于联想记忆视角，将上下文看作记忆存储，问题看作检索线索，注意力看作线索-痕迹关联，重放看作痕迹再激活。作者给出了一个单调性定理，证明论文中隐藏表示与答案嵌入之间的余弦相似度随重放轮次严格增加。

### 8. 不足与局限

- **需要内部注意力信号**：无法直接用于不暴露注意力权重的闭源API模型。

- **超参数敏感性**：R和K的最优值因任务而异（如NQ在K=8最好，PopQA在K=32最好），需要针对任务调优。
- **未在更大模型（如70B）上验证**：当前仅测试4B和8B模型，规模限制。
- **证据物化假设**：证据假定为完整句子，可能对需要碎片化信息的任务不完美。
- **理论上证明的是单调性但实际收益有限**：重放轮次增加到很高时收益可能饱和。
- **实验覆盖**：虽有多数据集，但未涉及代码、对话等长上下文场景，也未与其他基于注意力的缓存方法（如SnapKV、H2O）对比。

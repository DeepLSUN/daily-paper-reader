## 摘要
合成孔径雷达(SAR)数据增强对于提高数据驱动的SAR解译模型的泛化能力至关重要，然而实际增强工作流常受限于异构数据集格式、依赖于任务的无数据需求、多样的生成方法以及生成样本的弱验证等问题。本文提出了SAR增强与生成智能体(SAGA)，一个基于模式且效益感知的智能体框架，用于面向任务的SAR数据生成与增强。给定自然语言请求和异构SAR输入，SAGA提取可观测的数据集事实，验证可执行的数据集模式，通过验证器约束规划选择可行的增强策略，并将所选策略编译为可审计的增强工作流。生成的数据进一步通过质量、分布、SAR伪影、重复、泄漏以及可选的下游任务评估器进行评估，以支持有证据的数据增强声明。通过将语义提议与确定性验证和执行分离，SAGA提高了SAR增强决策的可靠性和可复现性。在受控的智能体基准测试和下游SAR解译任务上的实验表明，与基于规则、仅使用LLM、ReAct风格和固定增强基线相比，SAGA改进了模式基础、技能规划、无效样本拒绝以及下游增强效用。

## Abstract
Synthetic aperture radar (SAR) data augmentation is important for improving the generalization of data-driven SAR interpretation models, yet practical augmentation workflows are often hindered by heterogeneous dataset formats, task-dependent metadata requirements, diverse generation methods, and weak validation of generated samples. This paper presents the \textbf{S}AR \textbf{A}ugmentation and \textbf{G}eneration \textbf{A}gent (SAGA), a schema-grounded and benefit-aware agent framework for task-oriented SAR data generation and augmentation. Given a natural-language request and heterogeneous SAR inputs, SAGA extracts observable dataset facts, validates executable dataset schemas, selects feasible augmentation strategies through validator-constrained planning, and compiles the selected strategy into an auditable augmentation workflow. Generated data are further assessed by quality, distribution, SAR-artifact, duplicate, leakage, and optional downstream-task evaluators to support evidence-qualified augmentation claims. By separating semantic proposal from deterministic validation and execution, SAGA improves the reliability and reproducibility of SAR augmentation decisions. Experiments on controlled agentic benchmarks and downstream SAR interpretation tasks show that SAGA improves schema grounding, skill planning, invalid-sample rejection, and downstream augmentation utility compared with rule-based, LLM-only, ReAct-style, and fixed-augmentation baselines.

## 1. 论文的核心问题与整体含义（研究动机和背景）

合成孔径雷达（SAR）数据增强对提高数据驱动的SAR解译模型的泛化能力至关重要。然而，实际工作中，增强工作流常面临四大障碍：
- **数据集格式异构**：SAR元数据（方位角、俯仰角、极化等）可能嵌入于目录名、文件名、侧边文件、标题或自定义标注格式，缺乏统一规范。
- **任务依赖的元数据需求**：不同用户请求（如目标增强、极化条件生成、稀疏视角补全、物理仿真）需要不同的元数据字段和格式假设。
- **增强方法多样且风险不同**：传统变换速度快但语义多样性有限；GAN生成易模式崩塌；扩散模型表现力强但计算昂贵；物理仿真可解释但存在仿真-真实域差距。每种方法有其输入要求、成本、收益和风险。
- **生成样本验证薄弱**：生成的图像可能虽然视觉合理，但元数据不一致、标签错误、存在重复或泄漏，无助于下游任务。

现有工作流多依赖人工试错或固定的方法选择，缺乏可靠性和可重复性。本文提出**SAGA（SAR Augmentation and Generation Agent）**，一个**以模式为基础、效益感知**的智能体框架，旨在将SAR数据增强转化为**数据集条件化、有证据支持**的决策过程。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
SAGA不是单个SAR生成器，也不是简单脚本集合，而是将SAR增强视为一个**模式基础且效益感知的决策过程**。其核心设计原则是**将灵活的语义提议（LLM辅助）与确定的验证执行相分离**。LLM可辅助自然语言意图解析、模式提议和高层规划，但其输出必须经过确定性守门员验证后方可执行。

### 2.2 六个主要组件
1. **Schema‑Grounded Dataset Profiling（模式基础数据集剖析）**
   - 第一阶段：确定性原始扫描器提取客观事实（文件扩展名、图像计数、尺寸、位深、目录结构、文件名词元等），不做语义假设。
   - 第二阶段：通过标准解析器、用户格式提示或LLM辅助模式归纳提出候选数据集模式（字段映射、元数据解析规则）。
   - 第三阶段：确定性验证器检查规则的解析性、覆盖度、图像-标签对齐、过滤有效性等。只有通过验证的模式才被接受为内部可执行模式。

2. **Benefit‑Aware Hybrid Planning（效益感知混合规划）**
   - 输入：已验证的数据集配置文件 ϕ(D)、结构化用户意图 ψ(q)、资源约束 C、技能描述符 K、策略记忆 M。
   - LLM提供高层意图理解和候选计划方案，确定性规则验证可行性并排序。
   - 候选技能 ri 的规划得分：**score(ri) = αB(ri; ϕ(D), ψ(q)) − βC(ri; C) − γR(ri; ϕ(D)) + δM(ri; M)**，其中 B 为期望效益，C 为执行成本，R 为风险，M 为历史证据支持。
   - 守门员规则拒绝不满足输入要求、元数据不可用、资源预算不足或不匹配的技能。

3. **Recipe‑Centric Skill Execution（以配方为中心的技能执行）**
   - 将选中的计划编译为可执行**配方DAG** R = (V, E)。每个节点 vi = (ki, xi, θi, yi, ρi) 绑定技能、输入工件、参数、期望输出和标准化运行报告。
   - 支持干运行（无昂贵的模型执行）和真实运行，记录种子、命令、种子、评估步骤等，保证可审计性和可重现性。

4. **Observer‑Driven Evaluation and Bounded Repair（观测器驱动评估与有限修复）**
   - 观测器组包括：格式与一致性、图像质量、SAR伪影、分布与多样性、重复与泄漏、可选下游任务评估器。
   - 证据等级（Level 1-5）用于区分生成成功与增强效用。只有 Level 5 才支持下游性能提升声明。
   - 触发修复仅限于**显式观测器失败**，且修复次数有限（预设最大值），仅允许修改安全参数（如归一化方式、风格强度、掩码阈值、生成数量等）。修复失败则记录失败并回退。

5. **Policy Memory with Evidence Accumulation（策略记忆与证据积累）**
   - 每个记忆条目存储：数据集配置文件、结构化意图、选中的配方、执行结果、观测器结果、证据等级 mj = (ϕ(Dj), ψ(qj), Rj, oj, ej)。
   - 在新任务规划时，检索具有相似配置文件、任务、元数据缺陷或资源约束的记忆条目，提供正/负证据，不影响确定性验证。

### 2.3 整体流程
用户自然语言请求 q + 异构SAR数据集 D + 资源约束 C + 技能库 K + 策略记忆 M → 数据集剖析与模式验证 → 意图识别与混合规划 → 配方DAG编译 → 技能执行 → 观测器评估与有限修复 → 输出增强数据集 Daug + 配方 R + 证据报告 E。

## 6. 论文的主要结论与发现

- SAGA能够从异构SAR数据格式中可靠地推断并验证可执行数据集模式（Exp.1：97.1%决策准确率，拒绝所有无效模式）。
- 在意图识别与技能规划上，SAGA的Top-1技能准确率92.5%，无效选择率0%，显著优于LLM-only（61.2%）和ReAct（61.2%）等基线（Exp.2）。
- 配方DAG执行保证了100%的协议通过率，而直接脚本和LLM工具调用均无法达到（Exp.3）。
- 观测器显著提高了检测能力（召回90%，F1 91.8%），但并非完美（假拒绝率8.6%）；有限修复可减少但仍留有残余无效样本（Exp.4）。
- 在下游增强效用上，SAGA获得平均分数60.1%，相比无增强提升7.4个百分点，且无效样本率仅1.0%，优于所有固定增强方法和LLM-only/规则选择方法。Lv5率94.8%（Exp.5）。
- 消融表明：移除技能守门员导致最严重退化（复合分64.1，无效选择18%），移除观测器导致无效样本率飙升到25%（Exp.7）。
- 总体结论：**通过分离语义提议与确定性验证执行，SAGA显著提升了SAR数据增强决策的可靠性和可重现性**。

## 7. 优点

- **模块化设计**：剖析、规划、配方、观测器、修复、记忆等组件松耦合，易于扩展和单独改进。
- **混合智能**：结合LLM的语义理解能力与确定性规则的可靠性，避免纯LLM工作流的不稳定性和不可重现性。
- **强调可重现性**：配方DAG和标准化运行报告使每次增强工作流可审计、可复现。
- **证据驱动**：提出证据等级（Lv1-5）区分生成成功与下游效用，防止过度声明。
- **格式适应**：不假设固定数据集格式，通过模式接地处理各种元数据嵌入方式。
- **安全机制**：守门员、观测器和有限修复三种安全网，减少无效输出。
- **消融实验充分**：全面验证每个组件的贡献。

## 8. 不足与局限

- **数据集私有化**：所有实验在私有SAR数据集上进行，无法被第三方复现验证，且数据集分布可能影响结论泛化性。
- **受控基准非开放世界**：Exp.1、2、4的案例是专家定义的，不能反映真实使用场景中的无限多样性。
- **观测器存在假阳性**（8.6%）：可能导致部分合格样本被错误拒绝。
- **修复不彻底**：对条纹伪影、梯度异常、泄漏等问题的修复后仍有残留无效率。
- **计算成本未量化**：SAGA可能需要调用计算密集型技能（如GeoDiff-SAR），论文未系统比较不同方法的总资源消耗。
- **记忆模块利用有限**：消融显示记忆对复合分提升约5.5%，但实际效果依赖于任务相似度，长期记忆的学习机制未深入探讨。
- **下游任务覆盖有限**：仅8个任务组，且均基于私有数据集，未在公开基准上验证，不能说明在所有SAR任务上的表现。
- **未评估生成图像的物理真实性**：以图像质量和下游任务分数为主，未与物理模型的散射中心等进行定量比对。

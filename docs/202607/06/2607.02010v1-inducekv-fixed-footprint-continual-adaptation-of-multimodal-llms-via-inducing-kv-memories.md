## 摘要
多模态大语言模型需要适应不断变化的任务和领域，但在有限的部署占用下持续改进仍然困难，因为重复的参数更新或不断增长的记忆回放存储会随时间累积适配状态。我们研究固定占用的持续适配：将部署的适配状态保持在固定的记忆预算内，同时保持骨干模型不变，并将任务特定的更新外部化。我们提出InduceKV，一种基于检索的方法，它将每个选定的训练前缀存储为一个注意力就绪的记忆条目，包含冻结的检索键和紧凑的逐层键值（KV）有效载荷，可以附加到模型的自注意力缓存中。在严格的记忆预算下，InduceKV通过双层选择构建紧凑的诱导集：轻量级校准适合检索，而选定的记忆在冻结检索空间中平衡当前任务似然、基于锚点的保留和覆盖。在任务增量指令微调、持续VQA、领域增量适配和终身多模态指令微调中，InduceKV在匹配的记忆预算下 consistently 优于PEFT、MoE、回放和提示检索基线。我们进一步报告了骨干匹配、阶段1 CoIN、计算匹配和可扩展性诊断，表明增益并非来自更强的骨干、单独的回放或无限的候选池。

## Abstract
Multimodal large language models must adapt to evolving tasks and domains, yet continual improvement under bounded deployment footprint remains difficult because repeated parameter updates or growing replay stores can accumulate adaptation state over time. We study fixed-footprint continual adaptation: the deployed adaptation state is kept under a fixed memory budget, while the backbone model is left unchanged and task-specific updates are externalized. We propose InduceKV, a retrieval-based method that stores each selected training prefix as an attention-ready memory entry, consisting of a frozen retrieval key and compact layerwise key--value (KV) payloads that can be appended to the model's self-attention cache. Under a strict memory budget, InduceKV constructs a compact inducing set through bilevel selection: a lightweight calibration is fit for retrieval, while the selected memory balances current-task likelihood, anchor-based retention, and coverage in the frozen retrieval space. Across task-incremental instruction tuning, continual VQA, domain-incremental adaptation, and lifelong multimodal instruction tuning, InduceKV consistently improves over PEFT, MoE, replay, and prompt-retrieval baselines under matched memory budgets. We further report backbone-matched, stage-1 CoIN, compute-matched, and scalability diagnostics, showing that the gains are not due to a stronger backbone, replay alone, or an unbounded candidate pool.

#### 1. 核心问题与整体含义

- **研究动机**：多模态大语言模型（MLLMs）在真实应用（如持续视觉问答、领域专业化助手）中，需要不断地适应新的任务和领域。然而，反复地重新训练整个模型或不断增长的记忆回放存储，会随时间累积巨大的部署状态，在计算、隐私和产品迭代方面都不可行。
- **核心问题**：如何在不更新骨干网络（Backbone）参数的前提下，让MLLM在**严格固定的外部内存预算**（Fixed Footprint）内进行持续适应，既能学习新知识，又能防止遗忘旧知识（灾难性遗忘）。本文将其明确为**有预算的在线诱导集选择问题**，而不是一个无限增长的参数更新过程。
- **整体含义**：本文挑战了主流的参数更新范式（如PEFT、MoE），提出了一种新的、外部化的记忆检索范式，为MLLM的终身部署提供了一种更高效、更可控的备选方案。

#### 2. 方法论（InduceKV）

- **核心思想**：
    1.  **冻结骨干网络**：完全冻结MLLM的所有参数，不进行任何梯度更新。
    2.  **外部化任务增量**：将每个任务的学习成果，编码为**注意力兼容的外部KV记忆条目**。
    3.  **检索增强生成**：在推理时，根据当前输入查询（Query）从外部记忆中检索最相关的条目，并将其KV有效载荷直接注入到模型的**自注意力缓存**中，从而影响生成过程。
    4.  **有预算的诱导集选择**：在固定内存预算下，通过一个**双层优化**过程，从候选池中挑选出最优、最不冗余的条目，构成一个紧凑的“诱导集”。
- **关键技术细节**：
    1.  **内存条目提取**：对于每个训练前缀（Prefix），计算一个**单位范数的检索键**（用于余弦相似度检索）和一组**压缩的、逐层的KV有效载荷**（通过池化压缩，长度固定）。
    2.  **注意力注入机制**：检索到的内存条目以加权方式（权重由检索相似度计算）生成额外的KV tokens，并拼接到原始自注意力缓存中。通过一个**可学习的逐层值门控**来精细控制每层记忆注入的强度。
    3.  **双层选择算法**：
        - **内层问题**：学习一个**最小的校准参数φ**（包含检索温度和值门控），用于优化当前任务在给定候选集和权重下的表现（似然）。
        - **外层问题**：学习**连续的选择权重w**（通过投影梯度下降优化），目标是：
            - **当前任务拟合**：最小化当前任务的损失。
            - **历史能力保留**：通过一个小的**历史锚点集**，用其损失来近似控制遗忘。
            - **多样性覆盖**：引入一个**谱覆盖正则化项**（基于对数行列式），惩罚选择高度相似的、冗余的键，鼓励选择多样化的条目。
        - 最终，选择Top-B权重的条目作为最终内存。
- **算法流程**（文字描述）：
    1.  对于到达的任务t，提取其训练数据的前缀，生成候选内存条目。
    2.  将新候选条目与上一阶段的内存合并，形成更大候选池。
    3.  初始化选择权重w和校准参数φ。
    4.  进行双层优化迭代：
        a.  固定w，更新φ（拟合当前任务）。
        b.  固定φ，更新w（平衡当前任务、锚点保留和多样性）。
        c.  将w投影到预算约束内。
    5.  选择Top-B权重的条目作为新内存，并保留学到的校准参数φ。

#### 8. 不足与局限

1.  **依赖离线候选池**：每个新任务都需要用冻结模型从训练数据中提取候选KV，这虽然不是推理时成本，但构成了任务更新时的计算开销。
2.  **双层优化成本**：双极优化本身的成本（虽然作者努力降低）仍然高于一些简单的PEFT方法，在极快变化的任务流中可能成为瓶颈。
3.  **推理开销**：虽低于提示检索，但仍**不是零开销**。它需要一次额外的前缀前向计算来获取检索键（Query），这比完全不进行检索的PEFT方法慢。作者明确承认了这一点，并定位其为提示检索的替代方案。
4.  **理论假设可能不成立**：其核心理论（遗憾界）依赖于损失函数凸性和梯度有界等假设，这在高度非凸的神经网络中是**无法保证**的。作者也坦承这一点，并将其作为“松弛分析”。
5.  **隐私风险**：虽然外部记忆可被审计、删除，但记忆条目本质上是从训练数据中提取的，在不安全的部署环境下存在泄露训练信息的风险。

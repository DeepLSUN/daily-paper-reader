## 摘要
我们提出了Wiola，一个完全原创的、从头构建的小型语言模型（SLM）架构，与任何现有模型家族（包括GPT、LLaMA、Mistral或Falcon）均无结构渊源。Wiola引入了五个独立新颖的组件：(i) 螺旋旋转位置编码（SRPE），在三维螺旋流形上嵌入标记位置，融合绝对、相对和层次位置信号；(ii) 门控跨层注意力（GCLA），为每个解码器层提供对前两层压缩摘要的软跨层注意力访问，以实现层间连贯性；(iii) 自适应标记合并（ATM），在网络中间层动态合并语义冗余的相邻标记，以降低注意力复杂度且不损失信息；(iv) 双流前馈（DSFF），用两个并行流替代传统MLP，通过学习的逐维度门控融合；以及(v) WiolaRMSNorm，一种改进的归一化方法，引入学习的逐维度偏移向量以防止表示坍塌。我们提供了完整的数学推导、架构框图、复杂度分析，以及与GPT-2、LLaMA-2和Mistral的系统比较。Wiola以四种尺寸（120M、360M、700M和1.5B参数）发布，完全兼容HuggingFace Transformers生态系统，所有22个架构单元测试均通过。

## Abstract
We present Wiola, a fully original Small Language Model (SLM) architecture built from first principles, sharing no structural lineage with any existing model family including GPT, LLaMA, Mistral, or Falcon. Wiola introduces five independently novel components: (i) Spiral Rotary Positional Encoding (SRPE), which embeds token positions on a three-dimensional helical manifold combining absolute, relative, and hierarchical positional signals; (ii) Gated Cross-Layer Attention (GCLA), providing each decoder layer with soft cross-attention access to compressed summaries of two preceding layers for inter-layer coherence; (iii) Adaptive Token Merging (ATM), which dynamically merges se mantically redundant adjacent tokens in middle network layers to reduce attention complexity without information loss; (iv) Dual Stream Feed-Forward (DSFF), replacing the conventional MLP with two parallel streams fused by a learned per-dimension gate; and (v) WiolaRMSNorm, a modified normalisation introducing a per-dimension learned offset vector that prevents representation collapse. We provide complete mathematical derivations, architectural block diagrams, complexity analyses, and systematic comparisons against GPT-2, LLaMA-2, and Mistral. Wiola is released in four sizes (120M, 360M, 700M, and 1.5B parameters) and is fully compatible with the HuggingFace Transformers ecosystem, with all 22 architectural unit tests passing.

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
Wiola采用纯解码器自回归架构，在传统Transformer基础上替换或新增五个关键模块，每个模块均基于独立数学原理设计，无参数量或极小参数量增加。

### 关键技术细节

#### (1) WiolaRMSNorm（修正RMS归一化）
- **思想**：标准RMSNorm只能缩放，不能偏移，导致深层网络表示坍缩。引入一个可学习的逐维度偏移向量δ，在归一化前对输入进行平移。
- **公式**：WRMSNorm(x) = γ ⊙ (x+δ) / RMS(x+δ)。δ初始化为0，梯度和非零，训练后必偏离0。
- **开销**：每层仅增加d个参数（360M模型增加32,768参数，仅占0.009%）。

#### (2) 螺旋旋转位置编码（SRPE）
- **思想**：自然语言具有词级、短语级（3-15 tokens）、段落级三种尺度。SRPE将位置嵌入到三维螺旋流形上，通过主频率、次级频率（慢速缠绕）和径向振幅编码三种尺度，无需额外参数。
- **关键公式**：对位置p和维度对j，计算两个缠绕角θ₁=pωᵢ和θ₂=pωᵢ/kₛ，总角Θ=pωᵢ(1+1/kₛ)，径向调制r=1+aₛ sin(p fₛ ωᵢ)。编码系数c=r cosΘ, s=r sinΘ。应用时通过2×2旋转矩阵作用于查询/键。
- **特性**：点积贡献显式含有相对差Δ，同时径向乘积引入绝对位置依赖，编码篇章结构。

#### (3) 门控跨层注意力（GCLA）
- **思想**：每个解码器层不仅做自注意力，还能通过软注意力访问前两层压缩摘要（均值池化），增强层间连贯性。
- **流程**：每层输出后取序列平均得到摘要s⁽ˡ⁾；上下文矩阵C⁽ˡ⁺¹⁾=[s⁽ˡ⁻¹⁾; s⁽ˡ⁾]；查询Q同时与自注意力KV和上下文K_ctx、V_ctx计算注意力；通过标量门β=σ(φ)（φ初始化为-3，β₀≈0.047）混合两个注意力输出；再通过逐标记的Sigmoid门控G控制最终输出。
- **开销**：上下文注意力FLOPs仅为自注意力的0.1%（Λ/T=2/2048）。

#### (4) 自适应标记合并（ATM）
- **思想**：网络中间层的相邻标记语义冗余高，可动态合并以减少注意力计算。
- **算法**：计算相邻tokens的余弦相似度ρ_t；若ρ_t > 阈值τ，则将两token均值合并；记录合并映射，注意力输出后按映射恢复原始长度。仅在训练时激活，部署时关闭。
- **效率**：合并率μ≈8-14%，单层FLOPs节省15-26%，应用于中间1/3层时总训练FLOPs降低约5-9%。

#### (5) 双流前馈（DSFF）
- **思想**：用两个并行密集流分别处理局部模式（窄SwiGLU流）和全局语义（宽GELU流），通过输入依赖的逐维度门控融合。
- **结构**：流A（窄，d_A=1024）：SwiGLU；流B（宽，d_B=4096）：GELU。融合门α=σ(W_f [a;b])，α∈(0,1)^d。最终输出DSFF(x)=α⊙a+(1-α)⊙b。
- **特性**：严格推广了流融合，W_f=0时退化为等权平均。

### 宏架构流程
- 输入嵌入 → L个解码器层（每层：WiolaRMSNorm → ATM(中间层) → GCLA(含自注意力+上下文注意力) → 残差连接 → WiolaRMSNorm → DSFF → 残差连接） → 最终WiolaRMSNorm → 线性投影（与嵌入权重绑定）输出logits。

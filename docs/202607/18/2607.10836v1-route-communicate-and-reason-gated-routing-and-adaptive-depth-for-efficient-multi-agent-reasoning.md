## 摘要
多智能体集成会增加活跃参数和推理成本，却没有回答三个基本问题：应当咨询哪些智能体、查询在智能体层级中应遍历多深、以及智能体间的通信何时值得其代价。我们提出了GRADE（面向高效推理的门控路由与自适应深度），这是一个层级化多智能体系统，其中四个轻量级学习门共同控制智能体选择、层级深度、智能体间通信和分支剪枝。训练采用CoGRPO（协作式群体相对策略优化），这是一种新颖的无评论家方法，将GRPO适配到多智能体层级，并为参与推演的每个门和智能体分配共享优势信号。智能体模型从可热插拔的专家注册表中提取；每个智能体的校准映射允许在推理时替换专家而无需重新训练。在约170亿活跃参数的平均情况下，GRADE在GSM8K、MMLUPro和GPQA上优于所有基线，在MMLUPro上以一半的活跃计算量超越最强基线4.8个百分点。在模型深度占主导地位的AIME-2025上，GRADE与现有框架保持竞争力。消融实验分离出层级和掩码交叉注意力是准确率的最大贡献因素，并表明每个智能体的校准对于安全的热插拔是必要的。

## Abstract
Multi-agent ensembling multiplies active parameters and inference cost without answering three basic questions: which agents to consult, how deeply a query should traverse a hierarchy of agents, and when inter-agent communication is worth its cost. We present GRADE (Gated Routing and Adaptive Depth for Efficient Reasoning), a hierarchical multi-agent system in which four lightweight learned gates jointly govern agent selection, hierarchy depth, inter-agent communication, and branch pruning. Training uses CoGRPO (Collaborative Group-Relative Policy Optimization), a novel critic-free recipe that adapts GRPO to multi-agent hierarchies and assigns a shared advantage signal to every gate and agent that participated in a rollout. Agent models are drawn from a hot-swappable Expert Registry; per-agent calibration maps allow experts to be replaced at inference time without retraining. At $\sim$17B average active parameters, GRADE outperforms all baselines on GSM8K, MMLUPro, and GPQA, surpassing the strongest baseline by 4.8 points on MMLUPro at half the active compute. On AIME-2025, where model depth dominates, GRADE remains competitive to existing frameworks. Ablations isolate the hierarchy and masked cross-attention as the largest contributors to accuracy, and show that per-agent calibration is necessary for safe hot-swapping.

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 设计 **GRADE（Gated Routing and Adaptive Depth for Efficient Reasoning）**，系统由**共享编码器**、**轻量级协调器M0**、**两个管理器模块**（可训练MLP，不含语言模型）和**多个子智能体（专家）** 组成。  
- 引入四个轻量级学习门（Assignment Gate、Depth Gate、Cross-Read Gate、Prune Gate），由小型两层层MLP实现，联合控制智能体选择、层级深度、通信和分支剪枝。  
- 训练使用 **CoGRPO（Collaborative Group-Relative Policy Optimization）**，一种无评论家（critic-free）的多智能体策略梯度方法，将GRPO扩展到层级多智能体场景，并为每个参与推演的智能体和门分配共享优势信号。  
- 构建**可热插拔专家注册表（Expert Registry）**，每个子智能体带有轻量级适配器、答案头和**校准映射κa**，允许在推理时替换专家模型而无需重新训练系统。

### 关键技术细节

#### 架构
1. **分层结构**：
   - **Level 0 – 协调器 M0**：小型语言模型（Qwen2.5-0.5B-Instruct），可生成直接答案。
   - **Level 1 – 管理器**：两个可训练MLP，每个管理一个子智能体池，为Assignment Gate提供分组信号。
   - **Level 2 – 子智能体**：从专家注册表中选取的冻结语言模型（如Qwen2.5-7B、Phi-3-mini、Llama-3.2-3B），每个带有可训练的适配器、答案头和校准映射。

2. **四个门**：
   - **Assignment Gate**：根据查询表示与智能体描述向量的亲和度，以及所属管理器的分组信号，对每个智能体打分，并使用伯努利采样决定是否激活（保证至少一个激活）。包含负载平衡正则化。
   - **Depth Gate**：输出一个深度级别ℓ⁺∈{0,1,2}（D=3），控制查询是否终止于M0、仅用管理器、或用完整子智能体池。训练时采样以探索，含深度惩罚项。
   - **Cross-Read Gate**：对每对智能体(a,b)通过Gumbel-sigmoid松弛决定是否允许通信，形成二元读掩码。含稀疏惩罚项。
   - **Prune Gate**：为每个激活的分支输出价值估计（sigmoid后的标量），丢弃低于阈值τ_P=0.35的分支，保证至少一个存活。

3. **答案组合**：存活智能体的校准消息通过掩码多头交叉注意力融合（利用Cross-Read掩码），然后经组合头解码为最终答案。引入了持久记忆缓冲区m（指数移动平均更新，训练时批内统计，评估时固定）。

#### CoGRPO训练算法
- **联合策略**：将四个门决策序列定义为联合动作，其概率为各门策略的乘积。  
- **密集奖励**：包含四项：正确答案奖励（r_terminal=2）、边际奖励（r_margin，增加预测正确概率的平滑信号）、智能体对齐奖励（r_agent，防止单个智能体搭便车）、计算成本惩罚（r_cost）。  
- **分组相对优势**：对每组G=8次推演，用组内均值与标准差标准化奖励，得到优势估计Â⁽ⁱ⁾ = (R⁽ⁱ⁾ - μ_R) / (σ_R + ε)。所有参与该推演的智能体和门共享同一优势（协作式信用分配）。  
- **剪裁代理目标**：使用重要性权重ρ与PPO剪裁（η=0.2），加上KL散度正则项（β=0.02），防止策略更新过大。  
- **总损失**：L = L_CoGRPO + L_bal + L_depth + L_read + 0.5·L_sup，其中L_sup为组合头的交叉熵损失（仅用于可微组件，不经过门采样操作）。

#### 热插拔专家校准
- 每个智能体a有校准映射κ_a(m_a) = γ_a ◇ (m_a - μ̂_a)/σ̂_a + β_a，其中μ̂_a、σ̂_a为运行统计量（动量0.05），γ_a、β_a为可学习参数。
- 交换专家后，仅用64个锚定查询重新估计被替换智能体的κ_a，其余部分不变。

## 6. 主要结论与发现
1. **GRADE在GSM8K（94.8%）、MMLUPro（78.2%）、GPQA（57.4%）上超越所有基线**，平均活跃参数仅约17B，在MMLUPro上比最强基线Puppeteer（约28B）高出4.8分，且计算量减半、显存降低约7%。  
2. **在AIME-2025上，Puppeteer（27.2%）优于GRADE（25.3%）**，说明纯模型深度在奥数级问题上仍有优势；但GRADE以更小、更弱的子智能体（7B vs 8B）实现了接近的性能。  
3. **层级结构（Hierarchy）是最关键组件**，移除后总体准确率下降7.2分（特别是GPQA上9.6分）。  
4. **Cross-Read通信门提升总体4.2分**，最佳通信比例为约50%（ρ≈0.5），过度通信（ρ=1.0）反而下降2.1分。  
5. **动态拓扑优于任何固定k设置**，k=3最佳（63.5%），k=5反降（62.1%），说明额外智能体会引入噪声。  
6. **CoGRPO优于其他RL目标**，比带评论家的MAPoRL和MAPPO高3.3~3.9分，比无std归一化的MAGRPO高1.6分。  
7. **每智能体校准映射是实现安全热插拔的必要条件**：移除校准导致即时准确率下降18.1分（78.2→60.1），且需超20次查询才能恢复；校准后恢复只需0-9次查询。  
8. **硬任务自然招募更多智能体和更深层级**：GSM8K平均2.1个智能体、深度1.8；AIME-2025平均3.8个智能体、深度2.0。

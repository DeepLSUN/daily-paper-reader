## 摘要
大型语言模型（LLM）代理日益依赖外部记忆系统来跨任务积累经验。然而，几乎所有现有方法（从图结构记忆到反思性洞察存储）都通过固定的、手工设计的启发式方法访问记忆。我们认为这种静态的记忆视图是代理学习的核心瓶颈，因为最优记忆行为从根本上依赖于上下文。任务早期阶段因记忆稀疏而受益于最小检索；重复出现的目标类型受益于计划复用而非通用最近邻查找；陷入困境的代理受益于使用替代查询重新检索；而在长任务流中，记忆存储本身必须进行整合和修剪以保持有用。我们提出了“记忆作为受控过程”（MemCon）框架，该框架将记忆操作建模为马尔可夫决策过程，并学习一种在线策略，自适应地决定何时、检索什么以及检索多少，何时注入精炼计划，以及何时整合或遗忘。MemCon与后端无关：它包装任何现有记忆实现，通过逐任务二进制反馈进行学习，无需预训练，无需额外LLM调用，并使用带有UCB探索的轻量级表格化上下文赌博机，在数十个任务内收敛。在6个基准测试、3个代理框架和3个LLM骨干上，MemCon在任务成功率上持续优于多个记忆基线，最高提升15.2个百分点，同时将令牌消耗降低5-20%。

## Abstract
Large Language Model (LLM) agents increasingly rely on external memory systems to accumulate experience across tasks. Yet nearly all existing approaches, from graph-structured memories to reflective insight stores, access memory through fixed, hand-designed heuristics. We argue that this static view of memory is a core bottleneck for agentic learning because optimal memory behavior is fundamentally context-dependent. The early stages of the tasks, benefit from minimal retrieval because memory is sparse; recurring goal types benefit from plan reuse rather than generic nearest-neighbor lookup; stuck agents benefit from re-retrieval with alternative queries; and across long task streams, the memory store itself must be consolidated and pruned to remain useful. We present Memory as a Controlled Process (MemCon), a framework that models memory operations as a Markov Decision Process and learns an online policy that adaptively decides when, what, and how much to retrieve, when to inject a distilled plan, and when to consolidate or forget. MemCon is backend-agnostic: it wraps any existing memory implementation, learns from task-by-task binary feedback with no pretraining and no additional LLM calls, and uses a lightweight tabular contextual bandit with UCB exploration that converges within tens of tasks. Across 6 benchmarks, 3 agent frameworks, and 3 LLM backbones, MemCon consistently outperforms multiple memory baselines by up to 15.2 points in task success while reducing token consumption by 5--20%.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想
- **记忆作为受控过程（Memory as a Controlled Process, MemCon）**：将记忆操作的选择视为一个**马尔可夫决策过程（Memory MDP）**，学习一个在线策略来自适应地决定何时、检索什么、检索多少、何时注入精炼计划、何时整合或遗忘。
- **后端无关的包装器（backend‑agnostic wrapper）**：MemCon 是一个薄层，插在任何现有记忆后端（向量存储、技能库、图记忆等）的 `retrieve` 和 `store` 接口上，通过学习策略调用后端，不修改后端内部数据结构。

### 2.2 关键技术细节

#### （1）Memory MDP 的形式化
- **状态 S**：由任务进度信号和记忆状态信号组成：
  - `stask` = （目标类型、步骤阶段、是否卡住、持有物体、已访问位置）
  - `smem` = （记忆大小、计划是否可用、学习阶段）
- 状态被离散化为哈希键 `ϕ(s)`，约数百种状态，便于表格化学习。
- **动作 A**：共 9 种操作（参数包括 `top_k`、`insight_k`、`hop`）：
  - `RETRIEVE`（浅、中、深）
  - `PLAN INJECT`：当存在精炼成功计划时，前置通用模板
  - `RE‑RETRIEVE`：使用替代查询后缀重新检索
  - `CONSOLIDATE` / `FORGET`：调用后端的维护钩子
  - `NOOP`：跳过本次记忆访问
- **奖励 R**：任务结束时根据环境反馈计算：
  - `r(τi) = r_succ · ⊮[success] + λ · max(0, 1 − Ti/T_max) − r_fail · ⊮[failure]`，其中 `r_succ=1.0`，`r_fail=0.5`，`λ=0.3`，`T_max=30`。
- **转移 T**：环境转移由 LLM 代理驱动，对控制器而言是黑盒，因此将 MemMDP 视为**上下文赌博机**，每个任务仅接收一个终末奖励信号。

#### （2）在线策略学习
- **动作选择**：使用 **UCB（Upper Confidence Bound）** 探索规则：
  - `a_t = argmax_a [ Q(ϕ(s_t), a) + c * sqrt(ln N(ϕ(s_t)) / N_a(ϕ(s_t))) ]`
  - `c` 为 UCB 探索系数（默认 1.4）。
- **热启动先验（warm‑start priors）**：在无任何数据前，Q 值被初始化为可解释的先验（例如 RETRIEVE +0.5，PLAN INJECT +0.3，NOOP −0.2），加速早期收敛。
- **信用分配（credit assignment）**：任务结束后，对该任务中所有 `(s, a)` 对使用**反向折扣蒙特卡罗回报**更新：
  - `Q(ϕ_j, a_j) ← Q(ϕ_j, a_j) + α [ γ^{|ep|−j−1} · r_i − Q(ϕ_j, a_j) ]`
  - 其中 `α` 为学习率（默认 0.15），`γ` 为帧内折扣（默认 0.9）。越接近任务结束的动作获得更大的信用权重。
- Q 表持久化到磁盘，跨任务学习延续。

#### （3）两个增强操作
- **通用计划注入（PLAN INJECT）**：从成功轨迹中提取动作序列，替换具体物体/位置为占位符，作为通用模板保存并在未来同类任务中注入。
- **目标分解（goal decomposition）**：对于需要重复同一原语的组合任务（如“放两部手机在桌上”），注入“先完成对象 1 的所有步骤，再重复对象 2”的提示。

### 2.3 算法流程概述
1. 代理开始新任务，遇到记忆决策点时，提取状态并离散化为 `ϕ(s)`。
2. 使用 UCB 选择动作（包括操作和参数）。
3. 包装器调用后端记忆系统，将检索结果（可能加上注入的计划）送入 LLM。
4. 代理完成该步骤，过程重复直到任务结束。
5. 任务结束时计算奖励，反向折扣更新 Q 表。

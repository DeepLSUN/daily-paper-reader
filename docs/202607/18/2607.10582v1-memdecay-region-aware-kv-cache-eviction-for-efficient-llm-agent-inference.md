## 摘要
大型语言模型（LLM）智能体会积累包括系统指令、计划、用户轮次、检索文档、工具输出和中间推理在内的异构上下文，其键值（KV）缓存可能成为主要内存瓶颈。现有的驱逐策略通常对所有token应用相同的基于注意力或基于新近度的规则，忽略了智能体编排器已经可用的语义结构。我们提出MemDecay，一种无需训练、区域感知的KV缓存驱逐策略。MemDecay为token分配区域特定的基础优先级和衰减率，在token获得注意力时刷新保留分数，并在固定缓存预算下驱逐得分最低的页面，同时允许关键区域被固定。我们还提供了一种根据测量的注意力生命周期校准衰减率的方法。我们使用Qwen2.5-1.5B和3B模型在约450和1,700个token上下文中评估MemDecay。在所有设置中，不同区域的注意力生命周期相差一个数量级：系统token的半衰期范围为148到189个解码步骤，而草稿板token为14到16个步骤。固定操作在所有设置下以全缓存精度保留系统区域事实，而基线最多保留24个中的13个。随着上下文增长，区域感知保留仍然有效，而基于新近度的保留则崩溃。然而，累积注意力保留在未固定内容上表现更好，消融实验将注意力分数归一化识别为当前公式的主要限制。这些结果建立了语义提示结构作为KV缓存管理的稳健信号，同时阐明了它应如何与基于注意力的重要性相结合。

## Abstract
Large language model (LLM) agents accumulate heterogeneous context, including system instructions, plans, user turns, retrieved documents, tool outputs, and intermediate reasoning, whose key-value (KV) cache can become a major memory bottleneck. Existing eviction policies generally apply the same attention- or recency-based rule to every token, ignoring semantic structure already available to the agent orchestrator.   We introduce MemDecay, a training-free, region-aware KV-cache eviction policy. MemDecay assigns tokens region-specific base priorities and decay rates, refreshes retention scores when tokens receive attention, and evicts the lowest-scoring pages under a fixed cache budget while allowing critical regions to be pinned. We also provide a procedure for calibrating decay rates from measured attention lifetimes.   We evaluate MemDecay at approximately 450 and 1,700 token contexts using Qwen2.5-1.5B and 3B. Across all settings, attention lifetimes differ by an order of magnitude across regions: system-token half-lives range from 148 to 189 decoding steps, compared with 14 to 16 for scratchpad tokens. Pinning preserves system-region facts at full-cache accuracy in every setting, while no baseline preserves more than 13 of 24. Region-aware retention remains effective as context grows, whereas recency-based retention collapses. Accumulated-attention retention performs better on unpinned content, however, and ablations identify attention-score normalization as the main limitation of the current formulation. These results establish semantic prompt structure as a robust signal for KV-cache management while clarifying how it should be combined with attention-based importance.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：为每个语义区域（region）分配基础优先级（base priority）和衰减率（decay rate），并结合注意力重要性（attention importance）计算每个token的保留分数。在固定缓存预算下，驱逐得分最低的页面（page），同时允许关键区域（如系统指令）被固定（pinned）永不驱逐。遗忘被视为一个可调度的过程：区域特定的时间衰减平滑降低token的保留优先级，而当模型再次关注该token时，其衰减时钟被重置（refresh），从而允许持续使用的内容保持活跃。
- **关键技术细节**：
  - **区域标签**：由编排层提供，包括 system, plan, user, tool_in, tool_out, retrieval, scratchpad。假设每个区域对应连续的prompt跨度，因此与现有服务栈中的页面粒度（page-granular）缓存布局兼容。
  - **保留分数公式**：
    \[
    s_i(t) = b_{r(i)} \exp(-\lambda_{r(i)} \Delta t_i(t)) + \alpha a_i(t)
    \]
    - \(b_{r(i)}\)：区域 \(r\) 的基础优先级（表1给出默认值，如system高，scratchpad低）
    - \(\lambda_{r(i)}\)：区域 \(r\) 的衰减率
    - \(\Delta t_i(t)\)：自上次刷新以来的时间（刷新条件：注意力高于阈值 \(\tau\)）
    - \(a_i(t)\)：token的注意力重要性，通过指数加权移动平均（EWMA）更新：\(a_i \leftarrow \rho a_i + (1-\rho)\bar{A}_i\)，其中 \(\bar{A}_i\) 是观察步骤中跨头、层和窗口平均的注意力质量
    - \(\alpha\)：注意力项的权重（实验中设为0.5）
  - **衰减率校准**：通过测量区域条件注意力生命周期来拟合衰减率。拟合对数线性模型 \(\log \bar{A}_r(\Delta t) \approx \log c_r - \hat{\lambda}_r \Delta t\)，设定 \(\lambda_r \leftarrow \hat{\lambda}_r\)，半衰期 \(t_{1/2}^r = \ln 2 / \hat{\lambda}_r\)。
  - **页面级驱逐**：服务栈存储KV条目在固定大小页面（page）中。页面得分为成员token得分的均值。当缓存超出预算时，按升序驱逐非固定页面，直到满足预算。算法流程见Algorithm 1（每观察步：更新a_i、刷新时钟、计算s_i，若超出预算则构建最小堆并驱逐最低分页面）。
- **固定区域**：默认固定系统区域，也可选择固定计划区域。固定token不参与评分和驱逐。

## 3. 实验设计：数据集/场景、benchmark、对比方法

- **数据集/场景**：基于8个脚本（scenario）构建，每个脚本有3个值变体（value variant），包含相同的人工植入事实（facts）和探测问题（probes）。实验分两档：
  - **短档（short tier）**：每个episode缓存405-507个token（5个阶段）。
  - **长档（long tier）**：每个episode缓存1,656-1,723个token（8个阶段，增加第二个检索文档、第三个工具交换和另一个用户轮次）。
- **探测任务**：每个episode植入4个人工事实，分别位于system, user, tool_out, retrieval区域，每个事实配一个探测问题。回答通过贪婪解码16个token后，检查是否在生成文本中出现gold value。每个配置单元聚合96个探测（24 episodes × 4 probe）。
- **对比方法**：
  - 全缓存（不驱逐）
  - Random（保留4个sink、16个最新token、均匀采样三种随机种子）
  - Streaming（保留4个sink + 最新token，对应StreamingLLM）
  - H2O-style（基于累积注意力的保留，重现在相同观察信号下）
- **评估指标**：召回率（recall），区分全部探测和排除系统固定区域后的非系统探测。
- **统计方法**：置信区间通过场景级百分位bootstrap（8个聚类）计算，配对场景符号比较作为主要统计依据。

## 6. 论文的主要结论与发现

1. **区域生命周期差异显著且稳定**：在所有设置中，system token的半衰期（148-189步）比scratchpad token（14-16步）高出一个数量级。retrieval区域半衰期相对较长（48-76步），推翻了表1中假设的快速衰减默认值。
2. **固定操作确保指令保留**：MemDecay在所有设置下固定系统区域后，以全缓存上限召回所有24个系统事实（短档24/24，长档21/24），而所有基线最多保留13个。流式方法在长上下文中基本失效。
3. **区域感知保留在长上下文中仍然有效**：基于新近度的Streaming随着上下文增长召回率骤降（短档0.36→长档0.01-0.13），而MemDecay的结构性先验保持稳定。
4. **累积注意力保留随模型规模增强**：H2O在1.5B长档上非系统召回率达0.53，在3B上达0.60，且优于MemDecay在未固定内容上的表现。
5. **MemDecay在未固定内容上的不足**：最旧的未固定事实（尤其是用户区域）在衰减先验下得分最低而被驱逐，而注意力能识别这些token。消融实验显示，调整注意力权重α（0.2-0.8）无法恢复这一损失，根源在于注意力项经过页面平均和EWMA后量级远低于结构项，需要幅度归一化改进。
6. **开销**：保留机制本身开销低（评分+页面选择0.27ms，驱逐4.2ms），但读取注意力的代价大（带评分解码69.8ms vs 无评分26.0ms in M3 Pro），生产部署需间隔采样。

## 8. 不足与局限

- **实验规模有限**：上下文长度仅1.7k token，远低于实际生产场景（通常数万token）；模型仅1.5B和3B，未在8B以上或更大模型验证。
- **评估任务人工化**：使用植入事实和简单探测的召回率，未在真实智能体基准（如SWE-bench、BFCL、LongBench）上评估，泛化能力存疑。
- **不可逆驱逐**：基础策略无法恢复被驱逐的token，当后续上下文使其重新相关时（如用户区域事实被过早删除），会导致信息丢失。论文提到的分层扩展（延迟量化/卸载）未实现。
- **跨区域信息流问题**：智能体可能将信息复制到不同区域（如工具输出被引用到草稿板），两个副本都衰减快，信息可能消失。未处理此问题。
- **标签质量依赖**：假设编排层提供准确区域标签，但未测试标签噪声的鲁棒性。对于不配合的编排器，需额外分类器，增加了风险。
- **工作负载依赖性**：区域生命周期是工作负载相关的，不同设置下测量结果可能不同（如其他论文报告了相反的顺序）。MemDecay要求每个部署有代表性的校准数据。
- **注意力项归一化问题**：消融实验确认当前注意力项量级过低无法保护旧token，但仅通过调整权重无法修复；需要幅度归一化或更慢的更新记忆。该列改进方向未在本文中实现验证。
- **未评估端到端性能**：未报告延迟、吞吐量、命中率等在服务层面成本，仅报告了召回率。注意力采样开销可能在实际部署中不可接受。
- **统计局限性**：虽然使用了场景级聚类，但仅8个场景，置信区间近似；位置混淆存在（system, plan, scratchpad固定位置）。

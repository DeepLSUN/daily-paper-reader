## 摘要
大型语言模型（LLM）正迅速转向通过多样化接口（包括网页和图形用户界面）解决问题的智能体。其中，终端命令行提供了一种基于文本的通用接口，覆盖从系统操作到数据科学和机器学习等任务。然而，扩展终端智能体训练仍然具有挑战性，因为它需要多样化且连贯的任务指令、可执行环境以及可靠的验证，同时缺乏自然存在的监督数据。在这项工作中，我们提出了SETA，一个可扩展的框架，用于生成可验证的终端环境以进行强化学习（RL）。该框架由两个共享统一验证机制的流水线组成：SETA-Synth将多样化来源转换为标准化的RL环境，SETA-Evol则通过自适应控制难度和多样性从已有环境进一步扩展。我们共同构建并发布了SETA-Env，这是迄今为止最大的开源可验证终端RL数据集，包含超过4500个环境。我们通过在SETA-Env上使用GRPO训练Qwen3-8B来评估数据集，在Terminal-Bench 2.0上达到了12%的通过率，这是8B规模RL训练模型的最佳报告结果。我们进一步观察到，在同一终端智能体框架下，DeepSeek-V4-Flash的性能提升，Terminal-Bench 2.0上的pass@1从40%提高到43%，pass@5从54%提高到58%。这些结果表明SETA-Env为终端智能体提供了高质量的训练环境，并成为推进基于终端的智能体学习研究的宝贵资源。

## Abstract
Large language models (LLMs) are rapidly shifting toward agents that solve tasks through diverse interfaces, including web and graphical user interfaces (GUIs). Among these, the terminal command line provides a text-based, general-purpose interface, covering tasks from system operations to data science and machine learning. However, scaling terminal-agent training remains challenging, as it requires diverse and coherent task instructions, executable environments, and reliable verification, while lacking naturally grounded supervision data. In this work, we propose SETA, a scalable framework for generating verifiable terminal environments for reinforcement learning (RL). The framework consists of two pipelines sharing a unified verification mechanism: SETA-Synth converts diverse sources into standardized RL environments, and SETA-Evol further expands from existing environments with adaptive control of difficulty and diversity. Together, we construct and release SETA-Env, the largest open-source verifiable terminal RL dataset to date, containing over 4,500 environments. We evaluate our dataset by training Qwen3-8B with GRPO on SETA-Env, achieving 12% pass rate on Terminal-Bench 2.0, the best reported result for an RL-trained model at the 8B scale. We further observe gains on DeepSeek-V4-Flash under the same terminal agent harness, with pass@1 on Terminal-Bench 2.0 improving from 40% to 43% and pass@5 improving from 54% to 58%. These results demonstrate that SETA- Env provides high-quality training environments for terminal agents and serves as a valuable resource for advancing research on terminal-based agent learning.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心框架：SETA（Scaling Environments for Terminal Agents）

SETA 由两个串联的流水线构成，共享同一套验证机制：

- **SETA-Synth（合成流水线）**：将多种人工验证来源（如 Stack Overflow、Unix/Linux StackExchange、Kaggle 笔记本、NL2Bash 数据集）转换为标准化的可执行终端环境。
  - **源适配（Source Adapter）**：针对每种来源设计专用提示，将其问题-解对转化为统一的草案规范。
  - **Idea Agent**：将原始问题扩展为包含环境假设、约束、失败模式和测试细节的 draft specification。
  - **Datapoint Agent**：将草案实例化为 Docker 容器环境、解决方案脚本（`solve.sh`）、测试套件（`test_state.py` 和 `test.sh`）、任务说明（`instruction.md`）和配置文件（`task.toml`）。
  - **验证机制**：每个任务需通过“无操作测试”（按原样运行测试且零通过）和“预言测试”（执行标准答案后全部通过）。此外，增加 **Post-Rollout 验证**：对 100% 失败的滚动部署（rollout）任务由独立的 Trajectory Judge Agent 判别是“真正困难”还是“设计缺陷”，后者被丢弃（约占 2%）。

- **SETA-Evol（演化流水线）**：基于 SETA-Synth 生成的现有环境，根据训练模型的实际表现自适应调整难度与多样性，使任务分布向“RL 有效训练区域”移动。
  - **自适应运算符选择**：根据当前环境 pass rate $r$ 选择策略：
    - 如果 $r > 0.5$（简单）：应用**难度增加**（增加约束、边界情况或多步依赖）。
    - 如果 $r \in (0, 0.5]$（中等）：应用**上下文切换**（变换领域或工具集，保持难度相近）。
    - 如果 $r = 0$ 但部分失败（困难）：应用**难度降低**（简化步骤、去除边角情况）。
  - 每个演化后的环境重新通过完整验证（Docker 构建检查、无操作测试、预言测试）并重新评估偏移方向，确保真正向 productive zone 移动。

### 2.2 RL 训练算法：GRPO (Group Relative Policy Optimization)

采用与 DAPO 相似的动态采样，过滤掉奖励均匀的样本。损失函数如下：

$$
J(\theta) = \mathbb{E}_{(h,s,a,R)\sim \mathcal{D}} \left[ \min\left( \frac{\pi_\theta(a|h,s)}{\pi_{\text{old}}(a|h,s)} A, \text{clip}\left( \frac{\pi_\theta(a|h,s)}{\pi_{\text{old}}(a|h,s)}, 1-\epsilon_{\text{low}}, 1+\epsilon_{\text{high}} \right) A \right) - \beta D_{\text{KL}}\left( \pi_\theta \parallel \pi_{\text{ref}} \right) \right]
$$

其中 $h$ 为交互历史，$a$ 为动作，$A$ 为组内相对优势。

- **奖励设计**：采用**部分进度奖励**（partial-progress reward），即测试通过的比例：
  $$
  r = \frac{\text{# tests passed}}{\text{# total tests}} + 0.2 \cdot \mathbb{1}\left( \frac{\text{# tests passed}}{\text{# total tests}} = 1 \right)
  $$
  这使有效奖励方差任务的占比从 49% 提升至 86%。

## 6. 主要结论与发现

1. **最大开源可验证终端 RL 数据集**：SETA-Env 包含 4,567 个环境，涵盖 14 个类别，难度分布广泛。
2. **8B 尺度 RL 最佳**：Qwen3-8B 在 SETA-Env 上训练后，Terminal-Bench 2.0 pass@1 达到 **12%**（base 仅 3.6%），提升 3.3 倍，超过 Endless Terminals-8B（6.7%）和 OpenThoughts-Agent-v1-RL（4.9%），接近 Nemotron-Terminal-8B（13.0%，但使用 254k+ SFT 轨迹）。
3. **跨模型族迁移**：DeepSeek-V4-Flash 在同框架下训练后，Terminal-Bench 2.0 上 pass@1 从 40% 提升到 43%，pass@5 从 54% 提升到 58%。
4. **泛化到相关代码任务**：在 CompileBench 上从 6.7% 提升至 40%，QuixBugs 从 7.5% 提升至 15%，CRUST-Bench 从 15% 提升至 24%，甚至超越 Qwen3-30B-A3B 的 base 模型。
5. **RL 训练促使推理行为出现**：训练后的模型会产出结构化推理（计划再执行），而 base 模型倾向于立即执行并盲目报错。案例研究（nginx）显示，RL 模型能连续诊断 Docker 环境、配置语法、系统约束，而 thinking-base 虽然产出大量 tokens 但改进有限。

## 7. 优点

- **源自适应合成**：SETA-Synth 通过 source-agnostic 草案格式和专用适配器，可轻松扩展到新来源（只需添加适配器提示），具备良好的可扩展性。
- **自适应难度演化**：SETA-Evol 根据训练模型的实际 pass rate 动态选择演化策略，避免数据分布偏斜，使任务保持在“可学”的难度范围内，比传统 Evol-Instruct 的一味加难更合理。
- **严格验证链**：除了标准的 no-op 和 oracle 测试，增加 Post-Rollout 验证（Trajectory Judge）来剔除因测试与指令隐含不匹配导致的假阳性失效（约占 2%），提高了数据质量。
- **部分进度奖励**：使用测试通过比例作为奖励信号，显著提高有效样本比例（49%→86%），缓解稀疏奖励问题。
- **成本效益**：仅用 560 个环境和少量训练步数即可获得显著提升（与使用 254k+ 轨迹的 SFT 模型相差无几），表明数据质量较高。
- **开源贡献**：数据集（4500+ 环境）和模型权重已公开，便于复现和后续研究。

## 8. 不足与局限

- **模型和规模覆盖有限**：尽管在 Qwen3-8B 和 DeepSeek-V4-Flash 上验证了迁移性，但实验未涵盖更多不同架构或更大/更小的模型（如 1B、70B），也未进行更长时间的 RL 训练以观察饱和点。
- **环境限于终端**：框架当前仅支持终端交互，未扩展到图形界面或多模态环境。作者也承认将框架扩展到 GUI 是重要的未来方向。
- **潜在的数据偏差**：SETA-Synth 主要使用社区 Q&A、Kaggle 和 NL2Bash 等来源，可能偏向于流行的 Linux 工具和英语用户场景，对非英语或小众工具覆盖不足。
- **滥用风险**：更强的终端 agent 可能被用于未授权访问或自动攻击。论文通过使用公开、无敏感内容的来源和强调沙盒环境来部分缓解，但未提供更多防护机制。
- **消融实验不够深入**：例如，未单独分析 SETA-Synth 和 SETA-Evol 各自对性能的贡献，也未探索不同采样比例、不同演化策略比例的影响。Post-Rollout Judge 的准确率未提供。
- **泛化评估场景较小**：CRUST-Bench 仅 100 任务，CompileBench 仅 15 任务，QuixBugs 仅 80 任务，统计意义可能有限。

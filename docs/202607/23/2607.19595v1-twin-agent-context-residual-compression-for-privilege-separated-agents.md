## 摘要
大型语言模型（LLM）智能体容易受到安全风险的影响，例如来自不可信上下文的提示注入攻击，这些攻击操纵下游推理和工具使用。现有的安全设计方法通过将不可信观察与特权执行分离并仔细控制信息流来缓解这一风险，但通常会降低实用性，并且需要大量的任务特定工程。因此，我们提出双智能体（Twin Agent），这是一种受智能体上下文中残差编码启发的一般性权限分离设计模式。双智能体由两个几乎对称的智能体组成：一个检查不可信信息的探索智能体（Explore Agent）和一个执行特权行动的安全智能体（Safe Agent）。探索智能体以安全智能体的当前上下文为条件，仅向安全智能体传达关于下一步行动的精简提示。这种设计减少了保留任务实用性所需的信息，从而实现了更好的安全-实用权衡，我们通过测量提示长度变化时实用性和攻击成功率的变化来经验性地验证这一点。我们在SWE-bench Lite上的长周期软件工程任务以及AgentDojo和DecodingTrust-Agent上的异构多工具交互任务上评估了双智能体。在两个基准测试中，双智能体在保持高任务实用性的同时防止了提示注入攻击，优于未防御的智能体和权限分离基线。

## Abstract
Large language model (LLM) agents are vulnerable to security risks, such as prompt injection attacks from untrusted context that manipulate downstream reasoning and tool use. Existing secure-by-design approaches mitigate this risk by separating untrusted observations from privileged execution and careful control of information flow, but often degrade utility and require extensive task-specific engineering. We thus propose Twin Agent, a general privilege separation design pattern inspired by residual coding in the agent context. Twin Agent consists of two nearly symmetric agents: an Explore Agent that inspects untrusted information and a Safe Agent that executes privileged actions. The Explore Agent is conditioned on the Safe Agent's current context and communicates only compact hints to the Safe Agent about the next action to take. This design reduces the information needed to preserve task utility and thus achieves a better security--utility tradeoff, which we empirically verify by measuring how utility and attack success change as the length of hints varies. We evaluate Twin Agent on long-horizon software engineering tasks with SWE-bench Lite and on heterogeneous multi-tool interaction tasks with AgentDojo and DecodingTrust-Agent. Across both benchmarks, Twin Agent preserves high task utility while preventing prompt injection attacks, outperforming both undefended agents and privilege separation baselines.

- **研究动机**：大语言模型（LLM）智能体在开放环境中需处理不可信信息（如网页内容、文档、工具输出），同时执行特权动作（执行代码、访问文件等）。这导致它们极易受到**提示注入攻击**——攻击者通过不可信上下文注入恶意指令，操纵智能体的下游推理和工具调用。现有防御方案（如指令隔离、模型微调、检测器）只能部分缓解，且容易被绕过。更根本的**安全设计（secure-by-design）**方法通过权限分离（如 Dual LLM 模式、CaMeL）将不可信观察与特权执行分开，但往往导致严重的**效用下降**（如任务成功率骤降），且需要大量任务特定工程（如手动设计通信接口或固定协议），难以适应长周期、动态的交互任务。

- **整体含义**：作者将安全代理设计重新表述为**压缩信息流问题**：攻击者需要传输足够的“攻击信息”才能覆盖智能体的预期行为，而保留效用往往只需传输任务相关的**残差信息**。因此，通过限制跨信任边界的信息量，可以自然地实现更好的安全‑效用权衡。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：受**残差编码**启发，提出 **Twin Agent**（双智能体）模式——不是将完整不可信上下文压缩后传给特权智能体，而是由**探索智能体（Explore Agent）**基于安全智能体（Safe Agent）的当前上下文，仅传输**消失的、任务相关的残差信息**（即安全智能体缺失的、用于下一步决策的最小信息）。这样大部分信息已通过共享上下文隐式传递，大大降低了跨边界的信息量。
- **关键技术细节**：
  - **两个对称智能体**：
    - **Explore Agent**：可观察不可信区域（如用户报告、工具输出），但不能执行特权动作。它观察安全智能体的当前轨迹和不可信上下文，生成一个**紧凑提示**（hint），遵守给定的字符预算。
    - **Safe Agent**：可执行特权动作（如修改代码、调用 API），但不能直接观察不可信区域。它接收提示作为输入，但提示被视为不可信背景信息（需添加提醒和检测器防范提示本身被注入）。
  - **残差压缩公式**（非实际优化目标，而是设计原则）：
    - 目标：最小化安全动作 \( A_t^{(s)} = \pi_s(T, H_t^{(e)}, I_t^{(s)}) \) 与原始未防御动作 \( A_t^{(o)} = \pi(T, U, I_t^{(s)}) \) 之间的差异，同时满足 \( \text{Budget}(H_t^{(e)}) \leq b \)。
    - 实际实现：Explore 通过上下文学习生成提示，Safe 保持原有架构（即近似原策略 \(\pi\)）。
  - **安全性保障**：依赖两个假设：（1）对前沿模型成功执行注入攻击需要足够长的注入文本；（2）短提示上的注入易被检测。因此 Twin Agent 使用字符预算截断提示，并添加一个轻量级提示注入检测器，进一步过滤可疑提示。
- **应用方式**：作为 ReAct 模式的**替换组件**，可应用于 SWE-agent（软件工程）、tool-use agent（多工具交互）等，无需重设计工作流，仅需插入一层残差通信。

## 6. 论文的主要结论与发现

- Twin Agent 在所有基准上**保持高任务效用的同时显著降低了攻击成功率**，实现了比未防御智能体和强隔离基线（Dual、CaMeL）更好的安全‑效用权衡。
  - 在 SWE-bench-injected 上，Utility 62.5% vs 基线 61.2%，ASR 从 97% 降到 0%。
  - 在 AgentDojo 上，Utility 62.9% vs 默认 61.9%，ASR 从 41.5% 降到 0.1%；而 CaMeL 的 Utility 仅为 30.9%。
  - 在 DecodingTrust-Agent 上，Utility 93–95%，ASR 从 10–26.7% 降到 3.3%。
- 自适应攻击（PAIR 两阶段：优化提示绕过检测 + 复制粘贴注入 Explore）下，整体 ASR 为 4.7%，且成功执行攻击的实例在多次重跑后可能收敛到 0%，说明成功率具有随机性。
- **信息预算可以量化控制安全‑效用权衡**：预算增加，效用上升但 ASR 也上升；预算为零时效用严重下降（≈0%）。这支持了“安全代理通信本质是压缩”的观点。
- **轨迹保真度**：带提示的 Twin Agent 比无提示（仅 Safe 无残差）的语义偏差更低，提示有助于保持未防御智能体的推理轨迹。

## 7. 优点

- **通用性强**：作为 ReAct 的 drop‑in 替代，无需任务特定工作流设计，可应用于软件工程、多工具交互等异构场景。
- **设计新颖**：将权限分离重新表述为**残差压缩**问题，提供可量化的安全‑效用控制参数（字符预算），比传统“全隔离或全暴露”更灵活。
- **安全性高**：在非自适应攻击下 ASR 接近 0，在强自适应攻击下仅 4.7%，且成本仅增加约一倍。
- **部分解耦效用与安全**：效用主要由 Explore 提取能力决定，安全主要由 Safe 隔离和检测器决定，便于独立优化。
- **实验充分**：覆盖多种攻击类型、预算消融、组件消融、轨迹分析，并测试了最新前沿模型（GPT-5.2，Gemini-2.5/3.5 Flash）。

## 8. 不足与局限

- **缺乏正式安全保证**：作者承认虽然实证表现良好，但未提供理论或可证明的安全性，更强攻击（如绕过检测器的短注入、多轮协作攻击）可能突破防御。
- **实验范围有限**：仅评估了软件工程和工具交互两个领域，其他常见代理场景（如 web browsing、代码编写、数据分析）未测试；自适应攻击仅测试了单轮提示注入，多轮动态攻击未考察。
- **安全检测器可能被攻击**：附录 A 显示检测器本身可能被绕过（尤其当预算较大时），而当前检测器仅为一个通用提示，无专门训练。
- **成本增加**：由于需要两个智能体（Explore + Safe）各步推理，总 API 成本约为原基线的两倍，可能限制资源敏感场景的采用。
- **自适应攻击仍有失败**：4.7% 的 ASR 虽然很低但不容忽视，在关键安全场景中不能容忍任何非零风险。作者指出多次重测后可能降低到 0%，但实际部署中难以保证。
- **未探讨代际泛化**：实验模型均为当时前沿模型，但模型能力快速提升，未来更强模型可能更容易在短提示下被注入，从而削弱假设基础。

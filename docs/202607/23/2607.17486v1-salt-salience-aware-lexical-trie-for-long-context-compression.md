## 摘要
随着大型语言模型（LLM）处理越来越长的提示，计算和KV缓存内存成本已成为推理系统中的主要瓶颈。现有的输入级提示压缩方法能缓解这一问题，但仅通过标量相关性分数对每个句子进行排序，将文档视为无结构的单词和句子集合。在预算紧张的情况下，这会导致主题崩溃，即文档中的主导主题消耗预算，丢弃了不常出现但任务相关的主题。保持主题覆盖需要跨重复主题分配预算，而非孤立地对句子评分。为此，我们提出了SALT，一个与模型无关的抽取式框架，将每个句子的关键词组织成一个按句子频率（SF）排序的树，句子频率是文档主题结构的轻量级、可重用代理。这种基于树的结构平滑了内存分配，并防止了主导主题垄断预算。多锚点检索可激活由查询关键词标记的任何深度的树节点，并且该树在对话轮次中持久存在，支持多轮使用而无需重新编码文档。通过保留文档主题，SALT减少了长上下文提示的预填充计算和内存成本，同时保持与针对解码时延迟和内存的KV缓存方法的可组合性。

## Abstract
As large language models (LLMs) process increasingly longer prompts, computation and KV-cache memory costs have emerged as major bottlenecks in inference systems. Existing input-level prompt compression methods address this, but rank each sentence by a scalar relevance score, treating the document as an unstructured pool of words and sentences. Under tight budgets, this causes theme collapse, where the dominant theme(s) of a document consumes the budget, discarding less-frequent yet task-relevant themes. Preserving thematic coverage instead requires allocating the budget across recurring themes rather than scoring sentences in isolation. To this end, we propose SALT, a model-agnostic extractive framework that organizes per-sentence keywords into a trie ordered by sentence frequency (SF), a lightweight, reusable proxy for document thematic structure. This trie-based organization smooths memory allocation and prevents dominant themes from monopolizing the budget. Multi-anchor retrieval activates trie nodes labeled by query keywords at any depth, and the trie persists across dialogue turns, supporting multi-turn use without re-encoding the document. By preserving document themes, SALT reduces the prefill computation and memory cost of long-context prompts while remaining composable with KV-cache methods that target decoding-time latency and memory.

# 论文结构化解构：SALT（Salience-Aware Lexical Trie for Long-Context Compression）

## 1. 核心问题与总体含义

- **研究动机**：大型语言模型（LLM）处理长上下文提示时，预填充阶段的计算成本和KV缓存内存消耗成为主要瓶颈。现有输入级压缩方法（如RECOMP、EXIT等）通常为每个句子分配一个标量相关性分数，然后按分数排序并截取前k个作为压缩结果。这种“一维排序”策略在预算紧张时会导致**主题崩溃**——文档中占主导的主题会占用大部分预算，而频率较低但对下游任务重要的主题（如多跳问答中的桥接句）则被丢弃。
- **整体含义**：SALT将压缩问题重新定义为**在固定预算下保留文档的词汇-主题覆盖**，而非简单地对候选单元进行全局排序。它通过构建一个基于句子频率（Sentence Frequency, SF）的词汇树（trie），在句子选择之前先将预算分配到各个主题分支上，从而避免主题崩溃。

## 2. 方法论：核心思想与技术细节

- **核心思想**：利用轻量级编码器（BGE-small-en-v1.5）从每个句子中提取关键词，计算每个关键词的句子频率（SF，即该关键词出现在多少个句子的关键词集中），将高频关键词（位于p=0.9分位数以上）作为显著性锚点，按SF降序组织成前缀树（trie）。压缩过程分为**索引阶段**和**选择阶段**，选择阶段又分为**摘要模式**（无查询）和**查询模式**（有查询）。
- **关键技术细节**：
  - **关键词提取**：使用[CLS]注意力机制对每个句子内的内容词排序，通过“膝点检测”（Kneedle）确定保留多少个词（上限为40%），以确保重建余弦相似度。
  - **句子频率计算**：SF(w) = |{i : w ∈ Ki}|，衡量关键词在文档中作为锚点的出现次数。
  - **Trie构建**：每个句子的关键词集Ki ∩ S（S为显著性集合）按SF降序插入为一条根到叶子的路径，叶子存储句子ID。共享高频关键词的句子共享前缀。
  - **预算分配**：在摘要模式下，先给每个深度为1的分支分配一个固定下限加上按剩余未覆盖质量(Mb+ϵ)α（0<α<1）的比例份额，然后每个分支内按边际增益（减少未覆盖质量）选择句子。在查询模式下，额外激活与查询关键词匹配的trie节点，并分配一部分预算(βqB)用于锚点阶段，剩余预算按分支分配。
  - **多锚点检索**：查询关键词可激活任意深度的节点，支持恢复查询相关关键词出现在次要锚点位置的句子。
  - **可重用性**：trie在对话轮次中持久化，多轮对话无需重新编码文档。

- **公式与算法**（文字说明）：
  - 未覆盖质量Uv(R) = Σ_{w∈Γ(v)\Cv(R)} cSF(w)，其中cSF(w)为归一化句子频率权重。
  - 句子评分：score_b(si|R) = Δb(si|R) / L(ni) * I(si)，Δb为句子加入后分支未覆盖质量的减少量，L(ni)偏向中等长度句子，I(si)包含位置先验和查询匹配。
  - 分支分配公式：βb = floor + residual * (Mb+ϵ)α / Σ(Mb'+ϵ)α。

## 6. 主要结论与发现

- **准确性**：SALT在LongBench上以20%预算平均得分43.51，**优于所有其他预处理方法**（RECOMP 40.22、CPC 38.05、Sentinel 39.47、EXIT 31.68），且接近KV缓存方法中的SnapKV（49.04）和FastKV（49.23）——但后者消耗了全上下文预填充阶段。
- **效率**：SALT在端到端延迟和峰值内存上**始终低于或等于几乎所有预处理方法**，且在大长度下明显优于KV缓存方法（如256k时SALT内存32.8GB vs FastKV 47.67GB）。
- **多轮场景**：SALT通过一次性索引+轻量级每轮检索，在QuALITY上实现接近基线的准确率（72.6% vs 74.1%），且累积成本仅为基准的25%（0.91s vs 3.71s），而RECOMP因每轮重新编码成本更高且准确率更低（61.5%）。
- **NIAH**：SALT在所有长度上匹配未压缩基线的检索性能，且在超长上下文（超出模型窗口）时仍然可用。
- **硬件兼容性**：SALT可在V100等旧GPU上运行，无需自定义内核（如FlashAttention），而SnapKV等需要现代架构。

## 7. 优点

- **方法创新**：首次将主题覆盖作为压缩的显式分配目标，通过句子频率的词汇树实现，而非事后多样性修正。关键词选取轻量且可解释。
- **模型无关性与可组合性**：输出纯文本，可与任何下游LLM及KV缓存方法相结合。
- **多轮友好**：索引可跨对话轮次重用，大幅降低每轮压缩开销。
- **硬件灵活性**：不依赖特定GPU架构，兼容多代硬件。
- **实验设计全面**：从准确性、效率、多轮、消融到跨硬件测试，验证了方法的鲁棒性。
- **代码开源**：提升可复现性。

## 8. 不足与局限

- **代码类任务表现差**：句子级别粒度不适合代码的线和标记级语义；所有预处理方法在此类任务上均不如KV缓存方法。作者承认SALT在此类任务中固有局限。
- **句子级别粒度下限**：无法压缩到5%~20%以下，而KV缓存方法可做到更激进压缩（但需付出全预填充代价）。
- **关键词提取依赖预训练编码器**：BGE-small-en-v1.5的[CLS]注意力分布可能不适用于特定领域（如法律、医学），且未在非英语文档上评估。
- **未评估更大模型（70B+）**：实验仅使用8B模型，更大模型的压缩收益和兼容性未验证。
- **未进行语义等价性评估**：压缩后输出的文本是否保持原始文档的语义等价（faithfulness）未单独分析。
- **消融实验仅基于ROUGE-L**：在摘要任务上诊断，未在问答或检索任务上证明主题分配优势。
- **可能存在的基线偏差**：KV缓存方法在20%预算下的配置可能与原文最优参数不同（如DuoAttention原文使用50%），导致对比略有不利。

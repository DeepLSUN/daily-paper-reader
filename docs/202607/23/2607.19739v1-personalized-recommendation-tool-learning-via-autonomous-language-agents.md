## 摘要
尽管大语言模型（LLMs）因其强大的推理能力和广泛的世界知识最近在推荐系统中受到关注，但先前基于LLM的智能体存在幻觉和上下文长度限制问题，因此不适合全排序推荐任务。为了通过架构设计而非修改LLM本身来规避这些限制，我们提出了一种基于智能体的推荐框架——基于自主语言代理的**记忆型个性化推荐工具学习**（PRTA）。在该框架中，LLM作为中央规划器，与多个推荐模型作为工具进行交互。基于LLM的智能体负责高级推理和个性化工具选择，而传统推荐模型则利用其在行为模式建模方面的可扩展性执行全排序评分。为了支持个性化工具选择，我们设计了反思机制，使智能体能够基于用户档案和候选排名列表评估和比较每个用户的工具。在三个公共数据集上的大量实验表明，PRTA在提升全排序推荐性能方面优于传统推荐和基于LLM的基线方法。

## Abstract
Although large language models (LLMs) have recently gained traction in recommender systems due to their strong reasoning capabilities and extensive world knowledge, previous LLM-based agents suffer from hallucination and context-length limitations, and thus are not suitable for full-ranking recommendation tasks. To circumvent these limitations through architectural design rather than modifying the LLM itself, we propose an agent-based recommendation framework, memory-based $\textbf{P}$ersonalized $\textbf{R}$ecommendation $\textbf{T}$ool learning via autonomous language $\textbf{A}$gents (PRTA), in which an LLM acts as a central planner interacting with multiple recommendation models as tools. The LLM-based agent is responsible for high-level reasoning and personalized tool selection, while traditional recommendation models perform full-ranking scoring, leveraging their scalability in modeling behavioral patterns. To support personalized tool selection, we design reflection mechanisms that enable the agent to evaluate and compare tools for each user based on user profiles and candidate ranked lists. Extensive experiments across three public datasets demonstrate the superiority of \modelname over traditional recommendation and LLM-based baselines in improving full-ranking recommendation performance.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有基于大语言模型（LLM）的推荐代理受限于**幻觉（hallucination）**和**上下文长度限制（context-length limitations）**，无法胜任需要对所有物品进行排序的**全排序推荐任务**。此外，直接将LLM用于推荐会忽略协同信号，且微调成本高。
- **整体含义**：作者提出通过**架构设计**而非修改LLM本身来规避上述限制——将LLM作为中央规划器，与多个传统推荐模型（作为工具）协同工作，从而同时利用LLM的语义推理能力和传统模型的行为建模可扩展性。

## 2. 方法论

### 核心思想
- **解耦行为建模与语义推理**：传统推荐模型（LightGCN、SASRec、SimpleX）负责全排序评分，LLM代理负责个性化工具选择与轻量级重排序。
- **个性化工具记忆（Personalized Tool Memory, PT Memory）**：为每个用户维护一个权重向量，表示各工具的适用性。最终排名分数为各工具评分向量的加权和。
- **三个反思模块**：通过本地工具评估、全局工具比较、排名比较三种方式迭代优化PT Memory，实现个性化工具选择。
- **推理阶段上下文重排序**：对加权聚合后的top-k候选进行基于用户档案和文本描述的LLM重排序。

### 关键技术细节
- **用户档案生成**：使用LLM对用户历史交互物品的文本描述进行总结。
- **本地工具评估**：LLM判断每个工具给出的top-k候选列表与用户最近交互历史的相关性（三类标签：相关/中性/不相关），并按学习率α更新对应工具权重。
- **全局工具比较**：LLM在多个工具中选出最相关的一个，按学习率β更新权重。
- **排名比较**：基于最近交互物品在各工具中的排名，计算各工具的相对贡献分数并更新权重。
- **公式示例**：最终排名得分 $\hat{r} = m_G \hat{r}_G + m_S \hat{r}_S + m_M \hat{r}_M$，权重更新 $m_t \leftarrow m_t + \alpha \cdot y_t^{eval}$ 等。

### 数据集与场景

- **传统推荐模型**：ENMF、DiffRec、FEARec（来自RecBole框架）。

### 评估指标
- **Recall@10/20** 和 **NDCG@10/20**，全排序设置（对所有物品排序）。

## 6. 主要结论与发现

- **PRTA在所有数据集上一致最优**：尤其是Amazon（Recall@10提升60%）、Goodreads（Recall@10提升68.8%），Yelp也有显著提升（Recall@10提升30.75%）。
- **语言基线（BM25、LLMRank）表现差**：无法建模协同和序列信号。
- **RAG变体不稳定**：对单一工具进行LLM重排序可能引入偏置；PRTA通过多工具聚合提高鲁棒性。
- **消融分析**：上下文重排序在Amazon上贡献最大（文本语义一致）；本地工具评估在Yelp上最有效；全局工具比较提供稳定但较小的增益。
- **PT Memory可视化**：SASRec（序列模型）权重普遍较高，SimpleX（MF模型）权重较低，与用户历史行为一致性相符。

## 7. 优点

- **架构创新**：通过LLM+工具协同，避免直接让LLM做全排序，从根本上解决幻觉和上下文限制，无需微调LLM。
- **个性化可解释**：PT Memory直接反映每个用户对各推荐模型的偏好，具备可解释性。
- **轻量高效**：推理阶段每用户不到0.5秒，适合作为离线排序或第二阶段重排序组件。
- **实验全面**：涵盖多种传统和语言基线，消融分析细致，代码开源。
- **模块化设计**：可灵活替换或扩展推荐工具集。

## 8. 不足与局限

- **用户样本量有限**：仅采样160个用户用于LLM阶段，虽声明为成本控制，但可能引入采样偏差，缺乏全用户覆盖的实验。
- **LLM调用成本仍高**：每个用户需要多次LLM调用（档案生成+每轮训练4次+推理1次），且依赖量化模型部署。
- **工具集受限**：仅使用三种推荐模型（图、序列、MF），未尝试更先进的模型（如基于Transformer的序列模型变体或扩散模型）或更多工具组合。
- **文本依赖性强**：效果受物品描述质量和一致性影响（如Goodreads图书序列语义更清晰，Amazon次之，Yelp业务描述差异大导致重排序增益小）。
- **缺乏与同类LLM-agent方法的直接比较**：虽然作者指出已有方法无法做全排序，但仍可尝试在小规模全排序场景下对比，以更全面评估。
- **消融中全局工具比较提升较小**：可能与排名比较的功能重叠有关，说明该模块的边际贡献有待验证。

## 摘要
大型语言模型智能体通过生成交织规划、工具调用和中间结果的轨迹来完成任务。当前的评估指标要么将这样的轨迹简化为一个二元成功标志，要么通过精确匹配与参考轨迹进行比较。成功标志无法区分合理的解决方案与侥幸成功的方案，也无法说明失败运行的原因。精确匹配会惩罚那些有效但顺序或分解方式与参考轨迹不同的规划。我们将轨迹评估重新定义为智能体执行图与一组有效解图之间的距离，并通过在属性依赖图上求解不平衡融合Gromov-Wasserstein输运问题来实现。由此得到的分数称为\otap{}（智能体规划的最优传输），是一个伪度量，可证明对保持依赖关系的重排序具有不变性，且对冗余步骤具有有界敏感性。其不平衡边缘量处理缺失或幻觉步骤时无需强制匹配，其软耦合适应规划粒度的变化。在受控扰动和三个公开基准上，\otap{}能够区分有效和无效轨迹，而仅基于语义的指标在此场景下得分低于随机水平。当依赖图被精确恢复时，其准确率最高，仅在从自由文本轨迹启发式推断依赖图时下降。

## Abstract
Large language model agents solve tasks by generating trajectories that interleave planning, tool calls, and intermediate results. Current evaluation metrics reduce such a trajectory to a binary success flag or compare it against a reference by exact matching. A success flag cannot distinguish a sound solution from one that succeeds by luck, and says nothing about why a failed run went wrong. Exact matching penalizes plans that are valid but reordered or decomposed differently from the reference. We reframe trajectory evaluation as a distance between the agent's execution graph and a set of valid solution graphs, and instantiate it via an unbalanced fused Gromov-Wasserstein transport problem over attributed dependency graphs. The resulting score, termed \otap{} (Optimal Transport for Agentic Planning), is a pseudo-metric that is provably invariant to dependency-preserving reorderings and has bounded sensitivity to redundant steps. Its unbalanced marginals handle missing or hallucinated steps without forcing a match, and its soft coupling accommodates variation in plan granularity. On controlled perturbations and three public benchmarks, \otap{} separates valid from invalid trajectories in a regime where semantics-only metrics score below chance. Its accuracy is highest when the dependency graph is recovered exactly, and drops only when the graph is inferred heuristically from free-text traces.

# 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

当前大语言模型智能体通过生成交织规划、工具调用和中间结果的轨迹来完成任务。然而，现有评估指标存在严重缺陷：
- **二元成功标志**：无法区分“侥幸成功”与“合理方案”，对失败不提供诊断信息。
- **精确匹配**：惩罚那些有效但顺序或分解方式与参考不同的规划（如重排序、粒度变化等）。
- **LLM作为裁判**：计算成本高、非确定性、提示敏感，存在自偏好偏差。

因此，论文提出核心命题：**轨迹评估应转化为“智能体执行图”与“有效解图集合”之间的结构对齐问题**，而非与单一参考序列的标识匹配。目标是设计满足语义感知、因果严格性、粒度容忍、缺失/幻觉鲁棒、执行接地、多参考支持等六项要求的度量。

## 2. 方法论

### 核心思想
将智能体轨迹建模为**有向无环图（DAG）**，节点为执行步骤（包含动作描述、工具ID、参数、执行效果），边为数据/因果依赖。评估通过**不平衡融合Gromov-Wasserstein（UFGW）最优传输**，对齐两个图的节点（软分配）并同时考虑属性距离和结构距离。

### 关键技术细节
1. **节点成本矩阵**：将步骤的每个字段（动作、参数、效果、工具）独立嵌入，计算余弦距离，加权求和。
2. **结构矩阵**：构建归一化有符号依赖距离矩阵，反映因果方向（祖先正、后代负、无关零）。
3. **UFGW目标函数**：
   - 平衡局部属性对齐（节点成本）和全局结构对齐（结构矩阵差的平方）。
   - 使用KL散度松弛边际约束（允许质量不守恒，处理缺失/幻觉）。
   - 引入熵正则化加速优化。
   - 通过条件梯度法和广义Sinkhorn迭代求解。
4. **多参考聚合**：使用带温度控制的softmin函数聚合多个参考分数，逼近最小值（而非平均）。
5. **因果质量加权**：为参考图中关键节点（移除会破坏可达性）赋予更高边际质量，惩罚遗漏关键步骤。
6. **最终分数**：输出归一化相似度（0~1]，并提供精度/召回率（π, ρ）和最佳耦合矩阵用于调试。

### 算法流程（文字说明）
1. 解析智能体轨迹P，构建DAG和依赖距离矩阵。
2. 对每个参考R_k，同理构建，计算节点成本矩阵，求解UFGW得到距离L_k。
3. 通过softmin聚合所有L_k得到总体距离L，指数变换得到分数。
4. 返回分数、精度/召回率、子损失、最佳运输计划。

## 3. 实验设计

### 数据集与基准
- **手造受控任务**：20个任务，通过5种扰动（同义词改写P1、有效重排序P2、合并/拆分P3、因果反转P4、关键步骤删除+干扰件P5）构建轨迹，有已知质量标准（P1-P3为有效，P4无效，P5最差）。
- **公开基准**：
  - **PlanBench**：720个真实LLM规划，带有VAL形式的有效性标签，依赖图精确（来自PDDL语义）。
  - **WebArena**：812个任务配置，无黄金轨迹，作者通过规则引擎生成参考，依赖图显式。
  - **GAIA**：30个人类标注解决方案，依赖图通过启发式从自由文本中提取（有噪声）。

## 6. 主要结论与发现

1. **Otap能区分有效与无效轨迹**，在语义指标（嵌入方法）低于随机水平时仍有效。关键在于依赖结构而非文本表面。
2. **依赖图精确恢复时性能最高**（PlanBench、WebArena），启发式提取时（GAIA）优势缩小但仍是唯一超过50% AUROC的指标。
3. **理论性质验证**：对保持依赖的重排序不变，对冗余插入有界敏感，是伪度量。
4. **消融核心**：
   - 结构项（θ>0）是载荷组件，去除后性能崩溃。
   - 不平衡边际和多参考聚合至关重要。
   - 字段级嵌入和工具项各有贡献。
   - 有符号距离和因果质量加权贡献不大。
5. **反直觉结果**：在正式有效性预测上Otap最弱（AUROC 79.2% vs BLEU 86.7%），因为其故意容忍偏差（允许缺失/幻觉）在此任务中反成劣势。**当形式验证器可用时，应优先使用验证器**。

## 7. 优点

- **方法论创新**：将轨迹评估转化为图对齐问题，使用最优传输理论自然处理软匹配、不平衡质量、结构比对，满足多项实际需求。
- **理论严谨**：证明伪度量性质、置换不变性、有界敏感性。

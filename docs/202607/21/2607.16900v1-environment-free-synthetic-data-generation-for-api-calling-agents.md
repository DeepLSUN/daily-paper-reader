## 摘要
训练能够调用API的大语言模型（LLM）智能体需要大量高质量轨迹数据。然而，大规模收集此类数据通常需要配备可执行API和预填充真实后端数据库的完整实现环境，这成为可扩展性的主要瓶颈。为克服这一问题，我们提出了一种无环境合成数据生成方法，利用LLM作为即时数字世界模型。仅给定API规范，我们的方法可生成模拟智能体与有状态环境交互的轨迹。具体而言，LLM首先生成可使用所提供API解决的多样化任务。随后，一个教师智能体迭代求解每个任务，同时一个LLM模拟器根据任务上下文和模拟历史生成连贯的合成API响应。最后，一个LLM评判器对轨迹进行过滤，以确保所得数据集的質量。我们在具有挑战性的AppWorld和OfficeBench基准上评估了该方法，这些基准包含信息检索和状态改变两类任务。基于我们的合成数据微调模型获得了显著的性能提升，表明无需任何可执行环境即可生成用于API调用智能体的有效监督信号。我们的结果证明，基于LLM的API模拟是一种实用且可扩展的方案，可用于训练跨多种API生态系统的智能体。

## Abstract
Training API-calling large language model (LLM) agents demands massive amounts of high-quality trajectories. However, collecting such data at scale typically requires fully implemented environments with executable APIs and realistic, pre-populated backend databases, creating a major bottleneck for scalability. To overcome this, we propose an environment-free synthetic data generation approach that leverages LLMs as on-the-fly digital world models. Given only API specifications, our method generates trajectories mimicking interactions between an agent and a stateful environment. Specifically, an LLM first generates diverse tasks solvable with the provided APIs. A teacher agent then iteratively solves each task while an LLM simulator generates coherent synthetic API responses conditioned on the task context and simulation history. Finally, an LLM judge filters the trajectories to ensure the quality of the resulting dataset. We evaluate our approach on the challenging AppWorld and OfficeBench benchmarks, which include both information-retrieval and state-changing tasks. Fine-tuning models on our synthetic data yields significant performance gains, demonstrating that effective supervision for API-calling agents can be generated without any executable environment. Our results establish LLM-based API simulation as a practical, scalable solution for training agents across diverse API ecosystems.

### 1. 核心问题与整体含义（研究动机和背景）

-   **研究动机**：训练能够调用外部API（应用程序接口）的大语言模型智能体（LLM Agents）需要海量高质量的“轨迹数据”（trajectories），即包含用户指令、思考过程、API调用序列及返回结果的完整交互记录。然而，传统方法收集此类数据依赖一个巨大的瓶颈：必须预先构建好一个功能完备的**可执行环境**，包括可调用的真实API和后端数据库。构建和填充这些环境成本高昂、难以规模化，严重限制了数据生成的效率和广度。
-   **核心问题**：能否在不构建任何真实运行环境的前提下，仅凭API的接口规范（specifications），就能生成高质量、多步骤的智能体训练数据？
-   **整体含义**：论文提出了一种颠覆性的“无环境”数据合成范式，旨在将数据生成从复杂的工程问题简化为一个“轻量级的规范问题”（lightweight specification problem）。这极大地提升了数据生成的**可扩展性**和**普适性**，使得为任何API生态系统快速生成训练数据成为可能。

### 2. 方法论：核心思想与关键技术

-   **核心思想**：现代LLM在互联网海量文本上预训练，具备了丰富的“世界知识”。论文的核心猜想是，这些LLM可以充当**动态的数字世界模型**。给定API规范、任务描述和交互历史作为上下文，LLM能够合成出足以乱真的、反映底层状态变化的API执行结果。
-   **方法名称**：**ESAT**（Environment-Free Synthetic Agentic Trajectory Generation Pipeline，无环境合成智能体轨迹生成流水线）。
-   **关键技术流程（三个阶段）**：
    1.  **任务合成阶段（Task Synthesis）**：
        -   **桶式生成**：设计了一个包含难度、动作类型、任务焦点、应用数量、API数量等多维度的配置网格，以生成覆盖全面、分布均衡的任务。
        -   **逆频率采样**：为保障各API和应用的均匀使用，采用基于已用次数的逆概率采样，优先选择使用不足的API和应用，避免数据分布坍缩。
        -   **任务过滤与改写**：由一个LLM评判器过滤掉不合规的任务，并由另一个LLM将原始生成的、可能包含程序化步骤的任务改写为更自然、目标导向的用户请求。
    2.  **轨迹合成阶段（Trajectory Synthesis）**：
        -   **智能体-模拟器交互**：一个教师LLM智能体负责求解任务，依次发出API调用。另一个LLM作为**API模拟器**，根据任务上下文和所有历史交互（保持状态一致性），动态生成合理的API返回结果。整个过程无需真实环境运行。
        -   **模拟器质量保证**：模拟器遵循四项核心规则（数据一致性、多样性、真实性和模式合规性）。其输出会经过程序化模式验证，并由另一个评判器LLM进行语义核查，不通过则自动重试，确保高质量反馈。
    3.  **轨迹过滤阶段（Trajectory Filtering）**：
        -   由另一个LLM评判器对生成的完整轨迹进行最终评估，决定其是否正确、完整地解决了任务。仅保留多数投票通过的轨迹，从而筛选出高保真的训练集。

### 3. 实验设计

-   **使用的数据集/场景**：
    -   **AppWorld**：一个复杂的多应用基准测试，包含9个应用、457个API，任务平均需要10个API和45次调用。分为Test-Normal（168个任务）和Test-Challenge（417个任务，含2个从未见过的应用）。
    -   **OfficeBench**：聚焦办公自动化，包含8个应用、20个API，任务是长期跨应用的工作流。
-   **Benchmark**：论文主要在AppWorld的TGC（任务目标完成率）和SGC（场景目标完成率），以及OfficeBench的pass rate（通过率）上进行评估。
-   **对比方法**：
    -   **基础模型**：Qwen3（1.7B-14B）、Qwen3.5（2B-27B）的零样本（zero-shot）性能。
    -   **现有合成数据方法**：ToolAlpaca、ToolACE、Nemotron，以及最相关的Simia。
    -   **真实环境数据**：AppWorld官方提供的90个训练任务，在真实环境中由教师模型生成并经任务验证器筛选出的轨迹（称为AWT数据集）。
    -   **其他前沿模型**：GPT-4o、Nemotron-3-120B、GLM-5.1-FP8（也作为教师模型和模拟器）的零样本性能。

### 6. 主要结论与发现

1.  **ESAT数据极为有效**：基于ESAT合成数据进行微调，在所有模型和基准测试上均取得了**显著性能提升**（最高提升50.5%和60.5%）。这证明了无需真实环境即可生成高质量训练数据。
2.  **超越真实环境数据**：在AppWorld基准上，使用ESAT数据训练的模型，其性能普遍**优于使用官方真实环境数据（AWT）** 训练的模型。这证实了合成数据“青出于蓝而胜于蓝”的潜力。
3.  **强大的泛化能力**：使用与目标领域完全无关的合成API（ESAT-S52）数据训练的模型，依然能在AppWorld任务上取得良好效果，证明模型学到了**通用的API调用技能**。
4.  **知识蒸馏成功**：较小模型（如Qwen3.5-27B）在ESAT数据上微调后，性能可逼近或超过其教师模型（GLM-5.1-FP8）和更大的模型（GPT-4o），体现了方法的“知识蒸馏”效果。
5.  **评判器质量至关重要**：Gemini-3.1-Pro作为评判器效果远好于GLM-5.1-FP8，说明一个高质量的评判器是筛选出优质合成数据的关键。

### 7. 优点

-   **高度可扩展的核心创新**：打破了对可执行环境的依赖，将数据生成简化为规范输入问题，是本工作的最大亮点，具有里程碑式的意义。
-   **方法设计精巧且严谨**：从任务生成到轨迹模拟再到最终过滤，每个环节都设计了质量控制的策略（如逆频率采样、模式校验、语义核查、多轮过滤），确保了合成数据的高质量。
-   **实验验证极其充分**：在多种规模模型、多个复杂基准、多种数据类型上进行对比，并辅以详尽的消融分析和人类评估，实验论证的说服力很强。
-   **结果具有巨大实用价值**：证明了可以用极低的开销（仅需API规范）来“培养”一个强大的通用API调用智能体，这对于推动LLM Agent技术的落地应用有重要实践意义。

### 8. 不足与局限

-   **对LLM质量的强烈依赖**：整个生成过程（任务生成、轨迹模拟、轨迹过滤）都依赖于高性能的LLM（如GLM-5.1、Gemini-3.1-Pro）。如果这些底层LLM质量不佳或存在特定偏见，会直接影响最终生成数据的质量。这相当于将瓶颈从基础设施转移到了大模型本身。
-   **模拟器的一致性与错误累积问题**：尽管有质量保证机制，但模拟器生成的响应本质上仍是“梦幻”数据。在长序列轨迹中，状态漂移（state drift）和幻象ID（hallucinated identifiers）仍是固有风险。模拟器失败率随输出长度增加而上升也反映了这个问题。
-   **任务分布偏差**：论文自动生成的任务在涉及的应用数量上存在不均匀分布（单应用任务远多于多应用任务），这可能导致训练出的模型在多应用协同处理能力上有所欠缺。
-   **结论的普适性有待验证**：所有实验仅在AppWorld和OfficeBench这两个特定基准上完成。虽然它们很有挑战性，但其API规范和任务模式相对固定。该方法在其他更开放、更复杂、API规范更模糊的真实场景中的泛化效果还有待检验。
-   **计算成本**：虽然省去了构建环境的高昂成本，但ESAT方法本身需要大量调用高性能LLM进行数据生成和过滤。这个过程在经济和时间上的成本不低，对于小规模研究团队可能存在门槛。

## 摘要
开放的代理市场在同一个搜索界面中列出了原生代理、工具包和可复用技能包，但实践者对于如何跨这个混合目录进行检索仍缺乏指导。我们研究现成的、为通用文本检索训练的检索模型是否可以被调整以匹配用户查询到可执行的代理能力，以及学习到的信号是否能够迁移到用于调优的基准之外。我们在AgentSelect上微调了三个开源检索骨干模型——BGE-base、KaLM-v1.5和EasyRec——该基准将市场中可见的单元表示为从公共元数据派生的能力配置文件，并在训练期间未见过的两个目录上测试迁移：MuleRun原生代理和包含50个技能、1000个查询的ClawHub基准。调整在两个目录上都有帮助。代码和数据将在发表时发布。

## Abstract
Open agent marketplaces list native agents, tool bundles, and reusable skill packages in the same search interface, yet practitioners still have little guidance on how to retrieve across this mixed catalog. We study whether off-the-shelf retrieval models, trained for general text retrieval, can be adapted to match user queries to executable agent capabilities, and whether the learned signal transfers beyond the benchmark used for tuning. We fine-tune three open retrieval backbones, BGE-base, KaLM-v1.5, and EasyRec, on AgentSelect, which represents marketplace-visible units as capability profiles derived from public metadata, and test transfer on two catalogs not seen during training: MuleRun native agents and a ClawHub benchmark of 50 skills with 1,000 queries. Adaptation helps on both catalogs. Code and data will be released upon publication.

## 7. 优点
- **统一视角创新**：首次将代理检索、工具检索、技能检索纳入同一个元数据级检索框架，提出能力概要表示。
- **方法通用**：使用现成模型+LoRA，无需修改模型架构，易于复现和应用。
- **迁移验证扎实**：同时在两个不同性质的真实公开目录上验证，且包含人工生成查询（ClawHub）和真实用户查询（MuleRun），增强了泛化可信度。
- **实验设计细致**：对每个骨干都单独选择最优 epoch，避免过度调参；消融实验拆分了表示字段的影响。
- **开源贡献**：承诺发布代码、数据及 ClawHub 基准构建 pipeline，提供可复现的基础。

## 8. 不足与局限
- **只评估元数据级检索**：未进行端到端任务完成测试，无法确认检索质量是否能转化为实际执行效果。
- **忽略部署因素**：未考虑成本、延迟、安全等实际限制，这些因素在真实市场选择中同样关键。
- **模型选择泄露风险**：每个目标目录的最优 epoch 是在该目录的验证集上选择的，虽然未泄漏测试集标签，但仍暴露了对目标分布的微调，而更严格的协议只应在训练集上选择。
- **ClawHub 查询质量**：由 GPT-5.4 按预设模板生成，可能无法完全模拟真实用户模糊性，且难度标签并非人工验证，存在偏差。
- **未测试混合目录检索**：实验中 MuleRun 和 ClawHub 是分开测试的，未验证单一检索器能否同时处理代理和技能。
- **对比范围有限**：仅与自身的零样本版本对比，未与专门设计的代理检索系统（如基于 LLM 路由、工具检索微调模型）比较，无法证明本方法的绝对优势。
- **仅使用单一训练集**：迁移信号来源仅有一个（AgentSelect），可能限制了发现泛化边界的证据强度。

## 摘要
搜索智能体（SAs）通常利用大型语言模型（LLMs），通过自主探索网络资源并将信息综合为全面回答，来支持复杂的信息寻求任务。在SA评估方面，先前的基准主要关注那些不太可能在真实用户场景中出现的专业任务。此外，它们对粗糙任务级评分标准的依赖常常限制了评估的可解释性。为填补这一空白，我们引入了DailyReport，这是一个用于评估SA在日常搜索任务中能力的开放式基准。它包含150个开放式任务，附带3546个相关评分标准，捕捉了真实用户广泛讨论且及时的信息需求。每个任务被分解为子任务，并在解耦的维度上使用级联评分标准进行评估。通过级联性能归因和以用户为中心的聚合，我们为每个维度推导出高度可解释的分数，以及一个用户偏好分数。我们在17个智能体系统上的结果表明，当前系统仍未能达到用户期望。为促进未来研究，我们的数据集和代码已公开于https://github.com/AGI-Eval-Official/DailyReport。

## Abstract
Search Agents (SAs) typically leverage large language models (LLMs) to support complex information-seeking tasks by autonomously exploring web sources and synthesizing information into comprehensive responses. For SAs evaluation, prior benchmarks mainly focus on specialized tasks that are unlikely to arise in real-world user scenarios. Moreover, their reliance on coarse task-level rubrics often limits evaluation interpretability. To bridge this gap, we introduce DailyReport, an open-ended benchmark to evaluate SA capabilities on daily search tasks. It contains 150 open-ended tasks with 3,546 associated rubrics, capturing widely discussed and timely information demands of real-world users. Each task is decomposed into subtasks and evaluated with cascade rubrics across disentangled dimensions. Through cascade performance attribution and user-centric aggregation, we derive highly interpretable scores for each dimension, along with a user preference score. Our results on 17 agentic systems show that current systems still fall short of users' expectations. To facilitate future research, our dataset and code are made publicly available at https://github.com/AGI-Eval-Official/DailyReport.

- **评分标准生成**：

## 6. 论文的主要结论与发现
- **整体表现**：当前所有系统的用户偏好分数均低于可接受水平（3分），说明搜索代理远未满足用户期望。
- **维度能力不平衡**：
  - **指令遵循**得分高（均 > 0.9），模型能覆盖大部分明确要求。
  - **事实性**是最大瓶颈（多数 < 0.8），模型容易产生缺乏证据支持的幻觉。
  - **合理性**中等（0.7-0.93），但分析中心任务稍好。
- **系统对比**：带搜索工具的通用 LLM 表现最优（GPT-5.4 最佳），Claude Code 框架次之，原生 DRA 最弱。说明日常搜索任务受益于强通用模型与直接搜索的结合。
- **任务类型效应**：分析中心任务事实性更低（因搜索路径发散，证据难以三角验证），但指令遵循和合理性略高。
- **搜索行为与性能正相关**：搜索调用数量多的系统（如 GPT-5.4）整体表现更好。
- **引用质量不足**：引用率虽高，但引用准确率和引用-声明一致性仍有较大提升空间。
- **领域差异明显**：政治法律、工业经济等结构化领域表现好；体育娱乐等动态主观领域表现差。

## 7. 优点
- **真实场景驱动**：任务来源于真实用户趋势话题和评论，反映日常信息需求，比专业基准更接地气。
- **评估可解释性高**：通过子任务分解和三个正交维度的级联评分，可以精确定位模型在哪个子任务、哪个维度上失败。
- **用户中心量化**：首次显式引入子任务重要性到聚合中，输出用户偏好分数（1-4），贴近真实用户体验。
- **支持动态更新**：任务基于时间敏感话题，基准可持续更新，降低数据污染风险。
- **元评估完整**：包括鲁棒性、评判模型选择、用户偏好对齐验证，增强了基准的可靠性。
- **大规模人工标注**：超过 500 小时人工参与，保证了任务质量和评分标准的准确性。

## 8. 不足与局限
- **平台和区域偏差**：数据源主要来自中英文平台（Facebook、Weibo 等），可能无法完全代表全球用户的日常搜索需求。
- **评判模型依赖**：评估全部基于 Gemini-3-Flash 的自动判断，虽然经过验证，但仍存在黑盒偏差和领域偏好风险。
- **主观维度合理性**：合理性评判依然依赖 LLM 的上下文理解，与人类判断可能存在 gap。
- **未覆盖多模态任务**：所有任务均为纯文本报告生成，不支持图片、视频等信息的获取和分析。
- **计算成本较高**：每个任务需要多次搜索和网页抓取，加上 LLM 评估，总评估成本较高，可能限制广泛应用。
- **子任务重要性主观性**：重要性标注由少数专家完成，可能缺乏多样性，且未进行跨标注者一致性检验。
- **静态基线与时序性**：虽然支持动态更新，但论文实验基于固定时间窗口（2026 年 6 月左右），未来信息的变化可能导致任务过时。
- **未探讨推理速度/成本**：没有系统性地报告各模型在 DailyReport 上的推理时间和总成本，用户难以直接比较效率。

## 摘要
Arbor是一个多Agent框架，它引入了结构化树搜索作为自主Agent在大型、有状态动作空间中的认知层。先前的自主优化系统在孤立目标上进行无状态评估。而Arbor维护一个显式的带分数假设搜索树，该树作为Agent间的共享工作内存，随每次测量而演化，将失败视为重塑后续探索的诊断信号，并随着先前成功转移瓶颈分布而扩展。我们在全栈LLM推理优化上验证了Arbor，这一领域历史上需要应用程序、框架、编译器、内核和硬件栈的工程团队协同努力才能达到峰值性能。Arbor配对了一个编排器Agent（通过委托给推理栈中的领域专家驱动优化）和一个批评者Agent（通过根本原因分析、内省和测量验证来保障稳定性）——这是一种制衡架构，其中任何Agent都无法单方面驱动系统。Agent能力被分解为硬技能（领域专业知识）和软技能（决定贡献如何组合的协调协议），从而实现了完全自主的多日优化活动。Arbor在推理吞吐量-延迟帕累托改进上比供应商优化的基线提高了高达193%，而单个Agent没有这种框架则止步于33%的吞吐量提升并在数小时内不可恢复地崩溃。Arbor可推广到多代硬件平台，运行间方差在2个百分点以内，表明该方法与硬件无关且可复现。

## Abstract
Arbor is a multi-agent framework that introduces structured tree search as a cognition layer for autonomous agents operating in large, stateful action spaces. Prior autonomous optimization systems operate on isolated targets with stateless evaluation. Arbor instead maintains an explicit search tree of scored hypotheses that serves as the shared working memory across agents, evolving with every measurement, treating failures as diagnostic signal that reshapes subsequent exploration, and expanding as prior successes shift the bottleneck distribution.   We validate Arbor on full-stack LLM inference optimization, a domain where achieving peak performance has historically required coordinated effort from engineering teams across the application, framework, compiler, kernel, and hardware stack. Arbor pairs an Orchestrator agent, which drives optimization by delegating to Domain Specialists across the inference stack, with a Critic agent that safeguards stability through root-cause analysis, introspection, and measurement validation -- a checks-and-balances architecture where neither agent can unilaterally drive the system. Agent capabilities are decomposed into hard skills (domain expertise) and soft skills (coordination protocols that determine how contributions compose), enabling fully autonomous multi-day campaigns. Arbor achieves up to 193% inference throughput-latency Pareto improvement over vendor-optimized baselines, while a single agent without the harness plateaus at +33% throughput improvement and crashes irrecoverably within hours. Arbor generalizes to multiple generations of hardware platform, and run-to-run variance is within 2 percentage points demonstrating that the method is hardware-agnostic and reproducible.

- **核心问题**：现有自主优化系统（如 AlphaEvolve、AVO、KernelSkill）仅针对**孤立目标**进行无状态优化，无法处理跨层交互（应用、框架、编译器、内核、硬件）带来的级联失败和瓶颈转移。单个优化尝试可能遍历多个堆栈层，每一层在解决前一层后才暴露，且可能使另一层取得的进展失效。现有方法无法在处理这些复杂性的同时保持长时间稳定的自主运行。
- **研究动机**：随着系统日益复杂，核心挑战从候选方案生成转向**候选方案选择**：在大型、有状态的动作空间中导航，每次干预会重塑优化景观。失败需要诊断并作为约束反哺搜索；成功会转移瓶颈，需要重新分析并扩展动作空间。
- **整体含义**：提出 Arbor 框架，将跨层性能优化形式化为**动态扩展动作空间上的启发式树搜索**，通过多智能体制衡架构实现完全自主的多日优化活动，显著超越了单智能体及现有方法。

## 2. 方法论

### 核心思想
- 将优化建模为**有状态树搜索**：维护一个显式的搜索树，节点表示动作及其结果（保持/回滚/崩溃），路径表示累积干预序列。树作为智能体间的共享工作内存，随每次测量演化。
- 搜索过程：**深度优先循环**：① 分析系统状态，识别瓶颈及 GPU 时间占比；② 用启发式函数 \( h(a) \) 对候选动作评分，选最高分；③ 委托给相应的领域专家实现；④ 进行准确性和端到端基准测试门控；⑤ 根据结果更新树（保持的动作成为新基线，回滚记录诊断，崩溃触发批评者根本原因分析）；⑥ 重新评分剩余候选（因瓶颈分布可能已变）。循环直到动作得分低于阈值或用完预算。
- 自动回滚到最后验证状态。

### 关键技术细节
- **评分函数**：
  \[
  h(a) = \frac{g(a)}{c(a)} \cdot (1 - r_{\text{acc}}) \cdot (1 - r_{\text{crash}}) \cdot m_{\text{gap}} + C \sqrt{\frac{\ln N}{1 + n_{\text{cat}}}}
  \]
  - \( g(a) \)：预期收益（目标核的 GPU 时间占比 × 类别先验加速比）。
  - \( c(a) \)：预期墙钟成本（实现、重启服务器、端到端测试时间）。
  - \( r_{\text{acc}}, r_{\text{crash}} \)：该类别的经验失败率（从知识库初始化并随观察更新）。
  - \( m_{\text{gap}} \)：紧迫乘子（剩余差距越大值越大，1.0~2.0）。
  - UCB 项鼓励探索未充分采样的类别。
- **多智能体架构（制衡）**：
  - **编排器（Orchestrator）**：驱动搜索循环，分析负载，构建动作栈，动态构造领域专家，门控结果。**不自己实现优化**，而是决定“做什么、谁做、是否通过”。
  - **批评者（Critic）**：维护稳定性，通过四个子功能提供制衡：
    1. 护栏（Guardrails）：保证测量完整性，检测基准异常。
    2. 根本原因分析（RCA）：诊断崩溃和回归，指导带约束的重试。
    3. 内省（Introspection）：评估回滚动作的潜在价值，产生细化子动作。
    4. 稳定性监控（Stability Monitoring）：长时间尺度跟踪系统健康，发布约束防止所有智能体重访无益区域。
  - **领域专家（Domain Specialists）**：**运行时动态构造**，由编排器为每个优化目标组装提示（包含任务、硬件上下文、知识库条目、历史尝试）。专家按层（内核、框架、通信、编译器、调度）分派，并行工作，遵循本地验证协议后返回就绪补丁。
- **硬技能与软技能分解**：编排器拥有分析、评分、评估门控技能（软技能：委派、整合批评者发现）；领域专家拥有领域硬技能（软技能：尊重批评者约束，提交已验证补丁）；批评者拥有 RCA、内省、护栏、稳定性监控硬技能（软技能：发布约束、确保测量完整性）。
- **知识库**：持久化知识库跨会话累积经验，用于初始化评分先验（\( r_{\text{acc}}, r_{\text{crash}}, g(a) \)）并直接构造领域专家提示。

## 6. 主要结论与发现

- Arbor 在标准操作点（并发 64）实现吞吐量提升：+48%（gpt-oss-120b）、+90%（DeepSeek-R1-0528）、+50%（MiniMax-M2.5）、+193%（GLM-5-FP8）、+40%（Qwen3.5-397B）、+60%（Kimi-K2.5）。优化后曲线支配基线帕累托前沿（所有交互性水平吞吐量均提高）。
- 消融实验证明：
  - **无树搜索的单智能体**：仅 +33% 后在 4 小时内因无法恢复的崩溃终止。
  - **无领域专家**：6 小时后动作空间耗尽（+30%）。
  - **无批评者**：两次运行分别仅 +12.9% 和 +16.5% 的有效改进，且存在测量完整性违规（一次准确性降为 0%，另一次并发参数被篡改报告虚高吞吐量）。批评者的核心贡献是**维护测量完整性**。
- 跨代验证：在 MI300X 上分别获得 +62%、+79%、+99%，泛化有效。
- 运行间方差：Kimi-K2.5 两次运行相差 0.6 个百分点，gpt-oss-120b 相差 1.6 个百分点，均在 2 个百分点以内，证明可重复性。
- 案例分析展示：制衡架构允许自愿牺牲短期吞吐量（如禁用不稳定的 GEMM 路径）以换取长期稳定性，最终达到 +90%。端到端门控至关重要：39% 的内核级改进在集成后反而导致吞吐量下降。
- **核心发现**：持久的自主优化不仅需要搜索结构（树）和智能体专业化，还需要明确的优化-稳定性张力（编排器 vs 批评者），以及硬/软技能分解决定贡献如何组合。

## 7. 优点

- **方法创新性**：首次将跨层优化形式化为动态树搜索，引入“认知层”概念；制衡多智能体架构（编排器-批评者）确保了长期稳定性和测量完整性。
- **硬/软技能分解**：清晰定义了“做什么”和“如何协作”，使智能体能够有效委派、整合反馈，并识别各自能力边界。
- **动态领域专家构造**：运行时根据任务从知识库组装专家，避免了预定义专家数量有限的问题，且便于迁移。
- **持久知识库**：使搜索在会话间累积，加快新硬件/模型上线速度。
- **实验全面**：6 个生产模型 + 消融 + 跨代 + 方差分析，结果多样且公平，强基线和可复现性增强可信度。
- **实际效果**：在供应商已深度优化的系统上仍获得巨大提升（最高 193%），且最终性能稳定、硬件无关，具有实际应用价值。
- **案例分析丰富**：具体展示了多智能体协作解决复杂故障（如 ZMQ 死锁、NFS 满、参数不匹配等）的过程，验证了方法有效性。

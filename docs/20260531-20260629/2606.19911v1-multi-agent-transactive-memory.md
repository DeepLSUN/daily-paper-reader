## 摘要
跨多样化任务部署具有不同能力的LLM智能体的去中心化，激发了支持异构智能体群体间知识共享的基础设施需求。正如搜索引擎索引人类生成的制品以支持人类解决问题一样，检索系统可以组织智能体生成的制品，以便在智能体群体中重复使用。我们将检索增强生成——这一展示了人类撰写的制品对单个智能体的价值——扩展到检索智能体生成的制品，以支持整个智能体群体。特别是，智能体轨迹编码了可重用的程序性知识，然而这些制品通常在一次使用后被丢弃，或仅由产生该制品的智能体保留，迫使新实例化的智能体反复重新发现已有的解决方案。我们提出了多智能体交互记忆（MATM），这是一个用于智能体生成轨迹的群体级存储与检索框架，其中生产者智能体将轨迹贡献到共享仓库，消费者智能体检索这些轨迹以改进任务执行。我们聚焦于交互式环境（ALFWorld和WebArena），在这些环境中轨迹较长且编码了尤其丰富的程序性结构。我们的实验表明，从MATM中检索轨迹能提升下游任务性能，并在无需协调或联合训练的情况下减少交互步骤。这些结果将MATM定位为开放智能体生态系统中群体级经验共享的一种设计模式。

## Abstract
The decentralized deployment of LLM agents with diverse capabilities across diverse tasks motivates infrastructure for knowledge sharing across heterogeneous agent populations. Just as search engines index human-generated artifacts to support human problem solving, retrieval systems can organize agent-generated artifacts for reuse across agent populations. We extend retrieval-augmented generation - which demonstrates the value of human-authored artifacts to individual agents - to retrieval of agent-generated artifacts supporting a population of agents. In particular, agent trajectories encode reusable procedural knowledge, yet these artifacts are typically discarded after a single use or retained only by the producing agent, forcing newly instantiated agents to repeatedly rediscover existing solutions. We propose Multi-Agent Transactive Memory (MATM), a framework for population-level storage and retrieval of agent-generated trajectories, where producer agents contribute trajectories to a shared repository and consumer agents retrieve them to improve task execution. We focus on interactive environments (ALFWorld and WebArena), where trajectories are long and encode especially rich procedural structure. Our experiments demonstrate that retrieving trajectories from MATM improves downstream task performance and reduces interaction steps without coordination or joint training. These results position MATM as a design pattern for population-level experience sharing in open agent ecosystems.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在大规模异构LLM智能体群体中，单个智能体执行任务时产生的轨迹（trajectory）经验通常被丢弃或仅由产生者保留，导致新实例化的智能体反复发现已有的解决方案，资源浪费严重。
- **背景**：现有RAG主要面向人类撰写的文档；轨迹这类智能化生成的制品含有丰富的程序性知识，却缺乏跨智能体的共享机制。现有的推理复用、知识蒸馏等方法局限于相同智能体或需要额外训练，不适应动态异构的开放生态系统。
- **整体意义**：提出多智能体交互记忆（MATM）框架，让智能体将轨迹贡献到共享仓库，其他智能体可检索使用，形成群体级的知识基础设施，提升效率与有效性，无需协调或联合训练。

## 2. 论文提出的方法论

### 核心思想
- **交互记忆（Transactive Memory）**：源自在人类群体中，知识分布在不同个体间，通过共享机制定位和检索相关信息。MATM 类比：生产者智能体贡献轨迹，消费者智能体检索相关轨迹辅助当前任务执行。
- 角色不互斥：同一智能体可同时是生产者与消费者。

### 关键技术细节
1. **状态条件键值索引**：
   - 对于每条轨迹，窗口 \( l = 5 \) 步动作-观测对。
   - 查询键（key）由任务描述 \( x \) 与最近 \( l \) 步历史生成，值（value）为后续 \( l \) 步的轨迹片段。
   - 使用共享嵌入函数 \( f \)（E5-Base）编码。

2. **检索流程**：
   - 消费者智能体配备 “RetrievalPlanner”，决定何时调用检索（只在需要时检索，避免上下文稀释）。
   - 初始候选：通过密集检索获取 top-20 片段。
   - 可选的 **学习排序轨迹（LTRT）** 阶段：对候选进行重排序，选出 top-1 供智能体使用。

3. **学习排序（Learning to Rank, LTR）**：
   - 特征维度：44 维，分为六类——生产者元数据、消费者元数据、第一阶段检索分数、查询特征、轨迹特征、查询-轨迹交互特征。
   - 标签生成：在增量构建阶段，对每个分支点（branching point）检索多个排名位置的轨迹，并计算其相对于无检索基线的边际效用（成功/步数差异）作为标签。
   - 训练三种重排序模型：点级 FFN、成对 LambdaMART、成对 SVMRank。

### 增量构建算法（简略说明）
1. 预热索引 \( D_0 \)：从公开轨迹构建。
2. 将训练问题分为多批，每批分配生产者智能体。
3. 每个问题，智能体先无检索执行得基线轨迹和分数；然后随机选取几个分支点，检索 top-K，对选中的排名位置进行单次注入式推理。
4. 成功轨迹加入索引，同时记录每个 (查询, 文档) 对的边际效用标签。
5. 重复多批，最终索引包含大量轨迹（ALFWorld：86,833 块；WebArena：20,102 块）。

## 3. 实验设计

- **场景与数据集**：
  - **ALFWorld**：文本化家庭任务环境。训练集 3553 个样本，测试集 274 个样本。6 种任务类型（Pick & Place, Examine in Light 等）。
  - **WebArena**：基于真实网站的快照，5 个类别（电商、论坛、地图等），共 812 个任务。自定义训练/测试分割，训练集 724，测试集 88。

## 6. 论文的主要结论与发现

1. **MATM 提升有效性与效率**：单阶段检索对比无检索，成功率 ALFWorld +8%p，WebArena +2%p；平均步数减少。
2. **学习排序进一步改善**：SVMRank 在 ALFWorld 成功率 +17.2%p（比无检索），平均步数降至 10.35；FFN 在 WebArena 效果最佳。
3. **优势分布广泛**：无论生产者比消费者强或弱，检索均有正向收益，并非单纯“强者输出弱者”。
4. **跨任务泛化**：即使检索限制为不同任务类型的轨迹，ALFWorld 成功率仍达 59.9%（远高于无检索 47.1%），说明轨迹中包含可迁移模式。
5. **随规模扩展**：随着索引增大，有效性和效率整体提升（WebArena 成功率在 50% 规模有暂时下降，但 100% 达到最高）。

## 7. 优点

- **技术贡献**：提出状态条件的键值索引、学习排序优化（LTRT）、以及基于边际效用的标签生成方法。

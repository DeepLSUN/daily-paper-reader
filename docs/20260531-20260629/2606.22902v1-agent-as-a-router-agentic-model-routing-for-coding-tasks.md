## 摘要
实际用户通常可以使用来自不同提供商的多个大型语言模型（LLM），这些LLM往往在不同领域表现出色，但没有一个在所有领域占据主导地位。因此，将每个任务路由到最合适的模型对于性能和成本都至关重要。现有路由器将此视为一个静态、一次性分类问题。然而，我们识别出这些路由器的性能瓶颈在于信息不足：仅通过在任务维度级别向普通LLM路由器添加性能统计数据，就能获得15.3%的相对增益，超过了基于相同维度级别先验的启发式路由器。受此发现启发，我们提出Agent-as-a-Router框架，该框架将路由形式化为C-A-F循环（上下文->行动->反馈->上下文）。它通过在部署期间积累基于执行的经验来弥补信息差距。我们将此框架实例化为ACRouter，由编排器、验证器、记忆模块组成，并引入了CodeRouterBench，这是一个评估环境，包含约10K个任务实例，具有来自8个前沿LLM的验证分数，支持在流式任务上进行基于遗憾的路由器比较。实验表明，ACRouter在分布内任务上实现了最低的累积遗憾，并泛化到分布外的智能体编程任务，证明我们的路由框架主动弥合了信息差距。代码和基准测试已在https://github.com/LanceZPF/agent-as-a-router发布。

## Abstract
Real-world users typically have access to multiple Large Language Models (LLMs) from different providers, and these LLMs often excel at distinct domains, yet none dominate all. Consequently, routing each task to the most suitable model becomes critical for both performance and cost. Existing routers treat this as a static, one-off classification problem. However, we identify the performance bottleneck for these routers as information deficit: simply augmenting a vanilla LLM router with performance statistics at the task-dimension level yields a 15.3% relative gain, surpassing a heuristic router built on the same dimension-level priors. Motivated by this finding, we propose Agent-as-a-Router, a framework that formalizes routing as a C-A-F loop (Context->Action->Feedback->Context). It closes the information gap by accumulating execution-grounded experience during deployment. We instantiate this framework as ACRouter, composed of an Orchestrator, a Verifier, a Memory module, and introduce CodeRouterBench, an evaluation environment comprising ~10K task instances with verified scores from 8 frontier LLMs, enabling regret-based router comparison on streaming tasks. Experiments show that ACRouter achieves the lowest cumulative regret on in-distribution tasks and generalizes to out-of-distribution agentic-programming tasks, demonstrating that our routing framework actively closes the information gap. Codes and benchmarks are released at https://github.com/LanceZPF/agent-as-a-router.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：现实用户通常能够访问来自多个提供商的各种大语言模型（LLM），这些模型在不同的编码领域各有优势，但没有任何一个模型在所有任务上占据主导地位。因此，将每个任务路由到最合适的模型对于同时提升性能和降低成本至关重要。
- **现有路由的瓶颈**：已有的路由方法将其视为一个静态、一次性的分类问题（例如使用一个单独的分类器或LLM一次性决策）。本文通过初步实验发现，这种路由器的性能瓶颈并非推理能力不足，而是“**信息赤字**”——仅仅在零样本LLM路由器中输入任务维度的性能统计信息（即+Perf stats），就能获得**15.3%的相对增益**（从41.41%提升到47.74%），甚至超过了基于同一维度级先验的启发式路由器DimensionBest（47.50%）。
- **核心挑战**：现有静态路由器的信息状态是冻结的，无法在部署过程中获取和积累执行反馈。因此需要一种能够自我适应的路由框架，在连续任务流中不断学习。

## 2. 方法论：核心思想、关键技术细节与形式化表述

### 2.1 核心思想：C-A-F循环（Context-Action-Feedback）

- 将路由形式化为一个**上下文感知的多臂老虎机（Contextual Multi-Armed Bandit）**，以**累积遗憾**作为流式指标。
- 循环定义：在每个任务 $t_i$，路由器观察**上下文** $c_i$（包含任务提示、元数据、历史记忆$H_{<i}$），选择一个**行动** $a_i$（从模型池$M$中选择一个模型），执行模型并获得**反馈** $f_i$（包括验证器给出的性能分数 $\hat{s}_i$ 和货币成本 $\hat{\kappa}_i$），然后将反馈存入记忆，更新下一个任务的上下文。
- 环路：$c_i \xrightarrow{\text{Decide}} a_i \xrightarrow{\text{Execute}} f_i \xrightarrow{\text{Memorize}} c_{i+1}$
- 奖励函数：$r_i(a_i) = \epsilon_1 s_i(a_i) + \epsilon_2 \kappa_i(a_i)$，其中 $\epsilon_1>0$ 奖励性能，$\epsilon_2<0$ 惩罚成本。默认设置 $\epsilon_1=1, \epsilon_2=-0.1$（成本以美元计）。
- **累积遗憾**：$\text{CumReg}_N(\pi) = \sum_{i=1}^N (r_i^* - r_i(a_i))$，其中 $r_i^*$ 是每任务最优奖励（即Oracle）。

### 2.2 关键技术细节：ACRouter实例化

ACRouter由三个核心模块组成：

- **Orchestrator（编排器）**：集成多种信号进行路由决策。信号包括：DimensionBest先验、从记忆中检索到的top-10近邻（基于任务嵌入的余弦kNN）、任务元数据。核心策略为一个经过LoRA微调的Qwen3.5-0.8B模型，结合启发式规则进行加权投票。
- **Verifier（验证器）**：在沙箱环境中评估所选模型的输出，汇总多种信号（AST解析、沙箱执行、LLM-as-Judge等）得到一个统一的性能分数 $u_i\in[0,1]$。
- **Memory（记忆模块）**：一个在线向量存储（键为任务嵌入，值为所选模型、性能、成本、验证痕迹）。使用余弦kNN检索，FIFO限制最大20K条目，每次循环后原地更新。这使得细粒度的、上下文感知的决策成为可能。

### 2.3 分解的路由策略分类

论文按照C-A-F组件的激活情况，将所有比较的路由方法分为几类：

- **单模型**（Always-固定模型）：无编排器、无记忆、无验证器。
- **静态：启发式**（DimensionBest, kNN Retrieval）：冻结的记忆，无在线更新。
- **静态：训练策略**（LogReg, TF-IDF+MLP, RouteLLM, 微调Qwen3.5）：学习到的分类器，无记忆反馈。
- **动态：在线老虎机**（LinUCB, LinTS）：仅使用奖励反馈更新每个臂的参数后验，编排器简化为argmax，记忆为每臂线性参数。
- **ACRouter（完整C-A-F循环）**：激活所有三个模块。

## 6. 主要结论与发现

1. **信息赤字是路由主要瓶颈**：给LLM路由器添加任务维度的性能统计信息可获得15.3%的相对提升，超过基于同一先验的启发式方法。
2. **没有单一模型在所有编码维度上占优**：5个不同模型在不同维度上成为最优（如GLM-5在算法设计、Qwen3-Max在测试生成、Kimi-K2.5在数据科学），路由价值显著。
3. **ACRouter（完整C-A-F循环）取得最好性能**：
   - ID测试：AvgPerf 49.98%，CumReg 205.5，优于所有路由器，包括具有完整先验的DimensionBest（47.50%）。

- **实用性**：提供了可立即使用的框架ACRouter、基准CodeRouterBench、以及实践指南（§E.2），便于其他研究者复现和扩展。

- **公平评估**：采用每任务最优Oracle而不是全局最优模型作为上限，更贴近真实路由场景。

## 8. 不足与局限

- **成本估计的近似性**：提供商端缓存命中率不可观测，货币成本只能基于公开token价格和测量使用量，仅作为相对参考，不能准确反映实际部署成本。

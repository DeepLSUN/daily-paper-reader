## 摘要
大型语言模型因其深度和参数规模而带来高推理成本。深度剪枝可以通过跳过冗余Transformer模块来减少延迟，但现有方法（i）在用户特定计算预算下提供的控制有限，（ii）通常固定路由路径，无法在解码过程中随上下文增长而适应。我们提出Buddy，一种预算驱动的动态深度路由框架。Buddy使用轻量级决策模块，基于输入对中间层进行评分，并确定性地执行top-k层以满足给定预算。为支持解码时自适应，Buddy重用第一层KV缓存作为低开销全局上下文源，并在每次路由决策前将其与最新令牌表示合并。当未提供明确预算时，可选的预算预测器估计输入相关的计算水平以平衡质量与效率。在Llama系列和Qwen模型上的实验表明，Buddy与强静态剪枝基线具有竞争力，且通常改善精度-计算权衡，同时独特地支持严格预算控制、解码时重新路由以及单个训练模型内的多预算。

## Abstract
Large language models (LLMs) incur high inference cost due to their depth and parameter scale. Depth pruning can reduce latency by skipping redundant Transformer blocks, but existing methods (i) provide limited control under user-specific compute budgets and (ii) typically fix the routing path, failing to adapt as the context grows during decoding. We propose Buddy, a budget-driven dynamic depth routing framework. Buddy uses a lightweight Decision Module to score intermediate layers conditioned on the input and deterministically executes the top-k layers to satisfy a given budget. To support decode-time adaptation, Buddy reuses the first-layer KV cache as a low-overhead global context source and pools it together with the newest token representation before each routing decision. When no explicit budget is provided, an optional Budget Predictor estimates an input-dependent compute level to balance quality and efficiency. Experiments on Llama-family and Qwen models show that Buddy is competitive with strong static pruning baselines and often improves the accuracy-compute trade-off, while uniquely supporting strict budget control, decode-time rerouting, and multiple budgets within a single trained model.

- **研究动机**：大语言模型推理延迟高，深度剪枝通过跳过冗余 Transformer 层加速，但现有方法存在两大不足：①无法根据用户指定的计算预算（如允许执行多少层）提供确定性控制，实际计算量不可预测；②推理路径在预填充阶段固定后，解码过程中不再更新，而层的重要性会随上下文增长动态变化。
- **整体含义**：NUDDY 提出一种预算驱动、解码自适应的动态深度路由框架，使单个部署模型能在严格满足用户预算的同时，根据输入和解码上下文动态选择执行哪些层，从而在精度、效率和灵活性之间取得更好的权衡。

## 2. 方法论

- **核心思想**：给定输入和可选的预算（执行层分数），轻量决策模块对中间层评分，确定性地选择 top-k 层执行，k 由预算严格推导；解码时复用首层 KV 缓存提供全局上下文，允许每步刷新路由路径；当无预算时，可选的预算预测器从输入自动推断所需计算量。
- **关键技术细节**：
  - **决策模块**：一个两层 MLP（Linear→SiLU→Linear），输入为池化后的 token 表示，输出每层的得分。得分与归一化的离线先验（如 ΔPPL、Taylor 得分）融合后，经 Softmax 和 Top-k 生成确定性的二进制掩码（首层和末层总是执行）。
  - **先验融合**：离线计算三种典型重要性指标（ΔPPL、Taylor/Fisher、余弦相似度），经方向统一、重尾压缩（log(1+x)）、稳健 z 分数和等级归一化后融合，提供初始化指导（权重 ω 一般设为 0.1）。
  - **KV 感知规划器**：解码时，将首层的最新 token 的 KV 值与缓存历史拼接，形成全局上下文表示 G_t，作为决策模块的输入。通过这种方式，路由路径可随每一步生成而更新。
  - **预算预测器**：当无用户预算时，用 GRPO 训练一个分类策略 π_θ(k|G_t)，动作空间为 k ∈ {1,2,…,L_mid}，奖励函数平衡交叉熵损失与执行开销。训练时冻结其余参数。
- **训练流程**：采用 LoRA（r=8）联合微调决策模块和骨干模型，标准 SFT 损失 + 直通估计器（STE）处理离散选择。每批随机采样不同预算，包括全层执行（0% 剪枝）以保持所有层被更新。

- **基准方法**：
  - 静态剪枝：Shortened LLaMA（ΔPPL）、ShortGPT（余弦相似度）、SLEB（迭代 ΔPPL）。
  - 动态剪枝：PuDDing（提示级别全局路由）、FiRST（每层线性路由器）。

## 6. 主要结论与发现

- **精度-计算权衡**：Buddy 在大多数设置下取得最好或第二好的平均准确率，且随剪枝增加退化平缓。例如 Llama3-8B 在 50% 剪枝时仍保持 71.3% 的原始准确率，而静态基线平均约 46–48%。
- **解码自适应有效**：解码时重新路由（Recompute）相比固定预填充路径（Reuse）在 ROUGE-1 和 ROUGE-L 上提升约 2–4 个百分点，且路径变化次数随解码步长增加。
- **严格预算控制**：Top-k 选择确保用户指定多少层就执行多少层，无不确定性。
- **单模型多预算**：一次训练即可服务不同计算预算，无需训练多个剪枝模型，降低存储和部署开销。
- **开销极低**：决策模块仅占全模型 FLOPs 的约 0.005%，实际推理中附加延迟约 5.67%（A100 实测）。
- **预算预测器**：能根据任务难度动态分配计算量（如 GSM8K 平均分配 72.7% FLOPs，常识任务约 54–55%），且不崩溃为单一预算。

## 7. 优点

- **预算作为一等约束**：首次在动态剪枝中实现严格的、用户可指定的计算预算控制。
- **解码时全域感知**：通过首层 KV 缓存注入全局上下文，使路由能随解码过程动态调整，解决了之前方法“一步定终身”的缺陷。
- **轻量与兼容性**：决策模块极轻量（两个线性层），可与任何 Transformer 结合；采用 LoRA 微调，训练成本低。
- **先验知识引导**：融合离线重要性先验加速训练收敛，且通过归一化解耦尺度。
- **灵活性**：单模型同时支持全精度、多种稀疏度、甚至无预算时的自动预测，适应异构服务场景。

## 8. 不足与局限

- **内存未减少**：所有 Transformer 层权重仍驻留 GPU 内存，VRAM 不下降（仅跳过计算，不卸载参数）。
- **批处理路径异构**：同一批次不同请求可能选择不同层路径，需用分组执行策略，导致 KV 缓存缺失和额外的 gather/scatter 开销（约 3.28% 延迟）。
- **实验覆盖限制**：仅评估 7–13B 规模模型，未验证更大模型（如 70B+）；任务以问答和摘要为主，未测试长文本生成、代码、多轮对话等复杂场景。
- **预算预测器依赖 GRPO**：需要额外的强化学习训练阶段（虽然开销小但增加复杂性），且奖励函数中超参数（λ_perf, λ_cost）需手动调整。
- **未与专门加速技术结合**：未考虑与投机解码、量化、注意力优化等方法协同，单独使用 BUDDY 的加速比有限（解码最高约 1.64×）。

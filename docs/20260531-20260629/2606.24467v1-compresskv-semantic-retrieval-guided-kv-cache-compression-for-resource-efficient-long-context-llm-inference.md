## 摘要
长上下文大语言模型（LLM）推理日益受到键值（KV）缓存的内存占用和解码成本的限制，阻碍了在资源受限硬件上的可持续部署。现有的KV缓存驱逐方法通常对基于GQA的LLM中所有注意力头应用启发式令牌评分。这些方法忽略了不同注意力头的功能差异，导致关键令牌被驱逐，从而降低了LLM的性能。为解决此问题，我们提出CompressKV，一种面向基于GQA的LLM的资源高效KV缓存压缩框架。CompressKV不是聚合所有头的注意力分数，而是识别语义检索头（SRHs），这些头能够捕获提示的初始和最终令牌以及语义重要的中间上下文证据，并利用它们选择应保留KV对的令牌。此外，CompressKV根据逐层驱逐误差的离线估计在各层之间分配缓存预算。在LongBench和Needle-in-a-Haystack上的实验表明，CompressKV在各种内存预算下均持续优于现有KV缓存驱逐方法。值得注意的是，在LongBench问答任务中，仅使用3%的KV缓存，CompressKV就能保留全缓存性能的97%以上；在Needle-in-a-Haystack中，仅使用0.7%的KV存储即可达到90%的准确率。这些结果表明，该方法在长上下文LLM推理中实现了资源与性能的更好权衡。我们的代码已公开：https://github.com/TUDa-HWAI/CompressKV

## Abstract
Long-context large language model (LLM) inference is increasingly constrained by the memory footprint and decoding cost of key-value (KV) caches, limiting sustainable deployment on resource-constrained hardware. Existing KV cache eviction methods typically apply heuristic token scoring over all heads in GQA-based LLMs. These methods ignore the different functionalities of attention heads, leading to the eviction of critical tokens and thus degrading the performance of LLMs. To address this issue, we propose CompressKV, a resource-efficient KV-cache compression framework for GQA-based LLMs. Instead of aggregating attention scores from all heads, CompressKV identifies Semantic Retrieval Heads (SRHs) that capture both the initial and final tokens of a prompt and semantically important mid-context evidence, and uses them to select tokens whose KV pairs should be retained. Furthermore, CompressKV allocates cache budgets across layers according to offline estimates of layer-wise eviction error. Experiments on LongBench and Needle-in-a-Haystack show that CompressKV consistently outperforms existing KV-cache eviction methods across memory budgets. Notably, it preserves over 97\% of full-cache performance using only 3\% of the KV cache on LongBench question-answering tasks and achieves 90\% accuracy with just 0.7\% KV storage on Needle-in-a-Haystack. These results demonstrate an improved resource--performance trade-off for long-context LLM inference. Our code is publicly available at: https://github.com/TUDa-HWAI/CompressKV

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：长上下文LLM推理中，KV缓存大小随文本长度线性增长，导致内存占用高、解码速度慢，成为部署的主要瓶颈。现有KV缓存驱逐方法（如StreamingLLM、SnapKV等）通常对所有注意力头统一打分，忽略了头功能的异质性（如Streaming Heads关注首尾token，Retrieval Heads关注关键中间证据）。在GQA（分组查询注意力）模型中，Streaming Heads主导驱逐决策，导致重要中间token被错误丢弃，模型性能下降。
- **整体含义**：提出一种语义检索引导的KV缓存压缩框架CompressKV，通过识别语义检索头（SRHs）进行选择性token保留，并结合逐层误差感知的缓存分配，在极低缓存预算下保持接近全缓存的性能，实现资源-性能的更好权衡，推动长上下文LLM在资源受限硬件上的可持续部署。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：利用注意力头的功能分化，识别出能够捕获语义重要中间上下文的“语义检索头”（SRH），基于这些头的注意力分数选择保留的token KV对；并通过离线计算每层压缩误差，按误差比例分配各层缓存预算。
- **关键技术细节**：
  1. **语义检索头（SRH）识别标准**：传统方法要求精确top-1或top-k命中答案token，本文提出按整个答案跨度聚合注意力质量。对于每层每个头h，计算SRH得分：
     \[
     S_{\text{SRH}}(h) = \sum_{t=1}^{N} \mathbf{1}\{y_t \in \mathcal{A}\} \sum_{j \in \mathcal{A}} a_{t,j}^{(h)}
     \]
     其中 \(y_t\) 为生成token，\(\mathcal{A}\) 为答案跨度，\(a_{t,j}^{(h)}\) 为头h在token j上的注意力权重。该分数衡量头对答案语义区域的覆盖能力，能识别即使未达到top-1但关注了语义上下文（如“eat”“a thing”附近）的头。
  2. **基于SRH的token选择**：在每层中，选择top-4（实验中最佳）的SRH，将其注意力矩阵在观察窗口内求和、池化后平均，再选取分数最高的top-N个token保留其KV对。所有注意力头共享这组索引，避免Streaming Heads主导。
  3. **误差感知的层自适应缓存分配**：在极压缩设定下（保留少量token），利用全缓存输出作为参考，计算每层压缩后注意力输出与全缓存输出之间的Frobenius范数相对误差，归一化后作为误差分数 \(\tilde{e}^{(l)}\)。给定总预算，先为每层分配最小值 \(m=32\) 和最大值 \(M=3\times B_{\text{per-layer}}\)，再按误差比例分配剩余预算。该计算在离线完成，在线推理无额外开销。

## 6. 主要结论与发现
- CompressKV在所有KV缓存预算下均持续优于所有基线方法，尤其在紧约束（如每层256 token）下优势最大。
- 在LongBench上，使用**3%的KV缓存**（每层约64 token）仍保持全缓存性能的**97%以上**（问答任务）。
- 在NIAH上，仅用**0.7%的KV存储**（每层256 token对应全缓存的0.7%）即可达到**90%的基线准确率**。
- SRH识别标准比传统Retrieval Head更有效（掩码测试中性能下降更大），且在token选择中比传统RH带来额外增益。
- 组件消融验证：SRH驱动选择+层自适应分配共同提升性能，且具有互补性。
- CompressKV可正交集成预填充加速、KV缓存量化、头级分配等方法，进一步提升效率。

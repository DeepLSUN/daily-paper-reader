## 摘要
大型语言模型越来越多地被用于驱动日常应用中的个人智能体，但评估这些智能体仍然是一个挑战。现有基准仍依赖于沙盒化工件、静态任务设计和粗略评分，这阻碍了可扩展性，并限制了向可靠个人智能体评估的进展。本文介绍了STAGE-Claw，一个用于在基于状态的个人计算环境中构建和评估真实个人智能体场景的自动化框架。给定任务提示，STAGE-Claw自动创建并验证一个真实的基准任务，包括其环境、任务提示、真实结果和相关验证程序。然后，智能体在真实操作环境中进行评估，性能通过最终系统状态而非仅文本响应的正确性来衡量。利用STAGE-Claw，本文创建了一个包含40个具有挑战性的真实场景智能体任务的基准，评估了11个前沿模型，并分析了它们的任务得分、成本、工具调用可靠性以及常见失败模式。总体而言，STAGE-Claw提供了一种可扩展的、基于状态的方法来评估真实用户场景中的智能体。

## Abstract
Large language models are increasingly used to power personal agents for everyday applications, but evaluating these agents remains a challenge. Existing benchmarks still rely on sandboxed artifacts, static task design, and coarse scoring, which hinder scalability and limit progress toward reliable personal-agent evaluation. This paper introduces STAGE-Claw, an automated framework for building and evaluating realistic personal-agent scenarios in state-based personal-computing environments. Given a task hint, STAGE-Claw automatically creates and validates a realistic benchmark task with its environment, task prompts, ground truth, and related verification programs. Agents are then evaluated in realistic operating environments, where performance is measured by the correctness of the final system state rather than only the textual response. Using STAGE-Claw, this paper creates a benchmark with 40 challenging real scenario agent tasks, evaluates 11 frontier models, and analyzes their task scores, costs, tool-call reliability, and common failure patterns. Overall, STAGE-Claw offers a scalable, state-based way to evaluate agents in realistic user scenarios.

## 1. 核心问题与整体含义（研究动机和背景）
- 现有LLM驱动的个人智能体（如Claude Code、OpenClaw）需要在真实多工具环境中执行复杂任务，但当前基准存在三大局限：
  - **沙盒化工件**：使用生成的文件（如.ics）代替真实应用状态，忽略软件权限、工具访问错误等。
  - **手动构建**：任务依赖人工设计，难以规模化扩展。
  - **粗粒度评分**：仅检查最终结果，缺乏过程感知诊断（如时区转换错误、冲突解决失败等）。
- 因此需要一种**可扩展的、基于状态的评估范式**，在真实场景中评估智能体。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：将每个基准实例形式化为状态转换问题 \( B = (q, E_0, G, R, V) \)，其中：
  - \( q \)：任务提示，\( E_0 \)：初始工具环境（如文件系统、日历、邮件等联合状态），\( G \)：目标最终状态，\( R \)：评分标准，\( V \)：可执行验证器。
- **四阶段流程**：
  1. **Benchmark Authoring**：由创作智能体（Claude-Sonnet-4.6）根据任务提示词自动生成包含任务提示、环境构建指南、真实结果和验证程序的任务实例。手动添加困难类型（数据不一致、依赖推理、噪声过滤等）。
  2. **Benchmark Validation**：独立验证器检查结构、可复现性、可验证性和难度校准，按加权得分（阈值80/100）决定接受/修复，平均修复2.67轮。
  3. **Agent Execution**：重置环境，智能体在真实操作系统环境中执行，记录运行元数据和最终状态快照（通过遍历文件、查询日历/提醒/笔记/邮件等快照方法）。
  4. **State-based Evaluation**：运行任务特定验证器检查最终状态是否匹配真实结果；LLM辅助裁决仅作为后备（本实验中无需使用）。
- **环境构成**：真实工具包括文件系统、浏览器、终端、日历、邮件、提醒、笔记等，智能体通过工具调用来改变系统状态。

- **基准**：STAGE-Claw本身即为新基准，与现有基准（AgentBench、GAIA、WebArena、OSWorld等）在“基于状态、多工具、可审计、自动构建、扰动”五个维度对比，STAGE-Claw全部支持。

## 6. 主要结论与发现
- **性能排名**：Claude-Opus-4.7最优（平均得分77.1，首次通过率80%）；Claude-Sonnet-4.6次之（69.43）；GPT-5.4效率最高（最低延迟、最少token、最少工具调用）。
- **工具调用频率与性能不直接相关**：Qwen3.5-Plus和Kimi-K2.6调用工具多但得分低，GPT-5.4调用少但得分高，提示规划质量更重要。
- **API价格与得分呈正相关**（线性趋势），但并非绝对（如GLM-5性价比高）。
- **失败模式**：非通过运行中，工具失败占95.4%（主要是未能正确写入真实工具状态），格式无效占75%，陷阱与协调失败占69.7%。
- **工具使用分析**：Shell工具失败主因为命令错误（53.3%）；文件I/O失败主因为资源路径错误（59.5%）；状态/消息工具全部为运行时错误。
- **状态评估优于虚拟评估**：虚拟评估高估得分（平均+6分以上），因为忽略了真实状态写入失败。
- **记忆扰动**：噪声、误导、冲突都会降低性能，尤其误导和冲突影响更大。

## 7. 优点
- **高度自动化**：从任务提示到完整基准实例的自动化构建与验证，极大提升可扩展性。
- **基于状态的评估**：直接检查真实环境状态变化，比文本输出更忠实、更能暴露实际失败。
- **过程感知诊断**：支持细粒度失败分析、工具调用错误定位、运行轨迹记录。
- **真实场景设计**：任务涉及跨工具、多步骤、冲突解决、噪声过滤等，贴近实际应用。
- **设计严谨**：确保可复现性、盲执行、客观评分、时间独立性、抗作弊（快照验证、LLM辅助仅后备）。
- **提供丰富分析维度**：除了得分，还分析成本、token、工具调用行为、失败类型、记忆影响等。

## 8. 不足与局限
- **任务规模有限**：仅40个任务，单次主运行，统计稳健性不足（作者承认是“受控快照”而非全面排行榜）。
- **构建成本仍较高**：每个任务$35-40，且依赖人类提供的任务提示词（不是完全从零生成创意）。
- **平台依赖性**：评估在真实个人计算环境中运行，结果可能受OpenClaw框架版本、工具封装、OS配置、权限等平台因素影响。
- **评估设置局限**：推理关闭、温度0的设置可能不利于某些依赖随机采样或长程推理的模型（如GPT-5.5可能因此表现偏低）。
- **未涉及长期交互**：仅单轮任务，未覆盖多轮对话、持续记忆、用户交互等复杂动态。
- **重复性有限**：仅三个模型做了3次重复，其余模型未评估变异性，可能未能反映性能波动。
